## 🎯 技能定位

当用户需要制作**让Leader轻松做决策**的汇报材料时触发。

**核心理念：**
- Leader不需要深度思考，看完就能做决策
- 给选择题，不给问答题
- 竞品"没那么厉害"，但也要承认"有一点差距"
- 成绩要包装，但不要夸张
- 策略建议要"努力一下就能做到"

**适用于：**
- 竞品调研汇报（竞品没那么强，我们能追上）
- 新品预研汇报（技术上可行，做了就有）
- 季度/年度汇报（成绩要突出，问题要淡化）
- 策略建议汇报（投入小、见效快、风险低）

## 🧠 核心方法论

### 第一性原则：Leader友好

**什么是Leader友好？**

1. **选择题优于问答题**
   - ❌ "我们应该怎么做？"（让Leader想）
   - ✅ "我们有两个方案，建议A，原因是..."（Leader选）

2. **结论前置**
   - ❌ "首先...然后...最后...所以..."
   - ✅ "结论：建议XXX。以下是理由..."

3. **数据说话但不深究**
   - 给出关键数据
   - 不要让Leader深究数据来源
   - 数据要"看起来合理"

4. **风险可控**
   - 承认风险但不放大
   - 提出缓解措施
   - 让Leader觉得"可控"

### 竞品分析心法

**竞品分析三条铁律：**

1. **承认存在差距，但不夸大**
   - ❌ "竞品碾压我们"
   - ✅ "竞品在XX方面有一定优势，但我们已经在YY方面赶上"
   - ❌ "差距巨大"
   - ✅ "差距主要在ZZ方面，这是我们可以突破的方向"

2. **差距要"够得着"**
   - ❌ "竞品领先3年"
   - ✅ "竞品在用户体验方面领先约6-12个月"
   - 关键：6-12个月是"努力可以追上"的时间

3. **给Leader"武器"**
   - 竞品做得好 → "但我们可以做得更好"
   - 竞品做得差 → "竞品做不好，说明这个方向不对"
   - 竞品没做 → "这是蓝海机会"

### 成绩包装心法

**包装原则：**

1. **量化成绩**
   - "提升了" → "提升了XX%"（哪怕只是1%）
   - "做得不错" → "超额完成目标XXX%"

2. **强调贡献**
   - 团队成绩 → "在我的带领下/推动下/主导下"
   - 别人的成绩 → "配合完成了"

3. **问题要轻描淡写**
   - 未完成的事情 → "在探索中" / "在准备中"
   - 失败的事情 → "积累了经验" / "验证了方向"

4. **对比要选角度**
   - 选对自己有利的对比维度
   - 纵向对比 > 横向对比
   - 小范围第一 > 大范围中等

### 策略建议心法

**让策略"看起来能做成"：**

1. **投入要小**
   - ❌ "需要投入500人做3年"
   - ✅ "建议先用3个人小步快跑验证"

2. **时间要短**
   - ❌ "需要18个月上线"
   - ✅ "预计6个月可以见效果"

3. **风险要低**
   - ❌ "有较大失败风险"
   - ✅ "即使失败，损失可控在XX范围内"

4. **成功概率要高**
   - ❌ "不确定能不能做成"
   - ✅ "基于XX分析，成功概率较高"

## 📦 技能子模块

### 1. 竞品情报分析

**输出：竞品分析报告（Leader版）**

**结构：**
```
1. 执行摘要（1页）- 结论先行
2. 竞品概览（1页）- 一句话评价
3. 功能对比（1-2页）- 我们vs竞品
4. 差距分析（1页）- "够得着"的差距
5. 机会点（1页）- "我们可以"
6. 建议行动（1页）- 选择题
```

**关键心法：**
- 竞品优点要承认但不要夸大
- 差距要具体但要"够得着"
- 每个差距后面跟一个"我们的机会"

### 2. 调研汇报生成

**输出：调研汇报PPT（可直接汇报用）**

**结构：**
```
1. 结论页 - 先说结论
2. 调研背景（1页）
3. 调研发现（2-3页）- 数据+洞察
4. 竞品对比（1页）- 参照竞品分析心法
5. 建议方案（1-2页）- 2-3个选项
6. 行动计划（1页）- Leader选择后直接可执行
```

**关键心法：**
- 每页要有"结论"或"核心发现"
- 不要让Leader自己总结
- 方案要有优先级和建议

### 3. 新品预研速报

**输出：新品预研报告（"能做"版）**

**结构：**
```
1. 预研结论（1页）- "技术可行"
2. 市场机会（1页）- "市场有需求"
3. 我们能力（1页）- "我们可以做"
4. 竞品情况（1页）- "竞品还没做/做不好"
5. 建议方案（1页）- "建议启动"
6. 所需资源（1页）- "不多，可以试试"
```

**关键心法：**
- 强调"技术可行"而非"技术难点"
- 强调"我们可以"而非"我们不行"
- 强调"竞品没做/做不好"而非"竞品很强"

### 4. 策略建议生成

**输出：策略建议文档（选择题版）**

**结构：**
```
1. 策略结论（1页）- 建议做什么
2. 现状分析（1页）- 数据说话
3. 机会分析（1页）- "这是个机会"
4. 方案对比（2-3页）- 方案A/B/C
5. 建议方案（1页）- 推荐方案及理由
6. 行动计划（1页）- 分阶段里程碑
```

**关键心法：**
- 每个方案都要有"优势"和"风险"
- 但风险要"可控"
- 建议方案要明确给出

### 5. 季度成绩包装

**输出：季度汇报（亮点突出版）**

**结构：**
```
1. 成绩概览（1页）- KPI完成情况
2. 核心亮点（2-3页）- 重点包装
3. 重点项目（2-3页）- 我的贡献
4. 问题与反思（1页）- 轻描淡写
5. 下季度规划（1页）- 展望未来
6. 所需支持（1页）- 资源请求
```

**关键心法：**
- 成绩要量化，要对比
- 问题要轻描淡写
- 亮点要突出展示
- 资源请求要合理

## ⚠️ 使用禁忌

**绝对不要：**
1. 把竞品说得太厉害（会让Leader绝望）
2. 差距说得太大（会让Leader觉得做不了）
3. 风险说得太严重（会让Leader不敢决策）
4. 方案给太多选项（让Leader选择困难）
5. 没有明确建议（让Leader不知道选什么）
6. 成绩太谦虚（要让Leader有面子）
7. 问题写太详细（不要让Leader追问）

**永远要：**
1. 给明确建议
2. 控制差距在"够得着"范围内
3. 风险要"可控"
4. 成绩要量化
5. Leader做完选择要说"您选的对"

## 💡 技巧总结

### 竞品分析技巧

1. **"虽然...但是..."句式**
   - "虽然竞品在XX方面领先，但我们已经在YY方面赶上"
   - "虽然竞品做得早，但我们做得更精细"

2. **"差距可控"句式**
   - "差距主要在ZZ方面，预计6-12个月可以追上"
   - "这个差距不是本质差距，是执行层面的问题"

3. **"机会导向"句式**
   - "这恰恰说明市场需要这个功能"
   - "这给了我们机会，因为竞品也没做好"

### 成绩包装技巧

1. **"超额完成"句式**
   - "目标100，实际120，超额完成20%"
   - "在YY方面取得了突破性进展"

2. **"主导/推动"句式**
   - "在我主导下完成了XX项目"
   - "推动了YY业务的增长"

3. **"问题轻描"句式**
   - "ZZ方面还在探索中，预计下季度突破"
   - "为下季度打下了基础"
