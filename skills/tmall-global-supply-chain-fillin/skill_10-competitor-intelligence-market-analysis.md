# TL;DR

建立完整的竞品监控与市场情报体系，用数据驱动决策，知己知彼，在竞争中永远快人一步。

# 场景分析

## 典型场景
- **竞品价格战**：对手突然降价20%，要不要跟？怎么跟？
- **竞品上新**：对手推出爆款新品，自己的品类结构如何应对
- **市场机会发现**：某个细分品类增速极快，要不要切入
- **大促前情报**：了解竞品大促策略，提前制定竞争方案

## 核心挑战
1. **情报及时性难**：往往发现竞品动作时已经晚了
2. **数据量大但有效信息少**：监控了很多数据，但真正有价值的洞察不多
3. **价格监控受限**：平台反爬机制越来越严，数据获取越来越难
4. **分析深度不足**：知道竞品做了什么，但不知道为什么会这么做
5. **情报到行动的转化难**：有了情报但没有对应的行动计划

## 正确认知
- **竞品情报不是目的，服务于自身决策才是目的**
- **实时监控优于事后复盘，早知道意味着早行动**
- **情报的价值在于预测未来，不是描述过去**
- **数据采集是基础，分析洞察是核心，行动转化才是价值**


# 决策框架

## 第一步：建立竞品监控矩阵

### 竞品识别与分级

| 竞品层级 | 定义 | 监控粒度 | 响应优先级 |
|----------|------|----------|-----------|
| 核心竞品 | 同品类、同价格带、月销Top10 | **实时监控** | 最高 |
| 主要竞品 | 直接竞争、相似客群 | 每日监控 | 高 |
| 潜在竞品 | 新进入者、跨界竞争者 | 每周监控 | 中 |
| 关注竞品 | 长期观察、战略参考 | 月度分析 | 低 |

### 竞品监控清单建立
```
Step 1：确定核心竞品（3-5个）
→ 月销Top10中与我方品类重叠的
→ 价格带与我方重叠的
→ 流量来源相似的

Step 2：建立监控维度
→ 价格：日常价、活动价、券后价
→ 销量：GMV、销量、转化率
→ 流量：访客数、加购率、来源结构
→ 评价：好评率、差评内容、追评
→ 促销：活动参与、折扣力度

Step 3：设定预警阈值
→ 价格预警：竞品价格 < 我方价格 × 0.85
→ 活动预警：竞品折扣 < 我方折扣 × 0.9
→ 新品预警：竞品上架与我方重叠新品
```

## 第二步：确定数据采集方式

### 数据采集方式选择

| 数据类型 | 采集方式 | 频率 | 工具 |
|----------|----------|------|------|
| 店铺基础数据 | 平台API/生意参谋 | 每日 | 生意参谋 |
| 价格数据 | 爬虫/监控工具 | **实时或每日** | 慢慢买/情报通 |
| 销量数据 | 生意参谋/估算 | 每日 | 生意参谋 |
| 评价数据 | 平台抓取 | 每日 | 客服系统 |
| 活动数据 | 手动+工具 | 实时 | 监控工具 |
| 流量数据 | 生意参谋 | 每日/每周 | 生意参谋 |

### 采集优先级决策
```
竞品动态优先级：
1. 价格变动（影响最直接）→ **实时监控**
2. 新品上架（抢占市场）→ **实时监控**
3. 大促活动（影响流量）→ 活动前48小时密集监控
4. 评价变化（反映问题）→ 每日汇总
5. 流量波动（预判趋势）→ 每周分析
```

## 第三步：分析洞察与洞察深度决策

### 分析框架选择

| 分析场景 | 推荐框架 | 分析深度 | 输出 |
|----------|----------|----------|------|
| 日常监控 | 数据汇总+异常预警 | 表层 | 周报/日报 |
| 专题分析 | 波特五力+SWOT | 深层 | 月度分析报告 |
| 价格战应对 | 成本-毛利-竞品三角分析 | 中层 | 定价建议 |
| 新品竞争 | 竞品新品对比+用户评价分析 | 深层 | 应对策略 |
| 大促前分析 | 全维度对比+预判 | 全方位 | 大促作战方案 |

### 关键洞察提炼决策

```
竞品动作识别
    ↓
判断影响范围（对我方GMV/流量影响程度）
    ↓
├─ 影响 < 5% → 记录观察
├─ 影响 5-15% → 制定应对预案
└─ 影响 > 15% → **立即响应**（4小时内给方案）
```

## 第四步：制定竞争策略

### 竞争策略选择决策树

```
评估我方实力位置：
├─ 我方市场份额 > 竞品 → 领先者策略
│   └─ 防守为主 + 创新引领
├─ 我方市场份额 ≈ 竞品 → 挑战者策略
│   └─ 差异化进攻 + 抢夺份额
├─ 我方市场份额 < 竞品 → 跟随者策略
│   └─ 快速跟随 + 细分深耕
└─ 我方市场份额极小 → 补缺者策略
    └─ 专注细分 + 避免正面竞争
```

### 价格战应对决策

| 竞品降价幅度 | 我方成本空间 | 决策 | 备选方案 |
|-------------|-------------|------|----------|
| <10% | 充足 | 观察，不立即跟进 | - |
| 10-15% | 充足 | 评估后跟进 | 赠品替代降价 |
| >15% | 有限 | 不跟进，差异化竞争 | 服务升级/赠品策略 |
| >15% | 充足 | 可跟进，但要评估长期影响 | 限时限量的精准跟进 |

### 策略输出决策

| 分析类型 | 输出形式 | 分发对象 | 更新频率 |
|----------|----------|----------|----------|
| 竞品日报 | 简短摘要+关键变化 | 运营团队 | 每日 |
| 竞品周报 | 详细分析+数据+建议 | 运营+管理层 | 每周 |
| 月度分析 | 深度分析+趋势+策略 | 全管理层 | 每月 |
| 大促分析 | 竞品对比+作战方案 | 全团队 | 大促前 |

# 专业建议

## 典型场景
- 竞品突然降价30%，自己措手不及，GMV直接跌了20%
- 竞品上了一个新品，自己两周后才反应过来
- 大促前不知道竞品备了多少货，结果人家卖断了自己还有一堆库存
- 调价总是慢半拍，要么跟进了没利润，要么没跟进没流量

## 核心挑战
1. **信息滞后**：竞品动作往往比我们早几天甚至几周
2. **数据分散**：价格、销量、评价、促销，数据来源太多
3. **分析浅层**：有数据但不知道怎么用，只看表面
4. **反应太慢**：分析出来了但没有机制跟进执行

## 正确认知
- **情报是竞争优势的源泉**
- **知道竞品在想什么，比知道自己有什么更重要**
- **数据采集是基础，分析洞察是核心，快速响应是关键**
- **竞品监控不是一次性工作，是持续的系统性工程**

## 价格监控体系

### 价格监控维度

| 维度 | 内容 | 预警设置 |
|------|------|----------|
| 日常价格 | 日常售价 | 变动>5% |
| 活动价格 | 活动价、大促价 | 变动>10% |
| 券后价格 | 叠加优惠后价 | 实时监控 |
| 全网价格 | 各平台同款价格 | 低价预警 |

### 价格战应对策略

#### 被动跟进
| 情况 | 策略 |
|------|------|
| 竞品小幅降价（<10%） | 观察2-3天，若影响明显再跟进 |
| 竞品大幅降价（>15%） | 立即评估成本，准备跟进 |

#### 主动定价
| 策略 | 说明 |
|------|------|
| 成本定价 | 确保毛利率>20% |
| 价值定价 | 基于产品差异化和用户价值 |
| 竞争定价 | 基于竞品价格动态调整 |
| 心理定价 | 99/199等价格策略 |

#### 差异化竞争
| 策略 | 说明 |
|------|------|
| 服务差异化 | 比竞品提供更好的服务 |
| 赠品策略 | 比竞品提供更多赠品 |
| 会员专享 | 保护价格敏感用户 |
| 组合销售 | 提升客单价 |
