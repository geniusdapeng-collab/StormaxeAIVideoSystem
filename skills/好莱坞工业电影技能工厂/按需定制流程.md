# Skill Forge — 按需定制工作流

> 当用户说"我要拍深海潜水器"，系统怎么自动生成一个专项技能？

---

## 核心设计理念

Skill Forge（技能锻造引擎）是系统的"自我扩展机制"。

传统方式：
```
用户需求 → 告诉用户"我做不到" → 用户失望
```

Skill Forge方式：
```
用户需求 → 技能缺失识别 → Skill Forge自动锻造 → 新技能注册 → 搞定
```

**目标**：系统遇到未知领域时，不是拒绝，而是创造工具。

---

## 触发条件

以下任一条件满足时，触发 Skill Forge：

1. **显式新领域**：用户描述的场景/主题在现有技能库中无对应技能
2. **交叉领域需求**：用户需求跨越多个现有技能的边界，但每个现有技能都无法完整覆盖
3. **专业深度不足**：现有技能的输出在特定领域的专业知识深度不够
4. **新类型挑战**：用户要求处理全新类型的视觉元素（如"深海潜水器"、"太空零重力"）

---

## 锻造流程（6步）

### Step 1：问题域分解

**输入**：用户的原始需求描述（自然语言）

**处理**：
1. 提取核心关键词（主体、场景类型、特殊元素）
2. 映射到已有技能覆盖的知识领域
3. 识别未被覆盖的"知识空白区"

**输出**：
```
问题域分解报告：
  已有技能覆盖：Gaffer（光线）、DP（镜头）、Acting（表演）
  知识空白区：深海环境物理特性、压力下的光线行为、水下生物发光机制
  新技能定位：深海摄影专家 / 水下光学物理师
```

---

### Step 2：专业知识注入

**输入**：知识空白区描述 + v4.1约束包

**处理**：
1. 调用深度研究工作流（deep-research），对空白区进行专项研究
2. 研究内容包括：
   - 该领域的物理规律/化学反应/生物特性
   - 该领域在影视制作中的已有实践
   - 该领域的专业术语和参数体系
3. 研究结果沉淀为"专业知识库片段"

**输出**：
```
深海环境专业知识库片段：
  压力特性：每下潜10米增加1个大气压，深海生物适应高压
  光线行为：波长越深衰减越快，红色先消失，蓝色穿透最深
  生物发光：深海生物自发光是对黑暗的适应，发蓝光/绿光
  影视参考：《深海挑战》《大白鲨》的水下摄影技术
  专业术语：压载舱、潜水服、光纤照明、浑浊度
```

---

### Step 3：技能模板填充

**输入**：专业知识库片段

**处理**：
1. 选择最适合的基础模板：
   - **IATSE技术工会模板**：适合硬件/技术类技能（如"深海摄影指导"）
   - **SAG表演类模板**：适合生物/角色类技能（如"深海生物演员"）
   - **混合模板**：同时涉及技术和表演的技能
2. 按V9.0标准格式填充以下字段：
   - name（技能名称）
   - skill_id（技能唯一标识）
   - 核心能力（该技能能做什么）
   - 专业知识库（该领域特有的知识和数据）
   - 决策逻辑（在什么情况下做什么决策）
   - 输出中间产物（标准化的输出格式）
   - 与主链路对接（接入哪个Stage）

**输出**：
```yaml
---
name: 深海摄影指导
skill_id: skill_iatse_deep_sea_dp
version: 1.0.0
last_updated: AUTO-GENERATED

核心能力:
1. 深海光线模拟与参数化
2. 水下压强对拍摄设备的影响评估
3. 深海生物发光视觉化设计

专业知识库:
  光线衰减模型: [波长-深度对照表]
  生物发光色温: [蓝色450-495nm / 绿色495-570nm]
  压强参数: [不同深度的压强计算]
  影视参考: [《深海挑战》打光方案]

决策逻辑:
  深度<200m → 使用人造光源+后期蓝色调色
  深度200-1000m → 生物发光为主+微弱环境光
  深度>1000m → 全黑+生物发光唯一光源

输出中间产物:
  DeepSeaLightSheet:
    主光源:
    环境光源:
    生物发光:
    色调:

与主链路对接:
  Stage: 8（Prompt光影段）
  输入: Scene Card
  输出: DeepSeaLightSheet → 融合层
```

---

### Step 4：约束合规验证

**输入**：新技能草稿 + v4.1约束包

**处理**：
1. 逐项检查是否违反v4.1规范：
   - Scene Card必填字段是否完整
   - Shot Card叙事目的是否单一
   - Prompt长度是否在限制内（520字符）
   - Light Tier是否明确指定
2. 识别并修复约束冲突

**输出**：
```
约束合规报告：
  [✓] Scene Card光线总策略已定义
  [✓] Shot Card叙事目的单一性通过
  [✓] Prompt结构符合「主体→动作→环境→镜头→风格→约束」
  [✓] Light Tier明确指定为B档（深海低照神秘）
  [⚠] 警告：深海生物发光色温范围超出常规，需在约束段明确说明
  [修复] 已在输出模板中增加"特殊色温约束"字段
```

---

### Step 5：分支验证（test-render）

**输入**：新技能的输出中间产物

**处理**：
1. 将新技能纳入融合层
2. 选择一个测试镜头（优先选择与新技能领域匹配的场景）
3. 执行完整的分支链路验证：
   - 新技能输出是否产生专业增量
   - 与已有技能的融合是否顺畅
   - 是否产生冲突（冲突检测）
4. 渲染测试镜头
5. 评估渲染结果是否因新技能而提升

**输出**：
```
分支验证报告：
  测试镜头: SCxx-Sxx（深海神兽苏醒）
  新技能输出: DeepSeaLightSheet v1.0
  专业增量: [显著] 深海光线模拟准确度提升
  融合冲突: [无] 与Gaffer技能无冲突
  渲染结果: [通过] 情绪目标"敬畏"达成
  
  四维评估：
    运镜: 17/20
    光影: 18/20 ← 较基础版本+3分
    空间: 22/25
    情绪: 21/25 ← 较基础版本+5分
    总分: 78/100 ✓ 超过75分生死线
```

---

### Step 6：技能库注册

**通过验证？**

```
  ↙ 是                    否 → 退回Step 3重锻造
  ↓
  更新Skill Router声明式描述
  记录到技能注册表
  通知用户"新技能已就位"
  纳入主链路使用
```

**处理**：
1. 生成技能注册记录：
   - skill_id、name、version、生成日期
   - 对接的Stage
   - 输入输出Schema
   - 与其他技能的依赖关系
2. 更新Skill Router的声明式描述（让路由知道这个新技能的存在）
3. 输出注册完成通知

**输出**：
```
技能注册完成：
  skill_id: skill_iatse_deep_sea_dp
  name: 深海摄影指导
  version: 1.0.0
  生成日期: 2026-06-04
  状态: 已注册，可使用
  下次遇到深海场景将自动激活
```

## 异常处理

### 锻造失败

| 失败阶段 | 处理方式 |
|----------|----------|
| 专业知识研究超时 | 使用已有的通用知识临时替代，标记"需人工审核" |
| 约束合规验证失败 | 记录冲突点，等待人工介入裁定 |
| 分支验证失败 | 分析失败原因，调整专业知识库，重新锻造 |
| 无限循环锻造 | 3次锻造失败后停止，通知用户"该领域需要人工专家" |

### 人工介入点

以下情况需要人工确认，不能自动锻造：
1. 涉及真实武器/战争/政治内容的场景
2. 涉及医疗/法律等专业领域的建议
3. 涉及文化敏感性（如宗教、图腾、禁忌元素）
4. 3次自动锻造失败的领域

## Skill Forge 与现有系统的关系

```
┌─────────────────────────────────────────────────────────┐
│                    Skill Forge                          │
│                                                         │
│  用户需求 → 问题分解 → 专业研究 → 模板填充 →           │
│  约束验证 → 分支验证 → 注册/退回                       │
└─────────────────────────────────────────────────────────┘
                         ↓ ↑
┌─────────────────────────────────────────────────────────┐
│              Skill Router（技能路由）                    │
│                                                         │
│  声明式描述库 ← Skill Forge 注册记录                    │
│                                                         │
│  用户需求 → 路由判断 → 激活技能组合                     │
└─────────────────────────────────────────────────────────┘
                         ↓ ↑
┌─────────────────────────────────────────────────────────┐
│              融合层（Fusion Layer）                      │
│                                                         │
│  多技能输出 → 冲突检测 → 融合决策 → 主链路输入          │
└─────────────────────────────────────────────────────────┘
```

---

## 自动 vs 手动

| 场景 | 方式 | 说明 |
|------|------|------|
| 已知领域（已有技能覆盖） | 自动 | Skill Router直接路由 |
| 新领域但有通用模板 | 半自动 | Skill Forge自动锻造 + 人工审核 |
| 完全陌生领域 | 手动 | 需要人工提供专业知识输入 |
| 涉及敏感内容 | 禁止锻造 | 必须人工介入 |
