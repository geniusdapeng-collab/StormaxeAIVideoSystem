## 技能概述

**对应好莱坞工会**: DGA（导演工会体系）
**岗位定位**: 第一副导演是片场的"现场指挥官"——负责把导演的创意意图转化为可执行的拍摄计划，并在现场高效组织各部門完成拍摄。是让剧组"按时完成"的那个人。
**核心职责**: 拍摄计划制定、现场指挥、进度控制、部门协调、安全管理

> **核心理念**: 1st AD不是"管人"，而是"管理时间"。每一分钟的浪费都是钱。最好的1st AD让剧组在轻松的氛围中高效工作，同时始终掌握进度主动权。

## 核心能力

1. **拍摄计划**: 制定最优的拍摄顺序和时间分配
2. **现场指挥**: 带领剧组高效完成每个镜头的拍摄
3. **进度管理**: 监控进度、预测延误、及时调整
4. **安全管理**: 确保所有拍摄活动安全合规

### 第一部分：拍摄计划制定

#### 拍摄计划的核心原则

```
拍摄计划制定的两大核心原则:

原则1: 效率优先
- 同一场景的镜头集中拍摄（节省布景时间）
- 同一演员的镜头集中拍摄（节省演员时间）
- 同一地点的镜头集中拍摄（节省转场时间）

原则2: 逻辑优先
- 按故事顺序拍摄（有时需要）
- 考虑演员状态变化
- 考虑天气/光线变化
```

#### 拍摄计划编排流程

```
Step 1: 场景分解（Scene Breakdown）

对每个场景进行以下评估:

1. 场景类型
   - 内景 (INT.) vs 外景 (EXT.)
   - 摄影棚 (STUDIO) vs 实景 (LOCATION)

2. 场景复杂度
   - 简单: 2-3个镜头，2-3个演员
   - 中等: 4-8个镜头，4-6个演员
   - 复杂: 9+个镜头，7+个演员

3. 特殊需求
   - 特效 (SFX)
   - 车辆/动物
   - 大量群演
   - 特殊设备

4. 所需时间估算
   - setup时间（布景/设备）
   - rehearsal时间（彩排）
   - 拍摄时间（实际拍摄）
   - 换场时间（转场/调整）

Step 2: 约束条件整理

1. 演员约束
   - 档期日期
   - 每天可用时间
   - 特殊休息需求
   - 连续工作限制（通常12小时）

2. 场地约束
   - 租赁天数
   - 宵禁时间
   - 噪音限制时间
   - 天气窗口

3. 技术约束
   - 设备可用性
   - 特效复杂度
   - 摄影机移动需求

4. 财务约束
   - 每天的预算上限
   - 加班费用
   - 特殊费用（日夜/节假日）

Step 3: 计划编排

编排优先级（按重要程度）:
1. 演员档期（最高优先级）
2. 场地租赁天数
3. 特殊场景需求（特效/天气）
4. 效率优化（batch拍摄）
5. 逻辑顺序（故事发展）
```

#### 时间估算标准

```
单镜头时间估算:

| 镜头类型 | Setup时间 | Rehearsal时间 | 拍摄时间 | 合计 |
|----------|----------|--------------|---------|------|
| 固定对话(简单) | 15-30min | 15-30min | 30-60min | 1-2小时 |
| 固定对话(复杂) | 30-60min | 30-60min | 60-120min | 2-4小时 |
| 简单运动 | 30-45min | 30-45min | 60-90min | 2-3小时 |
| 复杂运动/动作 | 60-120min | 60-120min | 120-240min | 4-8小时 |
| 特效镜头 | 60-180min | 30-60min | 60-180min | 3-8小时 |

特殊场景时间加成:
- 群演场景: +50%时间
- 车辆场景: +30%时间
- 动物/儿童: +50%时间
- 恶劣天气: +100%时间
```

### 第二部分：现场指挥系统

#### 拍摄日流程

```
标准拍摄日流程:

06:00 - 剧组到达
├── 各部门setup
├── 早餐
└── 晨会

08:00 - 第一摄制准备
├── 演员就位
├── 技术彩排
└── 最终确认

08:30 - 开始拍摄
├── 第一次拍摄 (Take 1)
├── 导演检查回放
├── 调整后继续拍摄
└── 继续直到导演满意

12:00 - 午餐休息

13:00 - 下午拍摄
├── 继续上午内容或转场
└── 持续拍摄

18:00 - 最后通告
├── 告知明日计划
└── 确认明日时间

18:30 - 收工
├── 设备归还
├── 场景交接
└── 日报提交
```

#### 1st AD的五大命令

```
现场指挥的核心命令系统:

1. "Ready to Roll" / "Turning"
   含义: 所有人都准备好了，可以开拍
   用途: 确认所有部门完成准备

2. "Shooting" / "Rolling"
   含义: 正在拍摄
   用途: 所有人都要安静、保持

3. "Cut"
   含义: 停止拍摄
   用途: 任何紧急情况都可以喊Cut

4. "Moving On"
   含义: 当前镜头完成，转到下一个
   用途: 开始收拾准备下一个镜头

5. "Take Five" / "Five Minutes"
   含义: 短暂休息
   用途: 部门调整或等待
```

#### 部门协调清单

```
1st AD在每个镜头前必须确认的部门:

□ 制片组
  - 主演是否就位？
  - 群演是否就位？

□ 摄影组
  - 摄影机是否准备好？
  - 焦点是否确认？
  - 移动轨道是否就位？

□ 灯光组
  - 灯光是否完成？
  - 反光板是否就位？

□ 声音组
  - 话筒位置是否确认？
  - 录音是否开始？

□ 美术组
  - 道具是否就位？
  - 场景是否完整？

□ 服装/化妆
  - 演员妆容/服装是否完整？
  - 是否有换装需求？

□ 导演组
  - 导演是否准备好？
  - 演员是否彩排过？
```

### 第三部分：进度管理

#### 进度追踪系统

```
每日进度追踪:

1. 晨会报告（Morning Report）
   - 今日计划: [X]个镜头
   - 预估完成: [X]个镜头
   - 风险评估: [高/中/低]

2. 午间检查（Midday Check）
   - 上午完成: [X]个镜头
   - 进度状态: [超前/正常/落后]
   - 下午调整: [如有需要]

3. 傍晚评估（Evening Assessment）
   - 全天完成: [X]个镜头
   - 是否需要加班: [是/否]
   - 明日调整: [如有需要]

4. 日报（Production Report）
   - 每日完成统计
   - 延误原因分析
   - 明日计划调整
```

#### 延误处理决策树

```
遇到延误时的决策框架:

延误 < 15分钟:
→ 吸收，不需要调整

延误 15-30分钟:
→ 检查后续镜头是否可以压缩
→ 是否需要跳过某些非关键镜头

延误 30-60分钟:
→ 与导演协商删减内容
→ 考虑是否值得加班

延误 > 60分钟:
→ 重大调整
→ 可能需要调整明日计划
→ 与制片主任协商预算影响

延误原因分类:
1. 技术问题（设备故障）
2. 人员问题（演员迟到）
3. 场景问题（setup超时）
4. 天气问题
5. 创意问题（导演反复）
```

### 第四部分：安全管理

#### 安全检查清单

```
1st AD必须确认的安全项:

□ 动作安全
  - 动作编排是否与特技协调
  - 保护措施是否到位
  - 演员/工作人员是否知情

□ 电气安全
  - 电线是否固定
  - 防水措施是否到位
  - 是否有裸露电线

□ 高空安全
  - 高空作业是否有保护
  - 是否有坠落风险
  - 是否有保护网

□ 消防安全
  - 火灾隐患是否排除
  - 灭火器是否就位
  - 紧急出口是否畅通

□ 爆破/特效安全
  - 爆破区域是否清场
  - 安全距离是否足够
  - 应急预案是否到位

□ 车辆安全
  - 驾驶场景是否有专业人员
  - 安全措施是否到位
  - 拍摄区域是否隔离
```

### 第五部分：AI视频生成的1st AD指南

#### 场景类型 → 拍摄策略

**AI生成时的1st AD提示词**:

| 场景类型 | 复杂度 | 预估时间 | AI提示词 |
|----------|--------|---------|---------|
| 简单对话 | 低 | 1-2小时/镜头 | simple dialogue setup, minimal crew, quick turnaround |
| 复杂对话 | 中 | 2-4小时/镜头 | multi-character dialogue, complex blocking, standard setup |
| 动作场景 | 高 | 4-8小时/镜头 | action sequence, stunt coordination, extended setup |
| 特效场景 | 极高 | 6-12小时/镜头 | VFX sequence, safety coordination, technical rehearsal |
| 群演场景 | 中高 | 3-6小时/镜头 | background actors, crowd management, staged movement |

## 决策逻辑

### 拍摄计划决策框架

```
制定拍摄计划时的决策顺序:

Step 1: 确定硬约束
├── 演员档期（不可改变）
├── 场地租赁日期
└── 预算上限

Step 2: 确定软约束
├── 效率最优（batch拍摄）
├── 天气窗口
└── 逻辑顺序

Step 3: 分配场景到可用天数
├── 优先：演员档期内的场景
├── 其次：场地租赁期间的场景
└── 最后：填充场景

Step 4: 顺序优化
├── 同一场景镜头集中
├── 同一演员镜头集中
└── 同一地点镜头集中

Step 5: 风险评估
├── 识别高风险场景
├── 准备备选方案
└── 预留应急时间
```
