## 技能概述

**对应好莱坞后期制作**: 混录师是整个声音制作的"总指挥"——把所有声音元素（对白、拟音、环境音、音效、音乐）混合成一个有机的整体。是声音设计的最终守门人，决定观众最终听到什么。
**核心职责**: 最终混音、声道平衡、动态处理、空间塑造、格式输出

> **核心理念**: 混音不是"把声音调大调小"，而是"用声音画画"。每一个推子的位置、每一个参数的调整，都是在塑造观众的情绪体验。混录师是观众的"声音导演"。

## 核心能力

1. **混音架构**: 理解对白/音效/音乐三层结构，掌握层间平衡
2. **动态处理**: 压缩、限制、扩展，用动态控制情绪张力
3. **空间塑造**: 用混响、延迟、立体声定位塑造声音空间
4. **格式精通**: 5.1/7.1/ATMOS/Dolby Vision等多种输出格式

### 第一部分：混音架构——三层声音结构

#### 三层声音模型（电影工业标准）

```
电影声音三层结构:
┌─────────────────────────────────────────────────────────────────┐
│                         混音金字塔                               │
│                                                                  │
│                            ┌───────┐                            │
│                           │ 音乐层 │  ← Top: 音乐（MUSIC）       │
│                          ┌────────────┐                          │
│                         │   音效层   │ ← Middle: 音效（SFX）     │
│                        ┌────────────────┐                       │
│                       │     对白层      │ ← Base: 对白（DX）     │
│                      └────────────────────┘                       │
└─────────────────────────────────────────────────────────────────┘

核心原则:
- 对白层: 必须清晰，是叙事核心，永远不能被其他层"吃掉"
- 音效层: 支持叙事，创造真实感，是画面的声音延伸
- 音乐层: 塑造情绪，为场景增加情感维度
```

#### 层间平衡原则

**对白 vs 音乐平衡**:

| 场景类型 | 对白优先策略 | 音乐处理 | AI混音提示词 |
|----------|------------|---------|-------------|
| 对白为主 | 对白在前方，清晰突出 | 音乐后退，作为背景 | dialogue-forward, music recedes, supportive bed |
| 情感高潮 | 对白与音乐同等重要 | 音乐与对白并行 | music and dialogue equal, emotional peak |
| 音乐驱动 | 音乐是主角，对白作为音效 | 对白低沉、破碎 | music-driven, dialogue as texture, atmospheric |
| 沉默场景 | 几乎没有音乐 | 极简或无音乐 | sparse music, silence as design, tension |

**音效 vs 对白平衡**:

| 场景类型 | 对白策略 | 音效策略 | AI混音提示词 |
|----------|---------|---------|-------------|
| 安静对话 | 对白突出 | 极轻环境音 | quiet dialogue, minimal sound, intimate |
| 动作场景 | 对白层压缩增加可懂度 | 音效强烈但不对白覆盖 | dialogue cutting through, action sounds, clear speech |
| 紧张场景 | 对白微混响增加临场感 | 环境音增加压迫感 | tense atmosphere, dialogue with room, environmental pressure |

#### 混音音量单位与标准

**响度标准（Loudness Standards）**:

| 标准 | 应用 | 目标响度 | 单位 |
|------|------|---------|------|
| ATSC A/85 (北美TV) | 电视广播 | -24 LKFS | Loudness K-weighted, relative to Full Scale |
| EBU R128 (欧洲) | 欧洲广播 | -23 LUFS | Loudness Units relative to Full Scale |
| ACES/A6 | 电影后期 | -27 LKFS（混音）/-30 LKFS（影院） | 专业影院标准 |
| Streaming平台 | Netflix/Disney+ | -14 LUFS（综合）/-1 dB True Peak | 流媒体平台 |

**AI生成时的响度提示词**:
```
- broadcast standard loudness, -24 LKFS, ATSC A/85 compliant
- streaming optimized, -14 LUFS integrated, -1 dBTP max
- theatrical mix standard, -27 LKFS, cinema reference
```

### 第二部分：动态处理——声音的呼吸

#### 动态范围与电影情绪

```
动态范围定义:
动态范围 = 最大声与最小声的差值（dB）

电影动态的黄金法则:
- 安静场景: 动态范围大（小声很小，大声中等）= 亲密感
- 动作场景: 动态范围中等（小声被抬起，大声音量大）= 压迫感
- 惊悚场景: 动态范围极端（极端安静到突然巨响）= 震惊感
```

**动态范围与情绪映射**:

| 场景类型 | 动态范围 | 特点 | AI动态处理提示词 |
|----------|---------|------|----------------|
| 亲密对话 | 15-20dB | 变化柔和，无极端大声 | narrow dynamic range, intimate, consistent level |
| 日常叙事 | 20-25dB | 自然变化 | natural dynamics, moderate range |
| 悬疑紧张 | 25-30dB | 安静段很安静，突然爆发 | wide dynamics, quiet-to-loud contrast, suspense |
| 动作爆炸 | 30+dB | 极端爆发，瞬态强烈 | extreme dynamics, impact, transient |
| 音乐场景 | 跟随音乐 | 音乐决定动态 | music-driven dynamics, cinematic impact |

#### 动态处理器件

**压缩器 (Compressor)**:

| 参数 | 作用 | 典型设置 | AI提示词 |
|------|------|---------|---------|
| Threshold | 超过这个电平开始压缩 | 对白: -20dB, 音乐: -15dB | gentle compression, transparent |
| Ratio | 压缩比例 | 对白: 2:1-4:1, 音乐: 4:1-8:1 | moderate ratio, natural feel |
| Attack | 启动时间 | 快: <10ms（控制瞬态），慢: >50ms（保持自然） | fast attack, punchy / slow attack, smooth |
| Release | 释放时间 | 快: <100ms（紧密），慢: >200ms（平滑） | quick release, tight / slow release, natural |
| Knee | 软硬拐点 | 软: 音乐用，硬: 特殊效果 | soft knee, musical / hard knee, aggressive |

**限制器 (Limiter)**:

| 用途 | 目标 | AI提示词 |
|------|------|---------|
| 峰值控制 | 防止削波，限制在0dB以下 | transparent limiting, no clipping |
| 响度最大化 | 提升整体响度而不削波 | maximized loudness, brick wall limiting |
| 特殊效果 | 创造"压实"的声音 | compressed sound, pumping |

**扩展器/门限 (Expander/Gate)**:

| 用途 | 设置 | AI提示词 |
|------|------|---------|
| 降噪 | 扩展低信号，降低底噪 | noise gate, clean signal, floor noise reduction |
| 增加动态 | 扩大动态范围 | expander, expanded dynamics |
| 特殊效果 | 创造"跳动"的声音 | gated reverb, rhythmic expansion |

### 第三部分：空间塑造——声音的维度

#### 混响参数与空间感知

```
混响的本质:
混响 = 声音在空间中的"尾巴"
混响 = 直达声 + 早期反射 + 后期混响尾

混响五参数:
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  Predelay (预延迟)                                           │
│  └──────→  │                                                │
│            ↓                                                 │
│  Early Reflections (早期反射) ←──────┐                       │
│         │                            │                       │
│         ↓                            │                       │
│  Late Reverb (后期混响尾) ──────────┘                       │
│                                                              │
│  Decay Time (衰减时间) ← 声音从大声到无声的时间             │
│  Diffusion (漫射) ← 混响的"密度"                            │
│  Damping (阻尼) ← 高频衰减速度                              │
└──────────────────────────────────────────────────────────────┘
```

**混响类型与适用场景**:

| 混响类型 | 特征 | 适用场景 | AI混响提示词 |
|----------|------|---------|-------------|
| Hall (大厅) | 自然、空间大、衰减长 | 交响乐、大场景、史诗 | large hall reverb, natural space, orchestral |
| Room (房间) | 小空间感、自然 | 室内对话、日常场景 | small room reverb, intimate, natural |
| Chamber (录音室) | 干净、有染色 | 独奏、人声 | chamber reverb, clean, vocal |
| Plate (板式) | 干净、紧凑、金属感 | 人声、打击乐 | plate reverb, clean, metallic edge |
| Spring (弹簧) | 有特色、复古、弹跳感 | 复古电话、特殊效果 | spring reverb, vintage, boing |
| Algorithmic (算法) | 可调、干净、现代 | 任何需要精确控制的场景 | clean algorithmic reverb, precise control |
| Convolution (卷积) | 真实空间采样 | 需要真实空间感的场景 | convolution reverb, real space, IR-based |

#### 立体声与环绕声定位

**立体声声道定位**:

```
声像定位 (Stereo Panning):

左极端 ◄──────────────────────────────────────────► 右极端
  -63（几乎左）         C         +63（几乎右）
  
立体声定位原则:
- 对白: 中央（0）
- 音乐: 自然展开（左乐部分右乐部）
- 音效: 根据画面位置定位
- 环境音: 均匀分布在后方
```

**5.1环绕声配置**:

| 声道 | 缩写 | 位置 | 典型用途 |
|------|------|------|---------|
| 中央 | C | 正前方 | 对白（永远在中央） |
| 左前置 | L | 前左45° | 音乐主要部分、左侧音效 |
| 右前置 | R | 前右45° | 音乐主要部分、右侧音效 |
| 左环绕 | Ls | 侧后方 | 环境音、氛围 |
| 右环绕 | Rs | 侧后方 | 环境音、氛围 |
| 低音炮 | LFE | 前方（低频） | 低频效果音乐、爆炸 |

**AI环绕声提示词**:
```
- 5.1 surround mix, dialogue in center channel
- immersive sound, LFE channel active for impact
- surround atmosphere, environmental bed in surrounds
- cinematic panning, smooth movement across channels
```

### 第四部分：混音流程与决策框架

#### 专业混音工作流程

```
标准混音流程（Phase by Phase）:

Phase 1: 准备
├── 备份所有原始素材
├── 整理工程文件结构
├── 设置正确的电平基准
└── 熟悉所有素材（听一遍所有轨道）

Phase 2: 基础混音（Basic Mix）
├── 设置所有轨道初始电平
├── 设置声像定位（Pan）
├── 基础EQ（清除频率冲突）
├── 建立初步的对白/音效/音乐平衡
└── 建立混音的"框架"

Phase 3: 动态处理（Dynamic Control）
├── 对白层：确保可懂度，必要时压缩
├── 音效层：增强冲击力，控制瞬态
├── 音乐层：配合对白，平衡情绪
└── 全局：确保整体动态合适

Phase 4: 空间塑造（Spatial Design）
├── 添加混响（按场景需求）
├── 添加延迟（如需要）
├── 调整声像定位
└── 塑造声音的"深度"

Phase 5: 自动化（Automation）
├── 写入关键位置的电平变化
├── 写入声像变化（如需要）
├── 写入效果参数变化
└── 确保混音与画面同步

Phase 6: 检查与修正（QC）
├── 对白可懂度检查
├── 响度标准检查
├── 与参考版本对比
└── 修正任何问题

Phase 7: 输出（Delivery）
├── 导出各格式版本
├── M&E（国际版本音乐分离）
├── 5.1/ATMOS/Dolby等格式
└── 整理工程文件备份
```

#### 混音问题诊断

```
常见混音问题与解决方案:

问题1: 对白被音乐"吃掉"
├── 原因: 音乐电平太高/频率冲突
├── 解决: 
│   - Ducking（闪避）: 音乐在对白时自动降低
│   - EQ处理: 让音乐避开人声频率（2-4kHz）
│   - 音乐后退: 降低音乐电平
└── AI提示词: dialogue ducking music, music sidechain, speech clarity

问题2: 音效太"干"
├── 原因: 缺少空间感
├── 解决:
│   - 添加混响
│   - 添加环境音
│   - 增加反射声
└── AI提示词: add reverb, spatial enhancement, room tone

问题3: 动态范围过大（听疲劳）
├── 原因: 瞬态太强，响度变化太大
├── 解决:
│   - 压缩器控制瞬态
│   - 限制器控制峰值
│   - 降低整体动态
└── AI提示词: control dynamics, reduce peak, comfortable listening level

问题4: 低频"浑浊"
├── 原因: 低频过多，与其他元素冲突
├── 解决:
│   - 高通滤波非低频元素
│   - 低频通道控制
│   - 侧链低频压缩
└── AI提示词: clean low-end, frequency separation, clear bass
```

### 第五部分：AI视频生成的混音指南

#### 场景类型 → 混音策略

**室内安静对话场景**:

```
混音策略:
- 对白: 干净、靠前、轻微房间感
- 音乐: 轻柔背景，不干扰对白
- 音效: 极简，主要是房间环境
- 动态: 狭窄，音量变化小

AI混音提示词:
"intimate dialogue mix, clean speech, minimal music bed,
quiet room ambience, narrow dynamics, professional broadcast level,
-24 LKFS, dialogue-forward, natural room reverb"
```

**动作场景**:

```
混音策略:
- 对白: 压缩增加可懂度，Ducking处理
- 音乐: 强烈、节奏驱动、激励动作
- 音效: 强烈、冲击力、与画面同步
- 动态: 宽，有明显瞬态

AI混音提示词:
"action sequence mix, impactful sound effects, driving music bed,
dialogue cutting through explosions, wide dynamics,
cinematic impact, LFE channel active, theatrical level"
```

**悬疑惊悚场景**:

```
混音策略:
- 对白: 压抑感、轻微失真、增加紧张
- 音乐: 极简、长期渐变、音调不协和
- 音效: 细节音（心跳、呼吸、时钟）、寂静中的暗示
- 动态: 极端（极静到极响）

AI混音提示词:
"suspenseful mix, minimal music, tension through silence,
heartbeat and breathing, extreme dynamic range,
quiet-to-loud contrast, psychological horror sound design,
subliminal audio cues"
```

**浪漫场景**:

```
混音策略:
- 对白: 温暖、亲密、轻微混响
- 音乐: 突出、浪漫、弦乐为主
- 音效: 极少，主要是环境音
- 动态: 平滑、优美、变化细腻

AI混音提示词:
"romantic scene mix, lush orchestral music, intimate dialogue,
warm tone, smooth dynamics, gentle reverb, emotional depth,
music and dialogue in harmony, lyrical"
```

## 决策逻辑

### 混音决策树

```
接到混音任务：
  │
  ├── Step 1: 确定场景类型
  │     ├── 对白驱动（对话为主）
  │     ├── 动作驱动（音效为主）
  │     ├── 情绪驱动（音乐为主）
  │     └── 氛围驱动（环境音为主）
  │
  ├── Step 2: 确定混音优先级
  │     ├── 第一：对白可懂度
  │     ├── 第二：情绪/节奏
  │     └── 第三：空间真实感
  │
  ├── Step 3: 设计动态策略
  │     ├── 窄动态（亲密/安静）
  │     ├── 宽动态（悬疑/动作）
  │     └── 音乐驱动（跟随音乐）
  │
  ├── Step 4: 选择空间策略
  │     ├── 近距离（干声、亲密）
  │     ├── 中距离（自然）
  │     └── 远距离（空间感强）
  │
  └── Step 5: 设定响度标准
        ├── 广播TV: -24 LKFS
        ├── 流媒体: -14 LUFS
        └── 影院: -27 LKFS
```
