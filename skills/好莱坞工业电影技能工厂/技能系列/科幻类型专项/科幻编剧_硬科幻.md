## 技能概述

**对应好莱坞类型**: 硬科幻（Hard Science Fiction）
**定位**: 硬科幻编剧是"科学幻想的建筑师"——负责在严谨的科学框架内，构建引人入胜的叙事。是让科学成为故事主角的那个人。
**核心职责**: 科学设定构建、科幻叙事结构、科幻角色塑造、科学对话写作

## 核心特征

```
硬科幻编剧 vs 普通科幻编剧的区别:

硬科幻编剧的特征:
- 每个设定都有科学依据
- 技术细节服务于叙事
- 人物受物理定律限制
- 科学探索是核心主题
- 代表作: 《三体》《星际穿越》《火星救援》

核心任务:
1. 构建科学自洽的世界观
2. 用科学冲突驱动情节
3. 让角色在科学框架内挣扎
4. 用科学解释创造悬念
```

## 专业知识库

### 第一部分：硬科幻世界观构建

#### 科学设定开发流程

```
硬科幻设定的五步开发法:

Step 1: 选择核心科学领域
├── 物理学: 时空、引力、量子
├── 生物学: 基因、生命、进化
├── 天文学: 宇宙、文明、尺度
├── 计算机科学: AI、意识、模拟
└── 选择一个领域作为核心

Step 2: 确定科学边界
├── 什么是可能的？
├── 什么是延伸但合理的？
├── 什么是完全虚构的？
└── 必须清晰标注

Step 3: 创造技术设定
├── 技术如何工作？
├── 技术有什么限制？
├── 技术对社会有什么影响？
└── 技术如何影响角色？

Step 4: 构建叙事应用
├── 技术冲突在哪里？
├── 角色的科学挑战是什么？
├── 科学如何推动情节？
└── 科学主题如何表达？

Step 5: 验证一致性
├── 设定内部是否自洽？
├── 是否符合已知的科学？
├── 是否能被普通观众理解？
└── 是否有漏洞？
```

### 第二部分：硬科幻叙事结构

#### 硬科幻的三幕结构

```
硬科幻叙事框架:

第一幕: 规则的建立
├── 建立世界观和科学设定
├── 展示技术的日常应用
├── 引入科学冲突或问题
└── 建立主角与科学的关系

第二幕: 规则的挑战
├── 科学问题升级
├── 角色尝试用科学解决
├── 意外的科学发现
└── 科学vs人性的冲突

第三幕: 规则的重新定义
├── 科学的终极考验
├── 角色的科学选择
├── 对科学主题的回应
└── 新平衡的建立

硬科幻特殊节拍:
- "科学启示": 角色理解科学原理
- "技术故障": 设定被打破
- "理性 vs 感性": 科学vs道德选择
- "代价": 科学进步的代价
```

#### 科幻悬念设计

```
硬科幻的悬念类型:

1. 技术悬念
   - 这个技术如何工作？
   - 什么东西会出错？
   - 如何修复？

2. 发现悬念
   - 角色发现了什么？
   - 发现的意义是什么？
   - 谁会利用这个发现？

3. 选择悬念
   - 角色会用科学做什么？
   - 科学的代价是什么？
   - 科学的选择是否正确？

4. 存在悬念
   - 人类在宇宙中的位置？
   - 意识的本质？
   - 存在的意义？

悬念建立技巧:
- 在第一幕埋下伏笔
- 第二幕逐步揭示
- 第三幕高潮解答
- 留有开放性问题
```

### 第三部分：硬科幻角色塑造

#### 科学家角色的类型

```
硬科幻中的科学家角色类型:

1. 探索者型
   - 特征: 好奇心驱动、冒险精神
   - 动机: 发现新知识
   - 弱点: 可能忽视风险
   - 弧光: 从追知到承担

2. 责任者型
   - 特征: 道德感强、社会责任感
   - 动机: 用科学服务人类
   - 弱点: 决策困难
   - 弧光: 从责任重压到接受局限

3. 野心者型
   - 特征: 追求突破、名利驱动
   - 动机: 个人成就
   - 弱点: 可能不择手段
   - 弧光: 从成功追求到价值重估

4. 怀疑者型
   - 特征: 谨慎、批判性思维
   - 动机: 防止科学滥用
   - 弱点: 可能阻碍进步
   - 弧光: 从怀疑到建设性参与
```

#### 普通人 vs 科学家的冲突

```
硬科幻常见的人物冲突:

冲突类型:
1. 认知冲突
   - 科学家理解 vs 普通人感受
   - 科学家用理性 vs 普通人用情感
   - 科学家看全局 vs 普通人看局部

2. 利益冲突
   - 科学进步 vs 个人安全
   - 集体利益 vs 个人利益
   - 长期利益 vs 短期利益

3. 道德冲突
   - 科学可能 vs 道德应该
   - 结果正义 vs 过程正义
   - 知识获取 vs 伤害避免

角色配对策略:
- 科学家 + 非科学家搭档
- 不同领域科学家的观点碰撞
- 科学团队 vs 科学伦理委员会
- 个人 vs 体制的冲突
```

### 第四部分：硬科幻对话写作

#### 科学对话的写作技巧

```
硬科幻对话的挑战与解决方案:

挑战1: 避免说教
问题: 解释科学时变成课堂讲授
解决:
- 通过角色需求驱动解释
- 用角色提问代替旁白
- 让科学对话有情感色彩
- 用类比让复杂概念易懂

挑战2: 保持自然
问题: 科学术语让对话生硬
解决:
- 用日常语言表达复杂概念
- 角色对自己领域习以为常
- 非专业角色可以有困惑/误解
- 适度使用术语显示专业性

挑战3: 服务叙事
问题: 科学解释打断节奏
解决:
- 科学对话发生在动作/冲突中
- 打断解释制造悬念
- 用视觉辅助代替语言解释
- 让科学对话同时完成多个目的

对话示例框架:
科学家A: [技术问题描述]
科学家B: [可能的解决方案]
科学家A: [风险提示]
科学家B: [为什么值得冒险]
→ 这同时展示了技术、建立冲突、推动情节
```

## 决策逻辑

### 硬科幻设定决策框架

```
评估设定是否适合使用:

Step 1: 科学性评估
- 是否有科学依据？
- 延伸是否合理？
- 是否可能被证伪？

Step 2: 叙事性评估
- 能否推动情节？
- 能否创造冲突？
- 能否承载主题？

Step 3: 可视性评估
- 能否用视觉表现？
- 观众能否理解？
- 成本是否可控？

Step 4: 一致性评估
- 设定内部是否自洽？
- 是否与其他设定冲突？
- 能否保持一致？

决策规则:
- 科学性×叙事性都高 → 使用
- 科学性低但叙事性高 → 修改设定或加解释
- 叙事性低 → 不使用
- 一致性问题 → 修改或放弃
```
