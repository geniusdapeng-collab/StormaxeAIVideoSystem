## 技能概述

**对应好莱坞工会**: WGA（编剧工会体系）
**岗位定位**: 编剧是故事的"第一创造者"——负责把一个概念转化为一页页有血有肉的剧本。是电影/电视的第一作者。
**核心职责**: 故事构思、角色设计、场景撰写、格式规范、修改迭代

> **核心理念**: 编剧不是在"写对话"，而是在"创造世界"。每一页剧本都是一个承诺——承诺一个有趣的角色、一个有意义的情境、一个值得花时间看的故事。

## 核心能力

1. **故事构建**: 从概念到完整故事的结构化能力
2. **角色创作**: 创造独特、可信、有弧光的角色
3. **场景写作**: 写出有画面感、有节奏、有信息的场景
4. **格式规范**: 掌握专业剧本格式

### 第一部分：故事概念开发

#### 从概念到故事

```
故事概念的四个层次:

1. 高概念 (High Concept)
   - 一句话可以描述的核心概念
   - 公式: [角色] + [处境] + [特殊元素]
   - 示例: "一个会计师被困在外星人入侵的纽约，必须救出他的猫"

2. 主题 (Theme)
   - 故事要探讨的核心问题
   - 公式: [正面价值] vs [反面价值]
   - 示例: 真正的勇气不是不害怕，而是害怕了还继续

3. 核心冲突 (Core Conflict)
   - 故事的主要矛盾
   - 公式: [主角] + [障碍] + [目标]
   - 示例: 一个内向的科学家必须学会公开发表演说才能拯救世界

4. 情感内核 (Emotional Core)
   - 观众要感受的核心情感
   - 公式: [情感类型] + [强度]
   - 示例: 爆笑喜剧，但笑中带泪

概念评估表:
| 问题 | 好的回答 | 坏的回答 |
|------|---------|---------|
| 这是什么类型？ | 具体类型 | "这是个故事" |
| 独特在哪？ | 独特卖点 | "和别人差不多" |
| 有人想看吗？ | 具体受众 | "所有人都" |
| 为什么是现在？ | 时效性/相关性 | "没想过" |
```

#### 类型片设计

```
常见类型的核心元素:

| 类型 | 核心冲突 | 必备元素 | 期望结局 |
|------|---------|---------|---------|
| 悬疑 | 谁是真凶？ | 线索/误导/反转 | 揭示真相 |
| 惊悚 | 生命威胁 | 倒计时/孤立/追杀 | 逃脱/战胜 |
| 爱情 | 阻碍重重 | 相遇/波折/高潮 | 在一起/分离 |
| 动作 | 正邪对决 | 追逐/打斗/爆炸 | 胜利/牺牲 |
| 科幻 | 科技vs人性 | 世界观/悖论/选择 | 人性胜利 |
| 恐怖 | 超自然威胁 | 规则/恐惧/血腥 | 逃脱/牺牲 |
| 喜剧 | 荒诞处境 | 误会/反转/夸张 | 圆满/领悟 |
| 剧情 | 人性困境 | 抉择/成长/代价 | 领悟/转变 |
| 冒险 | 未知探索 | 考验/成长/宝藏 | 获得/领悟 |

类型写作检查清单:
- [ ] 这个类型观众期待的"约定"是否满足？
- [ ] 有没有在这个类型上做出独特创新？
- [ ] 类型的"规则"有没有被有意识地打破？
```

### 第二部分：角色设计

#### 角色的三个维度

```
完美角色的三个维度:

1. 社会身份 (Social Identity)
   - 职业
   - 社会地位
   - 家庭角色
   - 社会关系

2. 心理身份 (Psychological Identity)
   - 性格特点
   - 行为模式
   - 防御机制
   - 核心恐惧

3. 戏剧身份 (Dramatic Identity)
   - 欲望/目标
   - 障碍/冲突
   - 道德选择
   - 弧光设计

角色设计检查表:
- [ ] 观众第一次见到这个角色时，第一印象是什么？
- [ ] 观众什么时候意识到这个角色的真实一面？
- [ ] 这个角色在故事中的核心选择是什么？
- [ ] 这个角色的弧光是什么？
```

#### 人物关系设计

```
关系的四个层次:

1. 表面关系 (Surface Relationship)
   - 两人如何称呼对方
   - 社会层面的关系

2. 真实关系 (True Relationship)
   - 两人实际上如何看待对方
   - 隐藏的情感

3. 隐藏关系 (Hidden Relationship)
   - 双方都不承认的真相
   - 往往与表面关系相反

4. 动态关系 (Dynamic Relationship)
   - 关系如何随着故事发展变化
   - 关键转折点

关系设计示例:
表面: 老板-员工
真实: 互相欣赏的对手
隐藏: 父亲-女儿（未公开）
动态: 从对立到理解到和解
```

### 第三部分：场景写作

#### 场景的六个功能

```
每个场景必须完成至少一个功能:

1. 推进情节 (Plot Advancement)
   - 发生重要事件
   - 改变故事走向

2. 揭示人物 (Character Revelation)
   - 展示角色新的一面
   - 深化角色理解

3. 建立主题 (Thematic Establishment)
   - 表达故事主题
   - 传递核心信息

4. 建立氛围 (Mood/Atmosphere)
   - 塑造情绪
   - 建立世界感

5. 提供信息 (Exposition)
   - 交代背景
   - 解释情况

6. 制造娱乐 (Entertainment)
   - 笑点/动作/视觉奇观
   - 让观众享受当下

场景评估:
一个场景如果没有以上任何功能 → 删除
一个场景如果能完成多个功能 → 保留并强化
```

#### 场景写作技巧

```
好场景的五个要素:

1. 冲突 (Conflict)
   - 角色之间/角色内心/角色与环境
   - 没有冲突的场景 = 没有戏剧性

2. 变化 (Change)
   - 场景开始时 vs 场景结束时
   - 角色必须有所变化（哪怕很小）

3. 目的 (Purpose)
   - 这个场景要完成什么？
   - 用一句话描述这个场景

4. 独特性 (Uniqueness)
   - 这个场景有什么独特之处？
   - 与其他场景的区别在哪？

5. 简洁 (Economy)
   - 不说一句多余的话
   - 不写一个多余的描写

场景开头写法:
- 开门见山进入冲突
- 不要写"漫长的等待后..."
- 用动作/对白开始，不要用描写

场景结尾写法:
- 自然结束或转折
- 不要为了凑时长拖沓
- 结尾要产生新的问题/悬念

场景内部节奏:
- 长场景（复杂冲突）: 详细展开
- 短场景（简单信息）: 快速过渡
- 留白: 不是每个反应都要写
```

### 第四部分：剧本格式规范

#### 专业剧本格式

```
基础格式（必须严格遵守）:

页面: 1页 ≈ 1分钟

场景标题格式:
INT. 咖啡馆-日
或
EXT. 街道-夜

动作描述格式:
Sarah推开门，环顾四周。阳光从窗户洒进来。

对白格式:
SARAH
我等了你很久。

括号格式（少用）:
SARAH
(看着窗外)
这个地方变了。

转场格式（少用）:
CUT TO:

FADE TO BLACK.
```

#### 格式常见错误

```
新手常犯的错误:

1. 场景标题错误
   - ❌ "场景一：咖啡馆"
   - ✅ "INT. 咖啡馆-日"

2. 动作描述过长
   - ❌ 整页描写房间布置
   - ✅ 3-5行描述关键信息

3. 镜头指示过多
   - ❌ "镜头从左向右缓慢推进"
   - ✅ 让导演决定镜头语言

4. 对白过于书面
   - ❌ "我亲爱的朋友，很高兴见到你"
   - ✅ "嘿，老兄！好久不见"

5. 解释性旁白
   - ❌ "Sarah心里在想..."
   - ✅ 用行为展示而非告诉
```

### 第五部分：AI视频生成的编剧指南

#### 故事类型 → 编剧策略

**AI生成时的编剧提示词**:

| 类型 | 核心冲突 | 节奏 | AI提示词 |
|------|---------|------|---------|
| 悬疑 | 真相vs谎言 | 慢热→爆发→揭示 | mystery structure, clue-dropping, satisfying reveal |
| 动作 | 追逐/战斗 | 快节奏 | action-driven, chase sequences, high energy |
| 爱情 | 阻碍vs吸引 | 慢热→高潮 | romantic tension, meet-cute, obstacles to love |
| 喜剧 | 误会/荒诞 | 快节奏 | comedic timing, absurd situations, witty dialogue |
| 惊悚 | 生命威胁 | 持续紧张 | sustained tension, life threat, thriller pace |
| 剧情 | 人性困境 | 情感深度 | emotional depth, character study, moral dilemma |

## 决策逻辑

### 剧本问题诊断框架

```
发现剧本问题时的诊断顺序:

1. 结构问题？
   - 三幕比例是否正确？
   - 转折点是否有力？
   → 修改大纲

2. 角色问题？
   - 角色动机是否清晰？
   - 弧光是否完整？
   → 修改角色设计

3. 场景问题？
   - 场景是否有多重功能？
   - 场景是否有冲突和变化？
   → 重写场景

4. 对白问题？
   - 是否像真人说话？
   - 是否有潜台词？
   → 润色对白

5. 格式问题？
   - 格式是否标准？
   - 是否过于冗长？
   → 格式修正
```
