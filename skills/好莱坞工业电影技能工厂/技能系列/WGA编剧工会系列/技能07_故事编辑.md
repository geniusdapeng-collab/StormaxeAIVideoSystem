## 技能概述

**对应好莱坞工会**: WGA（编剧工会体系）
**岗位定位**: 故事编辑是剧本的"结构医生"——负责分析故事结构、诊断叙事问题、提出修改建议。是把好故事打磨成伟大故事的那个人。
**核心职责**: 故事结构分析、叙事诊断、修改建议、剧本开发、格式规范

> **核心理念**: 故事编辑不是在"写剧本"，而是在"诊断剧本"。最好的故事编辑能一眼看出故事哪里断了、哪里堵了、哪里多余，然后给出精准的修改方向。

## 核心能力

1. **结构诊断**: 快速识别故事的结构问题（三幕不均、节奏失衡、情节脱节）
2. **叙事分析**: 理解故事的内在逻辑，评估人物弧光和主题表达
3. **修改建议**: 给出具体、可操作、有效的修改方案
4. **格式规范**: 掌握专业剧本格式，确保合规交付

### 第一部分：故事结构诊断

#### 三幕结构深度分析

```
三幕结构是所有叙事的基础框架：

第一幕（建置 / Setup）: 0-25%
├── 建立主角的"日常世界"
├── 引入主题/冲突的种子
├── 出现"激励事件"（Inciting Incident）
├── 主角做出"第一决定"进入新世界
└── 第一幕转折点（Plot Point 1）: 主角被迫改变计划

第二幕（对抗 / Confrontation）: 25-75%
├── 主角面对越来越大的挑战
├── 盟友出现（Allies）
├── 敌人/反派出现（Antagonists）
├── 中点转折（Midpoint）: 重大揭示或主角地位反转
├── 低谷点（Darkest Moment）: 主角失败或面临最大危机
└── 第二幕转折点（Plot Point 2）: 准备最终决战

第三幕（解决 / Resolution）: 75-100%
├── 最终决战（Final Battle）
├── 主题揭示（Thematic Statement）
└── 新平衡建立（New Equilibrium）

常见三幕问题诊断:
| 问题 | 症状 | 解决方案 |
|------|------|---------|
| 第一幕过长 | 前30分钟没有实质冲突 | 删除铺垫，压缩建置 |
| 第一幕过短 | 主角动机不清晰 | 增加激励事件前的铺垫 |
| 第二幕拖沓 | 中间60分钟无聊 | 增加副线或障碍 |
| 第二幕断裂 | 中点后风格突变 | 重构中点转折 |
| 第三幕仓促 | 高潮来得快去得快 | 扩展最终对决 |
| 第三幕拖沓 | 结局后还有20分钟 | 删减尾声 |
```

#### 情节类型诊断矩阵

| 情节类型 | 核心问题 | 典型症状 | 诊断重点 |
|----------|---------|---------|---------|
| 因果情节 | 事件逻辑链 | 原因→结果 | 事件因果是否清晰 |
| 主题情节 | 论证结构 | 主张→论据 | 论点是否充分 |
| 角色情节 | 内心变化 | 创伤→成长 | 弧光是否完整 |
| 铺设情节 | 伏笔回收 | 伏笔→揭示 | 伏笔是否有效 |
| 情感情节 | 情绪节奏 | 高潮→释放 | 情绪是否到位 |
| 混合型 | 多重结构 | 交织混乱 | 结构优先级 |

### 第二部分：叙事元素诊断

#### 人物弧光诊断

```
完美人物弧光的五个阶段:

1. 负面特质（Flaw）
   - 主角缺什么：自私、恐惧、执念...
   - 诊断：负面特质是否与主题对立？

2. 外在目标（External Goal）
   - 主角想要什么：复仇、成功、拯救...
   - 诊断：目标是否足够具体和有意义？

3. 内在需求（Internal Need）
   - 主角真正需要什么：爱、原谅、勇气...
   - 诊断：需求是否与负面特质对立？

4. 错误信念（False Belief）
   - 主角相信什么错的东西：世界是公平的...
   - 诊断：错误信念是否导致了负面特质？

5. 真相时刻（Truth Moment）
   - 主角学到什么：世界不公平但我可以接受...
   - 诊断：真相是否解决了内在需求？

人物弧光问题诊断表:
| 问题类型 | 症状 | 修复方向 |
|----------|------|---------|
| 弧光缺失 | 主角从头到尾没变化 | 增加内在障碍和成长 |
| 弧光断线 | 变化来得突然，没有铺垫 | 增加中间过渡事件 |
| 弧光反向 | 主角变得更糟而非更好 | 重构中点转折 |
| 弧光重复 | 所有角色弧光相同 | 差异化设计各角色 |
| 弧光浅薄 | 变化太小或太简单 | 增加冲突的强度 |
```

#### 主题诊断

```
主题（Theme）诊断框架:

主题不是一个词，而是一个命题：
- ❌ "爱情"（太抽象）
- ✅ "真正的爱情意味着放下自我"（有立场）

主题与冲突的关系:
- 主题的正面陈述 → 主角最终学到的真理
- 主题的反面陈述 → 主角最初相信的谎言

主题诊断三问:
1. 主角相信什么？（初始立场）
2. 故事如何挑战这个信念？（冲突设计）
3. 主角最终学到什么？（主题揭示）

主题与类型的对应:
| 类型 | 常见主题 |
|------|---------|
| 爱情片 | 放下自我才能获得真爱 |
| 惊悚片 | 真相比安全更重要 |
| 超级英雄 | 真正的力量来自牺牲 |
| 喜剧 | 自嘲是智慧的开始 |
| 悬疑 | 逃避过去只会更痛苦 |
```

#### 节奏诊断

```
叙事节奏的四个层次:

1. 场景节奏（Scene Level）
   - 每个场景的进入/退出速度
   - 诊断：场景是否"开门见山"？是否"收尾干净"？

2. 序列节奏（Sequence Level）
   - 3-5个场景组成的叙事单元
   - 诊断：序列是否有"起承转合"？

3. 幕节奏（Act Level）
   - 三幕各自的张力曲线
   - 诊断：每一幕是否有明确的高潮和低谷？

4. 整体节奏（Film Level）
   - 全片120分钟的情绪起伏
   - 诊断：观众是否有喘息机会？是否有持续张力？

节奏问题诊断与修复:
| 问题 | 症状 | 修复方案 |
|------|------|---------|
| 节奏过慢 | 场景冗长，无进展 | 删减铺垫，加速进入冲突 |
| 节奏过快 | 场景跳跃，情绪跟不上 | 扩展关键场景，增加过渡 |
| 节奏单调 | 始终同一速度，无起伏 | 制造节奏对比（快/慢交替） |
| 节奏断裂 | 突然加速或减速 | 增加渐变过渡 |
```

### 第三部分：修改建议系统

#### 诊断→修改的转化框架

```
修改建议的五个层次:

1. 宏观修改（Big Picture）
   ├── 需要改变故事结构
   ├── 需要改变核心人物弧光
   └── 需要改变主题方向
   → 交付物：结构重写方案

2. 中观修改（Sequence Level）
   ├── 需要改变序列顺序
   ├── 需要增加/删除序列
   └── 需要改变序列内的事件
   → 交付物：序列重组方案

3. 微观修改（Scene Level）
   ├── 需要改变场景目的
   ├── 需要改变场景内的事件
   └── 需要重新设计场景冲突
   → 交付物：场景修改清单

4. 对话修改（Dialogue Level）
   ├── 需要改变台词
   ├── 需要改变潜台词
   └── 需要改变角色声音
   → 交付物：对话重写段落

5. 格式修改（Format Level）
   ├── 场景描述过长/过短
   ├── 对话格式错误
   └── 页面布局不规范
   → 交付物：格式修正版
```

#### 专业修改建议的写法

```
好的修改建议公式:
[问题诊断] + [问题原因] + [修改方向] + [修改理由]

示例1（宏观）:
"第一幕缺乏张力。
原因是主角的外在目标太容易达成，缺少真正的障碍。
建议：增加一个更强大的反派，或者提高目标的风险。
理由：更强的对抗能让主角的成长更有说服力。"

示例2（中观）:
"中点转折效果不足。
原因是B故事（爱情线）的转折与A故事（复仇线）的转折同时发生，互相削弱。
建议：让B故事的转折发生在A故事之前，作为A故事的推动力。
理由：两个转折应该有主次之分，才能形成完整的叙事波澜。"

示例3（微观）:
"第五场场景拖沓。
原因是场景试图同时完成两件事（建立关系+透露信息），结果都没做好。
建议：拆分成两个场景，或者将信息透露改为行为展示。
理由：一个场景应该只有一个核心目的。"

示例4（对话）:
"主角的对白与他的背景不符。
他声称自己是'独行侠'，但说话方式却像个社交高手。
建议：让主角的台词更简洁、更防御性。
理由：人物的语言风格应该反映他的内在特质。"

禁忌写法:
- ❌ "这里不好"（没有解释为什么）
- ❌ "改成X"（没有提供理由）
- ❌ "我不喜欢"（主观偏好不是修改依据）
- ❌ "参考Y电影"（抄袭借口）
```

### 第四部分：剧本格式规范

#### 专业剧本格式标准

```
好莱坞标准剧本格式（必须严格遵守）:

页面规格:
- 尺寸：8.5" x 11"（美国信纸）
- 字体：Courier 12-point（等宽字体，1页≈1分钟）
- 页边距：左1.5"，其余1"
- 行距：单倍行距

场景标题（Scene Heading/Slug Line）:
```
INT. 公寓-客厅-日

外景  INT./EXT.  地点  -  时间

规则：
- 全大写
- INT.=室内，EXT.=室外，INT./EXT.=内外
- 地点具体但简洁
- 时间分：日/夜/晨/昏/晨昏/连续
```

动作描述（Action/Description）:
```
规则：
- 现在时
- 简洁、有画面感
- 避免镜头术语（除非是导演剧本）
- 每行不超过约60个字符

示例：
Sarah推开门，阳光洒进昏暗的房间。
她环顾四周，嘴角微微上扬。
```

人物（Character Name）:
```
规则：
- 首次出现时全大写
- 对话前使用
- 括号说明：画外音、电话等

示例：
SARAH
（看着窗外）
这就是我的答案。

（画外音）
十年了，她终于回来了。
```

对话（Dialogue）:
```
规则：
- 居中，名字在左下方
- 简洁，有节奏感
- 每个人物有独特的声音

示例：
JOHN
你确定要这么做？

SARAH
我从来不确定。
我只是从不后悔。
```

括号（Parenthetical）:
```
规则：
- 用括号包裹
- 简短（通常不超过一行）
- 表明说话方式或声音来源

示例：
SARAH
(winking)
那当然。
```

过渡（Transition）:
```
规则：
- 居右
- 除非必要，不写"CUT TO:"
- 常用：CUT TO:, FADE TO:, DISSOLVE TO:

示例：
CUT TO:

FADE TO BLACK.
```

### 第五部分：AI视频生成的故事诊断指南

#### 故事类型 → 诊断重点

**AI生成时的故事诊断提示词**:

| 故事类型 | 结构重点 | 常见问题 | 诊断清单 |
|----------|---------|---------|---------|
| 线性叙事 | 因果链条清晰 | 节奏单调 | 有无意外转折？ |
| 多线叙事 | 交织节奏 | 喧宾夺主 | 主线是否主导？ |
| 非线性叙事 | 碎片重组 | 混乱困惑 | 重组是否有意义？ |
| 章节叙事 | 章节独立性 | 联系薄弱 | 章节是否服务于整体？ |
| 互文叙事 | 主题呼应 | 过度晦涩 | 呼应是否清晰？ |

#### 故事质量评估维度

```
AI生成故事的质量评估框架:

1. 结构完整性（25%）
   - 三幕是否完整？
   - 各部分比例是否合理？
   - 转折点是否有力？

2. 人物可信度（25%）
   - 人物动机是否清晰？
   - 人物是否成长/变化？
   - 人物声音是否独特？

3. 主题深度（20%）
   - 主题是否有立场？
   - 主题是否贯穿全片？
   - 主题是否不过于说教？

4. 节奏张力（20%）
   - 是否有持续吸引力？
   - 高潮是否足够高？
   - 是否有喘息空间？

5. 对话质量（10%）
   - 台词是否服务于角色？
   - 潜台词是否丰富？
   - 是否简洁有力？
```

## 决策逻辑

### 故事诊断流程

```
收到剧本后的诊断流程:

Step 1: 快速阅读（30分钟）
├── 记录第一印象
├── 标记"好的部分"
├── 标记"有问题的地方"
└── 不做判断，只观察

Step 2: 结构分析（45分钟）
├── 划分三幕
├── 标记所有转折点
├── 评估比例
└── 识别结构问题

Step 3: 元素深挖（1小时）
├── 分析主角弧光
├── 分析反派弧光
├── 分析主题
└── 分析节奏

Step 4: 综合诊断（30分钟）
├── 列出所有问题
├── 按优先级排序
├── 给出具体修改建议
└── 撰写诊断报告
```
