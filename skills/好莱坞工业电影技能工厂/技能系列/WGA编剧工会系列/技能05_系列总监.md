## 技能概述

**对应好莱坞体系**: 系列总监是电视/流媒体剧集的最高创意负责人——既是编剧，又是制作人，有时也执导。是把一个剧集从概念变成观众能看到的完整作品的那个人。
**核心职责**: 整体创意控制、编剧室管理、季结构规划、跨部门协调、品质把控

> **核心理念**: Showrunner不是一个职位，而是一种能力——同时理解创意、商业、技术和团队管理。最好的Showrunner让每个参与者都觉得自己的想法被听到了，同时确保最终作品是统一的、有灵魂的。

## 核心能力

1. **叙事架构**: 设计跨越整季/整系列的宏大叙事结构
2. **团队领导**: 管理编剧室，带领团队高效产出
3. **品质把控**: 确保每一集都符合整体标准
4. **跨部门协调**: 与制片、导演、平台方的沟通桥梁

### 第一部分：季结构设计

#### 季结构的基本类型

```
美剧季结构的四种经典模式:

1. 连续剧模式 (Serialized)
   - 特点: 剧情连续，每集紧密相连
   - 代表: 《黑道家族》《绝命毒师》《风骚律师》
   - 优点: 深度沉浸，粉丝粘性高
   - 风险: 新观众入门难，对编剧要求高

2. 单元剧模式 (Procedural/Episodic)
   - 特点: 每集独立故事，角色成长连续
   - 代表: 《CSI》《海军罪案调查组》《法律与秩序》
   - 优点: 入门简单，重播价值高
   - 风险: 剧情深度有限

3. 混合模式 (Hybrid)
   - 特点: 每集独立+季主线并行
   - 代表: 《X档案》《越狱》《神盾局特工》
   - 优点: 兼顾深度和入门
   - 风险: 两条线容易失衡

4. 有限剧集模式 (Limited Series)
   - 特点: 预定集数，讲完完整故事
   - 代表: 《真探》《东城梦魇》《切碎，切碎》
   - 优点: 叙事完整，风险可控
   - 风险: 没有后续空间
```

#### 季内结构设计

```
标准10集季的结构模板:

集数分配（经典三幕结构应用于季）:

第一幕（建立）: 第1-3集
├── 第1集: 设定世界，建立主要人物，引出核心冲突
├── 第2集: 深入角色关系，展示A/B故事
└── 第3集: 第一个重大转折，第一幕结尾钩子

第二幕（对抗）: 第4-7集
├── 第4集: 冲突升级，主角面临更大挑战
├── 第5集: 中点揭示，重大反转或信息
├── 第6集: 后果处理，新问题出现
└── 第7集: 逼近高潮，低谷点或重大损失

第三幕（解决）: 第8-10集
├── 第8集: 最终决战准备，所有线索汇聚
├── 第9集: 高潮对决，核心冲突爆发
└── 第10集: 结局与展望，填坑+留钩子

季内节奏规划:
- 每2-3集应有一个小高潮
- 每集结尾应有悬念钩子
- 第4-5集通常有重大转折（Mid-Season）
- 第8集开始进入最终冲刺
```

#### 多季叙事设计

```
跨季叙事的三种模式:

1. 递进式 (Progressive)
   - 特点: 每季独立完整，但层层递进
   - 代表: 《心理神探》《亚特兰大》
   - 适合: 主题驱动型剧集

2. 延续式 (Continuing)
   - 特点: 主角弧光跨越多季
   - 代表: 《绝命毒师》《权力的游戏》
   - 适合: 角色驱动型剧集

3. 进化式 (Evolving)
   - 特点: 每季有独立主题，但世界相连
   - 代表: 《黑镜》《阴阳魔界》
   - 适合: 概念驱动型剧集

跨季伏笔系统:
- 第一季埋下2-3个核心伏笔
- 第二季回收1-2个，埋下新的
- 第三季回收剩余+新伏笔
- 确保每季有"啊哈时刻"（Aha Moment）
```

### 第二部分：编剧室管理

#### 编剧室结构

```
标准编剧室配置:

1. Showrunner (系列总监)
   - 最终决策者
   - 负责与平台方沟通
   - 把控整体创意方向

2. Executive Producer (执行制片) x1-2
   - 通常由Showrunner兼任
   - 负责商业和制作层面

3. Co-Executive Producer (联合执行制片) x1-3
   - 高级编剧，可独立带队
   - 负责部分剧集开发

4. Supervising Producer (监制) x1-2
   - 资深编剧
   - 负责剧本质量

5. Producer (制片) x1-2
   - 负责具体制作流程

6. Story Editor / Staff Writer (剧本编辑/编剧) x3-6
   - 执行层编剧
   - 分头撰写具体场景

编剧室日常流程:
09:00 - 到达，个别讨论
10:00 - 全体编剧室会议（Outline讨论）
12:00 - 午餐
13:00 - 分头写作或小组讨论
17:00 - 进度汇报
18:00 - 收工（理想状态）
```

#### 编剧室工作方法

```
编剧室协作的核心方法:

1. 故事构思 (Story Breaking)
   目的: 共同确定故事走向

   方法:
   - 所有编剧围坐讨论
   - 用卡片或白板列出所有想法
   - 逐集讨论故事走向
   - Showrunner最终决定

   原则:
   - 没有坏主意（No Bad Ideas）
   - 辩论是为了更好，不是为了赢
   - 最终听Showrunner的

2. 大纲撰写 (Outlining)
   目的: 确定每集的具体故事线

   方法:
   - 按集分配负责编剧
   - 全体讨论大纲
   - Showrunner审核通过

   大纲内容:
   - 每场戏的目的
   - 角色在每场戏的变化
   - 关键对白要点
   - 时长估算

3. 分场写作 (Scene Writing)
   目的: 写出具体对白和动作描述

   方法:
   - 按集分配或按幕分配
   - 各自回家写
   - 定期汇报进度

4. 剧本朗读 (Table Read)
   目的: 通过朗读发现问题

   方法:
   - 所有编剧+核心演员参与
   - 完整朗读剧本
   - 朗读后讨论问题
   - 修改后进入拍摄准备
```

### 第三部分：季开发流程

#### 剧集开发时间线

```
标准10集剧集的开发流程（12-16个月）:

Phase 1: 开发期 (Month 1-4)
├── Month 1-2: 季整体概念开发
│     ├── 确定季主题
│     ├── 设计主要角色弧光
│     └── 规划季结构
├── Month 2-3: 编剧室开发
│     ├── 建立编剧室
│     ├── 逐集故事构思
│     └── 完成10集大纲
└── Month 3-4: 剧本撰写
      ├── 分配编剧
      ├── 逐集写作
      └── 剧本完成

Phase 2: 前期制作 (Month 5-8)
├── Month 5-6: 选角
│     ├── 主角选角
│     ├── 配角选角
│     └── 群演规划
├── Month 6-7: 场地/美术
│     ├── 确定拍摄场地
│     ├── 美术设计
│     └── 场景搭建
└── Month 7-8: 准备
      ├── 选定导演
      ├── 分集脚本完成
      └── 拍摄准备

Phase 3: 制作期 (Month 9-12)
├── Month 9-10: 第一阶段拍摄
│     ├── 第1-5集拍摄
│     └── 中期评估
├── Month 11-12: 第二阶段拍摄
      ├── 第6-10集拍摄
      └── 杀青

Phase 4: 后期制作 (Month 13-16)
├── Month 13-14: 剪辑
├── Month 14-15: 视效/调色
└── Month 15-16: 混音/交片
```

### 第四部分：与平台方的沟通

#### 平台方的需求管理

```
三大平台类型的特点:

1. 传统电视网 (HBO/Showtime/Starz)
   - 通常6-10集季
   - 创作自由度高
   - 商业压力相对小
   - 需要：高质量剧本，准时交付

2. 流媒体平台 (Netflix/Prime/Hulu)
   - 通常8-10集季
   - 数据驱动决策
   - 需要快速响应
   - 需要：观众参与度，话题性

3. 有线台 (FX/AMC/USA)
   - 通常10-13集季
   - 创作与商业平衡
   - 需要：稳定受众，奖项潜力

平台方的常见要求:
| 要求 | 应对策略 |
|------|---------|
| 增加动作场面 | 在不影响叙事的前提下添加 |
| 删减集数 | 压缩非核心场景 |
| 调整结局 | 确保主题不变的前提下修改 |
| 更换演员 | 在剧本中给出合理解释 |
| 加入热点话题 | 自然融入而非硬塞 |
```

### 第五部分：AI视频生成的Showrunner指南

#### 剧集类型 → 结构策略

**AI生成时的Showrunner提示词**:

| 剧集类型 | 结构 | 集数 | AI提示词 |
|----------|------|------|---------|
| 剧情惊悚 | 连续剧模式 | 8-10集 | serialized thriller structure, 8-10 episodes, intense arc |
| 情景喜剧 | 单元剧模式 | 22-24集 | episodic sitcom structure, 22+ episodes, self-contained stories |
| 悬疑破案 | 混合模式 | 10-13集 | hybrid procedural structure, main mystery + cases |
| 科幻史诗 | 连续剧模式 | 8-10集 | serialized sci-fi, epic multi-season arc |
| 律政剧 | 单元+连续混合 | 20-22集 | courtroom hybrid, case of the week + season arc |

## 决策逻辑

### 季规划决策框架

```
制定新一季时的决策流程:

Step 1: 回顾上一季
├── 收视/数据表现
├── 观众反馈
├── 奖项表现
└── 商业表现

Step 2: 确定新季方向
├── 主题选择
├── 角色弧光设计
├── 新角色规划
└── 旧角色去留

Step 3: 结构设计
├── 集数确定
├── 季结构类型
├── 核心冲突设计
└── 高潮设计

Step 4: 分配资源
├── 预算分配
├── 编剧室规模
├── 制作周期
└── 特殊需求

Step 5: 风险评估
├── 识别高风险元素
├── 准备备选方案
└── 建立应急预案
```
