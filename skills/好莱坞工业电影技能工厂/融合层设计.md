# 融合层设计 — Fusion Layer

> 多个技能的输出怎么融合成最终Prompt？谁来当裁判？冲突时怎么办？

---

## 融合层的定位

融合层是技能与主链路之间的"翻译和仲裁引擎"。

```
技能A输出 ─┐
技能B输出 ─┼─→ 融合层 ─→ 主链路（27 Stage）
技能C输出 ─┘    ↑
                  │
           冲突检测 + 仲裁 + 格式标准化
```

**核心职责**：
1. 接收多个技能的专业中间产物
2. 检测技能间的冲突
3. 标准化输出格式
4. 注入主链路的正确Stage

## 中间产物Schema（统一格式）

所有技能输出的中间产物必须符合以下Schema，这是融合的基础。

### Scene Card Schema

```yaml
SceneCard:
  scene_id: string          # SC01, SC02...
  function: enum             # setup/confrontation/revelation/payoff/turn
  objective: string          # 可验证的叙事结果
  emotional_curve:           # 时间轴情绪变化
    - timestamp: string      # 0s, 5s, 10s...
      emotion: string       # curiosity, unease, awe...
  light_tier: enum          # A/B/C/D
  color_script: string       # 主色系统
  axis_setup:               # 轴线设定
    primary_axis: string    # 左→右/右→左
    screen_direction: map    # 角色名: left/right/center
  pacing_trend: string      # 慢入—蓄力—爆发—余韵
```

### Shot Card Schema

```yaml
ShotCard:
  shot_id: string           # SC01-S01...
  scene_id: string          # 所属场次
  narrative_purpose: string # 单一叙事目的
  emotional_target: string  # 情绪目标
  subject: string           # 主体描述
  primary_action: string    # 主动作
  performance_objective: string  # 表演目标
  shot_size: enum           # ELS/FS/MS/MCU/CU/ECU/OTS/POV/Insert
  camera_position: enum     # eye_level/low_angle/high_angle/ground_level/subjective
  camera_movement: enum     # push/pull/pan/track/crane/orbit/POV/whip_pan/tilt/arc/handheld/static
  light_tier: enum          # A/B/C/D
  ofa: string               # 起幅锚点描述
  efa: string               # 落幅锚点描述
  continuity_mode: enum     # strict/soft/none
  performance_intensity: integer  # 1-5
```

## 融合决策树

当多个技能的输出进入融合层时，按以下顺序处理：

### 第一步：依赖排序

```
输入技能列表 → 按Stage顺序排序 → 确定融合顺序
```

**融合顺序优先级**：
1. **Screenwriter** → 最先（叙事结构决定一切）
2. **Character Analysis** → 第二（角色是一切的基础）
3. **Director** → 第三（导演意图是裁判）
4. **DP** → 第四（镜头语言服务叙事）
5. **Gaffer** → 第五（光影服务镜头）
6. **Acting Coach** → 第六（表演服务镜头语言）
7. **Sound Designer** → 第七（声音服务视觉）
8. **Colorist** → 最后（色彩是最后修饰）

### 第二步：冲突检测

检测以下冲突类型：

| 冲突类型 | 示例 | 检测方式 | 处理方式 |
|----------|------|----------|----------|
| 叙事-镜头冲突 | Screenwriter要求慢节奏，DP选择了快节奏运镜 | 情绪曲线 vs 运镜复杂度 | 提交Director裁定 |
| 光影-叙事冲突 | 情绪目标是"温暖"，Gaffer输出了冷色调 | 情绪目标 vs Light Tier | 提交Gaffer重新校准 |
| 表演-镜头冲突 | Acting Coach设计了高强度表演，DP选择了远景 | 表演强度 vs 景别 | 提交Director裁定 |
| 空间-连续性冲突 | 美术指导设计的空间与Scene Card不符 | Environment vs Scene | 提交Director裁定 |
| 声音-视觉冲突 | 声音主导层是music，视觉节奏是action | 声音主导层 vs 视觉节奏 | 提交Sound Designer重新校准 |

### 第三步：冲突解决

```
检测到冲突 → 判断冲突类型 → 选择解决策略
```

**解决策略**：

| 策略 | 适用场景 | 执行方式 |
|------|----------|----------|
| **导演裁定** | 叙事vs镜头/表演/空间类冲突 | 提交Director Agent裁决 |
| **专业回退** | 技能输出超出自身专业范围 | 该技能服从其他专业技能 |
| **叙事优先** | 任何与叙事冲突的输出 | 叙事输出优先级最高 |
| **最近优先** | 两个技能同等优先级 | 后调用的技能覆盖先调用的 |
| **权重叠加** | 多个技能都对某参数有权重 | 按权重计算最终值 |

### 第四步：格式标准化

将所有中间产物转换为统一的Prompt注入格式：

```yaml
PromptInjection:
  subject_and_binding: string    # 主体+角色锚点
  primary_action: string         # 主动作
  performance_focus: string      # 表演设计
  spatial_environment: string     # 空间环境
  camera_language: string         # 镜头语言
  lighting_and_material: string   # 光线材质
  sound_dialogue: string         # 声音对白
  closing_anchor: string          # 收束锚点
  constraints: string            # 负面约束
```

## 仲裁机制

### 三级仲裁体系

| 级别 | 仲裁者 | 管辖范围 | 时效 |
|------|--------|----------|------|
| **L1 自动裁决** | 融合层规则引擎 | 格式问题、默认值填充 | 实时 |
| **L2 专业裁决** | 对应专业技能Agent | 专业范围冲突 | <1秒 |
| **L3 导演裁决** | Director Agent | 跨专业冲突、创意方向 | 需人类确认 |

### 典型冲突的自动裁决规则

```
规则1：叙事目的 > 镜头炫技
  if DP_output.has_montage AND Scene_purpose == "establish":
    reject montage, use single_steady_shot

规则2：情绪基调 > 摄影师偏好
  if Gaffer_output.tone != SceneCard.emotional_tone:
    override Gaffer with SceneCard.tone

规则3：表演强度决定景别
  if Performance.intensity >= 4 AND shot_size == "FS":
    override shot_size = "CU"

规则4：Light Tier是强制约束
  if any_output.conflicts_with_light_tier:
    always enforce Light Tier
```

## 融合流程示例

**输入场景**：小G在深海神坛发现发光神兽

**技能输出集合**：

| 技能 | 输出 |
|------|------|
| Screenwriter | SceneCard：情绪curve=curiosity→awe，神圣显现时刻 |
| Director | 确认情绪曲线，要求"敬畏感"贯穿全镜 |
| DP | 景别=CU，运镜=static，机位=eye_level |
| Gaffer | LightTier=D，光源=Divine Glow，冷调青色 |
| Acting Coach | 表演强度=4，微表情=瞳孔慢慢放大 |
| Sound Designer | 声音主导层=silence，远处低频轰鸣 |

**融合处理**：

```
Step 1: 依赖排序
  Screenwriter(1) → Director(2) → DP(4) → Gaffer(5) → ActingCoach(6) → SoundDesigner(7)

Step 2: 冲突检测
  ✓ 叙事目的：深海神秘 → DP景别CU ✓ 一致
  ✓ 光影Tier D → 符合情绪"敬畏" ✓ 一致
  ✓ 表演强度4 → 景别CU ✓ 匹配
  ✓ 声音主导层=silence → 符合敬畏感 ✓ 一致
  结论：无冲突

Step 3: 格式标准化
  主体: @Image1 as Xiao G, 8yo Chinese boy, wide eyes showing curiosity
  主动作: 缓慢后退一步，目光锁定发光的深海神兽
  表演设计: 瞳孔缓慢放大，嘴巴微张，呼吸暂停
  空间环境: 深海神坛，巨大石柱间，蓝绿色生物发光粒子漂浮
  镜头语言: 固定机位，中近景，正面平视，缓慢推进
  光线: Light Tier D，神圣显现，青色光芒从神兽方向照射
  声音: 静默，仅远处低频轰鸣
  收束: 落幅于神兽的蓝色眼睛
```

**最终Prompt**（约380字符）：
```
@Image1 as Xiao G, 8yo Chinese boy, wide eyes showing curiosity and gradually dilating pupils, mouth slightly open, breath held. A slow step backward as the boy's gaze locks onto the glowing deep-sea deity. Bioluminescent blue-green particles drift through the ancient stone altar, massive stone pillars towering. Fixed medium close-up, eye-level, slowly pushing in. Light Tier D: divine manifestation, cold cyan glow emanating from the deity, no fill light, pure silhouette contrast. In silence with distant low-frequency rumble. Closing on the deity's blue luminescent eyes.
```

## 融合层与主链路对接

### Stage接入点

| Stage | 融合层输出 | 说明 |
|-------|-----------|------|
| Stage 3 | SceneCard | 进入Scene Card Gate |
| Stage 5 | ShotCards[] | 进入Shot Card Gate |
| Stage 8 | PromptInjection | 进入Prompt生成 |
| Stage 10 | DirectorReview | 导演审片反馈 |
| Stage 14 | SoundEventTable | 声音事件表 |

### 融合层输出格式（注入主链路）

```yaml
MainPipelineInput:
  stage: integer
  scene_id: string
  shot_cards: list[ShotCard]
  prompt_injection: PromptInjection
  constraints: list[string]
  metadata:
    skills_used: list[string]
    fusion_version: string
    conflict_count: integer
    director_approval: boolean
```

## 异常处理

### 融合失败

| 失败原因 | 处理方式 |
|----------|----------|
| 必需技能缺失 | 使用默认技能，标记"未完全优化" |
| 循环依赖 | 检测到环时选择最近一次输出，打破循环 |
| 技能输出格式错误 | 尝试自动修复，失败则回退到基础模板 |
| 超过3个冲突 | 升级到L3导演裁决 |

### 降级模式

当融合层遇到无法解决的复杂情况时：
1. 降级到"单技能模式"：只使用核心技能（Director+DP+Gaffer）
2. 记录降级原因
3. 通知用户"使用了简化配置"
