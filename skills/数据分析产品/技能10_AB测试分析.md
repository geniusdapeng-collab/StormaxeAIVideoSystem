## TL;DR

AB测试 = 测试设计 + 分组策略 + 效果评估 + 结果分析 + 决策应用

## 1. AB测试基础

### 1.1 什么是AB测试
```
AB测试定义：
AB测试是一种对照实验方法，通过将用户随机分成两组（A组和B组），对比不同方案的效果，从而做出数据驱动的决策。

AB测试的本质：
├── 对照实验：有实验组和对照组
├── 随机分组：排除选择偏差
├── 变量控制：只有一个变量不同
└── 效果对比：用数据说话

AB测试的价值：
├── 降低风险：用数据验证想法
├── 科学决策：避免拍脑袋
├── 持续优化：不断迭代改进
├── 用户导向：用户选择最优
└── 量化效果：准确评估价值

AB测试适用场景：
├── 产品改版：UI改版、流程优化
├── 功能上线：新功能测试
├── 营销活动：文案、渠道、优惠
├── 推荐算法：算法优化
└── 定价策略：价格测试

AB测试不适用场景：
├── 用户量太小：样本量不足
├── 周期太长：需要长期观察
├── 互相影响：会产生网络效应
└── 敏感场景：可能造成用户流失
```

### 1.2 AB测试流程
```
AB测试完整流程：

Step 1：明确目标
├── 想优化什么指标
├── 目标指标是什么
├── 预期提升多少
└── 接受标准是什么

Step 2：提出假设
├── 为什么要做这个改动
├── 改动会带来什么变化
├── 影响哪些指标
└── 形成假设

Step 3：设计实验
├── 确定实验分组
├── 确定流量分配
├── 确定实验周期
├── 设计实验方案
└── 技术实现

Step 4：运行实验
├── 灰度放量
├── 数据监控
├── 异常检测
├── 实验验证
└── 全量上线

Step 5：数据分析
├── 数据收集
├── 效果计算
├── 统计检验
├── 分群分析
└── 结论输出

Step 6：决策应用
├── 结论确认
├── 方案选择
├── 全量发布
├── 持续监控
└── 经验沉淀
```

## 2. 测试设计

### 2.1 指标设计
```
指标设计：

核心指标（北极星指标）：
├── 定义：衡量实验效果的核心指标
├── 要求：与业务目标一致
├── 特点：唯一、可衡量
└── 示例：转化率、点击率、GMV

辅助指标：
├── 定义：需要观察的其他指标
├── 作用：全面评估实验效果
├── 类型：正向指标、负向指标
└── 示例：DAU、留存、收入

护栏指标：
├── 定义：不能变差的指标
├── 作用：保护业务底线
├── 要求：必须保持稳定
└── 示例：核心功能使用率

指标选择原则：
├── 指标要可量化
├── 指标要与业务目标相关
├── 不要只看单一指标
├── 设置护栏指标
└── 明确判断标准

指标设计示例：
场景：优化注册流程

核心指标：
├── 注册转化率 = 注册成功数 / 访问注册页数

辅助指标：
├── 注册时长
├── 各步骤转化率
├── 跳出率

护栏指标：
├── 注册后7日留存
├── 注册后30日付费率
```

### 2.2 分组设计
```
分组设计：

分组原则：
├── 随机分配：保证同质性
├── 分组独立：避免互相影响
├── 流量均匀：保证样本量
└── 时间同步：同时进行实验

流量分配：
├── 分组数量：2组或以上
├── 流量比例：50:50 或 95:5
├── 特殊分组：小流量验证
└── 递进放量：从小到大

常见流量分配：
├── 50:50：标准分配，两组各50%
├── 95:5：大流量验证，小流量实验
├── 90:5:5：双实验组
└── 渐进放量：1% → 5% → 10% → 50%

分组变量：
├── 用户维度：user_id
├── 设备维度：device_id
├── 浏览器维度：cookie_id
└── 访问维度：session_id

分层实验：
├── 概念：同一流量层内互斥，不同层可重叠
├── 目的：提高实验效率
├── 要求：不同实验层使用不同分组变量
└── 示例：UI层实验 + 算法层实验
```

### 2.3 样本量计算
```
样本量计算：

为什么需要计算样本量：
├── 样本量太小：结果不可靠
├── 样本量太大：资源浪费
└── 科学计算：保证统计功效

影响因素：
├── 基准转化率：当前转化率
├── 最小检测效应：想检测到的最小变化
├── 显著性水平：α，通常为0.05
└── 统计功效：1-β，通常为0.8

计算公式（简化）：
n = 16 × σ² / δ²

其中：
├── n：每组所需样本量
├── σ：标准差
└── δ：最小检测效应

在线计算工具：
├── https://www.evanmiller.org/ab-testing/sample-size.html
├── https://www.calculator.net/sample-size-calculator.html
└── 各大平台自带计算工具

示例：
场景：点击率测试
├── 当前点击率：5%
├── 期望提升：相对提升20%
├── 目标点击率：6%
└── 需要样本量：约40万/组

样本量估计：
日均UV 10万，实验14天
→ 每组样本量 = 10万 × 14 / 2 = 70万
→ 可以满足要求
```

## 3. 效果评估

### 3.1 效果计算
```
效果计算：

绝对值变化：
实验组值 - 对照组值

相对值变化（提升率）：
(实验组值 - 对照组值) / 对照组值 × 100%

示例：
对照组转化率：5%
实验组转化率：6%
绝对提升：6% - 5% = 1%
相对提升：(6% - 5%) / 5% × 100% = 20%

计算公式：
转化率 = 转化用户数 / 曝光用户数
提升率 = (实验组转化率 - 对照组转化率) / 对照组转化率

分层指标计算：
每层转化率 = 每层转化用户数 / 每层曝光用户数
总转化率 = 总转化用户数 / 总曝光用户数
```

### 3.2 统计显著性
```
统计显著性：

概念：
统计显著性是指观察到的差异不是由随机因素引起的概率。

显著性水平（α）：
├── 通常设为0.05
├── 含义：有5%的概率是错的
└── p < 0.05 时认为显著

p值：
├── p < 0.05：差异显著
├── p < 0.01：差异高度显著
├── p >= 0.05：差异不显著
└── p < α：拒绝原假设

置信区间：
├── 95%置信区间
├── 表示真实值有95%的概率落在区间内
└── 区间不包含0表示显著

结果判断：
├── p < 0.05 且 提升 > 0：实验组显著更好
├── p < 0.05 且 提升 < 0：实验组显著更差
├── p >= 0.05：差异不显著，无法判断
└── 区间判断：区间不包含0表示显著

注意事项：
├── 统计显著 ≠ 业务显著
├── 即使显著，效果也需要评估
├── 不显著不等于没效果
└── 需要持续观察
```

### 3.3 结果解读
```
结果解读框架：

一、效果评估
├── 核心指标变化：提升多少
├── 统计显著性：是否显著
├── 置信区间：可能的范围
└── 实际意义：业务价值

二、辅助指标
├── 正向指标：是否提升
├── 负向指标：是否下降
└── 综合评估：整体效果

三、分群分析
├── 不同用户群效果
├── 是否有差异化
└── 优化方向

四、结论
├── 实验组是否更好
├── 是否全量发布
├── 后续建议
└── 经验沉淀

常见结果类型：
类型1：显著正向
→ 实验组显著更好，全量发布

类型2：显著负向
→ 实验组更差，保留原方案

类型3：不显著但正向
→ 继续观察或延长实验

类型4：不显著但负向
→ 考虑保留原方案

类型5：显著但有问题
→ 分析原因，决定是否发布
```

## 4. 常见问题

### 4.1 实验问题
```
常见实验问题：

问题1：样本量不足
├── 表现：置信区间很宽
├── 原因：流量少或周期短
├── 解决：延长实验周期或放弃
└── 预防：提前计算样本量

问题2：新奇效应
├── 表现：新用户效果好，老用户效果差
├── 原因：新用户好奇心
├── 解决：分层分析，按新老分析
└── 预防：关注长期效果

问题3：选择偏差
├── 表现：分组不均匀
├── 原因：分流算法问题
├── 解决：检查分流逻辑
└── 预防：使用正确的分流算法

问题4：辛普森悖论
├── 表现：分组好但总体差
├── 原因：分组分布不均
├── 解决：按分组分析
└── 预防：检查分组均匀性

问题5：外部因素干扰
├── 表现：效果波动大
├── 原因：外部事件影响
├── 解决：增加实验周期
└── 预防：避免敏感时段
```

### 4.2 决策陷阱
```
决策陷阱：

陷阱1：只看统计显著性
├── 问题：显著不等于有价值
├── 正确：看实际提升幅度
└── 示例：转化率从5%提升到5.1%

陷阱2：多重比较问题
├── 问题：比较太多容易犯错
├── 正确：控制比较次数
└── 解决：Bonferroni校正

陷阱3：幸存者偏差
├── 问题：只看成功的实验
├── 正确：记录所有实验
└── 解决：建立实验数据库

陷阱4：短期效应
├── 问题：只看短期数据
├── 正确：关注长期效果
└── 解决：设置观察周期

陷阱5：只看核心指标
├── 问题：忽略负面影响
├── 正确：全面评估指标
└── 解决：设置护栏指标
```
