## 1. 理解业务

### 1.1 业务理解框架
```
理解业务的框架：

商业模式画布：
├── 价值主张：提供什么价值
├── 客户细分：服务谁
├── 渠道通路：通过什么触达
├── 客户关系：怎么维护关系
├── 收入来源：怎么赚钱
├── 核心资源：需要什么资源
├── 关键业务：做什么事
├── 重要合作：和谁合作
└── 成本结构：花多少钱

理解业务的维度：
├── 业务本质
│   ├── 做什么生意
│   ├── 解决什么问题
│   └── 创造什么价值
├── 业务模式
│   ├── 怎么获客
│   ├── 怎么转化
│   ├── 怎么留存
│   └── 怎么变现
├── 业务流程
│   ├── 核心流程
│   ├── 关键节点
│   └── 效率瓶颈
├── 业务指标
│   ├── 核心指标
│   ├── 过程指标
│   └── 分解指标
└── 业务现状
    ├── 发展阶段
    ├── 竞争格局
    └── 趋势变化
```

### 1.2 业务模型
```
常见业务模型：

电商业务模型：
├── 流量入口：自然流量 + 付费流量
├── 转化漏斗：浏览 → 加购 → 下单 → 支付
├── 客单价：商品单价 × 购买数量
├── 复购：重复购买率
├── 成本结构：商品成本 + 运营成本 + 物流成本
└── 利润 = 销售额 - 成本

内容平台模型：
├── 内容生产：UGC + PGC + 专业内容
├── 内容分发：推荐 + 搜索 + 关注
├── 用户消费：浏览 → 互动 → 传播
├── 变现模式：广告 + 付费 + 电商
└── 用户价值：DAU × 人均时长

SaaS业务模型：
├── 获客漏斗：线索 → MQL → SQL → 成交
├── 续费模式：年费 + 续费率
├── 增购模式：增购模块 + 增购用户
├── 客户成功：激活 → 使用 → 续费
└── LTV = ARPA × 续费率 × 客户时长

交易平台模型：
├── 撮合效率：供需匹配
├── 变现模式：交易抽佣
├── 供需平衡：供给侧 + 需求侧
├── 用户粘性：复购 + 复访
└── 货币化率：收入 / GMV
```

## 2. 业务拆解

### 2.1 拆解方法
```
业务拆解方法：

拆解原则：
├── MECE原则：不重不漏
├── 层层拆解：直到可执行
├── 分类清晰：类别明确
├── 量化可测：可量化指标
└── 业务导向：符合业务逻辑

拆解维度：
├── 时间维度
│   ├── 年 → 季 → 月 → 周 → 日
│   └── 同比、环比
├── 空间维度
│   ├── 公司 → 业务线 → 产品 → 功能
│   └── 地域 → 部门 → 团队
├── 用户维度
│   ├── 整体 → 渠道 → 用户群 → 单用户
│   └── 新用户 → 活跃用户 → 付费用户
└── 流程维度
    ├── 整体 → 环节 → 步骤 → 操作
    └── 获客 → 转化 → 留存 → 变现

拆解示例：GMV拆解
GMV = 访客数 × 转化率 × 客单价
    = (新访客 + 老访客) × (订单数/访客数) × (销售额/订单数)
    = 渠道1访客 + 渠道2访客 + ... × 各渠道转化率 × 各品类客单价
```

### 2.2 业务公式
```
常见业务公式：

电商：
├── GMV = 访客数 × 转化率 × 客单价
├── 销售额 = GMV × 支付转化率
├── 毛利率 = (销售额 - 成本) / 销售额
├── ROI = 销售额 / 投放金额
└── 库存周转 = 销售成本 / 平均库存

内容平台：
├── DAU = 新用户 + 留存用户
├── 留存率 = 第N日留存 / 第0日新增
├── 人均时长 = 总时长 / DAU
├── 互动率 = 互动数 / 曝光数
└── 传播系数 = 传播带来的新用户 / 传播用户

SaaS：
├── MRR = 活跃客户数 × ARPU
├── ARR = MRR × 12
├── 续费率 = 续费金额 / 到期金额
├── CAC = 营销费用 / 新增客户
├── LTV = ARPU × 客户生命周期
└── CAC/LTV = < 0.3 较好

用户增长：
├── 自然增长 = 口碑传播 + SEO
├── 付费增长 = 投放获客
├── 裂变增长 = 老带新
├── 增长成本 = 总费用 / 新增用户
└── ROI = 用户LTV / 用户CAC
```

### 4.2 优化案例
```
优化案例：电商转化率优化

背景：
转化率2%，行业平均3%，目标提升到3%

Step1：拆解转化漏斗
├── 首页UV → 详情页UV：60%
├── 详情页UV → 加购：20%
├── 加购 → 提交订单：40%
├── 提交订单 → 支付成功：80%
└── 整体转化 = 60% × 20% × 40% × 80% = 3.84%

Step2：找到最大瓶颈
├── 详情页转化最低（20%）
└── 最大优化空间在详情页

Step3：分析详情页问题
├── 商品匹配度：用户找到想要的商品了吗
├── 商品展示：图片、价格、评价是否吸引
├── 信任度：用户是否信任
└── 行动引导：是否有足够的行动召唤

Step4：提出优化方案
├── 优化商品推荐算法
├── 优化商品主图和详情页
├── 增加用户评价展示
├── 增加限时优惠提示
└── 优化下单按钮和流程

Step5：方案优先级
├── P0：优化详情页转化（预期提升50%）
├── P1：优化推荐算法（预期提升30%）
└── P2：优化下单流程（预期提升20%）

Step6：落地执行
├── AB测试验证
├── 小流量上线
├── 监控数据
└── 全量推广

Step7：效果验证
├── 转化率从2%提升到3%
├── 达成目标
└── 持续优化其他环节
```

方法4：场景预测
├── 乐观场景
