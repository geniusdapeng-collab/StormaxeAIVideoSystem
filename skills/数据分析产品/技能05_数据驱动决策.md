## TL;DR

数据驱动决策 = 决策框架 + 数据收集 + 分析验证 + 结论输出 + 效果评估

## 1. 数据驱动决策基础

### 1.1 什么是数据驱动决策
```
数据驱动决策定义：
以数据为依据，通过数据分析得出结论，支持业务决策的决策方式。

数据驱动 vs 直觉决策：
├── 数据驱动：基于事实和证据
├── 直觉决策：基于经验和感觉
├── 数据驱动：客观、可量化、可验证
├── 直觉决策：快速、灵活、有经验时有效
└── 最佳实践：数据驱动 + 业务经验

数据驱动决策的价值：
├── 减少主观偏见
├── 提高决策质量
├── 量化决策效果
├── 积累决策经验
├── 促进团队共识
└── 持续优化迭代
```

### 1.2 决策类型
```
决策类型：

战略决策：
├── 定义：长期、全局性决策
├── 特点：影响大、周期长
├── 例子：市场进入、产品方向
├── 数据需求：市场数据、用户数据
└── 分析方法：趋势分析、预测分析

战术决策：
├── 定义：中短期、执行性决策
├── 特点：频率高、影响中等
├── 例子：营销策略、运营活动
├── 数据需求：运营数据、效果数据
└── 分析方法：对比分析、归因分析

运营决策：
├── 定义：日常、重复性决策
├── 特点：频率高、影响小
├── 例子：定价调整、客服处理
├── 数据需求：实时数据
└── 分析方法：规则分析、异常检测

决策层次与数据需求：
├── 高层决策：宏观数据、外部数据
├── 中层决策：业务数据、内部数据
└── 基层决策：操作数据、实时数据
```

## 2. 决策框架

### 2.1 通用决策框架
```
通用决策框架：

Step 1：明确决策问题
├── 识别问题：是什么问题
├── 定义边界：解决什么
├── 确定目标：达成什么
├── 明确约束：有哪些限制
└── 形成假设：可能的方案

Step 2：收集数据
├── 确定数据需求：需要什么数据
├── 识别数据来源：数据在哪里
├── 评估数据质量：数据可靠吗
├── 收集整理数据：获取和处理
└── 补充定性信息：补充非数据信息

Step 3：分析数据
├── 描述性分析：发生了什么
├── 诊断性分析：为什么发生
├── 预测性分析：会发生什么
├── 方案评估：各方案效果
└── 风险评估：有哪些风险

Step 4：得出结论
├── 综合分析结果
├── 给出建议方案
├── 说明支持证据
├── 预测可能结果
└── 提出风险预案

Step 5：决策执行
├── 执行方案
├── 跟踪效果
├── 评估决策
└── 总结经验
```

### 2.2 常用决策模型
```
常用决策模型：

模型1：SWOT分析
├── S (Strengths)：优势
├── W (Weaknesses)：劣势
├── O (Opportunities)：机会
└── T (Threats)：威胁

模型2：波特五力
├── 现有竞争者
├── 潜在进入者
├── 替代品威胁
├── 供应商议价能力
└── 买方议价能力

模型3：决策树
├── 明确所有选项
├── 评估每个选项
├── 量化概率和收益
├── 计算期望值
└── 选择最优方案

模型4：A/B测试
├── 提出假设
├── 设计实验
├── 分组测试
├── 数据分析
└── 得出结论

模型5：麦肯锡7步法
1. 定义问题
2. 分解问题
3. 优先排序
4. 分析议题
5. 关键假设
6. 数据验证
7. 总结建议
```

```
数据需求分析：

业务问题 → 数据需求 → 数据来源 → 数据获取

数据需求分析步骤：
1. 明确决策问题
2. 拆解关键要素
3. 确定所需数据
4. 评估数据可得性
5. 设计数据获取方案

常见数据需求：
├── 用户数据：画像、行为、反馈
├── 业务数据：交易、流程、效率
├── 市场数据：竞品、行业、趋势
├── 财务数据：收入、成本、利润

数据质量评估：
├── 准确性：数据正确吗
├── 完整性：数据完整吗
├── 一致性：口径一致吗
├── 时效性：数据新吗
└── 可得性：数据能获取吗
```

## 4. 分析验证

### 4.1 分析验证方法
```
分析验证方法：

描述性分析：
├── 发生了什么
├── 发生了什么变化
├── 现状是什么
└── 工具：Excel、BI报表

诊断性分析：
├── 为什么会这样
├── 原因是什么
├── 主要因素是什么
└── 工具：对比分析、相关分析

预测性分析：
├── 会发生什么
├── 趋势是什么
├── 预测结果是什么
└── 工具：趋势分析、预测模型

验证方法：
├── 对比验证：与其他数据对比
├── 逻辑验证：逻辑是否合理
├── 多角度验证：多个角度看
├── 敏感性分析：关键假设变化影响
└── 专家评审：专家审查

避免的分析错误：
├── 相关性≠因果性
├── 样本偏差
├── 选择性呈现
├── 确认偏误
└── 数据操纵
```

### 4.2 常见问题
```
数据分析常见问题：

问题1：数据量不足
├── 表现：数据太少无法得出结论
├── 解决：扩大数据范围、收集更多数据
└── 注意：没有足够数据就不下结论

问题2：数据质量问题
├── 表现：数据不准确、不完整
├── 解决：清洗数据、交叉验证
└── 注意：垃圾进垃圾出

问题3：选择性呈现
├── 表现：只呈现对自己有利的
├── 解决：完整呈现、客观分析
└── 注意：不要数据造假

问题4：过度解读
├── 表现：没有统计显著性却说显著
├── 解决：使用统计检验、谨慎结论
└── 注意：相关性不等于因果性

问题5：忽略背景
├── 表现：只看数据不看背景
├── 解决：结合业务背景理解数据
└── 注意：数据要放在场景中
```
