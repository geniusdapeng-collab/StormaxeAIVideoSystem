## 1. 指标基础

### 1.1 什么是指标
```
指标的定义：
指标是衡量业务状态和变化的量化标准，是数据分析的基础。

指标的构成：
├── 指标名称：叫什么（如DAU、GMV）
├── 指标口径：怎么算（如"当日去重的活跃用户数"）
├── 指标维度：怎么看（如地区、渠道、用户类型）
├── 指标单位：什么单位（如人数、金额、百分比）
└── 指标更新频率：什么时候更新

指标 vs 数据：
├── 数据是原始素材
├── 指标是基于数据的加工
├── 数据是原材料
├── 指标是成品
└── 好的指标体系是高效分析的前提
```

### 1.2 指标类型
```
指标类型：

绝对指标（总量）：
├── 定义：反映总规模
├── 例子：DAU、总用户数、总销售额
├── 特点：直观反映体量
└── 局限：难以比较

相对指标（比率）：
├── 定义：两个数值的比
├── 例子：转化率、留存率、增长率
├── 特点：便于比较
└── 局限：可能掩盖问题

先行指标：
├── 定义：预测未来的指标
├── 例子：搜索量、注册量
├── 特点：提前预警
└── 局限：相关性需验证

滞后指标：
├── 定义：结果性指标
├── 例子：付费用户数、GMV
├── 特点：反映结果
└── 局限：滞后发现

北极星指标：
├── 定义：最重要的指标
├── 特点：唯一、最核心
├── 作用：指引方向
└── 选择：可衡量、可持续、能反映用户价值
```

## 2. 设计原则

### 2.1 指标设计原则
```
指标设计六大原则：

原则1：可衡量
├── 可以用数据量化
├── 有明确的计算口径
├── 数据可以获取
└── 不能衡量的无法管理

原则2：有意义
├── 与业务目标相关
├── 能指导行动
├── 能发现问题
└── 避免虚荣指标

原则3：可理解
├── 名称清晰易懂
├── 定义明确无歧义
├── 业务人员能理解
└── 不要过度复杂

原则4：可行动
├── 指标变化能采取行动
├── 有明确的改善路径
├── 行动后指标能改善
└── 不要只看不改

原则5：可比较
├── 可以横向比较
├── 可以纵向对比
├── 可以对标行业
└── 便于评估效果

原则6：及时性
├── 数据能及时获取
├── 便于及时发现
├── 支持快速决策
└── 不要等到月底才知道
```

### 2.2 好指标特征
```
好指标的特征：

好指标特征：
├── 可量化：用数字表示
├── 可对比：可以比较
├── 可理解：清晰明了
├── 可行动：能指导行动
├── 可验证：能够验证假设
└── 可追踪：持续监控

坏指标特征：
├── 虚荣指标：好看但无用
│   └── 如：总用户数（不活跃的也算）
├── 复杂指标：难以理解
│   └── 如：加权综合评分
├── 滞后指标：事后才知道
│   └── 如：用户流失了才知道
├── 单一指标：以偏概全
│   └── 如：只看DAU不看质量
└── 割裂指标：缺乏关联
    └── 如：只看短期忽视长期
```

## 3. 构建方法

### 3.1 指标体系构建
```
指标体系构建四步法：

Step 1：明确业务目标
├── 业务是什么
├── 业务目标是什么
├── 关键成功因素是什么
├── 商业模型是什么
└── 输出：业务目标清单

Step 2：拆解关键路径
├── 用户旅程是什么
├── 关键节点有哪些
├── 核心动作是什么
├── 转化漏斗是什么
└── 输出：关键路径图

Step 3：设计核心指标
├── 目标指标：北极星指标
├── 过程指标：达成路径
├── 结果指标：最终结果
├── 分群指标：细分维度
└── 输出：指标清单

Step 4：建立指标体系
├── 分类：北极星指标 + 细分指标
├── 层级：集团 → 业务线 → 功能 → 模块
├── 关联：因果关系、支撑关系
├── 监控：数据埋点、报表体系
└── 输出：完整指标体系

指标分层：
├── L0：北极星指标（公司级）
├── L1：核心指标（业务线级）
├── L2：关键指标（功能级）
└── L3：基础指标（模块级）
```

### 3.2 指标拆解方法
```
指标拆解方法：

方法1：人货场拆解
适用于：电商、零售

人：用户相关
├── 新用户数
├── 活跃用户数
├── 付费用户数
└── 用户留存率

货：商品相关
├── SKU数
├── 动销率
├── 客单价
└── 商品曝光率

场：场景相关
├── 页面PV/UV
├── 访问时长
├── 跳出率
└── 场景转化率

---

方法2：人机料法环
适用于：制造业、生产

人：人员
├── 操作人员数
├── 培训完成率
└── 人员流失率

机：设备
├── 设备使用率
├── 故障率
└── 维修时间

料：物料
├── 物料合格率
├── 物料损耗率
└── 库存周转率

法：方法
├── 工艺执行率
├── 标准达成率
└── 良品率

环：环境
├── 环境达标率
├── 温湿度符合率
└── 清洁度

---

方法3：人财物拆解
适用于：企业通用

人：
├── 员工数
├── 人效
└── 人均成本

财：
├── 收入
├── 成本
├── 利润
└── ROI

物：
├── 资产
├── 库存
└── 周转
```

## 4. 常见场景

### 4.1 电商指标体系
```
电商指标体系：

L0：北极星指标
├── GMV = 访客数 × 转化率 × 客单价
└── 核心目标：提升销售额

L1：核心指标
├── 用户指标：DAU、新用户数
├── 流量指标：PV、UV、访问时长
├── 交易指标：GMV、订单数、客单价
├── 商品指标：动销率、库存周转
└── 服务指标：好评率、投诉率

L2：关键指标
├── 流量：广告点击率、落地页转化率
├── 转化：浏览-加购率、加购-付款率
├── 复购：复购率、复购周期
├── 用户：ARPU、LTV、流失率
└── 物流：发货及时率、配送时效

L3：细分指标
├── 渠道：各渠道ROI、转化率
├── 商品：各品类销售额、毛利率
├── 用户群：各等级用户占比、消费
└── 地域：各地区销售额、渗透率
```

### 4.2 内容平台指标体系
```
内容平台指标体系：

L0：北极星指标
├── 用户时长 = DAU × 人均时长
└── 核心目标：提升用户粘性

L1：核心指标
├── 用户指标：DAU、MAU、留存率
├── 内容指标：内容量、内容质量
├── 互动指标：点赞、评论、分享
└── 商业指标：广告收入、付费收入

L2：关键指标
├── 流量：新增用户数、获取成本
├── 留存：次日留存、7日留存
├── 活跃：DAU/MAU、时长分布
├── 互动：互动率、互动深度
└── 内容：发布量、消费量

L3：细分指标
├── 用户：各来源转化、新用户质量
├── 内容：各类型内容消费、完播率
├── 互动：各场景互动、内容热度
└── 商业：广告曝光、点击率、CPM
```

### 4.3 SaaS产品指标体系
```
SaaS产品指标体系：

L0：北极星指标
├── ARR = 年度经常性收入
└── 核心目标：提升收入

L1：核心指标
├── 获客指标：新签客户数、获客成本
├── 留存指标：客户留存率、流失率
├── 增购指标：增购收入、扩增率
├── 产品指标：功能使用、健康度
└── 财务指标：ARR、毛利率

L2：关键指标
├── 获客：MQL、CAC、转化率
├── 激活：首次使用、功能使用
├── 留存：月留存、流失预警
├── 增购：增购率、NRR
├── 产品：DAU/MAU、核心功能使用率
└── 成本：CAC、CTC、LTV

L3：细分指标
├── 渠道：各渠道获客效率
├── 产品：各功能使用、用户路径
├── 客户：各行业客户、流失原因
└── 销售：各阶段转化、销售周期
```

## 5. 落地实施

### 5.1 指标落地步骤
```

Step 1：指标定义
├── 明确指标名称

├── 定义统计维度
├── 说明数据来源
├── 确认更新频率

├── 确定埋点参数
├── 开发联调测试
├── 上线验证数据
└── 输出：埋点文档+数据

Step 3：报表开发
├── 设计报表结构
├── 开发数据报表
├── 配置数据预警
├── 建立仪表盘
├── 自动化更新
└── 输出：数据报表

Step 4：监控运营
├── 建立监控机制
├── 设置异常预警
├── 定期复盘分析
├── 持续优化迭代

└── 输出：运营机制
```
