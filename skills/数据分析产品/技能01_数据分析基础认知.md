## 1. 数据的本质

### 1.1 什么是数据
```
数据的定义：
数据是对客观事物的符号记录，是用于表示客观事物未经加工的原始素材。

数据的类型：
├── 结构化数据：表格数据，有固定格式
│   ├── 数值型：年龄、收入、销量
│   ├── 类别型：性别、城市、产品类别
│   └── 日期型：日期、时间
├── 非结构化数据：无法用表格存储
│   ├── 文本：评论、文章、聊天记录
│   ├── 图像：图片、照片、视频
│   └── 音频：语音、音乐、录音
└── 半结构化数据：介于两者之间
    ├── JSON：API返回数据
    ├── XML：配置文件
    └── HTML：网页数据

数据的特点：
├── 可量化：用数字表示
├── 可存储：保存在介质中
├── 可传输：在系统间传递
├── 可处理：通过工具分析
└── 可挖掘：发现规律价值
```

## 2. 数据分析流程

### 2.1 分析六步法
```
数据分析六步法：

Step 1：明确问题
├── 业务问题是什么
├── 需要回答什么问题
├── 期望得到什么结论
├── 如何定义成功
└── 形成分析假设

Step 2：数据收集
├── 需要什么数据
├── 数据在哪里
├── 如何获取数据
├── 数据是否可靠
└── 数据是否合规

Step 3：数据处理
├── 数据清洗：缺失值、异常值
├── 数据转换：格式统一
├── 数据整合：多源合并
├── 数据抽样：降低复杂度
└── 数据存储：便于分析

Step 4：数据分析
├── 描述性分析：是什么
├── 诊断性分析：为什么
├── 预测性分析：会怎样
├── 规范性分析：怎么做
└── 选择合适方法

Step 5：可视化呈现
├── 选择合适图表
├── 突出关键信息
├── 逻辑清晰
├── 视觉美观
└── 易于理解

Step 6：结论建议
├── 总结核心发现
├── 给出可行建议
├── 说明实施路径
├── 预测可能风险
└── 后续跟进计划
```

### 2.2 分析流程图
```
数据分析流程图：

    ↓

    ↓
```

## 3. 分析类型

### 3.1 按目的分类
```
分析类型（按目的）：

1. 描述性分析
定义：发生了什么
问题：是什么？有多少？
方法：计数、求和、平均、排名
示例：今天有多少用户注册？销售额是多少？

2. 诊断性分析
定义：为什么发生
问题：为什么？原因是什么？
方法：对比分析、交叉分析、相关分析
示例：为什么这个月销量下降？

3. 预测性分析
定义：会发生什么
问题：接下来会怎样？
方法：回归分析、时间序列、机器学习
示例：下个月的销售额预测

4. 规范性分析
定义：应该怎么做
问题：最佳策略是什么？
方法：优化模型、模拟分析
示例：如何定价才能最大化利润？
```

### 3.2 按方法分类
```
分析类型（按方法）：

对比分析：
├── 时间对比：同比、环比
├── 空间对比：不同地区、部门
├── 群体对比：不同用户群
└── 计划对比：实际vs计划

趋势分析：
├── 整体趋势：上升/下降/平稳
├── 季节性：周期性波动
├── 拐点分析：转折点识别
└── 预测外推：基于历史预测

分布分析：
├── 集中趋势：均值、中位数
├── 离散程度：方差、标准差
├── 数据分布：正态分布等
└── 异常识别：离群点检测

相关分析：
├── 相关关系：正相关/负相关
├── 相关程度：强/弱
├── 因果关系：需要验证
└── 影响因子：关键因素

漏斗分析：
├── 转化漏斗：每步转化率
├── 流失分析：用户流失点
├── 优化方向：提升转化
└── A/B测试：验证优化
```

## 4. 分析工具

### 4.1 工具类型
```
数据分析工具类型：

1. 表格工具
Excel/Google Sheets：
├── 适用：轻量分析、报表
├── 优点：普及、易用、灵活
├── 缺点：数据量大时卡顿
└── 核心技能：函数、数据透视表、图表

2. SQL数据库
MySQL/PostgreSQL：
├── 适用：数据提取、查询
├── 优点：处理大量数据
├── 缺点：需要写SQL
└── 核心技能：SQL查询

3. 编程工具
Python/R：
├── 适用：复杂分析、建模
├── 优点：灵活、强大
├── 缺点：学习成本高
└── 核心技能：数据处理、建模

4. BI工具
Tableau/Power BI/帆软：
├── 适用：可视化、报表
├── 优点：交互强、美观
├── 缺点：分析能力有限
└── 核心技能：可视化设计

5. 统计工具
SPSS/SAS：
├── 适用：统计分析
├── 优点：专业统计功能
├── 缺点：商业软件、成本高
└── 核心技能：统计分析
```

### 4.2 工具选择
```
工具选择原则：

原则1：匹配需求
├── 简单分析：Excel足够
├── 中等分析：SQL+Excel
├── 复杂分析：Python/R
└── 报表需求：BI工具

原则2：匹配技能
├── 会Excel：从Excel开始
├── 会SQL：SQL是基础
├── 会Python：进阶工具
└── 无基础：从简单工具学起

原则3：匹配场景
├── 日常工作：效率优先
├── 项目分析：需求导向
├── 建模需求：专业工具
└── 展示汇报：可视化工具

建议学习路径：
Excel → SQL → Python → 专业工具
```
