## 1. 对比分析

### 1.1 对比分析基础
```
对比分析定义：
通过对比发现差异，从而得出结论的分析方法。

对比分析类型：
├── 时间对比：不同时间段的对比
│   ├── 同比：今年vs去年
│   ├── 环比：本期vs上期
│   └── 特定日期对比：工作日vs周末
├── 空间对比：不同空间的对比
│   ├── 地域：不同城市/地区
│   ├── 部门：不同团队/业务线
│   └── 竞品：自己vs竞品
├── 群体对比：不同群体的对比
│   ├── 用户群：新用户vs老用户
│   ├── 渠道：不同渠道用户
│   └── 消费：高低价值用户
└── 计划对比：实际vs计划
    ├── 目标完成度
    ├── 预算执行率
    └── KPI达成率

对比分析原则：
├── 可比性：对比的对象要有可比性
├── 单一变量：一次只改变一个变量
├── 统一口径：数据定义要一致
└── 考虑背景：排除异常因素
```

### 1.2 对比分析应用
```
对比分析应用场景：

场景1：活动效果评估
活动前 vs 活动后
├── 指标：DAU、转化率、GMV
├── 分析：活动带来多少增量
└── 结论：活动效果如何

场景2：渠道质量评估
渠道A vs 渠道B
├── 指标：获客量、成本、留存
├── 分析：各渠道效率
└── 结论：优化渠道分配

场景3：版本优化评估
新版本 vs 旧版本
├── 指标：留存、功能使用
├── 分析：新版本是否更好
└── 结论：是否全量发布

场景4：AB测试
方案A vs 方案B
├── 指标：点击率、转化率
├── 分析：哪个方案更好
└── 结论：选择哪个方案

对比分析公式：
├── 绝对值变化 = 本期值 - 上期值
├── 相对变化 = (本期值 - 上期值) / 上期值
├── 完成度 = 实际值 / 目标值
└── 差异度 = A值 / B值
```

## 2. 趋势分析

### 2.1 趋势分析基础
```
趋势分析定义：
分析数据随时间变化的规律和趋势。

趋势类型：
├── 上升趋势：数据随时间增长
├── 下降趋势：数据随时间减少
├── 平稳趋势：数据基本不变
├── 波动趋势：周期性波动
└── 季节性趋势：周期性季节变化

趋势指标：
├── 增长量：绝对变化
│   └── = 本期值 - 基期值
├── 增长率：相对变化
│   └── = (本期值 - 上期值) / 上期值
├── 累计增长：累计变化
│   └── = 累计值 - 同期累计值
└── 复合增长率(CAGR)：年均复合增长
    └── = (期末值/期初值)^(1/年数) - 1

趋势分析维度：
├── 日趋势：按天观察
├── 周趋势：按周观察
├── 月趋势：按月观察
├── 季趋势：按季度观察
└── 年趋势：按年度观察
```

### 2.2 趋势分析应用
```
趋势分析应用场景：

场景1：业务监控
├── 日/周/月趋势监控
├── 发现异常波动
├── 及时预警处理
└── 持续跟踪优化

场景2：季节性分析
├── 识别季节性模式
├── 对比同期数据
├── 剔除季节因素
└── 发现真实趋势

场景3：增长分析
├── 计算增长率
├── 识别增长阶段
├── 分析增长驱动因素
└── 预测未来趋势

场景4：周期性分析
├── 识别周期模式
├── 预测周期性变化
├── 制定周期性策略
└── 合理规划资源

趋势分析方法：
├── 移动平均：平滑短期波动
├── 指数平滑：加权平均
├── 趋势线拟合：线性/指数拟合
├── 季节分解：分离趋势和季节
└── 预测模型：时间序列预测
```

## 3. 分布分析

### 3.1 分布分析基础
```
分布分析定义：
分析数据的分布形态，了解数据的集中程度和离散程度。

集中趋势指标：
├── 均值：数据的平均值
│   └── = 所有数据之和 / 数据个数
├── 中位数：排序后中间的值
│   └── 不受极端值影响
├── 众数：出现最多的值
│   └── 适用于类别数据
└── 分位数：特定比例的值
    ├── 25%分位数：Q1
    ├── 50%分位数：中位数
    └── 75%分位数：Q3

离散程度指标：
├── 方差：数据偏离均值的程度
│   └── = Σ(每个值-均值)² / n
├── 标准差：方差的平方根
│   └── = √方差
├── 极差：最大值-最小值
│   └── 最简单的离散指标
└── 四分位距：Q3-Q1
    └── 剔除极端值的影响

分布形态：
├── 正态分布：中间多两边少
├── 左偏分布：尾巴在左边
├── 右偏分布：尾巴在右边
└── 均匀分布：基本相等
```

### 3.2 分布分析应用
```
分布分析应用场景：

场景1：用户价值分布
├── 用户消费分布
├── 识别高价值用户
├── 制定差异化策略
└── RFM分层

场景2：收入分布
├── 收入区间分布
├── 了解收入结构
├── 识别收入异常
└── 制定收入策略

场景3：行为分布
├── 用户行为分布
├── 识别典型行为
├── 发现异常行为
└── 优化产品体验

场景4：指标分布
├── 转化率分布
├── 留存率分布
├── 识别优质/劣质来源
└── 优化获客策略

分布可视化：
├── 直方图：数值分布
├── 箱线图：统计分布
├── 密度图：概率密度
├── 饼图：占比分布
└── 帕累托图：二八法则
```

## 4. 相关分析

### 4.1 相关分析基础
```
相关分析定义：
分析两个或多个变量之间的关系。

相关关系类型：
├── 正相关：A增加B也增加
│   └── 例：广告投放增加，销量增加
├── 负相关：A增加B减少
│   └── 例：价格增加，销量减少
├── 无相关：A和B没有明显关系
│   └── 例：身高和收入（大部分情况）
└── 非线性相关：曲线关系
    └── 例：学习时间和成绩

相关系数：
├── r = 1：完全正相关
├── r = -1：完全负相关
├── r = 0：无相关
├── r > 0.7：强正相关
├── 0.3 < r < 0.7：中等相关
└── r < 0.3：弱相关

相关分析注意事项：
├── 相关不等于因果
├── 可能存在伪相关
├── 受样本量影响
├── 受异常值影响
└── 需要验证因果关系
```

### 4.2 相关分析应用
```
相关分析应用场景：

场景1：因素分析
├── 识别影响业务的关键因素
├── 分析各因素的影响程度
├── 优化关键投入
└── 例：分析影响GMV的因素

场景2：归因分析
├── 确定各渠道贡献
├── 评估各因素价值
├── 合理分配资源
└── 例：各营销渠道贡献

场景3：用户行为分析
├── 识别用户行为关联
├── 发现用户路径
├── 优化产品设计
└── 例：浏览和购买的关联

场景4：指标预测
├── 基于相关指标预测
├── 提前预警
├── 主动干预
└── 例：搜索量预测销量

相关分析步骤：
1. 明确分析目的
2. 选择分析变量
3. 收集相关数据
4. 计算相关系数
5. 验证统计显著性
6. 得出分析结论
```

## 5. 漏斗分析

### 5.1 漏斗分析基础
```
漏斗分析定义：
分析用户在关键路径上的转化情况。

漏斗分析要素：
├── 步骤：关键节点
├── 流量：每步的用户数
├── 转化率：上一步到下一步的比例
├── 流失率：流失的比例
├── 流失用户：流失了多少
└── 优化空间：可以提升多少

漏斗分析类型：
├── 转化漏斗：业务流程转化
│   └── 例：注册→开户→首投→复投
├── 行为漏斗：用户行为路径
│   └── 例：浏览→加购→下单→支付
├── 页面漏斗：页面浏览路径
│   └── 例：首页→列表→详情→下单
└── 渠道漏斗：获客转化路径
    └── 例：曝光→点击→注册→活跃

漏斗分析指标：
├── 每步转化率 = 本步用户数 / 上步用户数
├── 总体转化率 = 最终用户数 / 初始用户数
├── 每步流失率 = 流失用户数 / 上步用户数
├── 流失用户数 = 上步用户数 - 本步用户数
└── 优化空间 = 流失用户数 × 提升潜力
```

### 5.2 漏斗分析应用
```
漏斗分析应用场景：

场景1：电商转化漏斗
├── 浏览商品 → 加入购物车 → 提交订单 → 支付成功
├── 分析每步转化率
├── 识别流失最大环节
├── 针对性优化
└── 提升整体转化

场景2：用户注册漏斗
├── 访问注册页 → 开始注册 → 填写信息 → 注册成功
├── 分析注册流程
├── 优化注册体验
├── 提升注册转化率
└── 降低注册流失

场景3：产品激活漏斗
├── 下载 → 注册 → 首次使用 → 核心功能 → 活跃
├── 识别激活关键点
├── 优化新用户体验
├── 提升激活率
└── 提升用户价值

场景4：营销活动漏斗
├── 活动曝光 → 点击参与 → 完成转化 → 付费
├── 评估活动效果
├── 优化活动设计
├── 提升活动ROI
└── 积累活动经验

漏斗分析步骤：
1. 确定漏斗关键步骤
2. 收集各步骤数据
3. 计算转化率和流失率
4. 识别最大流失点
5. 分析流失原因
6. 提出优化建议
7. 落地执行验证
```
