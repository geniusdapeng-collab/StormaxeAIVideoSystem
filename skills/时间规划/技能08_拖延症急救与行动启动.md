## TL;DR

拖延解决核心：接纳现状→拆解任务→最小行动→即时奖励，四步打破拖延循环

## 1. 场景分析

### 请求摘要
用户拖延症发作，无法启动行动

### 显性需求
- 掌握拖延解决方法
- 获得可执行的行动方案

### 场景分类
- `scenario_classification`: 心理调适
- `urgency`: high

## 2. 拖延原因框架

### 常见拖延原因

| 原因 | 表现 | 解决方法 |
|------|------|----------|
| 完美主义 | 怕做不好 | 接受"完成>完美" |
| 困难感 | 任务太大 | 拆解到最小 |
| 无动力 | 不想做 | 找到意义 |
| 疲劳 | 精力不够 | 休息+启动 |
| 恐惧 | 害怕失败 | 行动化解 |

### 拖延循环

```
拖延 → 焦虑 → 逃避 → 更焦虑 → 更拖延
  ↓
打破：最小行动
```

## 3. 专业建议

### A. 2分钟启动法

#### 核心原则
```
如果任务2分钟内能做完 → 立即做
如果超过2分钟 → 先做2分钟
```

#### 操作步骤
```
1. 收到任务
2. 问：2分钟能开始吗？
3. 是：立即做
4. 否：做2分钟再说
```

### B. 最小行动

#### 拆解到不可能失败
```
原任务：写一份报告
最小行动：打开文档

原任务：健身1小时
最小行动：换上运动服

原任务：学习英语
最小行动：打开APP
```

### C. 动机激发

#### 利益可视化
```
1. 想象完成后好处
2. 写下完成的奖励
3. 告诉别人你要做
4. 想象不做的影响
```

### D. 即时奖励

#### 完成后立即奖励
```
1. 完成任务后
2. 给自己一个小奖励
3. 记录完成时刻
4. 积累成就感
```

### E. 环境设计

#### 减少阻力
```
1. 清除干扰物
2. 准备需要的工具
3. 设置启动仪式
4. 固定时间和地点
```
