## 一、流程总览

**视频生产 = 预生产 + 渲染/后期**

| 阶段 | 负责人 | 说明 |
|------|--------|------|
| **预生产** | 小G执行 + 大鹏把关 | 所有环节跑通，大鹏审核通过后进入渲染 |
| **渲染** | 小G提交 + 大鹏确认 | Seedance API调用，大鹏确认后才提交 |
| **后期制作** | 小G执行 | 拼接、转场、音效、字幕 |

**关键原则**：
- 预生产 = 生产，只是没点渲染按钮
- 所有环节必须全部跑通，不能跳过
- 预生产结尾有大鹏把关闸机
- 未经大鹏确认，严禁提交渲染

## 二、预生产完整流程（16+1个环节）

### 前置阶段：定妆照就绪检查（环节0）

#### 环节0：定妆照检查与生成
- **输入**：神兽ID
- **输出**：定妆照状态报告 + 定妆照文件（如需要生成）
- **负责人**：小G
- **流程分支**：
  - **有定妆照**：检查8张是否齐全，角度是否覆盖360°，锚点一致性是否通过
    - 全部通过 → 标记"定妆照已就绪"，进入环节1
    - 部分缺失 → 补充生成缺失角度，发送大鹏确认
  - **无定妆照**：启动定妆照生成流程
    1. 基于神兽档案15维度外观描述，生成8张多角度定妆照
    2. 锚点一致性校验（5维：面部/躯干/四肢/尾巴/特效）
    3. 发送大鹏审阅确认
    4. **大鹏确认"通过"** → 进入环节1
    5. 大鹏提出修改 → 重新生成 → 再次确认
- **定妆照清单**（8张）：
  1. 正面全身
  2. 正面半身
  3. 侧面全身
  4. 背面全身
  5. 前肢/武器特写
  6. 关键表情/状态
  7. 动态姿势
  8. 特殊能力展示
- **检查点**：
  - [ ] 8张全部生成成功
  - [ ] 角度覆盖360°
  - [ ] 锚点一致性通过（5维校验）
  - [ ] **大鹏确认"通过"**
- **⚠️ 重要**：
  - 此环节如果API欠费等外部原因阻塞，**标记为"阻塞"而非"跳过"**
  - 充值后立即补跑
  - **未经大鹏确认定妆照，严禁进入预生产环节1**

### 阶段2：导演创作（环节4-6）

#### 环节4：需求对齐闸机
- **输入**：PRD
- **输出**：需求对齐确认书
- **负责人**：小G执行，大鹏确认
- **对齐内容**：
  - 视频时长（60秒/90秒/120秒）
  - 镜头数量（6-12个）
  - 核心情感基调
  - 一镜到底数量（最少1个，最多2个）
  - 小G出场方式
- **检查点**：
  - [ ] 大鹏回复"确认"或"继续推进"

#### 环节5：故事板校验
- **输入**：PRD + 需求对齐确认
- **输出**：详细分镜故事板
- **负责人**：小G
- **故事板内容**：
  - 导演创作说明（基于PRD的调整）
  - 每个镜头的：编号/时长/叙事功能/情绪/景别/运镜/灯光/特效分级
  - 三幕结构校验
  - 情绪曲线图
  - 转场推荐
- **检查点**：
  - [ ] 三幕结构完整
  - [ ] 情绪曲线有起伏
  - [ ] 镜头连贯性通过

#### 环节5.5：镜头时间轴拆分（Shot Timeline Breakdown）【升级】
- **输入**：故事板（环节5输出）
- **输出**：秒级时间轴拆分表
- **负责人**：小G
- **拆分内容**：
  - 把每个镜头按秒拆分为子段落（0-3s / 3-7s / 7-10s等）
  - 每个子段落定义：画面内容/运镜方式/转场节点
  - 内部转场：match cut / hard cut / fade / 光爆闪切
  - 微运镜：极缓慢推进/拉远/固定+特效增强
- **示例**（S05化形之变10秒）：
  ```
  0-3s 准备期：九尾狐静立，固定机位极缓慢推进
  3-7s 变身期：毛发波动光透体出，固定+光效增强，光爆match cut
  7-10s reveal期：人形显现光芒收敛，极缓慢向后拉远，渐亮揭示
  ```
- **作用**：
  - Prompt可按子段落生成更精确描述
  - 后期拼接有明确剪辑点指导
  - 导演对每一秒都有精确控制
- **检查点**：
  - [ ] 每个镜头拆分为2-3个子段落
  - [ ] 每个子段落有明确画面内容
  - [ ] 内部转场节点定义清晰

#### 环节6：Schema校验
- **输入**：故事板 + 时间轴拆分表
- **输出**：Schema校验报告
- **负责人**：小G自动执行
- **校验内容**：
  - story-plan结构完整性
  - shot-prompt格式规范
  - character-asset引用正确
  - 时长总和=目标时长
  - 时间轴拆分与镜头时长匹配
- **检查点**：
  - [ ] 所有镜头时长之和 = 目标总时长
  - [ ] 每个镜头有明确的叙事功能
  - [ ] 没有重复或雷同镜头
  - [ ] 时间轴拆分时长 = 镜头总时长
