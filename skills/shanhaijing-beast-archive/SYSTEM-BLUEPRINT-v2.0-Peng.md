## 一、系统升级核心方向（vs 白皮书）

### 1.1 超越点总览

| 维度 | 白皮书设计 | 我们的升级 |
|------|-----------|-----------|
| **档案字段** | 10个基础维度 | **15个生产级维度**（增加Prompt工程专用字段） |
| **Prompt质量** | 60字简单描述 | **分层Prompt系统**（compact 40字 / standard 80字 / detailed 150字 / cinematic 200字） |
| **一致性校验** | 5维基础检查 | **8维深度校验**（增加材质纹理/动态特征/声音特征） |
| **Nirath融合** | 简单栖息地匹配 | **生态位深度嵌入**（食物链位置/能量来源/与其他神兽关系） |
| **小G交互** | 无 | **神兽-小G关系档案**（友好度/互动方式/剧情触发条件） |
| **生产就绪度** | 基础数据 | **Seedance直接可用**（含参考电影片段/镜头语言/特效参数） |
| **版本管理** | 无 | **档案版本追溯**（每次更新记录变更日志） |
| **批量导入** | 简单MD解析 | **智能富化引擎**（自动扩展/交叉引用/冲突检测） |

### 1.2 新增核心能力

1. **Prompt工程专用字段**
   - `promptLayers`: 分层Prompt（外观层/动作层/环境层/情绪层/技术层）
   - `seedanceOptimized`: 针对Seedance 2.0优化的490字精编Prompt
   - `negativePrompt`: 否定Prompt（防止出现的不想要元素）
   - `styleAnchors`: 风格锚点词（确保跨镜头风格一致）

2. **小G关系档案**
   - `xiaoGInteraction`: 小G与该神兽的互动方式
   - `affinity`: 友好度（-100到+100）
   - `firstEncounter`: 首次相遇剧情模板
   - `questTrigger`: 任务触发条件

3. **生产级元数据**
   - `renderComplexity`: 渲染复杂度评级（1-10）
   - `estimatedTokens`: 预估API调用token数
   - `recommendedRatio`: 推荐画幅比例
   - `lightingSetup`: 灯光配置建议
   - `cameraSettings`: 相机参数建议

4. **生态位深度嵌入**
   - `foodChain`: 食物链位置（生产者/消费者/顶级掠食者/分解者）
   - `energySource`: 能量来源（恒星辐射/地热/生物能/灵能）
   - `symbiosis`: 共生关系（与哪些神兽/植物/环境共生）
   - `territory`: 领地范围和行为模式

## 二、15个生产级维度详述

### 维度1-10：基础档案（继承并升级）

**① 神兽身份信息**
- 中文名称、拼音、别名
- 山海经出处（原文+译文）
- 神话地位、等级分类

**② 综合介绍**
- 200-300字精炼概述
- 核心定位一句话

**③ 外观描述（升级）**
- **核心锚点**：一句话精准描述（用于快速识别）
- **分层描述**：
  - L1 整体形态（全身比例/姿态）
  - L2 头部特征（面部/眼睛/角/耳）
  - L3 躯干特征（鳞片/毛发/纹理/颜色）
  - L4 四肢/尾部（爪子/蹄子/翅膀/尾巴）
  - L5 特殊标记（花纹/发光部位/独特器官）
- **材质纹理**：鳞片/毛发/皮肤的具体质感描述
- **动态特征**：行走/飞行/攻击时的体态变化

**④ 神通能力（升级）**
- 结构化列表（名称+描述+视觉表现+特效建议）
- 能力等级（普通/稀有/传说/创世）
- 能力触发条件

**⑤ 传说故事**
- 完整故事叙述
- 关键场景提取（可用于分镜）

**⑥ 象征寓意**
- 文化象征
- 现代启示

**⑦ 影视创作建议（升级）**
- 参考电影列表（具体到片段/镜头）
- 特效技术参数
- 色彩分级建议
- 声音设计方向

**⑧ Nirath融合设定（升级）**
- 栖息地精确坐标
- 生态位（食物链位置/能量来源）
- 与Nirath其他生物关系
- 与小G的相遇剧情

**⑨ 相关神兽网络**
- 血缘关系
- 敌对/友好关系
- 生态共生关系

**⑩ 档案元数据**
- 版本号
- 创建时间
- 审核状态
- 使用次数统计

### 4.2 引擎升级

**Prompt注入器 v2**
- 支持4层Prompt（compact/standard/detailed/cinematic）
- 自动根据镜头类型选择Prompt长度
- 支持多神兽同框的Prompt融合

**一致性守卫 v2**
- 8维校验（新增材质/动态/声音）
- 跨档案关联检查（确保相关神兽形象一致）
- 版本对比功能

**世界观校准器 v2**
- 生态位自动校验
- 食物链冲突检测
- 栖息地承载量计算

**运镜推荐器 v2**
- 基于体型+类型+场景三维推荐
- 支持小G同框的特殊构图
- 参考电影片段自动关联

**场景生成器 v2**
- 时间×天气×生态位三维组合
- 动态生态描述（其他生物活动）
- 声音环境生成

**新增：生态模拟器**
- 神兽在Nirath的行为模拟
- 与小G的交互剧情生成
- 任务链自动构建

### 4.3 集成升级

```javascript
// 导演管线集成示例
class BeastArchiveIntegration {
  // 自动检测并注入
  processStoryPlan(storyPlan) {
    const beasts = this.detectBeasts(storyPlan);
    
    for (const beast of beasts) {
      // 加载完整档案
      const archive = this.loadArchive(beast.id);
      
      // 注入分层Prompt
      storyPlan = this.injectPromptLayers(storyPlan, archive);
      
      // 校验生态位
      this.validateEcosystem(storyPlan, archive);
      
      // 推荐运镜
      this.recommendCinematography(storyPlan, archive);
      
      // 生成小G交互
      this.generateXiaoGInteraction(storyPlan, archive);
    }
    
    return storyPlan;
  }
}
```

## 五、质量标准

### 5.1 档案质量检查清单

- [ ] 外观描述 ≥ 300字，包含5个层次
- [ ] 能力描述 ≥ 5项，每项有视觉表现
- [ ] 故事叙述 ≥ 300字，有完整情节
- [ ] Nirath融合 ≥ 500字，包含生态位
- [ ] Prompt分层 ≥ 4层，每层有明确用途
- [ ] 小G交互有明确友好度和触发条件
- [ ] 参考电影 ≥ 3部，具体到片段
- [ ] 生态位描述完整（食物链/能量/共生）
- [ ] 版本日志可追溯
- [ ] Seedance 490字Prompt通过校验

### 5.2 世界一致性标准

- [ ] 栖息地符合Nirath地理
- [ ] 生态位不冲突（两个顶级掠食者不同领地）
- [ ] 能量来源符合Nirath物理规则
- [ ] 与小G的关系符合剧情时间线
- [ ] 相关神兽的描述互相印证
