## 一、模块概述

### 1.1 模块定位

本模块是ShanhaiStory Forge v2.0-Peng的核心子系统，负责将原始异兽JSON数据转化为**15维度深度精修版档案**，每只档案约12000-14000字节，质量评分50/50，可直接用于Seedance视频生产管线。

### 1.2 核心能力

- **输入**：原始异兽JSON数据（含名称、外观、能力、传说等基础信息）
- **输出**：15维度完整版Markdown档案（含Prompt工程套件、小G交互档案、生产就绪参数）
- **标准**：P3批次深度精修标准（v2.0-Peng）
- **质量**：50/50满分标准，可直接投入生产

### 1.3 模块架构

```

├── 输入层：原始JSON数据解析
```

## 三、生成流程规范

### 3.1 输入要求

**原始JSON数据必须包含：**
```json
{
  "id": "神兽ID（英文）",
  "name": "中文名称",
  "nameEn": "英文名称",
  "no": "编号",
  "category": "类别",
  "origin": "出处",
  "nameDetails": {
    "chinese": "中文名",
    "pinyin": "拼音",
    "aliases": ["别名1", "别名2"]
  },
  "appearance": {
    "summary": "外观摘要",
    "fullDescription": "外观详细描述"
  },
  "abilities": ["能力1", "能力2"],
  "stories": ["传说1", "传说2"],
  "symbolism": ["象征1", "象征2"],
  "relatedBeasts": ["相关神兽1", "相关神兽2"],
  "nirath": {
    "habitat": "Nirath栖息地",
    "adaptation": "外星生态适应",
    "humanRelation": "与人类关系",
    "storylines": ["科幻故事线1", "科幻故事线2"]
  }
}
```

### 3.2 生成步骤

**Step 1：数据解析**
- 读取原始JSON数据
- 提取核心信息（名称、外观、能力、传说等）
- 确定神兽类别（守护/凶兽/瑞兽/灾兽等）

**Step 2：内容生成（按维度顺序）**
1. 身份信息（基于JSON直接提取+补充）
2. 综合介绍（基于JSON摘要+扩展）
3. 外观描述（基于JSON外观描述扩展至700+字）
4. 神通能力（基于JSON能力扩展至5项，每项加科幻原理）
5. 传说故事（基于JSON传说扩展至3则，加视觉高潮点）
6. 象征寓意（基于JSON象征扩展+现代启示）
7. 影视创作建议（基于神兽特征设计）
8. Nirath融合设定（基于JSON Nirath数据扩展）
9. 相关神兽网络（基于JSON相关神兽+扩展）
10. 档案元数据（自动生成）
11. Prompt工程套件（基于外观+动作+环境生成）
12. 小G交互档案（基于神兽性格设计）
13. 生产就绪参数（基于渲染复杂度评估）
14. 生态位档案（基于栖息地设计）
15. 版本审计日志（自动生成）

**Step 3：质量检查**
- 检查15维度完整性（缺一不可）
- 检查字数标准（12000-14000字节）
- 检查Prompt字数（490字精编Prompt）
- 检查科幻原理术语（至少5个具体术语）
- 检查Nirath融合度（栖息地具体参数+3条科幻故事线）
- 检查小G交互完整性（友好度+相遇剧情+任务+羁绊+奖励）

**Step 4：输出归档**
- 生成Markdown文件：`{id}-v2.0-Peng-final.md`
- 存入目录：`docs/`
- 更新索引：`INDEX.md`
- 记录审计日志

### 3.3 禁止事项

- ❌ 严禁跳过任何维度（15维度缺一不可）
- ❌ 严禁使用占位符文本（如"九尾狐完整描述，490字精编Prompt"）
- ❌ 严禁使用中国传统符号（阴阳、八卦、太极、五行等）
- ❌ 严禁使用模拟数据或假数据
- ❌ 严禁批量生成降质（必须逐只精修）
- ❌ 严禁少于12000字节

## 四、Prompt工程规范

### 4.1 分层Prompt系统

**外观层（80字）**
- 基于外观描述提炼核心视觉特征
- 包含：体型、毛色/肤色、头部特征、标志性器官

**动作层（60字）**
- 基于动态特征提炼核心动作
- 包含：行走/飞行/游动方式、标志性动作、攻击/防御姿态

**环境层（60字）**
- 基于Nirath栖息地提炼环境特征
- 包含：地理位置、气候特征、标志性景观

**情绪层（40字）**
- 基于象征寓意提炼核心情绪
- 包含：威严/优雅/恐怖/神圣等

**技术层（50字）**
- 统一技术参数
- 包含：IMAX级史诗渲染、镜头规格、体积光、ACES色彩管理

### 4.2 490字精编Prompt生成规则

1. **基于外观层**：提取体型、毛色、头部、标志性器官描述
2. **基于动作层**：提取核心动作、标志性姿态
3. **融合环境层**：加入Nirath栖息地特征
4. **加入情绪层**：表达核心情绪
5. **加入技术层**：统一技术参数
6. **加入免责声明**：DISCLAIMER: This is a completely fictional digital creature generated by AI, NOT a real animal.
7. **字数控制**：严格控制在490字左右（允许±20字误差）

### 4.3 否定Prompt标准（9条）

1. 真实动物/真实[神兽类别]
2. 可爱/萌/卡通风格
3. 现代场景/城市/科技建筑
4. 卡通/动画/皮克斯风格
5. 低质量/模糊/过度曝光/噪点
6. 人类演员/真实人脸
7. 西方魔幻/欧式风格
8. 恐怖血腥/丧尸风格
9. 色彩鲜艳/高饱和/快乐氛围

### 4.4 角色一致性锚点（5维）

**面部锚点**：
- 眼睛颜色/形状（必须保留）
- 瞳孔特征（微光/竖瞳等）
- 面部轮廓
- 标志性表情

**躯干锚点**：
- 主体颜色/材质（必须保留）
- 体型特征
- 标志性器官（翅膀/尾巴/角等）
- 纹理细节

**四肢/翅膀锚点**：
- 四肢/翅膀形态（必须保留）
- 爪子/蹄子/翅膀特征
- 标志性动作

**尾巴锚点**（如有）：
- 尾巴数量/形态（必须保留）
- 尾巴动作
- 尾巴特效

**特效锚点**：
- 标志性能力视觉表现
- 环境互动效果
- 情绪表达特效
