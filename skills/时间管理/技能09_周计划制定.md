## TL;DR

周计划 = 回顾上周 + 确定目标 + 分解任务 + 分配时间 + 留有缓冲 + 灵活调整

## 1. 周计划认知

### 1.1 为什么要做周计划
```
周计划的价值：
├── 承上启下：连接月计划和日计划
├── 目标清晰：知道本周要完成什么
├── 时间保障：确保重要的事有时间做
├── 减少焦虑：知道一周要做什么
├── 便于复盘：知道一周做了什么
└── 培养习惯：建立周期规划习惯
```

### 1.2 周计划常见问题
```
周计划常见问题：
├── 计划太满：没有留buffer
├── 不切实际：完不成
├── 太空洞：只有大目标没分解
├── 没有优先级：什么都要做
├── 不灵活：变化就打乱
├── 只制定不执行：计划成空想
└── 不知道做什么：没有方向
```

### 1.3 周计划原则
```
周计划原则：
├── 聚焦目标：每周2-3个核心目标
├── 分解任务：大目标分解成小任务
├── 分配时间：把任务分配到每天
├── 留有缓冲：预留10-20%时间
├── 考虑会议：预留会议时间
├── 灵活调整：根据实际调整
└── 执行优先：计划了就要执行
```

## 2. 周计划制定

### 2.1 周计划时间
```
周计划时间安排：

周初制定（30-60分钟）：
├── 时间：周日晚上或周一早晨
├── 内容：制定本周计划
├── 方式：回顾+规划
└── 工具：纸笔或数字工具

周中调整（10分钟）：
├── 时间：周三或周四
├── 内容：检查进度，调整计划
└── 方式：快速回顾

周末复盘（30分钟）：
├── 时间：周五晚上
├── 内容：回顾本周完成情况
├── 方式：对照计划检查
└── 为下周做准备
```

### 2.3 目标设定
```
周目标设定：

目标类型：
├── 项目型：完成某个项目/里程碑
├── 能力型：学习某个技能
├── 习惯型：养成某个习惯
└── 问题型：解决某个问题

目标设定标准：
├── 具体：知道要完成什么
├── 可衡量：知道怎么算完成
├── 可实现：踮脚能够到
├── 相关：和季度/月度目标相关
├── 有时限：本周内能完成
└── 聚焦：2-3个核心目标

目标示例：
❌ "提升能力"（太模糊）
✅ "完成XX项目的需求文档"（具体）

❌ "好好学习"（太空洞）
✅ "每天学习英语30分钟"（具体可执行）
```

## 3. 任务分解

### 3.1 分解方法
```
任务分解：

分解原则：
├── 具体：每个子任务清晰
├── 可执行：马上就能做
├── 有成果：做完有可见成果
├── 时间短：每个不超过2小时
└── 独立：子任务可以独立完成

分解示例：

❌ 大任务：完成XX项目
✅ 小任务分解：
├── 周一：完成项目计划
├── 周二：需求调研
├── 周三：需求文档初稿
├── 周四：评审需求文档
└── 周五：根据评审修改

分解后更容易：
- 每天知道要做什么
- 不会觉得无从下手
- 完成子任务有成就感
```

### 3.2 任务分配
```
任务分配到天：

分配原则：
├── 重要任务优先
├── 考虑精力周期
├── 留有缓冲时间
├── 考虑会议安排
└── 不要太均匀

分配示例：

周一：
├── 9-11点：重要任务1（高效时段）
├── 11-12点：处理邮件
├── 14-16点：重要任务2
└── 16-18点：常规任务

周五：
├── 上午：重要任务
├── 下午：收尾工作
└── 下班前：周复盘

考虑精力：
├── 周一/周五精力可能较低
├── 周二/周三精力较高
├── 把重要任务放精力高的日子
```

### 3.3 考虑固定事项
```
考虑固定事项：

每周固定事项：
├── 例会：周例会、周评审
├── 汇报：周报、月报
├── 沟通：固定团队沟通
├── 邮件：邮件处理时间
├── 复盘：周复盘
└── 休息：给自己留休息时间

处理方式：
├── 提前规划到日历
├── 估算时间
├── 围绕固定事项安排其他任务
└── 不要让固定事项填满
```

## 4. 执行技巧

### 4.1 每日执行
```
每日执行：

早晨规划（10分钟）：
├── 查看今日任务
├── 确定3件最重要的事
├── 预估时间
└── 排列顺序

白天执行：
├── 从最重要的事开始
├── 用番茄钟保持专注
├── 记录完成情况
└── 适时调整

晚间收尾（10分钟）：
├── 标记完成的任务
├── 调整明天的任务
├── 记录未完成的原因
└── 为明天做准备
```

### 4.2 进度跟踪
```
进度跟踪：

每日检查：
├── 今日任务完成了吗
├── 有什么阻碍
├── 需要调整吗
└── 明天要做什么

周中调整：
├── 周三检查本周目标完成进度
├── 完成顺利：继续保持
├── 进度落后：分析原因
├── 调整计划：推迟或增加资源
└── 如果无法完成：确定优先顺序

调整原则：
├── 优先保证核心目标
├── 可推迟次要任务
├── 不要因为调整放弃
└── 记录调整原因
```

### 4.3 应对变化
```
应对变化：

正常变化：
├── 临时任务加入：评估优先级后安排
├── 会议增加：压缩其他任务
├── 任务延期：调整后续计划
└── 计划延期：调整到其他天

大幅变化：
├── 重新评估本周目标
├── 推迟非核心任务
├── 必要时加班
└── 反思计划合理性

处理原则：
├── 保持冷静
├── 评估影响
├── 调整计划
├── 沟通协调
└── 继续执行
```

## 5. 周复盘

### 5.1 复盘内容
```
周复盘内容：

完成情况：
├── 核心目标完成了吗？
├── 完成的任务有哪些？
├── 未完成的任务有哪些？
└── 完成率是多少？

做得好的：
├── 哪些任务执行得好？
├── 哪些习惯坚持得好？
├── 有什么有效的方法？
└── 可以继续做什么？

需要改进的：
├── 哪些任务没完成？
├── 计划为什么没完成？
├── 执行中有什么问题？
└── 需要改变什么？

下周改进：
├── 下周的核心目标是什么？
├── 需要调整计划方法吗？
├── 有什么要特别注意的？
└── 需要什么资源？
```

### 5.3 复盘技巧
```
复盘技巧：

诚实面对：
├── 不要自我欺骗
├── 承认做不好的地方
├── 但不要过度自责
└── 目的是改进，不是评判

找到原因：
├── 分析未完成的原因
├── 是计划问题还是执行问题
├── 是外部原因还是内部原因
└── 找到真正的原因才能改进

持续改进：
├── 基于复盘调整下周计划
├── 尝试不同的方法
├── 记录有效的方法
└── 逐步建立自己的节奏
```
