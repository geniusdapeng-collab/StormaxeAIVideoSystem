## 2. 需求分析框架

### 需求来源

| 来源 | 方法 | 适用场景 |
|------|------|----------|
| 用户访谈 | 深度1v1 | 核心用户 |
| 问卷调研 | 大规模 | 验证假设 |
| 数据分析 | 行为数据 | 优化方向 |
| 竞品分析 | 市场调研 | 功能参考 |
| 内部需求 | 业务方 | B端产品 |

### 需求层次

```
1. 业务需求：业务目标是什么
2. 用户需求：用户想要什么
3. 功能需求：具体做什么
4. 非功能需求：性能/安全等
```

## 3. 专业建议

### A. 用户研究

#### 研究方法
```
1. 访谈法：深入了解痛点
2. 观察法：看用户怎么用
3. 问卷法：量化验证假设
4. 数据法：行为数据分析
5. 竞品法：市场参考
```

### B. 需求收集

#### 收集方法
```
1. 需求池：统一收集入口
2. 模板规范：统一格式
3. 来源标记：知道从哪来
4. 状态跟踪：处理进度
5. 定期清理：删除无效需求
```

### C. 需求筛选

#### 筛选方法
```
1. 真实性：是不是真需求
2. 价值：带来什么价值
3. 可行性：能不能实现
4. 成本：需要多少资源
5. 优先级：紧急重要程度
```

### D. 优先级排序

#### 排序方法
```
1. KANO模型：基本/期望/兴奋
2. MoSCoW：必须/应该/可以/不会
3. ICE：影响/信心/容易
4. RICE：影响/信心/努力/收益
```
