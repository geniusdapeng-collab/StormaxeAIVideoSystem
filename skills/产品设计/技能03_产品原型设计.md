## 2. 原型设计框架

### 原型保真度

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 低保真 | 线框图 | 早期验证 |

| 高保真 | 视觉稿 | 最终确认 |

### 页面要素

```
1. 导航：怎么跳转
2. 内容：展示什么
3. 交互：怎么操作
4. 状态：不同情况
5. 异常：出错怎么办
```

## 3. 专业建议

### A. 页面梳理

#### 梳理方法
```
1. 核心页面：主要功能页
2. 辅助页面：支持功能页
3. 通用页面：设置/帮助等
4. 异常页面：报错页等
5. 页面关系：层级结构
```

### B. 流程设计

#### 设计方法
```
1. 用户流程：从哪到哪
2. 页面流转：怎么跳转
3. 分支流程：不同情况
4. 返回路径：怎么回去
5. 中断处理：退出怎么办
```

### C. 信息布局

#### 布局原则
```
1. 层次分明：主次分明
2. 逻辑清晰：易理解
3. 操作方便：触手可及
4. 留白适当：不拥挤
5. 一致性：统一规范
```

### D. 交互设计

#### 设计要点
```
1. 手势：点击/滑动/拖拽
2. 动画：过渡效果
3. 反馈：操作响应
4. 提示：引导说明
5. 容错：允许撤销
```
