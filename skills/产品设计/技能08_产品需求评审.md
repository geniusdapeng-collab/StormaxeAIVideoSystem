## 2. 评审框架

### 评审类型

| 类型 | 时机 | 参与方 |
|------|------|--------|
| 需求评审 | 开发前 | 产品/研发/测试 |
| 设计评审 | 设计后 | 产品/设计/研发 |
| 上线评审 | 上线前 | 产品/测试/运维 |
| 复盘评审 | 上线后 | 全团队 |

### 评审流程

```
1. 评审前：准备文档/预约时间
2. 评审中：讲解/讨论/决策
3. 评审后：记录问题/跟进解决
4. 问题闭环：确认修复/验证
5. 持续优化：改进评审流程
```

### A. 评审前准备

#### 准备要点
```
1. 文档完善：提前发出
2. 时间预约：提前预约
3. 参会人员：确定参与
4. 评审议程：明确流程

```

### B. 评审中控场

#### 控场技巧
```
1. 背景介绍：先讲背景
2. 目标明确：再讲目标
3. 讲解逻辑：主线清晰
4. 讨论有序：先讨论后决策
5. 决策明确：当场做决定
```

### C. 问题处理

#### 处理方法
```
1. 记录问题：全部记录
2. 问题分类：技术/业务/设计
3. 优先级：高/中/低
4. 责任人：指定人
5. 解决时间：约定时间
```

### D. 评审后跟进

#### 跟进要点
```
1. 发出纪要：及时发出
2. 问题跟踪：进度跟进
```
