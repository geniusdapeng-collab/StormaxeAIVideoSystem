## TL;DR

复盘核心：目标回顾→结果评估→原因分析→经验总结→改进落地，五步完成产品复盘

### 显性需求

### 场景分类
- `scenario_classification`: 产品管理
- `urgency`: normal

## 2. 复盘框架

### 复盘类型

| 类型 | 时机 | 重点 |
|------|------|------|
| 版本复盘 | 每次发版后 | 本版问题 |
| 项目复盘 | 项目完成后 | 项目经验 |
| 季度复盘 | 每季度 | 整体复盘 |

### 复盘流程

```
1. 目标回顾：当初目标是什么
2. 结果评估：实际结果如何
3. 原因分析：为什么会这样
4. 经验总结：学到了什么
5. 改进落地：下次怎么做
```

## 3. 专业建议

### A. 目标回顾

#### 回顾要点
```
1. 原始目标：当初想做什么
2. 衡量标准：怎么算成功
3. 预期资源：需要什么支持
4. 预期风险：可能的问题
5. 目标调整：过程中有无调整
```

### B. 结果评估

#### 评估维度
```
1. 目标达成：完成了多少
2. 质量情况：效果好不好
3. 时间进度：按时完成没
4. 资源使用：成本超支没
5. 用户反馈：用户满意吗
```

### C. 原因分析

#### 分析方法
```
1. 5Why分析法：追根究底
2. 鱼骨图：分类找原因
3. 颗粒度：细化到具体
4. 主客观：区分内外部
5. 数据驱动：用数据说话
```

### D. 改进落地

#### 落地方法
```
1. 改进清单：列出行动项
2. 责任人：指定负责人
3. 完成时间：设定截止
4. 跟进机制：定期检查
5. 效果验证：确认改进有效
```
