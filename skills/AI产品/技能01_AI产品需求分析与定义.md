### 需求结构

```
1. 背景：为什么做

3. 用户：谁会用
4. 场景：在什么情况下
5. 功能：具体做什么
6. 优先级：先做哪个
7. 验收：怎么算完成
```

### A. 用户研究

```
1. 访谈：深入了解需求
```

### B. 场景拆解

#### 拆解方法
```
1. 主流程：核心使用路径
2. 分支流程：异常情况
3. 边界条件：限制情况
4. 极端情况：压力测试
```

### C. 功能定义

#### 定义要素
```
1. 输入：用户提供什么
2. 处理：AI怎么理解/生成
3. 输出：返回什么结果
4. 交互：怎么展示/操作
5. 异常：出错怎么办
```
