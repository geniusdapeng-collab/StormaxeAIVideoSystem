## 2. 竞品分析框架

### 竞品分类

| 类型 | 说明 | 分析重点 |
|------|------|----------|
| 直接竞品 | 同一赛道 | 功能/体验/价格 |
| 间接竞品 | 替代方案 | 解决同一需求 |
| 潜在竞品 | 新进入者 | 动向/融资/团队 |

### 分析维度

```
1. 产品：功能/体验/技术
2. 商业：盈利模式/定价
3. 用户：规模/画像/评价
4. 运营：推广/渠道/活动
5. 团队：背景/资源/融资
```

## 3. 专业建议

### A. 信息收集

#### 收集方法
```
1. 产品体验：深度使用
2. 用户评价：App Store/知乎
3. 数据分析：SimilarWeb/七麦
4. 行业报告：艾瑞/易观
5. 招聘数据：拉勾/Boss
```

### B. 分析方法

#### 分析框架
```
1. 核心功能：解决什么问题
2. 用户体验：交互/视觉
3. 技术能力：AI效果/性能
4. 商业模式：如何盈利
5. 市场表现：排名/增速
```

### C. 市场定位

#### 定位方法
```
1. 细分市场：选择切入点
2. 差异化：找到独特价值
3. 价值主张：核心宣传点
4. 目标用户：服务谁
5. 定价策略：怎么收费
```

### D. 竞争策略

#### 策略选择
```
1. 差异化：做得不同
2. 成本领先：价格优势
3. 聚焦：专注细分
4. 生态协同：借力生态
5. 先发优势：快速占领
```
