## 2. 技术框架

### AI能力评估

| 维度 | 内容 | 评估要点 |
|------|------|----------|
| 准确性 | AI输出质量 | 能否满足要求 |
| 稳定性 | 表现一致性 | 波动大不大 |
| 响应速度 | 处理时间 | 能否接受 |
| 成本 | 资源消耗 | 是否可控 |
| 局限性 | 边界在哪 | 什么做不到 |

### 技术沟通要素

```
1. 背景：为什么做
2. 目标：做成什么样
3. 约束：有什么限制
4. 风险：可能的问题
5. 验收：怎么算完成
```

## 3. 专业建议

### A. 技术理解

#### AI产品经理需了解
```
1. 大模型能力：能做什么
2. 提示词工程：怎么用好
3. 模型微调：怎么定制
4. 评估方法：怎么测效果
5. 成本构成：钱花在哪
```

### B. 需求表达

#### 清晰表达要点
```
1. 用户故事：谁在什么场景做什么
2. 验收标准：怎么算完成
3. 示例输入输出：具体例子
4. 边界情况：例外怎么处
5. 优先级：哪些必须哪些可选
```

### C. 技术评估

#### 评估维度
```
1. 可行性：技术上能实现吗
2. 工作量：需要多久
3. 风险点：可能有什么问题
4. 替代方案：有没有简单方案
5. 效果预期：能做到什么程度
```

### D. 协作方法

#### 高效配合技巧
```
1. 提前沟通：需求前置讨论
2. 小步快跑：先做最小版本
3. 及时反馈：问题尽早暴露
4. 数据驱动：用数据说话
5. 相互尊重：理解彼此压力
```
