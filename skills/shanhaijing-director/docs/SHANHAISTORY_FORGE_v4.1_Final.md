# 第一部分：总则与制作原则

## 1.1 规范定位

本文档用于规范 **山海经视频生产系统 / Nirath 系统** 的场次设计、镜头设计、Prompt 构建、角色控制、审片、渲染与交付流程。本规范以 **镜头生产结果与成片成立** 为核心。

本规范适用于：
- 山海经神话系列
- 统一 Seedance 视频生产平台
- 角色连续叙事型短片
- 需要跨镜头一致性与系列化产能的 AI 影视项目
- 专业影视制作团队的标准化生产管理

---

## 1.2 核心目标

系统目标为以下五项：

1. **镜头意图清晰 (Clear Shot Intention)**
 每个镜头必须知道"为什么存在"，而不是只描述"画面里有什么"。

2. **叙事服务优先 (Narrative-First Approach)**
 运镜、光影、材质、声音都必须服务于叙事与情绪，不得喧宾夺主。

3. **角色与世界观稳定 (Character & World Consistency)**
 角色一致性、环境逻辑、系列风格必须稳定，但允许在戏剧需要下产生层次变化。

4. **成片可交付 (Delivery-Ready Output)**
 所有规范最终都必须服务于"可审、可改、可剪、可交付"。

5. **整体优先于单镜炫技 (Holistic Over Individual)**
 单镜优秀但不服务整体叙事，不视为合格产出。

---

## 1.3 制作原则

| 原则 | 英文术语 | 说明 |
|------|----------|------|
| 意图优先 | Intention Priority | Prompt 不是目的，镜头表达才是目的 |
| 场次优先于单镜 | Scene Over Shot | 镜头必须服务场次功能，不得只追求单镜漂亮 |
| 主次清晰 | Clear Hierarchy | 每个镜头只允许一个主视觉中心 |
| 可读性优先 | Readability First | 主体、动作、空间关系必须一眼可辨 |
| 连续性分级 | Continuity Grading | 不是所有镜头都必须帧级连续，按叙事需求决定 |
| 风格统一而非单调统一 | Coherent But Not Monotonous | 同一世界观内允许光线、节奏、尺度变化 |
| 声画协同 | Sound-Picture Synergy | 画面、对白、环境声、音乐共同建立情绪 |
| 可控优先于堆砌 | Controllable Over Complex | 禁止为了"丰富"而增加无效信息 |
| 表演可信 | Believable Performance | 动作不等于表演，角色反应必须成立 |
| 交付导向 | Delivery-Oriented | 所有生成策略都必须考虑后期与交付 |

## 2.3 专业景别标准 / Standard Shot Size Classification

本规范采用影视行业标准景别分类，与 Shot Card 中的"景别"字段对应：

| 景别代码 | 中文名称 | 英文名称 | 画面范围 | 叙事功能 |
|----------|----------|----------|----------|----------|
| ELS | 大全景 | Extreme Long Shot | 人物占画面高度 1/4 以下 | 建立世界观、空间规模、环境全貌 |
| LS / FS | 全景 | Long Shot / Full Shot | 人物全身 + 完整环境 | 展示角色与环境关系、人物动作全貌 |
| MFS | 中全景 | Medium Full Shot | 人物膝部以上 | 兼顾动作与表情，适合肢体表演 |
| MS | 中景 | Medium Shot | 人物腰部以上 | 动作与关系镜，对话场景标准景别 |
| MCU | 中近景 | Medium Close-Up | 人物胸部以上 | 表情与情绪镜，强调面部表演 |
| CU | 近景 / 特写 | Close-Up | 人物肩部以上 / 面部填满画面 | 表情、情绪、心理变化核心镜 |
| ECU | 大特写 | Extreme Close-Up | 眼部/嘴部/关键物件细节 | 关键物件、眼神、转折细节 |
| OTS | 过肩镜头 | Over-The-Shoulder | 一角色肩部上方拍摄另一角色 | 对话场景标准机位，建立空间关系 |
| POV | 主观视角 | Point of View | 角色所见视角 | 沉浸、恐惧、探索，全片限 0-2 个 |
| Insert | 插入镜头 | Insert Shot | 手部动作 / 物件细节 / 文字 | 补充信息、强调关键道具 |

**景别变化规则**：
- 一个镜头内景别变化最多 1-2 次（如推镜头从 FS 推到 MCU）
- 不允许在短时长内（<5秒）频繁跨级切换（如 CU → ELS → CU）
- 近景/特写必须有明确表演理由

# 第三部分：场次执行卡（Scene Card）规范

## 3.1 Scene Card 的地位

Scene Card 是 Shot Card 的上游控制文档。所有镜头必须归属于某个场次，所有场次必须先定义其叙事功能、情绪方向和视觉总策略，再拆分镜头。

核心生产链路：

```
故事需求 → Scene Card → Shot Card → Prompt → 渲染结果 → 审片 → 后期 → 交付
```

**专业影视制作对应**：

```
剧本/大纲 → 场次分解表 (Breakdown) → 分镜表 (Storyboard) → 拍摄单 (Shot List) → 实拍/渲染 → 样片审看 (Dailies) → 剪辑 → 母版交付
```

---

## 3.2 Scene Card 的核心目标

Scene Card 用于解决以下问题：
- 这一场戏存在的理由是什么（场次功能 / Scene Function）
- 观众在场次结束后必须知道什么（场次目标 / Scene Objective）
- 这场戏的情绪如何推进（情绪曲线 / Emotional Arc）
- 光线、色彩、声音、空间逻辑是否统一（视觉总策略 / Visual Strategy）
- 哪些镜头必须严格连续，哪些不必（连续性规划 / Continuity Planning）
- 本场的主要风险和资源重点在哪里（风险评估 / Risk Assessment）

---

## 3.3 Scene Card 必填字段

| 字段 | 必填 | 说明 | 影视专业等效 |
|------|------|------|--------------|
| 场次编号 | 是 | 如 SC01 / SC03 / SC02A | Scene Number |
| 场次功能 | 是 | 建立、推进、冲突、揭示、回收 | Scene Function / Beat |
| 场次目标 | 是 | 这一场戏结束后观众必须知道什么 | Scene Objective / Story Point |
| 情绪曲线 | 是 | 从什么情绪进入，走向什么情绪 | Emotional Arc / Tone |
| 场景地点 | 是 | 神坛、林地、洞穴、山谷等 | Location / Set |
| 时间状态 | 是 | 白天 / 黄昏 / 夜 / 非现实时空 | Time of Day / Temporal State |
| 主要角色 | 是 | 本场涉及角色 | Cast List |
| 场次主冲突 | 否 | 人与环境、人与神兽、人与自我 | Central Conflict |
| 场次视觉母题 | 否 | 裂缝、风、菌光、赤金脉冲等 | Visual Motif |
| 场次光线总策略 | 是 | 本场主 Light Tier 及变化逻辑 | Lighting Strategy / Key Light Plan |
| 场次色彩策略 | 是 | 本场主色系统及强调色 | Color Script / Palette |
| 场次声音总策略 | 否 | 风声主导 / 低频共鸣主导 / 寂静主导 | Sound Design / Audio Landscape |
| 节奏走势 | 是 | 慢入—蓄力—爆发—余韵 | Rhythm Arc / Pacing |
| 轴线设定 | 是 | 主轴方向、视线关系、机位限制 | Axis of Action / 180-Degree Rule |
| 需要严格连续的镜头段 | 否 | 哪几镜必须 strict continuity | Continuity Lock |
| 场次风险点 | 是 | 最容易失控的系统问题 | Risk Assessment |
| 交付重点 | 否 | 本场重点保哪几个镜头 | Delivery Priority / Hero Shots |

---

## 3.4 Scene Card 编写规则

### 3.4.1 场次目标必须可验证

**错误写法**：
- 让场面很震撼
- 营造高级感

**正确写法**：
- 让观众确认神坛不是废墟，而是正在苏醒的活体结构
- 让观众理解小G与刑天的第一次遭遇改变了他的认知，从"探险"变为"敬畏"

**专业标准**：场次目标必须是一个"可验证的叙事结果"——观众在场次结束后能陈述出来的事实或情感变化。

### 3.4.2 情绪曲线必须有方向

**错误写法**：
- 神秘、震撼、史诗、悬疑

**正确写法**：
- 好奇 → 不安 → 敬畏
- 平静 → 警觉 → 恐惧 → 求生

**专业标准**：情绪曲线必须是时间轴上的连续变化，不是标签堆砌。每个情绪节点应至少对应一个镜头。

### 3.4.3 场次策略优先于镜头自由发挥

一场戏内各镜头可有变化，但不得违反该场次已定义的：
- 光线总策略
- 色彩总策略
- 屏幕方向主逻辑
- 主冲突与情绪走势

**专业标准**：这是导演的"场次纪律"——DP 和编剧可以在策略框架内创作，但不能突破框架。

## 3.5 Scene Card 示例（专业版）

```
================================================================================

================================================================================
场次编号: SC02
场次功能: 异常揭示 (Revelation)
场次目标: 让观众意识到神坛不是废墟，而是活体结构

时间状态: 暮色将尽，环境低照 (Magic Hour fading to Night)
主要角色: 小G (主角)、神坛 (环境角色)
场次主冲突: 儿童视角的未知逼近 (Man vs. Unknown)
场次视觉母题: 裂缝、赤金脉冲、漂浮孢子
场次光线总策略: Light Tier B (神秘低照) 过渡到 D (神圣显现)

 Color Script: 低饱和冷暗 → 局部赤金异常 → 赤金爆发
场次声音总策略: 低频环境共鸣先行，石层开裂作为情绪爆点
节奏走势: S01观察(节奏2) → S02接近(节奏3) → S03确认(节奏4) → S04显现(节奏5)
轴线设定: 小G保持左至右推进方向
 神坛建立正轴后不随意翻轴
 180度线：以小G-神坛连线为轴，摄影机不跨越
需要严格连续的镜头段: S02→S03 (动作连续)
场次风险点: 发光元素吞没结构、角色比例漂移、神坛空间方位不稳定
交付重点: S03 (确认镜)、S04 (爆发镜 / Hero Shot)
================================================================================
```

| Stage 8 | Prompt 绑定 | 生成结果偏离设计意图 |
| Stage 10 | 导演审片 | 叙事/表演/节奏问题未拦截 |
| Stage 11 | 预生产测试 | 技术风险未暴露，正式渲染大规模失败 |
| Stage 13 | 连续性复核 | 镜头接不上，剪辑困难 |
| Stage 15 | 输出交付检查 | 格式/标准/合规问题导致返工 |

# 第五部分：Shot Card 镜头执行卡规范

## 5.1 Shot Card 的地位

Shot Card 是每个镜头的主文档。Prompt 只是 Shot Card 在生成模型上的展开形式。所有镜头必须先有 Shot Card，后有 Prompt。

**专业标准**：Shot Card 相当于传统影视的"镜头单 (Shot List)" + "摄影指导备注 (DP Notes)" 的结合体。

---

## 5.2 Shot Card 必填字段

| 字段 | 必填 | 说明 | 专业等效术语 |
|------|------|------|--------------|
| 镜号 | 是 | 如 S01 / S04，完整格式：SCxx-Sxx | Shot Number |
| 所属场次 | 是 | 如 SC02 | Scene Reference |
| 镜头优先级 | 是 | P1 / P2 / P3 / P4 / P5 | Shot Priority |
| 叙事目的 | 是 | 该镜头要告诉观众什么 | Story Purpose |
| 情绪目标 | 是 | 该镜头希望观众感受到什么 | Emotional Target |
| 主体 | 是 | 谁/什么是主视觉中心 | Subject / Principal |
| 主动作 | 是 | 主体在做什么 | Primary Action |
| 表演目标 | 是 | 角色内在状态或反应目标 | Performance Objective |
| 表演强度 | 是 | 1-5 | Performance Intensity |
| 空间关系 | 是 | 主体与环境/他者的关系 | Spatial Relationship |
| 景别 | 是 | ELS/FS/MFS/MS/MCU/CU/ECU/OTS/POV/Insert | Shot Size |
| 机位 | 是 | 平视/俯视/仰视/贴地/主观 | Camera Angle |
| 运镜 | 是 | 推/拉/摇/移/升降/环绕/固定 | Camera Movement |
| 屏幕方向 | 是 | 左→右 / 右→左 / 向镜头 / 离镜头 | Screen Direction |
| 视线方向 | 否 | 看向何处，用于反打或接续 | Eyeline Direction |
| 光线档位 | 是 | Light Tier A/B/C/D | Lighting Tier |
| 色温/色彩倾向 | 否 | 冷暖主调、强调色、开尔文参考 | Color Temperature / Palette |
| 第一视觉重点 | 是 | 观众第一眼必须看到什么 | Primary Point of Interest |
| 起幅锚点 (OFA) | 是 | 起始构图与主体状态 | Opening Frame Anchor |
| 镜内节拍点 | 否 | 2-4 个关键动作节拍 | In-Shot Beats |
| 建议入点 | 否 | 剪辑切入建议 (帧/秒) | Suggested In-Point |
| 落幅锚点 (EFA) | 是 | 结束构图与下一镜连接点 | Ending Frame Anchor |
| 建议出点 | 否 | 剪辑切出建议 (帧/秒) | Suggested Out-Point |
| 连续性模式 | 是 | strict / soft / none | Continuity Mode |
| 转场意图 | 是 | 动作接续/视线接续/光线接续/情绪接续/声音先接/时间跳转 | Transition Intent |
| 声音事件 | 否 | 风声、脚步、低鸣、对白等 | Sound Events |
| 风险点 | 是 | 最易生成失败的部分 | Risk Factors |
| 失败容忍项 | 否 | 可接受的轻微偏差 | Acceptable Deviations |
| 是否 Hero Shot | 是 | 是否关键镜头 | Hero Shot Flag |

## 5.3 Shot Card 编写规则

### 5.3.1 叙事目的必须单一明确

**错误写法**：
- 展示环境、交代人物、埋悬念、制造高潮、完成反转

**正确写法**：
- 展示小G首次意识到神坛并非废墟，而是仍在"呼吸"
- 建立刑天从地底升起的压迫感，让观众第一次理解神兽的体量

**专业标准**：每个镜头只回答一个问题。如果镜头需要回答两个问题，拆成两个镜头。

### 5.3.2 第一视觉重点必须唯一

每镜头只能有一个第一视觉重点，例如：
- 小G抬头瞬间的眼神
- 刑天从地底升起的躯体
- 神坛裂缝里的发光脉搏

**不得同时设定多个视觉中心。**

### 5.3.3 动作不等于表演

动作是外在行为，表演是内在状态在画面上的体现。每镜必须区分：
- **主动作 (Primary Action)**：身体或物体行为
- **表演目标 (Performance Objective)**：观众应感知到的心理状态

**示例**：
- 主动作：小G后退一步
- 表演目标：不是"后退"，而是"本能的恐惧让他想逃离，但好奇心让他停住"

### 5.3.4 镜头复杂度必须受控

当主体动作复杂度较高时，运镜复杂度应自动降一级。

**禁止在短镜头中同时叠加**：
- 多角色同权重
- 大幅动作
- 强发光异常
- 环绕运镜
- 多次景别变化

**专业标准**：这是"复杂度预算"原则——每个镜头有固定的复杂度额度，动作、运镜、特效、多角色共享这个额度。

### 5.3.5 风险点必须写实

**示例**：
- 多角色同框可能导致主角漂移
- 发光元素过多可能淹没主体
- 大动作 + 环绕运镜易造成画面混乱
- 视线方向不清可能导致反打失败
- 低机位仰视可能导致角色比例失真
- strict continuity 下角色外观细微变化会被放大

# 第六部分：Prompt 构建规范

## 6.1 Prompt 的定位

Prompt 是模型执行文本，不是创作母文档。它必须忠实服务于 Scene Card 与 Shot Card，而不是脱离场次意图自由发挥。

Prompt 不负责替代导演判断，不承担场次结构、表演设计和剪辑策略的全部工作。

**专业标准**：Prompt 相当于摄影指导给摄影机的"技术指令"——它执行 Shot Card 的设计，但不创造设计。

---

## 6.2 Prompt 基本结构

推荐采用以下顺序：

```
1. 主体与绑定 (Subject & Character Anchor)
2. 主动作 (Primary Action)
3. 表演或反应重点 (Performance / Reaction Focus)
4. 空间环境 (Spatial Environment)
5. 镜头语言 (Camera Language)
6. 光线与材质 (Lighting & Material)
7. 声音/对白（如有）(Sound / Dialogue)
8. 收束锚点 (Closing Anchor / EFA)
```

---

## 6.3 Prompt 构建原则

| 原则 | 说明 | 专业标准 |
|------|------|----------|
| 关键信息前置 | 前 80-120 字优先放主体、动作、核心视觉意图 | 模型注意力曲线：前 1/3 权重最高 |
| 一镜一主旨 | 不允许一个镜头承载多个相互竞争的事件 | 单镜头单事件原则 (Single Event Per Shot) |
| 正向表达优先 | 尽量描述想要什么，不堆叠不想要什么 | "Show, don't tell" 的 Prompt 版本 |
| 具象优先 | 用可视化词汇替代抽象概念 | 可执行词汇 > 形容词 |
| 可见差异优先 | 每一句都应对应可见变化，否则删除 | 每句必对应像素变化 |
| 少而准优于多而散 | 不追求冗长，追求权重聚焦 | 聚焦 > 覆盖 |
| 忠于场次策略 | 单镜描述不得偏离所在场次的光线、色彩、方向、情绪总逻辑 | 场次纪律 (Scene Discipline) |

---

## 6.4 推荐结构模板

### 6.4.1 单主体镜头模板

```
@Image1 as the main character, [角色锚点].
[主体动作 + 当前状态].
[表演或反应重点].
[环境与空间关系].
[机位 + 景别 + 运镜].
[光线档位与材质重点].
[如有对白，用双引号].
[落幅画面锚点].
```

### 6.4.2 场景主导镜头模板

```
[场景主体] dominates the frame.
[核心变化/动作].
[角色在空间中的相对位置].
[镜头运动与揭示顺序].
[光线、雾气、材质].
[结尾落点].
```

### 6.4.3 对话镜头模板 (Dialogue Shot)

```
@Image1 as [角色锚点], [角色位置].
[角色动作 + 表演状态].
[视线方向，指向画面某处].
[景别: MCU 或 CU].
[机位: 平视或微仰].
[光线: 符合本场 Light Tier].
"[对白内容]".
[反应节拍: 说完后的微表情或动作].
```

---

## 6.5 长度控制原则

本规范不把"写满"为目标。Prompt 长度控制标准：

| 场景类型 | 建议长度 | 等效影视标准 |
|----------|----------|--------------|
| 简单动作镜 / Insert | 120-220 字符 | 插入镜头、过渡镜 |
| 标准叙事镜 | 220-420 字符 | 常规叙事、对话 |
| Hero Shot | 320-520 字符 | 关键镜头、高潮镜 |
| 特殊长镜 / 多阶段镜 | 按需扩展，但必须保证主次清晰 | 长镜头、复杂运动 |

### 强制规则：
- 不得为了提高利用率机械填充
- 若镜头主信息已完整，可接受低于旧版 90% 的利用率
- 长度超限时，按以下优先级保留：
 1. 主体与绑定 (不可删除)
 2. 主动作 (不可删除)
 3. 表演目标 (删除前需导演批准)
 4. 空间关系 (可简化但不可删除)
 5. 运镜 (可简化)
 6. 落幅锚点 (不可删除)
 7. 光线/材质 (可压缩为 Light Tier 代码)
 8. 声音 (可删除，由后期补充)

---

## 6.6 Prompt 质量判定标准

一个合格 Prompt 必须回答以下七个问题：

1. **谁是主角？** — 主体识别清晰
2. **正在发生什么？** — 主动作明确
3. **角色在怎么反应？** — 表演目标可感知
4. **相机怎么看？** — 机位、景别、运镜明确
5. **光线是什么逻辑？** — Light Tier 可推断
6. **观众第一眼看哪里？** — 第一视觉重点唯一
7. **这一镜结束时停在什么状态？** — 落幅锚点完整

**若不能回答以上任意一个问题，则 Prompt 不合格，需重写。**

# 第七部分：角色一致性与定妆照绑定规范

## 7.1 核心目标

角色一致性是系列内容的生命线。本规范强调：**角色一致性不等于表情、角度、光线完全不变**，而是角色识别特征稳定，且表演可变化。

**专业标准**：这对应传统影视的"演员连续性 (Actor Continuity)"——服装、发型、妆容保持一致，但表演可以随情绪变化。

---

## 7.2 定妆照输入规则

定妆照通过 `content` 数组的 `reference_image` 方式传入。推荐每角色 3-5 张高质量参考，而不是机械堆到 9 张上限。

**推荐组合（专业定妆照标准）**：

| 照片类型 | 用途 | 拍摄标准 |
|----------|------|----------|
| 正面照 (Front) | 面部识别基准 | 平视，正面，中性光，面部占画面 50% |
| 四分之三侧面 (3/4 Profile) | 角度识别 | 45° 侧转，展示面部轮廓 |
| 侧面照 (Profile) | 轮廓识别 | 90° 侧转，展示鼻梁、下颌线 |
| 全身照 (Full Body) | 服装与体型 | 正面，全身入镜，展示完整服装 |
| 关键表情照 (Expression) | 表演参考 | 角色标志性表情或情绪状态 |

---

## 7.3 @ 绑定规则

推荐写法：

```
@Image1 as the main character, Chinese boy 8yo, round face, black hair,
brown eyes, khaki pants, green jacket.
```

规则：
- `@Image1` 尽量前置
- 角色描述紧跟其后
- 仅保留最有辨识度的 3-5 个特征
- 不要在角色锚点里混入风格、情绪、镜头信息

---

## 7.4 角色锚点写法规范

角色锚点应包含：

- 年龄/年龄段
- 脸部特征
- 发型
- 眼睛特征
- 服装主色
- 1-2 个高识别特征

**不建议长期保留的内容**：
- 镜头状态词 (close-up, wide shot 等)
- 情绪词 (scared, happy 等)
- 冗余美学词 (beautiful, stunning 等)
- 过多服饰细节 (除非为叙事关键)

---

## 7.5 多角色场景规则

多角色同框时：
- 必须定义唯一主视觉中心
- 非主角色只保留识别性最低限度描述
- 不允许多角色都使用同等高权重锚点
- 若镜头的叙事重点是关系，不是细节，优先锁定站位和体量关系，而非所有角色细节

**专业标准**：这是"焦点分配"原则——摄影机一次只能有一个主要焦点，Prompt 也是。

---

# 第八部分：Production Bible 生产圣经规范

## 8.1 生产圣经的地位

Production Bible 用于统一系列项目中的角色、环境、道具、服装、色彩与视觉母题。其目标不是限制创作变化，而是防止系列世界观漂移。

Production Bible 是 Scene Card 与 Shot Card 的基础约束层。

**专业标准**：Production Bible 相当于传统影视的"系列 bible"或"设定集"，是维护系列一致性的核心文档。

---

## 8.2 Production Bible 构成

| 模块 | 内容 | 专业等效 |
|------|------|----------|
| Character Bible | 角色识别特征、服装主设定、状态边界 | Character Design Bible |
| Environment Bible | 主要场景结构、材质、光源逻辑、禁漂移项 | Location Bible / Set Design |
| Props Bible | 关键道具、符号、神坛构件、遗迹元素 | Prop Bible |
| Costume Bible | 角色服装轮廓、主色、变化边界 | Costume Design Bible |
| Color Bible | 系列主色系统、异常强调色、禁用色漂移 | Color Bible / LUT 规范 |
| Motif Bible | 裂缝、菌丝、风压、石纹、脉冲等视觉母题 | Visual Motif Library |

---

## 8.3 Character Bible 规则

角色需固定以下内容：
- 基础年龄与体型
- 脸型与发型
- 主服装轮廓
- 主色与识别点
- 神态边界与表演风格边界

可变化内容：
- 表情
- 姿态
- 光照效果
- 污损程度
- 环境附着物

---

## 8.4 Environment Bible 规则

每个重要场景至少定义：

| 字段 | 说明 | 专业标准 |
|------|------|----------|
| 场景名称 | 神坛外围 / 地下裂谷 / 雾林 | Location Name |
| 空间关键词 | 高耸、环形、裂缝、悬浮孢子 | Spatial Keywords |
| 结构识别物 | 主祭坛、断柱、石桥、深井 | Landmark Elements |
| 主色系统 | 青灰、土褐、赤金 | Primary Palette |
| 禁止漂移项 | 不得出现现代设施、金属科技感、城市光污染 | Forbidden Elements |
| 地面逻辑 | 湿石、泥土、碎裂石板 | Ground Surface |
| 空气逻辑 | 雾、尘、微粒、孢子浓度 | Atmosphere |
| 光源逻辑 | 自然天光 / 裂缝内发光 / 火源 | Light Sources |

---

## 8.5 Props / Costume Bible 规则

- 关键道具与祭坛构件要有可重复识别的轮廓和纹理
- 小G服装主套装不可无理由变色
- 神兽出场相关痕迹必须统一，如风压、鳞纹、低鸣、地面裂痕
- 道具细节为叙事服务，不得喧宾夺主

---

## 8.6 Color Script 规则

每集需定义色彩推进逻辑，而不是由镜头各自决定。

**示例**：

| 场次 | 色彩方案 | 叙事功能 |
|------|----------|----------|
| S00-S01 | 青灰 + 土褐 | 现实探索 |
| S02-S03 | 低饱和冷暗 + 局部赤金异常 | 异常逼近 |
| S04 | 赤金爆发 + 深黑对比 | 高潮显现 |
| S05 | 余烬金 + 夜色蓝灰 | 余韵回收 |

规则：
- 每场最多 1 个主色系统 + 1 个强调色
- 强调色必须服务叙事事件
- 神圣显现色应保持系列一致性
- 不得因单镜"好看"破坏全片色彩结构

**专业标准**：Color Script 对应传统影视的"色彩脚本"，由美术指导和 DP 共同制定，是全片色彩管理的总纲。

---

# 第九部分：镜头语言与运镜规范

## 9.1 核心原则

运镜不是装饰，必须回答：**为什么这样看？**

**专业标准**：每个运镜决策必须有叙事理由。没有叙事理由的运镜是"炫技"，不是"创作"。

---

## 9.2 六类基础运镜及叙事用途

| 运镜 | 英文 | 用途 | 不宜滥用的场景 | 复杂度等级 |
|------|------|------|----------------|------------|
| 推 | Push In | 发现、确认、逼近情绪 | 纯展示大场景 | 中 |
| 拉 | Pull Out | 揭示空间、制造渺小感 | 高情绪首次进入 | 中 |
| 摇 | Pan | 搜索、跟视线、揭示横向信息 | 复杂多事件并发 | 低 |
| 移 | Track / Dolly | 陪伴行动、并行跟进 | 静态神性展示 | 中 |
| 升降 | Crane / Boom | 重建空间、展现规模 | 高频连续使用 | 高 |
| 环绕 | Orbit | 凝视、神性、关系张力 | 普通信息镜、动作混乱镜 | 高 |
| 固定 | Static / Lock-off | 稳定观察、让动作自己说话 | 需要动态揭示时 | 低 |

---

## 9.3 运镜选择规则

### 允许使用的依据
- 建立空间关系
- 强化情绪
- 引导观众视线
- 连接前后镜头
- 展示角色行动过程

### 禁止使用的情形
- 仅为"显得高级"
- 镜头中已有足够复杂动作仍叠加复杂运镜
- 相机运动与主体运动方向冲突，导致读图困难

### 复杂度联动规则

| 主体动作复杂度 | 推荐运镜复杂度 | 示例 |
|----------------|----------------|------|
| 高（打斗、追逐、变形） | 低（固定或简单推/拉） | 刑天苏醒时，固定机位让动作主导 |
| 中（行走、对话、探索） | 中（摇、移、缓推） | 小G穿越林地时，平行跟踪 |
| 低（静坐、凝视、等待） | 高（环绕、升降、长推） | 小G凝视神坛时，缓慢环绕强化神性 |

**镜头清晰度优先于"运动感炫技"。**

---

## 9.4 景别策略

| 景别 | 代码 | 用途 | 典型时长 |
|------|------|------|----------|
| 大全景 | ELS | 建立世界观、空间规模 | 3-5s |
| 全景 | FS | 展示角色与环境关系 | 2-4s |
| 中全景 | MFS | 动作与肢体语言 | 2-4s |
| 中景 | MS | 动作与关系镜、对话标准 | 2-4s |
| 中近景 | MCU | 表情、情绪、心理变化 | 2-4s |
| 近景/特写 | CU | 关键物件、眼神、转折细节 | 1-3s |
| 大特写 | ECU | 极端细节、冲击瞬间 | 1-2s |

规则：
- 一个镜头的景别变化最多 1-2 次
- 不允许在短时长内频繁跨级切换
- 近景/特写必须有明确表演理由

---

## 9.5 机位策略

| 机位 | 英文 | 角度 | 用途 |
|------|------|------|------|
| 平视 | Eye Level | 0° | 中性观察，最常用 |
| 低机位仰视 | Low Angle | +15° ~ +45° | 神性、压迫、力量 |
| 高机位俯视 | High Angle | -15° ~ -60° | 渺小、观察、脆弱 |
| 贴地机位 | Worm's Eye | +60° ~ +90° | 速度感、压迫临场 |
| 俯拍 | Bird's Eye | -90° | 抽象、图案、命运感 |
| 主观机位 | POV | 角色视角 | 沉浸、恐惧、探索 |

**FPV / 主观机位规则**：
- 整片建议 0-2 个
- 只有在角色体验优先时使用
- 不得作为"新鲜感补丁"滥用

---

## 9.6 焦距与透视参考

虽然 AI 视频生成不直接使用物理镜头，但使用镜头焦距术语可以帮助描述画面透视关系：

| 焦距范围 | 类型 | 画面特征 | 适用场景 |
|----------|------|----------|----------|
| 16-24mm | 超广角 | 强烈透视变形，景深极大 | 空间建立、夸张透视 |
| 28-35mm | 广角 | 自然透视，较大视野 | 环境展示、行走跟踪 |
| 50mm | 标准 | 接近人眼透视 | 对话、中性叙事 |
| 85-135mm | 中长焦 | 压缩空间，浅景深 | 肖像、情绪镜、分离背景 |
| 200mm+ | 长焦 | 强烈压缩，极浅景深 | 远距离观察、偷拍感 |

---

# 第十部分：光线、色彩、空间与材质规范

## 10.1 光线分级系统（Light Tier）

本规范采用光线分级，而非机械要求所有镜头"高亮"。

| 档位 | 名称 | 英文 | 色温参考 | 反差比 | 用途 |
|------|------|------|----------|--------|------|
| A | 明亮探索 | Bright Exploration | 5600K (日光) | 低 (2:1) | 探索、发现、儿童向友好镜头 |
| B | 神秘低照 | Mystery Low-Key | 3200-4000K | 中 (4:1) | 未知、悬疑、逼近前夜 |
| C | 对抗高反差 | Contrast Drama | 混合色温 | 高 (8:1+) | 冲突、觉醒、威压 |
| D | 神圣显现 | Divine Manifestation | 非现实色温 (金/青) | 极高 (16:1+) | 显灵、启示、史诗瞬间 |

规则：
- 每镜必须指定一个光线档位
- 同一场次内允许相邻镜头切换，但必须说明原因
- 低照度镜头允许存在，但主体轮廓和动作必须可辨

**专业标准**：Light Tier 对应传统影视的"照明策略"，由 DP 根据场景情绪设计。反差比 (Contrast Ratio) 是衡量光线质量的核心指标。

---

## 10.2 光线描述要求

光线必须写清：

- **主光方向 (Key Light Direction)**：侧光、逆光、顶光、底光
- **明暗关系 (Light-to-Shadow Ratio)**：反差比
- **色温倾向 (Color Temperature)**：暖 (3200K) / 冷 (5600K+) / 非现实色温
- **特殊光效 (Special Effects)**：体积光、孢子辉光、逆光边缘光、光脉

**不建议写法**：
- 很有电影感
- 非常梦幻
- 特别高级

**建议写法**：
- warm backlight (3200K) cuts through the canopy, outlining the boy's shoulder
- low golden pulse (2200K) from the altar crack illuminates the drifting spores with rim light
- cold blue fill (8000K) from the left balances the warm key, creating a 4:1 contrast ratio

---

## 10.3 发光元素使用规则

发光元素属于**叙事强调工具**，不是基础照明默认项。除关键异常镜外，不应作为全场普遍光源。

规则：
- 发光必须服务于事件节点
- 发光不得平均铺满画面
- 发光强度不得吞没主体轮廓与结构细节
- 同场中过多发光点会削弱神异感，需控制密度

---

## 10.4 色彩控制要求

色彩必须服从：
1. 场次色彩策略
2. 系列 Color Script
3. 主体可读性
4. 情绪逻辑

不允许：
- 为追求艳丽而破坏世界观
- 单镜无理由跳出场次色彩系统
- 高饱和强调色长期常驻

---

## 10.5 空间层次要求

每镜头应至少具备以下三层中的两层：

- **前景 (Foreground)**：增加纵深、框定主体、增加质感
- **中景 (Middle Ground)**：主体所在平面，叙事核心区域
- **背景 (Background)**：说明规模、方位或情绪

规则：
- 背景不是装饰，必须帮助说明规模、方位或情绪
- 大场景镜头必须建立远近关系
- 发光元素不得平均铺满画面，必须服务层次

---

## 10.6 材质规范

材质描写服务于"可信度"，不是堆词。

**优先级**：
1. 与剧情直接相关的材质
2. 主体接触到的材质
3. 镜头近处可读材质

**示例**：
- wet rocky cliff face with visible sediment layers
- cracked stone surface with glowing mineral veins
- fibrous mycelium strands with bioluminescent tips
- weathered altar surface with ancient carved patterns

**不建议大量堆叠**：
- 十种不同表面质感同时出现
- 与镜头无关的宏观材质百科式描述

---

# 第十一部分：连续性、轴线与转场规范

## 11.1 连续性模式

本规范将连续性分为三档：

| 模式 | 定义 | 适用场景 | 技术实现 |
|------|------|----------|----------|
| strict | 使用前镜尾帧或高度接近状态作为下镜起点 | 连续动作、连续表演 | first_frame / last_frame 绑定 |
| soft | 延续角色状态/光线/空间逻辑，不强制同构图 | 多数常规叙事镜 | 角色锚点 + Light Tier 延续 |
| none | 仅保持叙事连贯，不要求帧级连续 | 时空切换、建立镜、跳切 | 无特殊技术绑定 |

默认：`soft`

---

## 11.2 连续性优先级

连续性优先级依次为：

```
叙事连续 > 角色连续 > 空间连续 > 光线连续 > 构图连续
```

当各类连续性无法同时满足时，优先保证前位项。

---

## 11.3 何时使用 strict

仅在以下情况下使用：
- 同一动作被拆成两镜 (如走路从 A 点到 B 点)
- 同一角色表演需要无缝延续 (如表情变化连续)
- 无缝转场是镜头卖点 (如长镜头模拟)
- 镜头间需要精确接动作 (如接物、击打)

---

## 11.4 禁止强制 strict 的场景

以下情况不得强制帧级衔接：
- 时空转换
- 轴线重建
- 环境建立镜
- 情绪跳变
- 需要重新分配构图主导权

---

## 11.5 起幅与落幅锚点

每镜头必须明确：

### 起幅锚点（Opening Frame Anchor / OFA）
- 主体位置 (Subject Position)
- 景别 (Shot Size)
- 初始运动状态 (Initial Motion State)
- 光线状态 (Lighting State)

### 落幅锚点（Ending Frame Anchor / EFA）
- 最终主视觉对象 (Final Primary Subject)
- 最终构图重心 (Final Composition Center)
- 情绪停点 (Emotional Resting Point)
- 下一镜接点 (Next Shot Connection Point)

---

## 11.6 屏幕方向规则

每场戏需设定主体主要运动方向：

| 方向 | 含义 | 叙事功能 |
|------|------|----------|
| 左 → 右 | 前进 / 接近 / 主动 | 主角主动推进 |
| 右 → 左 | 受阻 / 退却 / 逆行 | 主角遭遇阻力 |
| 向镜头 | 压迫 / 逼近 / 降临 | 威胁 approaching |
| 离镜头 | 离开 / 被吞没 / 失落 | 离去或逃避 |

规则：
- 同一行动段落内，主角屏幕方向不得频繁反转
- 如需反转，必须用中性镜头、正面镜头、大全景重建轴线
- 神兽或环境主压迫物也必须有稳定方向逻辑

---

## 11.7 视线接续规则

若前镜头角色视线指向画外，后一镜可接：
- 被注视对象 (What they are looking at)
- 同方向空间 (Same direction space)
- 反打视角 (Reverse angle / OTS)

**不得出现**：
- 无理由改变注视方向
- 近景视线与下一镜空间方位矛盾
- 角色看向高处，下一镜却是低处物体，且无中间桥接

---

## 11.8 轴线重建规则

允许翻轴，但必须通过以下任一方式重建：
- 正轴大全景 (Neutral Wide Shot)
- 明显绕行镜头 (Arcing Shot)
- 主体穿越镜头中线 (Subject Crosses Frame)
- 中性正面镜头 (Neutral Frontal Shot)

**180度轴线原则**：在任何对话或互动场景中，摄影机不可跨越角色间的 180 度线，除非使用上述重建方法。

---

## 11.9 转场关系定义

每镜头必须标注转场意图之一：

| 转场类型 | 说明 | 专业术语 |
|----------|------|----------|
| 动作接续 | 前镜动作在下镜中延续 | Action Match Cut |
| 视线接续 | 前镜视线指向下镜主体 | Eyeline Match |
| 光线接续 | 前镜光效在下镜中延续 | Light Bridge |
| 情绪接续 | 情绪状态延续 | Emotional Continuity |
| 声音先接 | 下镜声音先出现，画面后切 | Sound Lead / L-Cut |
| 空间对比 | 前镜与下镜形成空间反差 | Contrast Cut |
| 时间跳转 | 明显的时间跳跃 | Time Jump / Jump Cut |

---

# 第十二部分：节奏曲线与镜头编排规范

## 12.1 节奏曲线的必要性

单镜头再优秀，如果全片节奏失衡，成片依然会显得平、乱、疲劳。每集必须设计 **Rhythm Arc**。

**专业标准**：Rhythm Arc 对应传统影视的"节奏脚本"或"剪辑脚本"，由导演和剪辑师共同设计。

---

## 12.2 节奏的构成

节奏不仅由运镜速度决定，也由以下因素共同构成：

- **信息密度** (Information Density)：每帧包含的新信息量
- **镜头长度** (Shot Duration)：单镜持续时间
- **剪辑频率** (Cutting Frequency)：单位时间内的剪辑次数
- **声音张力** (Audio Tension)：音量、频率、节奏的变化
- **表演停顿** (Performance Pause)：角色动作中的静止时刻
- **光线变化节拍** (Lighting Rhythm)：明暗变化的节奏感

---

## 12.3 节奏四维标注

每镜头在 Shot Card 阶段必须标注 1-5 分：

| 维度 | 1 分 | 5 分 | 说明 |
|------|------|------|------|
| 节奏等级 | 静 (静止画面) | 爆发 (快速剪辑+强运动) | 整体节奏感受 |
| 信息密度 | 极简 (单一元素) | 高密 (多角色+多动作+多特效) | 画面复杂度 |
| 情绪强度 | 平稳 (中性情绪) | 强烈 (极端情绪) | 情感冲击力 |
| 运动强度 | 静止 (固定画面) | 强运动 (多方向快速运动) | 画面动态程度 |

---

## 12.4 建议曲线

| 镜号 | 作用 | 节奏等级 | 信息密度 | 情绪强度 | 运动强度 |
|------|------|----------|----------|----------|----------|
| S00 | 钩子 / 入口 | 3-4 | 3 | 3-4 | 2-3 |
| S01 | 建立人物与空间 | 2-3 | 2 | 2 | 2 |
| S02 | 引入异常 | 3 | 3 | 3 | 3 |
| S03 | 累积期待 | 3-4 | 3-4 | 4 | 3-4 |
| S04 | 高潮爆发 | 5 | 4-5 | 5 | 4-5 |
| S05 | 回收 / 余韵 / 悬念 | 2-3 | 2 | 3 | 1-2 |

规则：
- 连续 3 镜处于同等节奏强度时需人工确认
- 高潮前应有蓄力镜 (Build-up)
- 高潮后应有回落镜 (Release)，避免整片顶格输出
- 节奏变化应平滑，避免 1→5 的跳跃（除非为特殊效果）

---

# 第十三部分：片头镜头（S00）规范

## 13.1 S00 的目标

S00 只承担以下三项中的任意两项，不得全包：
- 品牌识别 (Brand Identity)
- 情绪钩子 (Emotional Hook)
- 世界观导入 (World Introduction)

---

## 13.2 S00 的两种合法模式

### 模式 A：Title-first (标题优先)
适用于品牌感、史诗感开场

优先顺序：
1. 标题 (Title / Logo)
2. 情绪钩子 (Emotional Hook)
3. 场景氛围 (Atmosphere)

### 模式 B：Character-first (角色优先)
适用于角色驱动开场

优先顺序：
1. 角色动作 (Character Action)
2. 神兽预告 (Creature Teaser)
3. 标题弱化出现 (Subtle Title)

---

## 13.3 S00 执行规则

- 不强制 3 秒内同时完成标题、角色、神兽、免责声明、开场白
- 免责声明优先通过字幕模板或平台元数据处理
- 若标题需被完整阅读，需给出足够停留时间 (建议 ≥ 2 秒)
- 开场白只在角色/神兽是开场主导时使用

---

## 13.4 S00 审核项

| 检查项 | 标准 |
|--------|------|
| 主目标明确 | 仅承担 2 项核心任务 |
| 阅读性 | 标题可被读取（停留时间 ≥ 2s） |
| 钩子强度 | 前 1.5 秒建立兴趣 |
| 信息不过载 | 不出现多重竞争焦点 |
| 声画一致 | 声音强化而非干扰画面 |
| 品牌合规 | 平台标识与品牌元素符合规范 |

---

# 第十四部分：声音、对白与混音策略规范

## 14.1 声音策略总则

声音不再被视为单纯"生成附带项"，而是镜头情绪的一部分。采用 **双阶段策略**：

1. **生成期参考音频（Audio Preview）**
2. **交付期声音精修（Delivery Mix）**

**专业标准**：这对应传统影视的"同期声 (Production Sound)" 与 "后期混音 (Post Mix)" 工作流。

---

## 14.2 生成期策略

可使用 `generate_audio=true` 生成：
- 对白参考 (Dialogue Reference)
- 环境音参考 (Ambience Reference)
- 节奏参考 (Tempo Reference)
- 口型参考 (Lip-sync Reference)

对白建议放入双引号中：
```
小G看向裂缝，轻声说："它还活着……"
```

---

## 14.3 交付期策略

最终交付允许进行专业声音后期，包括：
- 降噪 (Noise Reduction)
- 环境声补充 (Ambience Fill)
- 音乐结构调整 (Music Edit)
- 对白清理 (Dialogue Cleanup)
- 响度统一 (Loudness Normalization)
- 转场声桥 (Sound Bridge)

**禁止**：
- 完全推翻镜头原始表演意图
- 用后期声音弥补镜头设计缺失

**交付响度标准**：

| 指标 | 标准值 | 说明 |
|------|--------|------|
| Integrated Loudness | -16 LUFS | 网络平台标准 |
| True Peak | <= -1 dBTP | 防止削波 |
| Loudness Range | 8-15 LU | 动态范围控制 |

---

## 14.4 声音主导层规则

对白、环境声、音乐三者不得同时争夺情绪主导权。每镜头应指定一个主声音层：

| 主导层 | 适用场景 | 混音优先级 |
|--------|----------|------------|
| Dialogue 主导 | 对话场景、叙事关键对白 | 对白 > 环境 > 音乐 |
| Ambience 主导 | 环境建立、空间沉浸 | 环境 > 音乐 > 对白 |
| FX 主导 | 动作场景、特效冲击 | 特效 > 环境 > 音乐 |
| Music 主导 | 情绪高潮、蒙太奇 | 音乐 > 环境 > 对白 |
| Silence / Near Silence 主导 | 神性显现、恐惧、敬畏 | 静默 > 细微环境 |

---

## 14.5 声桥规则

可通过以下方式建立镜头连接：
- 低频异常声先于画面出现 (Sound Lead)
- 裂缝声延续进入角色反应镜 (Sound Bridge)
- 神兽呼吸作为跨镜连接 (Motif Sound)
- 风声作为空间统一线索 (Ambient Unity)

---

## 14.6 静音与留白策略

不是每个高潮都应堆满音乐。以下场景可受控使用静音或留白：
- 神性显现前的停顿
- 角色意识到重大真相的瞬间
- 余韵镜头
- 恐惧或敬畏的悬停点

**专业标准**：静默是声音设计中最有力的工具。恰当的留白比不恰当的填充更有冲击力。

---

## 14.7 旁白使用规则

默认不使用旁白作为主叙事手段。但以下场景可受控启用：
- 世界观导入
- 时间跳转说明
- 儿童向辅助理解
- 片尾总结

**旁白限制**：
- 不得重复画面已明确表达的信息
- 字数与时长受控（建议不超过画面时长的 30%）
- 必须通过导演审片批准

---

# 第十五部分：系统约束与内容安全规范

## 15.1 约束分层原则

本规范将约束分为两层：

### A. 系统规则层
用于生成前拦截：
- 敏感内容
- 版权风险
- 禁用元素
- 不合法素材

### B. Prompt 表达层
用于模型引导：
- bright readable subject separation
- single clear main character
- clean organic surfaces
- cinematic family-friendly tone

---

## 15.2 禁用元素

系统规则层禁用：
- 红眼/火焰瞳孔
- 真实品牌/IP角色
- 过度机械/科幻元素
- 不合规人脸素材
- 无授权真人素材
- 不适龄暴力/成人内容

---

## 15.3 风格黑名单的处理方式

不建议在 Prompt 主体中大量堆叠：
- no dark
- no blur
- no artifact
- no this no that

优先用正向表达替代。如需保留少量经验证有效的短负向，可作为系统尾部短语，但不得喧宾夺主。

---

# 第十六部分：长度控制、压缩与截断规范

## 16.1 长度控制原则

长度控制以**镜头有效信息完整**为目标，而非机械追求满额。

---

## 16.2 压缩优先级

超长时按以下顺序压缩：

1. 冗余修饰词
2. 重复环境形容
3. 次要材质细节
4. 可省略的次级动作
5. 次级光效描述

**不得优先删除**：
- 主体识别
- 主动作
- 表演目标
- 镜头运动
- 起幅/落幅锚点
- 角色绑定信息

---

## 16.3 截断规则

必须优先在：
1. 中文标点
2. 英文标点
3. 空格

处截断。禁止截断半个语义单元。禁止为截断破坏角色绑定结构。

---

## 16.4 长度健康区间

| 状态 | 说明 | 处理建议 |
|------|------|----------|
| 优 | 信息完整，主次清晰，无明显冗余 | 直接提交 |
| 可用 | 略长或略短，但不影响执行 | 微调后提交 |
| 警告 | 关键信息埋没或信息不足 | 需重写或补充 |
| 阻断 | 主体/动作/运镜/锚点丢失 | 必须重写，不得提交 |

---

# 第十七部分：质量评审与导演审片规范

## 17.1 评审原则

文本阶段的质量，不以"格式合格"代替"镜头成立"。

---

## 17.2 五维成片导向评分

| 维度 | 权重 | 说明 | 评审标准 |
|------|------|------|----------|
| 可读性 | 25% | 主体、动作、空间是否一眼可辨 | 3秒内能识别主体和动作 |
| 可控性 | 20% | 是否易于生成稳定结果 | 参考历史成功率与风险点评估 |
| 可剪性 | 20% | 是否方便接前后镜头 | 落幅锚点清晰，转场意图明确 |
| 情绪命中率 | 20% | 是否准确传达预期情绪 | 与 Scene Card 情绪目标对比 |
| 记忆点 | 15% | 是否有鲜明视觉钩子 | 是否有"一眼难忘"的元素 |

**评分等级**：

| 总分 | 等级 | 处理建议 |
|------|------|----------|
| 90-100 | 优秀 (Excellent) | 直接通过，可作为系列标杆 |
| 75-89 | 良好 (Good) | 通过，记录优化建议 |
| 60-74 | 合格 (Acceptable) | 需修改后复审 |
| 40-59 | 不合格 (Below Standard) | 需重写 Shot Card 和 Prompt |
| 0-39 | 阻断 (Blocking) | 退回重新设计，不得渲染 |

---

## 17.3 导演审片附加检查

每镜头导演必须回答：
1. 这一镜存在的理由是什么？(Why this shot?)
2. 第一眼看哪里？(Where does the eye go first?)
3. 如果删掉这镜，故事损失什么？(What is lost if cut?)
4. 这镜的落幅能否自然接下一镜？(Does the EFA connect?)
5. 是否存在更简单、更准确的拍法？(Is there a simpler way?)
6. 这镜是否"好剪"而不是仅仅"好看"？(Editable or just pretty?)

---

## 17.4 阻断条件

以下任一出现即不得进入渲染：
- 主视觉中心不明确
- 多个动作竞争
- 运镜与动作冲突
- 起幅/落幅缺失
- 多角色同权重混乱
- 角色绑定不完整
- 明显不利于剪辑衔接
- 屏幕方向或视线接续逻辑不成立
- 违反系统约束或内容安全规范

---

## 17.5 审片版本定义

| 版本 | 英文 | 内容 | 用途 | 审批人 |
|------|------|------|------|--------|
| 技术审看 | Tech Review | 仅检查生成稳定性 | 技术排错 | 后期总监 |
| 导演审看 | Director Review | 检查表演、叙事、节奏 | 创作判断 | 导演 |
| 连续性审看 | Continuity Review | 检查方向、光线、接续 | 系列统一 | 摄影指导 + 后期总监 |
| 声音审看 | Sound Review | 检查对白、声桥、氛围 | 声音定版前 | 声音设计师 |
| 最终综合审看 | Final Master Review | 最终综合审查 | 出片决策 | 制片总控 + 导演 |

---

# 第十八部分：渲染执行规范

## 18.1 渲染模式

| 模式 | 英文 | 用途 | 分辨率 | 质量等级 |
|------|------|------|--------|----------|
| 预生产测试 | Pre-Production Test | 低成本验证镜头可行性 | 480p / 720p | 草稿 (Draft) |
| 正式渲染 | Principal Rendering | 生产级生成 | 720p | 生产 (Production) |
| Hero 渲染 | Hero Rendering | 关键镜头高质量版本 | 1080p | 精品 (Premium) |

---

## 18.2 分辨率策略

| 场景 | 建议分辨率 | 说明 |
|------|------------|------|
| 预生产 | 480p / 720p | 快速迭代，成本控制 |
| 常规镜头 | 720p | 标准生产分辨率 |
| Hero Shot / 片头 / 高潮镜 | 1080p | 关键镜头与宣发物料 |
| 宣发物料 | 1080p 或更高（若平台支持） | 宣传使用 |

---

## 18.3 镜头优先级与资源分配

| 级别 | 名称 | 英文 | 用途 | 资源策略 |
|------|------|------|------|----------|
| P1 | 叙事核心镜 | Story Critical | 不可删除的叙事核心镜 | 最高测试与重试预算 |
| P2 | 情绪关键镜 | Emotion Critical | 情绪成立关键镜 | 中高预算 |
| P3 | 视觉标志镜 | Visual Signature | 宣发/记忆点镜头 | 允许更高分辨率和更多版本 |
| P4 | 功能连接镜 | Utility Shot | 连接镜、建立镜、补位镜 | 控制成本，追求稳定 |
| P5 | 可替代镜 | Replaceable | 可替代镜头 | 低预算，必要时删改 |

---

## 18.4 连续性参数执行

如需 strict continuity：
- 开启 `return_last_frame=true`
- 使用上一镜尾帧作为下一镜首帧输入
- 必须在 Shot Card 标注理由

若为 soft continuity：
- 保留角色与光线逻辑即可
- 不强制使用上一镜尾帧

---

## 18.5 重试原则

重试优先处理：
1. 主体识别失败
2. 动作不准
3. 角色漂移
4. 光线失控
5. 空间关系错乱
6. 视线与方向不稳定

**禁止用"整体重写极简 Prompt"作为第一反应。**

**重试策略**：

| 失败类型 | 第 1 次重试 | 第 2 次重试 | 第 3 次重试 |
|----------|-------------|-------------|-------------|
| 主体识别失败 | 加强角色锚点 | 检查 reference_image | 修改 Shot Card 主体设计 |
| 动作不准 | 简化动作描述 | 拆分为两个镜头 | 重新设计动作 |
| 光线失控 | 明确 Light Tier | 减少发光元素 | 调整色彩描述 |
| 角色漂移 | 检查 strict 输入 | 增加锚点特征 | 改用 soft continuity |

---

# 第十九部分：生成失败诊断与返工机制

## 19.1 核心原则

失败镜头的处理应优先定位问题源，而不是盲目重试。返工应遵循"先诊断层级，再修改执行对象"的原则。

---

## 19.2 失败分类

| 类型 | 表现 | 优先判断问题源 | 修改对象优先级 |
|------|------|----------------|----------------|
| 主体失败 | 主角不像、主次不清 | 角色锚点 / 主视觉权重 | Prompt → Shot Card |
| 动作失败 | 动作不连贯、不准确 | 动作描述过复杂 / 节拍不清 | Shot Card → Scene Card |
| 空间失败 | 前后景混乱、方位不对 | 空间关系描述不足 | Shot Card |
| 光线失败 | 过曝、主体淹没 | Light Tier + 发光控制失衡 | Shot Card → Prompt |
| 连续性失败 | 接不上上一镜 | 锚点不明确 / continuity 选错 | Shot Card |
| 表演失败 | 情绪不对、反应假 | 表演目标缺失或过度 | Shot Card |
| 剪辑失败 | 单镜可看但拼不起来 | 落幅设计无效 / 节奏错位 | Shot Card → Scene Card |

---

## 19.3 返工优先顺序

1. 检查 Scene Card 是否错误
2. 检查 Shot Card 是否错误
3. 检查 Prompt 是否偏离 Shot Card
4. 检查参考图 / 连续性输入是否冲突
5. 判断是否应改分镜，而非继续重试

---

## 19.4 可继续重试的情况

- 主体识别轻微偏差
- 光线层次可修正
- 运镜接近正确但节奏略偏
- 局部材质失真但不破坏主信息

---

## 19.5 应改 Prompt 的情况

- 第一视觉重点不明确
- 动作描述存在两段竞争
- 环境与角色争夺主导权
- 表演描述混入过多抽象词

---

## 19.6 应改 Shot Card 的情况

- 镜头目标太多
- 同时要求"大动作 + 环绕 + 多角色 + 强发光"
- 起幅和落幅在逻辑上无法共存
- 运镜、表演、景别设计互相冲突

---

## 19.7 应改分镜的情况

- 连续多次失败都集中于同一镜头机制
- 镜头设计超出当前模型稳定区
- 单镜承载过多叙事信息
- 本镜头可被更简单镜头替代

---

# 第二十部分：后期制作与最终交付规范

## 20.1 后期目标

后期不是补锅，而是完成镜头间统一、声音精修、字幕与交付整理。

---

## 20.2 后期内容

包括但不限于：
- 粗剪编排 (Assembly Edit)
- 节奏调整 (Pacing)
- 连续性复核 (Continuity Check)
- 字幕烧录 (Subtitle Burn-in)
- 声音混音 (Audio Mix)
- 音量标准化 (Loudness Normalization)
- 审片版与交付版输出 (Review & Master Output)

---

## 20.3 声音分层建议

后期声音建议按以下逻辑组织：
- Dialogue（对白）
- Ambience（环境声）
- FX（特效声）
- Music（音乐）
- Narration（旁白，若有）

即使平台生成的是混合参考音频，交付期仍应按分层思路处理。

---

## 20.4 输出规格

### Master 交付版

| 参数 | 规格 | 说明 |
|------|------|------|
| 分辨率 | 1920×1080 | Full HD |
| 帧率 | 24fps | 电影标准 |
| 画幅比 | 16:9 | 标准宽屏 |
| 编码 | H.264 High Profile | 通用兼容 |
| 音频编码 | AAC 48kHz | 标准音频 |
| 响度 | -16 LUFS | 网络平台标准 |
| 峰值 | <= -1 dBTP | 防止削波 |
| 色彩空间 | Rec. 709 | 标准高清色彩空间 |
| 文件格式 | .mp4 | 通用格式 |

### Review 审片版

| 参数 | 规格 | 说明 |
|------|------|------|
| 分辨率 | 1280×720 | HD Ready |
| 镜号显示 | 是 | 画面角落叠加镜号 |
| 时间码 | 是 | 画面角落叠加时间码 |

# 附录 A：Seedance API 执行要点

## A.1 基本策略
- 参数优先用 request body 传入
- `first_frame/last_frame` 与 `reference_image` 模式不得混用
- 需要角色一致性时优先多模态参考
- 需要严格首尾指定时优先首尾帧模式
- 有声视频成本更高，测试阶段可按需关闭

## A.2 推荐参数策略
- `duration`：按镜头意图设置，不为凑时长
- `resolution`：720p 常规，1080p 关键镜头
- `generate_audio`：测试阶段可开，终版可按交付策略决定
- `return_last_frame`：仅 strict continuity 需要时开启
- `seed`：用于近似复现，不承诺完全一致

# 附录 E：180度轴线规则图示 / 180-Degree Rule Illustration

```
场景：小G (A) 与 神坛 (B) 对话/互动

俯视图：

 摄影机安全区
 (The Safe Side / Green Zone)
 * 180°线
 A o<============>o B
 (小G) (神坛)
 \ /
 \ * /
 \ 180° /
 \ /
 \ /
 \ /
 \/
 摄影机禁区
 (The Forbidden Side / Red Zone)

规则说明：
1. 180° 线由 A-B 连线定义
2. 摄影机必须始终位于 180° 线的一侧 (绿色安全区)
3. 跨越 180° 线会导致屏幕方向反转，观众空间迷失
4. 如需跨越，必须通过中性镜头重建轴线

正确接法 (遵守 180° 线)：
 镜1: A→B (A在左，B在右)
 镜2: B→A (B在右，A在左) — 屏幕方向一致 ✓

错误接法 (跨越 180° 线)：
 镜1: A→B (A在左，B在右)
 镜3: B←A (B在左，A在右) — 屏幕方向反转 ✗

轴线重建方法：
 - 正轴大全景 (Neutral Wide Shot)
 - 主体穿越画面 (Subject Crosses Frame)
 - 明显绕行镜头 (Arcing Movement)
```
