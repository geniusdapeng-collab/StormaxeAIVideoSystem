# BeastMind Engine v1.0-Peng — 兽魂叙事引擎设计方案
> **状态**: 设计草案，待大鹏审阅确认
> **设计时间**: 2026-05-25
> **定位**: 替代现有 Pipeline Stage-5 剧本生成模块，服务于山海经神话系列
> **核心超越点**: StoryCraft Engine v1.0的异兽POV视角 + 8项结构性升级

---

## 一、对StoryCraft Engine的评估：取其所长，弃其所短

### 1.1 StoryCraft的核心贡献（必须保留）

**① 异兽POV视角转向** — 这是质的突破
- 从"人类遇到怪兽"到"孤独守护者遇到第一个不怕它的人"
- 情绪权重从3分→8分，观众代入对象从旁观者变为共情者
- **BeastMind完全继承此洞察，并进一步深化**

**② 5节拍结构** — 工业级故事骨架
- 钩子→深入→裂缝→翻转→余韵，每一帧有叙事功能
- 比现有"展示→展示→展示→转折→和解"高级太多
- **BeastMind继承骨架，但改为自适应结构**

**③ TwistValidator质量拦截** — 量化反馈机制
- 70分阈值，低于则拦截重写
- **BeastMind继承拦截思想，但改为"内嵌式反转"（见第3节）**

### 1.2 StoryCraft的5个结构性缺陷（BeastMind必须规避）

| # | 缺陷 | 后果 | BeastMind解决方案 |
|---|------|------|------------------|
| 1 | 工期估5-6工作日，实际需10-12个 | 计划失控 | 模块化开发，每模块2天可独立验证 |
| 2 | 60秒要求"完全认知断裂" | 24秒推翻前24秒，密度超载 | **自适应断裂强度**：60秒=部分断裂，90秒+=完整断裂 |
| 3 | 视觉转化器"零台词"与DialogueDistiller生成台词矛盾 | 设计冲突，后期返工 | **P0-P2纯视觉零台词**，情绪通过视觉行为表达 |
| 4 | 8模块线性串联，任一失败则回退 | 依赖链过长，循环重写 | **核心2模块+可选4插件**，失败 graceful degrade |
| 5 | 台词方案A/B抉择导致未来 rework | 中文台词与Nirath语体系冲突 | **永久零台词设计**，异兽"语言"=视觉行为语法 |

## 二、BeastMind Engine核心哲学："兽负"（The Beast's Burden）

### 2.1 超越"POV"的概念

StoryCraft说："让观众代入异兽"。
BeastMind说：**"异兽不是主角，它的'负担'才是主角。"

核心公式：
```
Story = Beast's Burden + Human Catalyst + Ecological Revelation

兽负（Burden）: 异兽在Nirath生态系统中承担的功能/角色/诅咒
催化剂（Catalyst）: 人类（小G）的到来，打破了平衡
揭示（Revelation）: 观众最终理解"兽负"的真相
```

### 2.2 与现有世界观的无缝融合

Nirath十大生态区不是背景板，每个生态区都有"生态压力"：
- S01 浮空晶簇山脉：能量过载，需要调节者
- S02 流光虹脉河：光脉污染，需要净化者
- S03 青丘灵原：孢子扩散，需要控制者
...

**每只异兽的"兽负"=它在生态系统中的功能。**
观众看到的"可怕特征"=生态功能的极端表现。

**示例（饕餮重构）：**
- StoryCraft: "饕餮吃掉毒素保护生态" → 功能说明，类似旁白解释
- BeastMind: "饕餮的胃袋是Nirath的消化器官，它的饥饿是生态系统的代谢节律。小G看到的'吞噬'，其实是'循环'。"

区别：StoryCraft用概念解释反转，BeastMind用**生态逻辑**自然呈现反转。

## 三、架构设计：2核4插件，非线性依赖

### 3.1 总体架构

```
输入: 异兽档案 + Nirath生态区数据 + 时长约束 + 小G角色设定
  ↓
【核1】兽魂原型机 (ArchetypeForge) — 永远运行
  → 输出: 异兽心智原型 + 兽负定义 + 生态功能映射
  ↓
【核2】自适应节拍器 (AdaptiveBeat) — 永远运行
  → 输出: N个叙事节拍（时长自适应）+ 断裂强度分级
  ↓
可选插件（根据项目启用）:
  【插1】生态嵌入器 (EcologyEmbedder)
    → 将兽负与Nirath生态区数据绑定，生成生态级视觉描述
  【插2】双轨误解机 (DualMisunderstanding)
    → 生成4层认知落差（观众/人类/异兽/生态）
  【插3】视觉语法引擎 (VisualGrammar)
    → 将每个节拍转化为纯视觉动作+尺度跳跃方案
  【插4】情绪共振仪 (EmotionResonance)
    → 量化情绪曲线，验证≥2个峰值且Beat-(N-1)≥70%
  ↓
输出: 完整故事计划 (story-plan.json，兼容现有Pipeline Stage-6+)
```

### 3.2 与现有Pipeline的集成点

**替换Stage-5: 剧本生成**
- 现有: 结构化fallback生成简单5镜叙事
- BeastMind: 生成增强型story-plan.json，新增字段:
  - `beastPsyche`: 异兽心智原型对象
  - `burden`: 兽负定义
  - `ecologicalRole`: 生态功能映射
  - `beats[].misunderstandingLayers`: 4层认知落差
  - `beats[].visualScale`: 视觉尺度目标（微观/中观/宏观/生态级）
  - `beats[].emotionalTarget`: 精确情绪目标（含强度值0-100）

**Stage-6+ 完全兼容**
- 时长分配器读取 `beats[].emotionalTarget` 加权分配
- 运镜控制读取 `beats[].visualScale` 选择镜头语法
- 渲染引擎读取 `beastPsyche` 注入情绪基调到Prompt

## 四、核1：兽魂原型机 (ArchetypeForge)

### 4.1 核心思想

**不用每次生成心理画像，而是基于异兽本质特征匹配原型。**

40只异兽 → 8个心智原型 → 每个原型有预建心理模型。

### 4.2 八大原型

| 原型 | 核心驱动力 | 兽负本质 | 代表异兽 | 反转模式 |
|------|-----------|---------|---------|---------|
| **守望者** | 责任感 | 守护某物/某地，被误解为占有欲 | 烛龙、白泽 | "它不是在阻挡，是在保护" |
| **囚徒** | 赎罪/惩罚 | 承受诅咒/封印，被误解为恶意 | 饕餮、穷奇 | "它的痛苦是自愿的牺牲" |
| **园丁** | 养育/修复 | 维护生态平衡，被误解为破坏 | 帝江、当康 | "它的'破坏'是修剪" |
| **信使** | 传递/警告 | 携带信息，被误解为威胁 | 毕方、鸾鸟 | "它的出现是预警，不是灾难" |
| **隐士** | 避世/孤独 | 自我隔离，被误解为傲慢 | 九尾狐、麒麟 | "它的远离是保护人类" |
| **戏谑者** | 测试/筛选 | 设下考验，被误解为残忍 | 狻猊、梼杌 | "它的攻击是试炼" |
| **记忆体** | 保存/记录 | 承载历史，被误解为停滞 | 玄武、貔貅 | "它的'沉睡'是铭记" |
| **失衡者** | 渴求/缺失 | 寻求补全，被误解为贪婪 | 饕餮（另一面向）、混沌 | "它的渴求是对失去的哀悼" |

### 4.3 原型→心理模型自动生成

给定原型，自动生成:
```javascript
{
  archetype: "守望者",
  burden: "守护汤谷双日平衡，防止光脉暴走",
  motivation: "如果双日失衡，整个Nirath生态系统将崩溃",
  fear: "被误解为独占光脉的力量，被猎杀",
  desire: "被理解：我不是要控制光，我是光的一部分",
  memory: "上一次双日风暴，我失去了同类",
  judgmentMode: "遇到生命体 → 评估对光脉的影响 → 决定引导/阻挡/忽视",
  reversal: "观众以为它在'统治'光脉 → 实际它在'承受'光脉的负荷"
}
```

**与现有异兽档案的映射:**
- 读取异兽档案的 `signatureFeatures`、`weaknesses`、`mythOrigin`
- 通过关键词匹配原型（如"守护"→守望者，"诅咒"→囚徒，"吞噬"→园丁/失衡者）
- 若匹配失败，fallback到通用原型（守望者）

## 五、核2：自适应节拍器 (AdaptiveBeat)

### 5.1 核心思想

**StoryCraft的5×12s是固定模板。BeastMind根据时长、原型复杂度、生态嵌入深度，动态生成节拍数量和时长分配。**

### 5.2 时长→节拍映射

```
30s（短视频）: 3节拍 — 钩子(10s)→裂缝(10s)→翻转(10s)
45s: 4节拍 — 钩子(8s)→深入(10s)→裂缝(12s)→翻转(15s)
60s（标准）: 5节拍 — 钩子(8s)→深入(10s)→裂缝(12s)→翻转(18s)→余韵(12s)
90s: 7节拍 — 钩子(8s)→日常(8s)→深入(10s)→裂缝(12s)→翻转(20s)→揭示(15s)→余韵(17s)
120s+: 9节拍 — 增加"代价"和"传承"节拍
```

### 5.3 断裂强度自适应

**StoryCraft要求所有时长都"完全认知断裂"→BeastMind分级:**

| 时长 | 断裂模式 | 观众体验 | 示例 |
|------|---------|---------|------|
| 30-60s | **部分断裂** | "它可能不是我想的那样" | 观众开始怀疑，但不完全推翻 |
| 90s | **渐进断裂** | "原来如此……但还有更多" | 多层反转，每层揭开一部分 |
| 120s+ | **完全断裂** | "我完全误解了它" | StoryCraft级别的彻底推翻 |

**60秒标准版的"部分断裂"设计:**
- Beat-1~2: 建立"异兽是威胁"的初始印象
- Beat-3: 出现矛盾信号（异兽行为不符合"威胁"逻辑）
- Beat-4: **部分断裂** — 观众意识到"它可能不是威胁，但还不确定是什么"
- Beat-5: 揭示兽负，观众理解"原来它是在保护"

**关键差异**: 不是"推翻前24秒"，而是**"前24秒的理解被重新框架化"**。观众不觉得自己"错了"，而是觉得自己"只看到一半"。

## 六、插件1：生态嵌入器 (EcologyEmbedder)

### 6.1 核心思想

**异兽的每一个行为都与其生态功能绑定，视觉描述自带叙事逻辑。**

### 6.2 示例：烛龙在汤谷（S01）

**传统描述（无生态嵌入）:**
"烛龙睁开眼睛，光芒照亮了峡谷。小G感到震撼。"

**生态嵌入描述:**
"烛龙睁眼不是为了照亮——它的眼睑是Nirath大气中光脉粒子的过滤膜。
睁眼=释放过量光脉（生态排毒），闭眼=吸收光脉（生态蓄能）。
小G看到的'睁眼'，其实是Nirath的'呼吸'。"

### 6.3 自动生成逻辑

```javascript
function embedEcology(beast, habitat) {
  // 读取Nirath Bible中habitat的生态压力
  const pressure = habitat.ecologicalPressure; // e.g., "光脉过载"
  
  // 读取beast的原型兽负
  const burden = beast.psyche.burden; // e.g., "守护双日平衡"
  
  // 生成生态行为映射
  return {
    behavior: "睁眼",
    ecologicalFunction: "释放过量光脉，防止大气光脉浓度超标",
    visualMetaphor: "Nirath的呼吸——吸气（闭眼蓄能）→呼气（睁眼排毒）",
    humanMisunderstanding: "它在展示力量/发怒",
    ecologicalTruth: "它在维持光脉平衡，保护整个生态系统"
  };
}
```

**视觉Prompt自动增强:**
- 输入: "烛龙睁开眼睛，光芒照亮峡谷"
- 输出: "烛龙的眼睑如光脉过滤膜缓缓张开，过量光脉粒子从瞳孔如金色潮汐般涌出——这不是攻击，是Nirath大气在'呼气'。小G脚下的晶簇地面因光脉浓度降低而发出微弱的舒缓脉动"

## 七、插件2：双轨误解机 (DualMisunderstanding)

### 7.1 核心思想

**StoryCraft只有"观众误解→反转"单层结构。BeastMind设计4层认知落差，每一层独立翻转，形成共振。**

### 7.2 四层认知结构

```
Layer 1: 观众层（Audience）
  "这异兽看起来好可怕"
  → 反转: "它的可怕是为了保护更脆弱的东西"

Layer 2: 人类层（小G）
  "它要攻击我？/它在阻止我？"
  → 反转: "它是在引导我避开危险"

Layer 3: 异兽层（Beast POV）
  "这个人类是威胁吗？/是来帮助我的吗？"
  → 反转: "他不需要我的保护，但他需要我的理解"

Layer 4: 生态层（Ecosystem）
  "异兽和人类的对立"
  → 反转: "它们是同一生态系统的两个互补节点"
```

### 7.3 叙事效果

**传统单层反转:**
"原来饕餮不是坏兽" → 观众感到一次惊讶

**BeastMind四层共振反转:**
- 观众以为"坏兽"→实际"守护者"（惊讶）
- 同时意识到"小G也误会了它"（同情小G）
- 同时看到"饕餮也在判断小G"（双重POV）
- 最终理解"它们的对立是生态系统的设计"（哲学升华）

**情绪效果**: 不是一次"wow"，而是**层层递进的"啊……原来……天啊……"**。

## 八、插件3：视觉语法引擎 (VisualGrammar)

### 8.1 核心思想

**StoryCraft先有文字叙事，后检查"是否视觉化"。BeastMind从视觉出发——每个节拍首先是一个视觉动作。**

### 8.2 视觉行为语法

每个节拍由3个视觉元素定义：

```javascript
{
  beat: "裂缝",
  visualAction: "烛龙将巨爪插入地面光脉晶簇",  // 做了什么
  visualScale: "微观→宏观",                     // 尺度跳跃
  visualContrast: "破坏动作→生长结果",           // 动作与结果的反差
  emotionalCarrier: "金色光脉从裂缝中如根系般蔓延" // 承载情绪的画面
}
```

### 8.3 尺度跳跃矩阵

StoryCraft没有系统化的尺度设计。BeastMind建立尺度跳跃矩阵：

| 节拍 | 建议尺度 | 效果 |
|------|---------|------|
| 钩子 | 宏观/生态级 | "这是什么？" |
| 深入 | 中观/场景级 | "它在做什么？" |
| 裂缝 | 微观→宏观跳切 | "细节 contradicts 整体印象" |
| 翻转 | 宏观+微观叠化 | "整体理解被细节重构" |
| 余韵 | 生态级定格 | "它在系统中的位置" |

**尺度跳跃=叙事节奏**: 每次尺度跳跃都是一次认知刷新。

## 九、插件4：情绪共振仪 (EmotionResonance)

### 9.1 核心思想

**StoryCraft的情绪校准是"事后检查"。BeastMind的情绪曲线是"预计算+实时调整"。**

### 9.2 原型→基准情绪曲线

每个原型有预建的情绪曲线模板：

```javascript
// 守望者原型基准曲线
const guardianCurve = {
  hook: { emotion: "好奇", intensity: 40 },      // 观众: "这是什么？"
  deepening: { emotion: "紧张", intensity: 60 }, // 观众: "它有危险吗？"
  crack: { emotion: "困惑", intensity: 50 },     // 观众: "它的行为不太对……"
  flip: { emotion: "敬畏", intensity: 85 },        // 观众: "原来它在守护……"
  resonance: { emotion: "感动", intensity: 70 }   // 观众: "它的孤独……"
};
```

### 9.3 自适应调整

根据生态嵌入深度和双轨误解层数，自动调整：
- 生态嵌入越深 → 翻转节拍情绪强度+10%
- 4层误解全开 → 增加一个"顿悟"子峰值（翻转节拍内）
- 时长缩短 → 压缩前段，保护翻转和余韵

### 9.4 拦截规则

- 翻转节拍情绪强度<70% → 拦截，要求增强生态嵌入或增加误解层
- 相邻节拍情绪差<15% → 警告"缺少情绪峰值"
- 曲线没有≥2个明显峰值 → 拦截

## 十、零台词设计：异兽的"视觉语言"

### 10.1 永久零台词原则

**P0-P2阶段**: 纯视觉叙事，零台词。
**P3+阶段**: 仅允许"情绪标点"——单个词或短句在高潮点出现（如叹息、低吼的名词化）。

**异兽的"语言"是其行为模式：**

| 原型 | 日常行为 | 警觉行为 | 接纳行为 |
|------|---------|---------|---------|
| 守望者 | 缓慢巡视，目光跟随光脉 | 挡在人类与危险之间 | 让出道路，目光柔和 |
| 囚徒 | 蜷缩，自我封闭姿态 | 防御性展示伤痕 | 首次暴露脆弱部位 |
| 园丁 | 用身体/工具处理环境 | 加速"工作"，驱赶人类 | 邀请人类参与"工作" |

**人类（小G）的理解过程通过视觉呈现：**
- 误解阶段: 小G后退/握紧指南针（恐惧肢体语言）
- 怀疑阶段: 小G停顿/歪头（困惑肢体语言）
- 理解阶段: 小G模仿异兽动作（镜像神经元激活）
- 共鸣阶段: 小G主动靠近/伸手（信任肢体语言）

## 十一、与现有系统的集成方案

### 11.1 替换Stage-5（剧本生成）

现有链路:
```
Stage-5: 剧本生成（结构化fallback）
  → Stage-6: 时长分配
```

升级后链路:
```
Stage-5: BeastMind Engine 剧本生成
  → 5.1: ArchetypeForge（核1）→ 输出beastPsyche
  → 5.2: AdaptiveBeat（核2）→ 输出beats结构
```

```
ArchetypeForge ───┬──→ AdaptiveBeat ───┬──→ 合并输出

                  └──→  EmotionResonance ─┘（可选）
```

### 12.2 开发计划

|------|------|--------|---------|

**总计: 9个工作日**（比StoryCraft估的5-6现实，比实际10-12紧凑，比BeastMind建议的10-12更聚焦）。

### 12.3 最小可行版本（MVP）

**如果只允许做2天，出什么？**

**答案: ArchetypeForge + AdaptiveBeat 的MVP版本**
- 手工编写8个原型的JSON心理模型（1天）
- 手工编写3个时长模板（30s/60s/90s）的节拍结构（半天）
- 集成到Stage-5，替换现有fallback（半天）

**MVP验证标准:**
- 跑一集饕餮60s → 故事结构比现有版本明显更饱满
- 反转不依赖台词解释 → 纯视觉可理解
- 与Stage-6+完全兼容 → 链路不中断

| **情绪曲线** | 事后校准 | **预计算+实时调整** | 从检查到设计 |
