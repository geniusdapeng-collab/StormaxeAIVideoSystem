## TL;DR

工作汇报核心：结论先行→数据支撑→问题解决→请求支持，四步搞定领导汇报

## 1. 场景分析

### 请求摘要
用户需要进行工作汇报

### 显性需求
- 掌握汇报方法
- 获得可执行的汇报方案

### 场景分类
- `scenario_classification`: 职场沟通
- `urgency`: normal

## 2. 汇报类型框架

### 汇报类型

| 类型 | 时长 | 重点 |
|------|------|------|
| 日/周报 | 5分钟 | 进展+问题 |
| 月报 | 15分钟 | 成果+计划 |
| 项目汇报 | 30分钟 | 进度+风险 |
| 述职 | 30分钟 | 业绩+成长 |

### 汇报结构

```
1. 结论先行：先说结果
2. 关键数据：核心指标
3. 进展说明：做了什么
4. 问题分析：遇到什么
5. 解决思路：准备怎么干
6. 支持请求：需要什么
```

## 3. 专业建议

### A. 数据呈现

#### 数据原则
```
1. 关键指标：只展示核心
2. 对比分析：同比/环比
3. 可视化：图表优先
4. 结论明确：数据说明什么
5. 异常标注：突出异常值
```

### B. 问题呈现

#### 问题结构
```
1. 问题描述：发生了什么
2. 影响分析：有什么影响
3. 原因分析：为什么发生
4. 解决方案：准备怎么做
5. 时间计划：什么时候解决
```

### C. 向上沟通

#### 沟通原则
```
1. 换位思考：领导关注什么
2. 结论先行：先说结果
3. 带着方案：问题+解决方案
4. 简洁明了：控制时间
5. 主动同步：不要等问
```

### D. 汇报准备

#### 准备清单
```
1. 了解目的：领导想听什么
2. 梳理内容：列出要说的
3. 数据核实：确保准确
4. 模拟演练：检查效果
5. 预判问题：准备答案
```
