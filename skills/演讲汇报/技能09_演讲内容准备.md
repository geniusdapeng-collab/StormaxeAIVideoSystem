## TL;DR

内容准备核心：明确目标→收集素材→设计故事→优化表达，四步打造优质内容

## 1. 场景分析

### 请求摘要
用户需要准备演讲内容

### 显性需求
- 掌握内容准备方法
- 获得可执行的内容方案

### 场景分类
- `scenario_classification`: 内容创作
- `urgency`: normal

## 2. 内容准备框架

### 内容结构

```
1. 核心信息：1句话总结
2. 关键观点：3-5个要点
3. 支撑材料：每个观点的支撑
4. 故事案例：增强吸引力
5. 金句：强化记忆
```

### 素材类型

| 类型 | 来源 | 用途 |
|------|------|------|
| 数据 | 行业报告/公司数据 | 增强可信度 |
| 案例 | 亲身经历/行业案例 | 增强共鸣 |
| 故事 | 个人/他人故事 | 情感连接 |
| 金句 | 名言/自创 | 强化记忆 |
| 图片 | 相关图片/图表 | 视觉辅助 |

## 3. 专业建议

### A. 素材收集

#### 收集方法
```
1. 行业报告：获取数据
2. 亲身经历：挖掘个人故事
3. 网络搜索：补充信息
4. 访谈：获取他人经验
5. 观察：发现身边案例
```

### B. 故事设计

#### 故事结构
```
1. 背景：时间/地点/人物
2. 冲突：遇到什么问题
3. 解决：怎么解决的
4. 结果：带来什么改变
5. 启示：学到什么
```

### C. 金句创作

#### 金句类型
```
1. 对比式：正反对比
2. 排比式：结构重复
3. 比喻式：形象比喻
4. 反问式：引发思考
5. 总结式：高度概括
```

### D. 内容优化

#### 优化原则
```
1. 精简：去掉冗余
2. 聚焦：突出重点
3. 生动：使用比喻
4. 易懂：避免专业术语
5. 情感：引发共鸣
```
