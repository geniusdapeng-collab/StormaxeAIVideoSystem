## 2. 指标体系框架

### 业务指标金字塔

```
            商业目标
               ↑
        业务核心指标
               ↑
       运营过程指标
               ↑
      细分维度指标
```

### 核心指标体系

| 层级 | 指标 | 说明 |
|------|------|------|
| 商业 | GMV, 收入, 利润 | 最终价值 |
| 用户 | DAU, MAU, 新增, 留存 | 用户规模 |
| 转化 | 注册率, 转化率, 复购率 | 转化效率 |
| 成本 | CAC, 获客成本, 物流成本 | 成本控制 |
| 质量 | NPS, 评分, 投诉率 | 用户满意度 |

## 3. 专业建议

### A. 数据采集

#### 埋点方案
```javascript
// 电商关键事件
const events = {
  'page_view': ['页面', '来源'],
  'product_view': ['商品ID', '品类'],
  'add_to_cart': ['商品ID', '数量', '价格'],
  'checkout': ['订单ID', '金额'],
  'purchase': ['订单ID', '商品列表', '总价']
};
```

#### 数据源整合
```
1. 业务数据：订单/用户/商品
2. 行为数据：埋点/日志
3. 财务数据：收入/成本
4. 外部数据：市场/竞品
```

### B. 分析框架

#### 对比分析
```
1. 同比：同期对比
2. 环比：连续周期对比
3. 竞品对比：市场份额
4. 目标对比：完成率
```

#### 漏斗分析
```
用户路径：访问→注册→首购→复购→忠诚

转化率 = 下一级/上一级
流失率 = 1 - 转化率
```

#### 同期群分析
```
按获客时间分组
追踪用户随时间的留存/收入变化
```

### C. 数据可视化

#### 报表原则
```
1. 关键指标突出
2. 趋势清晰可见
3. 支持下钻
4. 实时更新
```

#### 常用图表
```
趋势：折线图
对比：柱状图
构成：饼图/环形图
分布：直方图
关系：散点图
```

### D. 数据驱动决策

#### 决策流程
```
1. 定义问题
2. 收集数据
3. 分析洞察
4. 提出假设
5. 验证测试
6. 决策行动
```

#### A/B测试
```
步骤：
1. 设定指标
2. 分配流量
3. 收集数据
4. 统计显著性
5. 决策上线
```
