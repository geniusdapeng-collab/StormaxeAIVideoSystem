# V5.9 子维度深挖模板

> **核心理念：每个子维度独立深挖 600-1000 字，像专业分析师一样逐项拆解**

---

## 子维度深挖标准结构

每个子维度按以下结构展开：

### 1. 事实陈述（200-300 字）
- 客观描述该子维度的现状
- 列出关键事实和事件
- 标注数据来源

### 2. 数据分析（200-400 字）
- 定量分析该子维度的关键指标
- 用数据表格呈现
- 对比历史趋势和竞品数据

### 3. 对比参照（100-200 字）
- 与同类技术/产品/市场进行对比
- 指出差异和原因

### 4. 小结（100-200 字）
- 用 1-2 句话总结该子维度的核心发现
- 标注置信度

## 子维度深挖的字数分配

| 子维度序号 | 建议字数 | 内容侧重 |
|-----------|---------|---------|
| 第 1 个 | 800-1000 字 | 基础事实 + 核心数据 |
| 第 2 个 | 700-900 字 | 技术/市场深入分析 |
| 第 3 个 | 700-900 字 | 竞争/对比分析 |
| 第 4 个 | 600-800 字 | 用户/需求分析 |
| 第 5 个 | 600-800 字 | 趋势/前瞻分析 |
| 第 6-7 个 | 600-800 字 | 补充维度（可选） |

**每章节子维度总计：4000-6000 字（底线），最佳 5000-8000 字**

## 数据标注规范

### 来源标注格式

```
数据来源：{机构名称}，《{报告名称}》，{发布年份}，{URL（如有）}
```

### 可信度评级

| 等级 | 标准 | 示例 |
|------|------|------|
| 高 | peer-reviewed 论文、官方财报、权威机构报告 | Nature 论文、10-K 财报、Gartner 报告 |
| 中 | 行业媒体报道、厂商官方文档、第三方评测 | TechCrunch、厂商白皮书 |
| 低 | 匿名来源、社交媒体、未验证数据 | Reddit 讨论、未署名博客 |

### 置信度标注

| 等级 | 标准 |
|------|------|
| 高 | 有充分数据支撑，多个独立信源交叉验证 |
| 中 | 有一定数据支撑，但样本量有限或信源单一 |
| 低 | 基于推测或少量间接证据，需更多数据验证 |
