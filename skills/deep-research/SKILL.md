# 深度调研工作流 V5.9 — 客观深度分析版

> **核心引擎**：`deep_research_v5_9.py`（V5.9 独立运行版）
> 
> **V5.9 核心升级：客观分析替代特稿叙事（2026-05-06）**
> 
> V5.9 = 递归深钻机制 + 客观分析报告体 + 结构化分析框架：
> 1. **客观报告体**：禁用所有文学性创作手法（场景开场/人物引语/金句收束/叙事张力/矛盾揭示），改用客观分析、数据驱动、结构化框架
> 2. **子维度深挖**：每章节拆分为 5-7 个子维度（技术/市场/竞争/用户/政策/经济/趋势），每个子维度独立深挖 600-1000 字
> 3. **分析框架强制**：每章节至少使用 2 种分析框架（SWOT/PESTEL/价值链/波特五力/技术栈拆解/商业模式画布等）
> 4. **数据表格强制**：每章节至少包含 2 个结构化数据表格（竞品对比/市场规模/技术参数/时间线等）
> 5. **字数硬约束**：每章节 4000-6000 字（底线），最佳 6000-10000 字，深度报告 10000-15000 字。全报告不少于 30000 字，最佳 60000-80000 字，深度报告 100000+ 字
> 6. **证据链强制**：每个判断必须有数据或来源支撑，禁止无依据断言

## V5.9 客观分析范式

### 核心原则

**不是**：场景开场 → 叙事深描 → 矛盾揭示 → 洞察收束（特稿文学）
**而是**：问题定义 → 分析框架 → 数据拆解 → 证据链 → 结构化结论（客观分析）

### 禁用清单

| 类型 | 禁用内容 | 替代方案 |
|------|---------|---------|
| 文学手法 | 场景开场（"2022年11月30日，OpenAI的Slack群里..."） | 直接定义问题或给出核心发现 |
| 文学手法 | 人物直接引语（"XX说：'...'"） | 用第三方报告/论文的客观数据替代 |
| 文学手法 | 金句收束（"大模型输在了经济学里"） | 用结构化结论或行动建议替代 |
| 文学手法 | 叙事张力/矛盾揭示（"有一个对比令人深思..."） | 用数据表格或分析框架客观呈现 |
| 文学手法 | 修辞性问句（"这意味着什么？"） | 直接给出分析结论 |
| 模糊表述 | "大约/可能/据说" | 用具体数据，或标注"推测"并注明可信度 |
| 空话 | "值得注意的是/不容忽视的是" | 直接陈述事实或数据 |

### 章节写作结构（客观分析体）

每个章节按以下结构展开：

| 部分 | 内容要求 | 字数占比 |
|------|---------|---------|
| **1. 问题定义与范围界定** | 明确本章分析的核心问题、边界、关键变量 | 10% |
| **2. 分析框架** | 明确使用的分析框架（SWOT/PESTEL/价值链等），说明为什么选这个框架 | 10% |
| **3. 子维度深钻** | 按 5-7 个子维度逐项深挖，每个子维度 600-1000 字 | 50% |
| **4. 数据与证据** | 结构化数据表格、技术参数对比、市场数据、时间线 | 15% |
| **5. 批判性审视** | 指出当前分析的局限性、数据缺口、反方观点 | 10% |
| **6. 结构化结论** | 用要点式总结核心发现，标注置信度 | 5% |

### 子维度拆解规则

每个章节必须拆分为 5-7 个子维度。子维度的选择根据课题性质动态确定，但以下维度优先：

**技术类课题**：
1. 技术架构与原理
2. 技术成熟度与演进路线
3. 性能指标与基准对比
4. 技术瓶颈与挑战
5. 开源生态与闭源对比
6. 未来技术趋势

**市场类课题**：
1. 市场规模与增长驱动
2. 产业链结构与价值链分析
3. 竞争格局与市场份额
4. 用户画像与行为分析
5. 商业模式与盈利路径
6. 区域市场差异

**政策类课题**：
1. 政策环境与监管框架
2. 合规要求与标准体系
3. 政策对市场的影响
4. 国际政策对比
5. 未来政策趋势

**通用维度**：
- SWOT 分析（优势/劣势/机会/威胁）
- PESTEL 分析（政治/经济/社会/技术/环境/法律）
- 波特五力模型
- 价值链分析
- 商业模式画布

### 数据表格强制要求

每个章节至少包含 2 个结构化数据表格。表格类型包括但不限于：

| 表格类型 | 示例 |
|---------|------|
| 竞品对比表 | 产品名称 | 核心参数 | 价格 | 市场份额 | 优势 | 劣势 |
| 时间线表 | 时间 | 事件 | 影响 | 来源 |
| 市场规模表 | 年份 | 市场规模 | 增长率 | 数据来源 |
| 技术参数表 | 指标 | 产品A | 产品B | 产品C | 备注 |
| SWOT 表 | 类别 | 具体内容 | 影响程度 |
| 商业模式表 | 要素 | 描述 | 案例 |

### 证据链要求

- 每个判断必须有数据或来源支撑
- 数据标注来源（URL、报告名称、发布机构、日期）
- 低可信度内容标注可信度等级（高/中/低）
- 定量数据标注测试条件和样本量
- 禁止引用无法验证的来源
- 禁止捏造数据
