## TL;DR

职场写作 = 邮件沟通 + 工作汇报 + 方案撰写 + 会议记录 + 跨部门沟通

## 1. 邮件写作

### 1.1 邮件结构
```
职场邮件结构：

邮件三段式：
├── 第一段：为什么要写这封邮件（目的）
├── 第二段：具体内容（事项）
└── 第三段：需要对方做什么（行动）

邮件要素：
├── 主题：简洁明了，说明邮件目的
├── 称呼：尊称，如"张总"、"王经理"
├── 正文：分段清晰，重点突出
├── 附件：如有，注明
└── 签名：姓名、职位、联系方式

回复原则：
├── 及时回复：24小时内
├── 直接回答：不要绕弯子
├── 简洁明了：不要废话
├── 有始有终：闭环
└── 专业礼貌：注意语气
```

### 1.2 邮件类型
```
邮件类型与模板：

类型1：请求邮件
主题：[请求]协助确认XX事项

尊敬的XX：

关于[具体事项]，需要您的协助。

[背景说明]
[具体需求]
[时间要求]

麻烦您在[日期]前回复，非常感谢！

祝好
[你的名字]

---

类型2：通知邮件
主题：【通知】关于XX事项的通知

各位同事：

现将XX事项通知如下：

一、[事项1]
二、[事项2]
三、[事项3]

如有疑问，请联系[联系方式]。

谢谢！

[你的名字]

---

类型3：汇报邮件
主题：【周报】XX项目进展-2024.01.15

领导：

本周[项目名称]进展如下：

已完成：
- [事项1]
- [事项2]

下周计划：
- [事项3]
- [事项4]

问题与风险：
- [问题1]，正在[解决方案]

请知悉！

[你的名字]

---

类型4：感谢邮件
主题：感谢您的XX（帮助/支持/指导）

XX：

感谢您在我[做某事]期间提供的[帮助类型]。

[具体感谢内容]
[具体帮助带来的价值]

祝您工作顺利！

[你的名字]
```

### 1.3 邮件礼仪
```
邮件礼仪：

主题：
├── 简洁明了
├── 说明邮件内容
├── 可加前缀：【通知】【请求】【回复】
└── 不要空白或随意

称呼：
├── 正式场合用"尊敬的"
├── 熟悉可用"您好"
├── 不知道名字用职位
└── 不要用"hey"、"hi"

正文：
├── 分段清晰
├── 每段不超过5行
├── 重点信息加粗
├── 避免长篇大论
└── 重要事项编号

回复：
├── 及时回复
├── 回答所有问题
├── 不要只说"好的"
├── 重要事项确认
└── 抄送相关人员

发送：
├── 检查收件人
├── 检查附件
├── 检查错别字
├── 合适时间发送
└── 避免深夜发送
```

## 2. 工作汇报

### 2.1 汇报类型
```
工作汇报类型：

日报：
├── 内容：今日完成+明日计划
├── 频率：每日
├── 特点：简短、快速
└── 工具：企业微信/钉钉

周报：
├── 内容：本周完成+下周计划+问题
├── 频率：每周
├── 特点：阶段性总结
└── 工具：邮件/文档

月报：
├── 内容：本月成果+下月计划+数据
├── 频率：每月
├── 特点：全面复盘
└── 工具：邮件/PPT

项目汇报：
├── 内容：项目进展+问题+计划
├── 频率：按节点
├── 特点：聚焦项目
└── 工具：邮件/当面汇报

述职报告：
├── 内容：工作成果+能力展示+规划
├── 频率：晋升/年度
├── 特点：全面展示
└── 工具：PPT
```

### 2.2 周报写法
```
周报写法：

周报结构：
├── 本周完成
├── 下周计划
├── 问题与风险
└── 需要支持

周报模板：
```
【周报】2024.01.08-01.12 | XX

一、本周完成
1. [项目/任务] - [完成情况] - [结果/数据]
2. [项目/任务] - [完成情况] - [结果/数据]
3. [项目/任务] - [完成情况] - [结果/数据]

二、下周计划
1. [任务1]
2. [任务2]
3. [任务3]

三、问题与风险
- [问题1]，影响：[影响]，解决方案：[方案]
- [问题2]，影响：[影响]，解决方案：[方案]

四、需要支持
- [需要什么支持]
- [需要什么资源]

五、数据指标（可选）
- [核心数据]
```

周报技巧：
├── 数据说话：量化成果
├── 结果导向：突出价值
├── 问题坦诚：不要只报喜不报忧
├── 计划具体：可执行
└── 格式规范：专业
```

述职报告结构：

述职报告模板：

## 5. 跨部门沟通

### 5.1 沟通原则
```
跨部门沟通原则：

原则1：目标对齐
├── 明确共同目标
├── 找到共同利益
├── 强调双赢
└── 减少对立

原则2：换位思考
├── 理解对方立场
├── 理解对方压力
├── 理解对方语言
└── 寻求共识

原则3：数据支撑
├── 用数据说话
├── 避免情绪化
├── 客观理性
└── 专业说服

原则4：提前沟通
├── 不要等到最后一刻
├── 提前打招呼
├── 预留时间
└── 从容沟通

原则5：书面确认
├── 重要事项书面确认
├── 邮件留底
├── 避免扯皮
└── 责任明确
```

### 5.2 沟通模板
```
跨部门沟通模板：

发起请求：
"[XX部门]的同学好，

关于[事项]，需要贵部门协助。

背景：[背景说明]
需求：[具体需求]
时间：[时间要求]
价值：[对双方的价值]

麻烦评估一下可行性，
如有疑问随时沟通。

谢谢！"

---

处理冲突：
"理解贵部门的顾虑，
这确实是[问题]。

但我们面临的情况是[情况]，
这个问题如果不解决，会影响[影响]。

我的建议是[方案]，
您看是否可行？"

---

推进协作：
"XX好，

跟进一下[事项]的进展：

之前确定的方案是[方案]，
约定的完成时间是[时间]。

目前进展如何？
有没有遇到什么问题？"

---

表达感谢：
"XX好，

感谢贵部门在[事项]上的支持！

正是因为有你们的配合，
我们才能[成果]。

期待下次合作！
"
```
