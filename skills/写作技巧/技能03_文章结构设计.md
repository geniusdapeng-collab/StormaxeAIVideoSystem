## TL;DR

文章结构 = 确定类型 + 选择结构 + 设计骨架 + 填充内容 + 检查优化

## 1. 文章结构类型

### 1.1 按功能分类
```
文章结构类型：

信息传递型：
├── 定义：传递信息、说明情况
├── 结构：总分总、清单式
├── 特点：清晰、有条理
├── 代表：说明书、工作报告
└── 重点：信息准确、结构清晰

说服论证型：
├── 定义：说服读者接受观点
├── 结构：观点+论据+论证
├── 特点：有观点、有论据
├── 代表：议论文、评论文章
└── 重点：论据充分、论证严密

故事叙述型：
├── 定义：讲述故事、传递情感
├── 结构：起承转合、叙事线
├── 特点：有故事、有情感
├── 代表：小说、叙事散文
└── 重点：情节吸引人、情感动人

情感表达型：
├── 定义：表达情感、引起共鸣
├── 结构：情感线、意象组合
├── 特点：情感真挚、意象丰富
├── 代表：散文、诗歌
└── 重点：情感真实、表达细腻

综合型：
├── 定义：多种目的兼有
├── 结构：根据目的灵活组合
├── 特点：功能全面
├── 代表：大多数公众号文章
└── 重点：主次分明、层次清晰
```

### 1.2 按篇幅分类
```
不同篇幅的结构：

短文（500-1000字）：
├── 结构：简单直接
├── 开头：直接入题
├── 中间：1-2个要点
├── 结尾：总结升华
└── 时间：30分钟

中等（1500-2500字）：
├── 结构：完整丰富
├── 开头：引入+观点
├── 中间：3-4个要点
├── 结尾：总结+升华
└── 时间：1-2小时

长文（3000-5000字）：
├── 结构：完整系统
├── 引言：背景+问题
├── 中间：5-6个要点
├── 结论：总结+建议
└── 时间：3-4小时

超长文（5000+字）：
├── 结构：系列化
├── 引言：深度背景
├── 主体：多个章节
├── 结论：系统总结
└── 时间：半天到一天
```

## 2. 经典结构模式

### 2.1 总分总结构
```
总分总结构（最经典）：

适用场景：
├── 议论文
├── 说明文
├── 职场文章
└── 实用性文章

结构框架：

总（开头）：
├── 亮出核心观点
├── 吸引读者注意
├── 引出下文
└── 字数：10-15%

分（中间）：
├── 2-4个分论点
├── 每个分论点独立成段
├── 论据充分
├── 过渡自然
└── 字数：70-80%

总（结尾）：
├── 总结核心观点
├── 升华主题
├── 行动号召（可选）
└── 字数：10-15%

示例结构：
标题：《沟通是职场最重要的能力》

总：
职场中，最重要的能力是什么？很多人会想到专业能力、执行力。但我认为，沟通能力才是职场最重要的能力。

分：
1. 沟通决定信息传递效率
   - 案例：信息不对成导致的项目失败
2. 沟通影响团队协作
   - 案例：沟通不畅导致的团队矛盾
3. 沟通能力决定职业发展
   - 案例：善于沟通的人更容易晋升

总：
沟通不是天生的，是可以习得的。学会沟通，就是学会职场的第一课。
```

### 2.2 递进结构
```
递进结构（层层深入）：

适用场景：
├── 分析问题
├── 深度思考
├── 解决问题
└── 评论文章

结构框架：

是什么（现象）：
├── 描述现象
├── 列举表现
└── 引起关注

为什么（原因）：
├── 分析原因
├── 多个角度
└── 深挖本质

会怎样（影响）：
├── 正面影响
├── 负面影响
└── 长远影响

怎么办（对策）：
├── 解决思路
├── 具体方法
├── 行动建议
└── 总结升华

示例：
标题：《为什么你总是拖延》

是什么：
每次计划好的事情，总是拖到最后一刻才开始...
为什么：
1. 完美主义：害怕做得不够好
2. 恐惧失败：担心结果不如预期
3. 即时满足：刷手机比工作更爽
4. 任务模糊：不知道从哪里开始

会怎样：
短期：任务完成质量差、压力大
长期：自信心下降、机会流失

怎么办：
1. 拆解任务：从5分钟开始
2. 设定截止时间：给自己压力
3. 消除干扰：手机放远一点
4. 奖励机制：完成任务后奖励自己
```

### 2.3 并列结构
```
并列结构（平铺直叙）：

适用场景：
├── 分享经验
├── 列举清单
├── 方法论总结
└── 人物特写

结构框架：

观点1：
├── 观点
├── 案例/数据
└── 小结

观点2：
├── 观点
├── 案例/数据
└── 小结

观点3：
├── 观点
├── 案例/数据
└── 小结

总结：
├── 综合观点
└── 升华主题

示例：
标题：《高效人士的三个习惯》

习惯一：早起
早起让你拥有不被打扰的时间，可以做最重要的事。村上春树每天4点起床写作，乔布斯每天6点开始工作。

习惯二：清单
用清单管理任务，让大脑专注于思考而不是记忆。每天上班前列清单，按重要性排序。

习惯三：复盘
每天结束前花10分钟复盘：今天完成了什么？有什么可以改进？明天要做什么？

总结：
高效不是天生的，是一个个好习惯累积的结果。从今天开始，培养这三个习惯。
```

### 2.4 时间线结构
```
时间线结构（线性叙事）：

适用场景：
├── 个人故事
├── 成长经历
├── 项目复盘
├── 事件回顾
└── 传记文章

结构框架：

过去：
├── 背景介绍
├── 发生了什么
└── 那时的状态

转折：
├── 关键事件
├── 转折点
└── 为什么改变

现在：
├── 现在的状态
├── 取得的成果
└ └── 学到的东西

未来：
├── 展望
├── 行动计划
└── 鼓励读者

示例：
标题：《我是如何从月薪3000到年薪50万的》

过去：
五年前，我刚毕业，在一家小公司做行政，月薪3000。那时候我觉得，这就是我的命。

转折：
转机出现在第三年。一次偶然的机会，我接触到了数据分析，开始自学Excel、SQL、Python。

现在：
现在，我在一家互联网公司做数据分析负责人，年薪50万。

未来：
如果你现在月薪3000，不要放弃。改变从现在开始。
```

### 2.5 对比结构
```
对比结构（正反对照）：

适用场景：
├── 评论文章
├── 产品评测
├── 选择分析
└── 观点碰撞

结构框架：

A面：
├── 描述A
├── 特点
├── 优势
├── 劣势
└── 适用场景

B面：
├── 描述B
├── 特点
├── 优势
├── 劣势
└── 适用场景

对比分析：
├── 相同点
├── 不同点
├── 各有优劣
└── 适用不同

结论/我的选择：
├── 根据场景选择
├── 我的建议
└── 没有绝对的好坏

示例：
标题：《打工还是创业？》
```

## 3. 开头设计

### 3.1 开头原则
```
开头设计原则：

原则1：吸引注意力
├── 前三秒决定读者去留
├── 开头要有钩子
├── 不要废话
└── 直接入题

原则2：引出主题
├── 让读者知道这篇文章讲什么
├── 不要让读者猜
├── 明确价值
└── 降低阅读门槛

原则3：激发兴趣
├── 制造悬念
├── 引发共鸣
├── 引发好奇
├── 引发需求
└── 提供价值承诺
```

### 3.2 开头方法
```
开头八种方法：

1. 痛点引入
"你是不是也遇到过这种情况..."
"相信很多人都有这样的烦恼..."
适用：职场、成长、情感

2. 数据引入
"据统计，80%的人都遇到过..."
"最新数据显示，..."
适用：商业、科技、数据分析

3. 故事引入
"上周，我遇到了一件让我..."
"我有一个朋友，他..."
适用：个人成长、职场、情感

4. 观点引入
"很多人认为...，但我不这么认为"
"有一种观点说...，其实..."
适用：评论、观点、议论

5. 疑问引入
"为什么你明明很努力，却..."
"有没有一种方法，能..."
适用：方法论、教程、解答

6. 名言引入
"正如乔布斯所说..."
"有一句话我一直记在心里..."
适用：成长、励志、管理

7. 场景引入
"想象一下这样的场景..."
"试想一下，如果你..."
适用：商业、战略、未来

8. 直接入题
"今天，我想跟你聊聊..."
"这篇文章，聊聊..."
适用：所有类型
```

## 4. 结尾设计

### 4.1 结尾原则
```
结尾设计原则：

原则1：收束有力
├── 回扣主题
├── 总结全文
├── 不要拖沓
└── 戛然而止

原则2：升华主题
├── 上升到更高层面
├── 引发思考
├── 留下余韵
└── 不要落入俗套

原则3：行动号召
├── 告诉读者要做什么
├── 具体可执行
├── 不要空洞
└── 提供路径
```

### 4.2 结尾方法
```
结尾八种方法：

1. 总结升华
"总之，...这不仅...而且..."
"归根结底，...才是..."

2. 金句收尾
"记住，..."
"正如那句话所说，..."

3. 行动号召
"从今天开始，..."
"不妨试试..."

4. 呼应开头
"回到开头的问题，..."
"还记得开头提到的..."

5. 开放式结尾
"至于结果如何，..."
"这个问题的答案，..."

6. 展望未来
"未来已来，..."
"十年后的你，..."

7. 感恩感谢
"感谢你的阅读，..."
"感谢一路陪伴..."

8. 互动提问
"你有什么想法？"
"你遇到过类似的情况吗？"
```

## 5. 过渡设计

### 5.1 过渡方法
```
段落过渡方法：

1. 关联词过渡
"除了...之外，..."
"然而，..."
"更重要的是，..."

2. 数字过渡
"第一点...第二点...第三点..."
"首先...其次...最后..."

3. 承接过渡
"说完...，我们再来看..."
"基于以上分析，..."

4. 转折过渡
"但是，..."
"然而，..."
"不过，..."

5. 因果过渡
"因此，..."
"所以，..."
"于是，..."

6. 设问过渡
"那么，如何...？"
"为什么...？"
```

### 5.2 段落衔接
```
段落衔接技巧：

技巧1：重复关键词
在段落开头重复上一段的关键词

技巧2：承上启下
每个段落结尾埋下下一段的伏笔

技巧3：逻辑连接
使用"因为...所以..."
"虽然...但是..."

技巧4：时间连接
"之后"、"然后"、"接着"

技巧5：空间连接
"上面...下面..."
"这里...那里..."
```
