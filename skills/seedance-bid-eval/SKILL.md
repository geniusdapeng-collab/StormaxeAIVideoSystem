# Seedance BidEval — 内容比稿技能 v1.0.0-Peng

## 定位

**渲染配额是稀缺资源，LLM是无限的。**

BidEval 是视频生产流水线的"闸门"，在提交 Seedance 渲染之前，从 N 个候选方案中：
1. **选出最佳方案**（送渲染）
2. **给出修改意见**（反馈给生成系统优化）

## 架构位置

```
用户 → 方案脚本制作系统（生成 N 个方案）
                    ↓
              BidEval 比稿闸门
                    ↓
           ┌────────┴────────┐
           ↓                 ↓
      最优方案送渲染     反馈给生成系统
           ↓                 ↓
      渲染 → 后期 → 交付    优化后再生成
```

## 四大评测维度

| 维度 | 权重 | 子维度 |
|------|------|--------|
| 需求对齐度 | 30% | 主题覆盖、情感基调、关键场景、角色设定、规格匹配 |
| 剧本质量 | 25% | 结构清晰度、角色弧光、冲突设计、节奏感、转场自然度 |
| Seedance合规 | 25% | 提示词长度、6步公式、敏感内容、参数合法性、角色一致性 |
| 艺术感染力 | 20% | 画面想象力、情感共鸣、独特性、镜头语言、张力节奏 |

## 用法

```bash
node bid-eval.js evaluate --input <方案目录> --user-need "用户需求描述" --output <输出文件>
node bid-eval.js help
```

## 输入输出

**输入：** `bid-plans/` 目录下放置 `plan-01.json`, `plan-02.json`... 等 story-plan.json 文件

**输出：** JSON 格式，包含胜出方案、各方案得分、修改建议
