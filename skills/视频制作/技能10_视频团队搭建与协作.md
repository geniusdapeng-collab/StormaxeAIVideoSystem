## 2. 团队架构框架

### 团队规模配置

| 规模 | 角色配置 | 产能 |
|------|----------|------|
| 1-2人 | 全能型 | 2-3条/周 |
| 3-5人 | 编导+拍摄+剪辑 | 5-10条/周 |
| 5-10人 | 分工细化 | 15-30条/周 |
| 10人+ | 专业化+项目制 | 50条+/周 |

### 核心角色

```
1. 编导：内容策划/脚本
2. 拍摄：摄影/灯光
3. 剪辑：后期制作
4. 运营：发布/数据
5. 主播：出镜表演
```

## 3. 专业建议

### A. 角色分工

#### 职责定义
```
编导：
- 内容策划
- 脚本创作
- 现场统筹

拍摄：
- 画面拍摄
- 设备管理
- 灯光布置

剪辑：
- 视频剪辑
- 特效制作
- 调色配乐

运营：
- 数据分析
- 账号运营
- 粉丝互动
```

### B. 协作流程

#### 标准流程
```
1. 选题会：确定本周选题
2. 策划会：脚本讨论
3. 拍摄日：素材采集
4. 剪辑日：后期制作
5. 审片会：内容审核
6. 发布：数据监控
```

### C. 效率工具

#### 工具推荐
```
1. 素材管理：NAS/网盘
2. 项目管理：飞书/Notion
3. 创意协作：在线文档
4. 资产共享：团队盘
```

### D. 绩效考核

#### 考核维度
```
1. 产量：产出数量
2. 质量：数据表现
3. 效率：交付时效
4. 协作：团队配合
```
