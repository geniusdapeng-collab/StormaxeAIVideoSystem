## TL;DR

用户画像 = 基础属性 + 行为特征 + 偏好特征 + 价值分层 + 标签体系

## 1. 画像基础

### 1.1 画像数据类型
```
用户画像数据类型：

基础属性：
├── 人口属性：年龄、性别、地域
├── 社会属性：职业、收入、教育
├── 设备属性：手机型号、系统版本
├── 注册信息：注册渠道、注册时间
└── 认证信息：实名认证、芝麻信用

行为特征：
├── 访问行为：访问频率、访问时长
├── 浏览行为：浏览深度、浏览路径
├── 交易行为：下单、支付、退款
├── 互动行为：点赞、评论、分享
└── 搜索行为：搜索词、搜索频率

偏好特征：
├── 类目偏好：喜欢的商品/内容类目
├── 价格偏好：消费水平、价格敏感度
├── 品牌偏好：喜欢的品牌
├── 功能偏好：使用的核心功能
└── 内容偏好：喜欢的内容类型

价值分层：
├── 活跃度：DAU、WAU、MAU
├── 忠诚度：复购次数、复购周期
├── 消费力：ARPU、LTV
├── 影响力：分享次数、邀请人数
└── 风险度：退款率、投诉率
```

用户画像构建流程：

Step 1：数据采集

├── 行为特征：频率、时长、路径

└── 时间特征：最近、趋势、周期

├── 推荐标签：用于个性化推荐
├── 风控标签：用于风险控制
└── 营销标签：用于精准营销
```

### 2.2 标签开发
```
标签开发方法：

规则标签：
IF 年龄 BETWEEN 18 AND 25 THEN '青年'
IF 最近30天购买次数 >= 3 THEN '高复购'
IF 平均订单金额 > 500 THEN '高消费'

统计标签：
基于历史数据的统计值
用户平均订单金额 = SUM(订单金额) / COUNT(DISTINCT 订单ID)

模型标签：
流失概率 = predict_logistic(特征)
用户性别 = predict_gender(设备型号, 行为)
用户生命周期阶段 = predict_lifecycle(行为)

标签权重：
行为权重：浏览1分、收藏2分、加购3分、购买5分
时间衰减：最近7天权重1.0，7-30天0.8，30-90天0.5
```

用户分群方法：

规则分群：
├── 基于单一维度：活跃度分层
├── 基于多维度：RFM模型

├── 基于事件：事件序列

└── RFM：最近、频率、金额

预测分群：
├── 流失风险分层
├── 转化潜力分层
├── 付费意愿分层
└── 基于机器学习预测
```

### 3.2 RFM模型
```
RFM模型分析：

R（Recency）：最近一次消费
├── 定义：最近一次购买距今时间
├── 越近越好
└── 客户活跃度指标

F（Frequency）：消费频率
├── 定义：一定周期内的购买次数
├── 越频繁越好
└── 客户忠诚度指标

M（Monetary）：消费金额
├── 定义：总消费金额或平均金额
├── 越高越好
└── 客户价值指标

RFM分群：
| R | F | M | 用户类型 | 运营策略 |
|---|---|---|----------|----------|
| 高 | 高 | 高 | 价值用户 | 重点维护 |
| 高 | 低 | 高 | 潜力用户 | 提升频率 |
| 低 | 高 | 高 | 忠诚用户 | 唤回 |
| 低 | 低 | 高 | 消费用户 | 激活 |
| 高 | 高 | 低 | 活跃用户 | 提升客单 |
| 高 | 低 | 低 | 新用户 | 培育 |
| 低 | 高 | 低 | 习惯用户 | 提升价值 |
| 低 | 低 | 低 | 流失用户 | 唤醒 |
```
