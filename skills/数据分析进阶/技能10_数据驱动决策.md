## 1. 数据驱动基础

### 1.1 数据驱动框架
```
数据驱动决策框架：

DECIDE模型：
D - Define：定义决策问题
E - Explore：探索数据
C - Choose：选择分析方法
I - Identify：识别洞察
D - Decide：做出决策
E - Evaluate：评估效果

CRISP-DM流程：
1. 业务理解
2. 数据理解
3. 数据准备
4. 建模分析
5. 模型评估
6. 部署应用

数据分析流程：
1. 提出问题
2. 设计分析
3. 获取数据
4. 分析数据
5. 得出结论
6. 报告建议
7. 跟踪效果
```

### 1.2 数据驱动层次
```
数据驱动层次：

Level 1：描述性分析

组织成熟度：
```

## 2. A/B测试

### 2.1 实验设计
```
A/B测试设计：

测试类型：
├── A/B测试：两个版本对比
├── A/B/N测试：多个版本对比
├── 多臂老虎机：自适应测试
└── 裂变测试：用户分组

实验要素：
├── 假设：明确的零假设和备择假设
├── 指标：核心指标 + 护栏指标
├── 样本：用户划分 + 流量分配
├── 时长：样本量 + 测试周期
└── 分桶：用户粒度随机

流量分配：
├── 流量比例：50/50、80/20、95/5
├── 流量分配：分层实验、互斥实验
├── 分桶策略：UserID哈希、Cookie哈希
└── 流量校验：基础特征分布一致

分层实验：
├── 层与层正交
├── 同一层互斥
├── 不同层可叠加
└── 帝企鹅实验平台
```

### 2.2 实验分析
```
A/B测试分析：

统计分析：
├── 样本量计算
├── 统计功效分析
├── 显著性检验
├── 置信区间
└── 效果量

检验方法：
├── 两比例z检验：转化率
├── 两样本t检验：均值
├── Mann-Whitney U：非正态
└── 卡方检验：分布

结果解读：
| 指标 | 对照组 | 实验组 | 变化 | P值 | 置信区间 |
|------|--------|--------|------|-----|----------|
| 转化率 | 3.2% | 3.5% | +9.4% | 0.02 | [0.1%, 0.5%] |
| UV | 10000 | 10000 | - | - | - |

显著判断：
├── P < 0.05：统计显著
├── 置信区间不含0：显著
├── 实际业务显著性
├── 综合判断
└── 多指标综合评估

常见问题：
├── 多重检验问题
├── 新颖性效应
├──  primacy效应
├── 外部因素影响
└── 短期vs长期
```
