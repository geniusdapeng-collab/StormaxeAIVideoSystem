## TL;DR

归因分析 = 归因模型选择 + 数据采集 + 模型计算 + 结果解读 + 决策应用

## 1. 归因模型

### 1.1 常用归因模型
```
归因模型类型：

首次触达归因：
├── 100%归因给第一个渠道
├── 优点：简单直接
├── 缺点：忽略后续贡献
└── 适用：拉新渠道评估

末次触达归因：
├── 100%归因给最后一个渠道
├── 优点：转化归因明确
├── 缺点：忽略首次和中间
└── 适用：效果类渠道

线性归因：
├── 平均分配给所有渠道
├── 优点：公平合理
├── 缺点：忽略渠道重要性
└── 适用：中间渠道价值评估

时间衰减归因：
├── 越近的渠道权重越高
├── 公式：指数衰减
├── 优点：考虑时间因素
├── 缺点：最近渠道权重过高
└── 适用：短决策周期

位置归因（U型）：
├── 首尾各40%，中间20%
├── 优点：首尾重要
├── 缺点：中间分配较少
└── 适用：通用场景

数据驱动归因：
├── 基于数据计算权重
├── 使用Shapley值
├── 优点：最准确
├── 缺点：计算复杂
└── 适用：有足够数据
```

### 1.2 模型选择
```
归因模型选择原则：

业务特性：
├── 决策周期：长→数据驱动，短→末次
├── 渠道数量：少→线性，多→U型
├── 数据量：多→数据驱动，少→U型
└── 评估目的：拉新→首次，复购→末次

渠道特性：
├── 拉新渠道：品牌广告、内容
├── 转化渠道：效果广告、搜索
├── 辅助渠道：社交、邮件
└── 各渠道重要性

数据条件：
├── 点击数据完整 → 选更复杂模型
├── 点击数据缺失 → 简化模型
├── 数据量足够 → 数据驱动
└── 数据量不足 → U型或线性
```

## 2. 归因数据

### 2.1 数据采集
```
归因数据采集：

用户行为数据：
├── 点击数据：渠道、时间、关键词
├── 访问数据：来源、媒介、广告创意
├── 转化数据：下单、注册、付费
├── 跨设备数据：设备关联
└── 跨渠道数据：渠道关联

渠道数据：
├── 曝光数据：展示次数
├── 点击数据：点击次数
├── 消耗数据：广告花费
├── 转化数据：直接转化
└── 归因转化：分配后转化

数据关联：
├── 用户ID关联
├── 设备ID关联
├── Cookie关联
├── 订单ID关联
└── 时间窗口关联

数据质量：
├── 数据完整性
├── 数据准确性
├── 数据一致性
├── 数据时效性
└── 数据去重
```

### 2.2 数据处理
```
归因数据处理：

用户路径构建：
用户A: 搜索 → SEM → 社交 → SEM → 转化
用户B: 信息流 → 搜索 → 转化
用户C: 直接访问 → 转化

转化归因：
├── 转化类型：注册、下单、付费
├── 转化窗口：1天、7天、30天
├── 归因窗口：点击归因、曝光归因
└── 归因排除：直接流量排除

渠道匹配：
├── Channel：渠道大类
├── Source：流量来源
├── Medium：营销介质
├── Campaign：广告活动
├── Ad Content：创意
└── Keyword：关键词

去重处理：
├── 点击去重：同用户同渠道多次点击
├── 转化去重：同一转化只归因一次
└── 展示去重：同用户同广告多次展示
```

## 4. 归因挑战

### 4.1 归因难点
```
归因分析难点：

数据挑战：
├── 数据孤岛：各平台数据不互通
├── 数据延迟：转化有延迟
├── 数据缺失：部分数据不可得
├── 数据冲突：平台数据不一致
└── 数据造假：刷量等行为

用户挑战：
├── 跨设备：PC、手机、平板
├── 隐私保护：IDFA等受限
├── 行为碎片：路径不完整
├── 决策周期：周期长短不一
└── 无痕访问：无法追踪

渠道挑战：
├── 渠道重叠：多渠道同时影响
├── 间接作用：品牌广告等
├── 协同效应：渠道组合效果
├── 竞争干扰：竞品影响
└── 外部因素：经济环境等

模型挑战：
├── 模型选择：哪个模型准确
├── 权重设置：各渠道权重
├── 归因窗口：多长合适
├── 模型假设：是否符合实际
└── 模型迭代：持续优化
```
