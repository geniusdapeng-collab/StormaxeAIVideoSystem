## 1. 认识异常

### 1.1 异常类型
```
异常类型分类：

点异常（Point Anomaly）：
├── 单个数据点偏离
├── 例如：异常高的销售额
├── 最常见的异常类型
└── 易于检测

上下文异常（Contextual Anomaly）：
├── 在特定上下文才是异常
├── 例如：凌晨的高流量
├── 需要结合上下文判断
└── 较难检测

集体异常（Collective Anomaly）：
├── 单个点正常，但集体异常
├── 例如：连续多天下降
├── 需要序列分析
└── 复杂检测

业务异常：
├── 数据采集错误
├── 系统故障导致
├── 业务本身变化
├── 作弊/刷单
└── 活动影响

正常变异：
├── 季节性波动
├── 周期性变化
├── 正常增长/下降
└── 正常波动范围
```

### 1.2 异常场景
```
业务中的异常场景：

流量异常：
├── UV/ PV突增/突降
├── 访问时长异常
├── 跳出率异常
├── 页面报错突增
└── 接口响应变慢

转化异常：
├── 转化率突增/突降
├── 下单量异常
├── 支付失败率上升
├── 加购率异常
└── 注册转化异常

交易异常：
├── 客单价异常
├── 订单金额异常
├── 退款率上升
├── 优惠券使用异常
└── 折扣力度异常

用户异常：
├── 用户活跃度异常
├── 新增用户异常
├── 留存率异常
├── 流失率突增
└── 复购率异常

行为异常：

```

## 2. 检测方法

### 2.1 统计方法
```
统计异常检测：

Z-Score方法：
├── 计算：z = (x - μ) / σ
├── 阈值：|z| > 3 为异常
├── 优点：简单直观
├── 缺点：假设正态分布
└── 适用：单指标检测

IQR方法：
├── Q1 = 25%分位数
├── Q3 = 75%分位数
├── IQR = Q3 - Q1
├── 异常：< Q1 - 1.5*IQR 或 > Q3 + 1.5*IQR
├── 优点：不受极端值影响
├── 缺点：只考虑单变量
└── 适用：分布较稳定数据

Grubbs检验：
├── 检测单变量离群点
├── 适合正态分布
├── 统计检验方法
└── 适合精确检测

Dixon检验：
├── 小样本离群点检测
├── 适合样本量n<30
└── 适合服从正态分布

分位数方法：
├── 定义正常范围
├── 低于P5或高于P95
├── 可调整分位数
└── 适合有明确边界
```

### 2.2 机器学习方法
```
机器学习异常检测：

孤立森林（Isolation Forest）：
├── 基于决策树
├── 异常点更容易被孤立
├── 高维数据适用
├── 参数少，效果好
└── 适合大规模数据

One-Class SVM：
├── 单类分类器
├── 学习正常数据模式
├── 异常点偏离模式
├── 适合高维数据
└── 核函数选择重要

LOF（局部离群因子）：
├── 计算点与邻域的密度
├── 密度远低于邻居为异常
├── 适合密度不均数据
├── 可解释性好
└── 计算复杂度较高

DBSCAN：
├── 聚类方法
├── 非聚类点为异常
├── 自动确定聚类数
├── 适合任意形状
└── 参数敏感

移动平均+标准差：
├── 计算滚动均值和标准差
├── 超出n个标准差为异常
├── 可调整窗口期
├── 适合时序数据
└── 简单有效
```

## 3. 阈值设定

### 3.1 阈值原则
```
阈值设定原则：

业务导向：
├── 基于业务理解
├── 考虑业务容差
├── 异常会影响业务
└── 阈值服务于业务

数据驱动：
├── 分析历史数据
├── 了解正常波动范围
├── 计算波动标准差
└── 基于统计设定

可调节性：
├── 阈值不是固定的
├── 根据业务阶段调整
├── 上线初期可以宽松
├── 稳定后可收紧
└── 定期回顾调整

告警平衡：
├── 漏报成本：异常未发现
├── 误报成本：正常告警
├── 平衡检全率和准率
└── 动态阈值

监控反馈：
├── 收集告警反馈
├── 统计误报漏报
├── 持续优化阈值
└── 迭代完善
```

### 3.2 阈值类型
```
阈值类型选择：

固定阈值：
├── 设定固定值
├── 简单直接
├── 不随数据变化
└── 适合有明确标准

动态阈值：
├── 基于历史数据
├── 周期性更新
├── 适应数据变化
├── 计算移动平均
└── 考虑季节性

自适应阈值：
├── 机器学习自动学习
├── 自动调整参数
├── 适应数据分布
└── 复杂但准确

多条件阈值：
├── 多个条件同时满足
├── 组合判断
├── 更精确识别
└── 减少误报

分级阈值：
├── 关注：±10%
├── 警告：±20%
├── 严重：±30%
└── 紧急：±50%

示例：
流量告警阈值：
├── 关注：日环比±5%
├── 警告：日环比±10%
├── 严重：日环比±20%
└── 紧急：日环比±30%
```

## 4. 异常处理

### 4.1 处理方式
```
异常处理方式：

数据清洗：
├── 删除异常值
├── 替换为缺失值
├── 替换为均值/中位数
├── 插值填充
└── Winsorize（缩尾）

不处理：
├── 异常是真实业务
├── 业务本身变化
├── 需要分析原因
└── 保留用于分析

标记处理：
├── 标记为异常
├── 不参与统计
├── 单独分析
└── 保留原始数据

业务判断：
├── 确认是否是业务异常
├── 排查原因
├── 采取行动
└── 记录经验

告警处理：
├── 发送告警通知
├── 值班人员处理
├── 快速响应
└── 问题排查
```

### 4.2 处理流程
```
异常处理流程：

Step 1：确认异常
├── 核实数据准确性
├── 检查数据采集
├── 确认计算逻辑
└── 排除系统问题

Step 2：分类异常
├── 点异常 vs 上下文异常
├── 可解释 vs 不可解释
├── 一次性 vs 持续性
└── 轻微 vs 严重

Step 3：分析原因
├── 时间维度：什么时间开始
├── 渠道维度：哪个渠道
├── 用户维度：哪类用户
├── 产品维度：哪个产品
└── 地域维度：哪个地域

Step 4：采取措施
├── 运营措施：调整策略
├── 技术措施：修复bug
├── 产品措施：优化功能
├── 风控措施：防止作弊
└── 数据措施：修正数据

Step 5：复盘总结
├── 异常原因总结
├── 处理过程记录
├── 经验教训沉淀
├── 阈值优化调整
└── 预防措施建立
```
