## TL;DR

需求 = 场景 → 痛点 → 动机 → 方案

## 1. C端需求特点

### vs B端关键区别
| 维度 | C端 | B端 |
|------|-----|-----|
| 决策者 | 个人 | 组织 |
| 需求来源 | 体验驱动 | 业务驱动 |
| 付费 | 个人 | 企业 |
| 替换成本 | 低 | 高 |
| 决策周期 | 短 | 长 |

### C端需求特征
| 特征 | 说明 |
|------|------|
| 感性 | 情绪价值重要 |
| 场景化 | 特定场景触发 |
| 碎片化 | 需求散落 |
| 伪需求多 | 容易被表象迷惑 |

## 2. 需求挖掘方法

### 用户调研
| 方法 | 适用 | 深度 |
|------|------|------|
| 访谈 | 深度需求 | 深 |
| 问卷 | 量化验证 | 中 |
| 观察 | 真实行为 | 深 |
| 数据 | 验证假设 | 中 |

### 需求类型
```markdown
## 需求分类

### 刚性需求
- 解决核心问题
- 没有就换产品

### 柔性需求
- 体验提升
- 有则更好

### 兴奋需求
- 超预期
- 惊喜功能
```

### 场景分析法
```markdown
## 场景分析模板

### 场景描述
- 时间：
- 地点：
- 人物：
- 事件：

### 痛点挖掘
- 当前怎么做？
- 哪里不方便？
- 期望是什么？

### 机会点
- 产品能解决吗？
- 解决后价值多大？
```

## 3. 需求验证

### 验证方法
| 方法 | 成本 | 准确度 |
|------|------|--------|
| 竞品分析 | 低 | 中 |
| 数据验证 | 中 | 高 |
| MVP测试 | 高 | 高 |
| 用户反馈 | 低 | 中 |

### 需求优先级
| 维度 | 评估项 |
|------|--------|
| 价值 | 用户收益多大 |
| 频次 | 发生多频繁 |
| 成本 | 开发难度 |
| 差异 | 竞品有吗 |
