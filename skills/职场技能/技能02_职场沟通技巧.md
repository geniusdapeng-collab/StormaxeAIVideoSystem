## TL;DR

职场沟通 = 结论先行 + 逻辑清晰 + 数据支撑 + 换位思考

## 1. 沟通基础原则

### 1.1 职场沟通黄金法则

| 法则 | 说明 | 应用场景 |
|------|------|----------|
| 结论先行 | 先说结论再说原因 | 汇报、请求 |
| MECE原则 | 相互独立，完全穷尽 | 分析、汇报 |
| 电梯演讲 | 30秒说清重点 | 偶遇领导 |
| 5W1H | 何时何地何人何事为何如何 | 任务传达 |

### 1.2 沟通场景分类

```
沟通场景
├── 向上沟通（汇报、请示、建议）
├── 平级沟通（协作、请求、支持）
├── 向下沟通（布置任务、反馈、激励）
└── 对外沟通（客户、供应商、合作伙伴）
```

## 2. 向上沟通技巧

### 2.1 工作汇报方法

#### 汇报结构
```
1. 结论先行（1句话）
   └── "项目进度正常，预计本周上线"
   
2. 关键进展（3点以内）
   ├── 已完成：XX功能开发
   ├── 进行中：YY功能测试
   └── 待解决：ZZ问题需要协调
   
3. 需要支持（如果需要）
   └── "需要XX部门配合，预计明天完成"
   
4. 下步计划
   └── "明天开始ZZ功能开发"
```

#### 不同情境的汇报方式

| 情境 | 汇报重点 | 时长 |
|------|----------|------|
| 日常进度 | 完成什么/下周计划 | 5分钟 |
| 问题求助 | 问题+原因+建议方案 | 3分钟 |
| 项目汇报 | 整体进展+关键节点+风险 | 10分钟 |
| 年度汇报 | 成绩+贡献+规划 | 15分钟 |

### 2.2 请求领导支持

#### 请求公式
```
背景 + 请求 + 价值 + 替代方案（可选）
```

#### 示例
```
背景：Q3用户增长目标压力大，目前留存率偏低
请求：申请增加10万预算用于用户补贴
价值：预计可提升留存率15%，增加DAU 5万
替代：如预算紧张，可优先在核心用户群试运营
```

## 3. 平级沟通技巧

### 3.1 跨部门协作

#### 协作沟通公式
```
共同目标 + 具体需求 + 时间要求 + 互惠承诺
```

#### 示例
```
"为了Q3 GMV增长30%的共同目标（共同目标），
需要你们提供XX数据支持（具体需求），
请在周五前完成（时间要求），
后续我们可以提供YY数据作为回报（互惠承诺）"
```

### 3.2 请求帮助的技巧

#### 请求公式
```
请求 + 原因 + 时间 + 感谢
```

#### 示例
```
"XX，能否帮我看下这个问题（请求）
因为客户明天要反馈（原因）
请在今天下班前帮忙看下（时间）
非常感谢！（感谢）
"

"XX，有个紧急情况需要你支持（请求）
客户那边很着急，明天要上线（原因）
今天下午3点前需要你的帮助（时间）
真的很感谢你的支持！（感谢）
"
```

### 3.3 拒绝的技巧

#### 拒绝公式
```
肯定 + 原因 + 替代方案 + 态度
```

#### 示例
```
"这个需求很好（肯定）
但目前我们团队人力紧张，已经排满了（原因）
建议可以放到下个版本考虑，或者简化一下需求（替代方案）
真的很抱歉，希望理解（态度）
"
```

## 4. 向下沟通技巧

### 4.1 布置任务

#### 任务传达5步法
```
1. 背景：为什么要做这件事
2. 目标：达成什么结果
3. 范围：具体做什么
4. 资源：有什么支持
5. 反馈：如何汇报和沟通
```

### 4.2 反馈技巧

#### 反馈公式
```
具体行为 + 影响 + 期望
```

#### 正向反馈示例
```
"你上次的方案做得很好（具体行为）
让客户非常满意，已经决定续约了（影响）
继续保持，期待你更多好方案（期望）
"
```

#### 改进反馈示例
```
"上次报告中数据部分有些不一致（具体行为）
导致我汇报时被领导追问（影响）
下次汇报前我们可以先对齐一下数据（期望）
"
```

## 5. 书面沟通技巧

### 5.1 邮件沟通

#### 邮件结构
```
主题：核心内容+动作要求
正文：
├── 开头（1句话）- 目的
├── 正文（3段）- 详情
└── 结尾（1句话）- 行动

格式：
- 段落不超过5行
- 重点内容加粗
- 重要数据用表格
```

#### 邮件礼仪清单
```
✅ 主题清晰，一眼知道内容
✅ 正文简洁，重要信息前置
✅ 附件命名规范：文件+日期+版本
✅ 发送前检查收件人、抄送人
✅ 重要邮件发送后电话确认
```

### 5.2 微信/钉钉沟通

#### 工作群沟通原则
```
✅ 消息结构化：分段、列点
✅ 重要信息加@提醒
✅ 长消息分多条发送
✅ 及时回复，如无法回复先确认收到
❌ 不发大段语音（除非紧急）
❌ 不发与工作无关内容
❌ 不用表情包过度（可适当使用活跃气氛）
```

## 6. 会议沟通技巧

### 6.1 会议发言技巧

#### 发言公式
```
观点 + 论据 + 行动建议
```

#### 示例
```
"我认为这个方案可行（观点）
原因有三点：第一XX，第二YY，第三ZZ（论据）
建议先在A渠道试点，根据数据再决定是否推广（行动建议）
"
```

### 6.2 会议纪要要点

#### 纪要模板
```
【会议主题】
时间：
参会人：
主持人：

【议程】
1. XX议题 - 结论：...
2. YY议题 - 结论：...

【行动项】
| 事项 | 负责人 | 完成时间 |
|------|--------|----------|
| ... | ... | ... |

【下次会议】
时间：
主题：
"
```

## 7. 沟通能力自检清单

### 自检问题
```
□ 结论是否先说？
□ 逻辑是否清晰？
□ 数据是否有支撑？
□ 是否考虑了对方立场？
□ 是否明确了行动？
□ 是否确认对方理解？
```

### 常见错误
```
❌ 说了很多但没重点
❌ 只说自己，不考虑对方
❌ 没有明确结论
❌ 不确认是否被理解
❌ 沟通后不跟进
```
