## TL;DR

问题解决 = 准确定义问题 + 分析根本原因 + 生成解决方案 + 评估决策 + 实施验证

## 1. 问题分析框架

### 1.1 问题定义
```
问题定义公式：
问题 = 期望状态 - 实际状态

定义问题4要素：
1. 是什么：具体问题是什么？
2. 在哪：发生在哪个环节？
3. 何时：从什么时候开始？
4. 多严重：影响范围和程度？
```

### 1.2 5Why分析法
```
方法：连续问5个"为什么"，找到根本原因

示例：项目延期了
Why1: 为什么延期？→ 设计稿反复修改
Why2: 为什么反复修改？→ 需求频繁变更
Why3: 为什么需求变更？→ 没有需求评审流程
Why4: 为什么没有评审？→ 产品设计经验不足
Why5: 为什么经验不足？→ 团队缺乏设计规范 ← 根本原因

解决方案：建立设计规范和需求评审流程
```

### 1.3 逻辑树分析法
```
方法：将问题逐层分解为子问题

示例：如何提升用户留存
├── 原因1：内容不够好
│   ├── 选题不够精准
│   ├── 内容深度不够
│   └── 更新频率低
├── 原因2：体验不够好
│   ├── 加载速度慢
│   ├── 操作流程复杂
│   └── 界面不够美观
└── 原因3：竞品更好
    ├── 功能更丰富
    └── 价格更优惠
```

## 2. 问题分析工具

### 2.1 鱼骨图（因果分析）
```
鱼骨图结构：
       ┌─────────┐
       │  问题   │
       └────┬────┘
    ┌─────┼─────┐
    │     │     │
 ┌──┴──┐┌┴──┐┌─┴──┐
 │人员 ││方法││设备│
 └──┬──┘└┬──┘└─┬──┘

分析维度：
├── 人：谁、多少人、什么角色
├── 方法：怎么做、流程是什么
├── 设备：用什么工具、设备
├── 材料：用了什么、原料
├── 环境：什么地点、什么时间
└── 测量：怎么测量的、数据
```

### 2.2 对比分析
```
对比维度：
├── 时间对比：之前vs现在
├── 预期vs实际：目标vs结果
├── 横向对比：自己vs竞品
└── 内部对比：A方案vsB方案

对比分析步骤：
1. 确定对比指标
2. 收集对比数据
3. 计算差异
4. 分析原因
```

### 2.3 数据分析
```
常用分析方法：
├── 趋势分析：数据随时间变化
├── 分布分析：数据的分布情况
├── 漏斗分析：各环节转化率
├── 相关分析：因素之间的关系
└── 对比分析：不同群体的差异

数据来源：
├── 用户行为数据
├── 业务数据
├── 调研数据
└── 竞品数据
```

## 3. 解决方案生成

### 3.1 方案生成方法
```
头脑风暴原则：
□ 不要批评，任何想法都可以
□ 追求数量，越多越好
□ 鼓励发散，组合创新
□ 延迟判断，先收集再评估

方案收集维度：
├── 内部方案：自己团队能做的
├── 外部方案：借助外部资源的
├── 短期方案：快速见效的
├── 长期方案：系统性解决的
└── 混合方案：长短结合的
```

### 3.2 方案评估
```
评估矩阵：
| 方案 | 收益 | 成本 | 时间 | 风险 | 综合 |
|------|------|------|------|------|------|
| A    | 高   | 中   | 短   | 低   | ⭐⭐⭐⭐ |
| B    | 中   | 低   | 中   | 中   | ⭐⭐⭐ |
| C    | 高   | 高   | 长   | 高   | ⭐⭐ |

评估维度：
├── 可行性：技术/资源/能力是否可行
├── 效果：能解决多少问题
├── 成本：投入多少资源
├── 周期：需要多长时间
├── 风险：有什么风险
└── 副作用：会有什么负面影响
```

## 4. 决策方法

### 4.1 决策框架
```
决策步骤：
1. 明确决策目标
2. 收集必要信息
3. 生成备选方案
4. 评估各方案
5. 选择最优方案
6. 制定执行计划
7. 跟踪和调整
```

### 4.2 决策类型
```
决策分类：
├── 确定型：信息充分，可计算最优
│   └── 方法：数据分析、线性规划
├── 风险型：知道各种结果概率
│   └── 方法：期望值分析、决策树
├── 不确定型：不知道结果概率
│   └── 方法：乐观/悲观/后悔值法
└── 竞争型：有竞争对手
    └── 方法：博弈论、策略分析
```

### 4.3 决策原则
```
决策原则：
├── 数据驱动：基于数据而非感觉
├── 理性分析：避免认知偏差
├── 快速迭代：小步快跑，快速验证
├── 备选方案：永远有Plan B
└── 止损思维：设定可接受损失
```

## 5. 问题解决实践

### 5.1 问题解决流程
```
问题解决8步法：
1. 识别问题：发现并定义问题
2. 分析问题：找到根本原因
3. 设定目标：明确要达成什么
4. 生成方案：列出可能的解决方案
5. 评估方案：比较各方案优劣
6. 选择方案：确定最优方案
7. 实施方案：按计划执行
8. 验证结果：确认问题已解决
```

### 5.2 常见问题模式
```
问题模式库：
├── 偏差问题：结果偏离预期
│   └── 方法：对比分析找差异
├── 缺口问题：目标未达成
│   └── 方法：层层分解找瓶颈
├── 变异问题：结果不稳定
│   └── 方法：控制图分析
├── 根本问题：反复出现
│   └── 方法：5Why找根因
└── 机会问题：改进空间
    └── 方法：创新思考
```
