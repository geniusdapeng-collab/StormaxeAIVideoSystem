## TL;DR

跨部门协作 = 明确目标 + 理解立场 + 互惠互利 + 及时沟通 + 结果导向

## 1. 跨部门协作挑战

### 1.1 常见问题
```
跨部门协作障碍：
├── 目标不一致：各部门的KPI不同
├── 优先级冲突：资源有限，时间冲突
├── 沟通不畅：信息不对称，理解偏差
├── 责任不清：边界模糊，互相推诿
├── 文化差异：不同部门工作风格不同
└── 利益冲突：可能影响部门利益
```

### 1.2 协作成功要素
```
协作成功公式：
成功 = 共同目标 + 清晰职责 + 互惠互利 + 高效沟通 + 信任关系
```

## 2. 协作前准备

### 2.1 了解对方
```
协作前了解：
├── 对方部门的核心KPI是什么
├── 他们最关心什么
├── 他们有什么困难
├── 他们的工作节奏
├── 关键决策人是谁
└── 他们的沟通偏好
```

### 2.2 准备材料
```
协作准备清单：
□ 明确的目标和诉求
□ 清晰的时间要求
□ 完整的方案/需求文档
□ 预期结果和收益
□ 需要的支持清单
□ 备选方案（如果有）
```

## 3. 协作沟通技巧

### 3.1 沟通框架
```
协作沟通公式：
共同目标 + 具体需求 + 时间要求 + 互惠承诺 + 感谢

示例：
"为了Q3 GMV增长30%的共同目标（共同目标）
需要运营部提供XX支持（具体需求）
请在周五前完成（时间要求）
后续我们可以提供YY数据作为回报（互惠承诺）
谢谢！（感谢）
"
```

### 3.2 请求支持技巧
```
请求优先级：
├── P0紧急：直接影响业务，立即处理
├── P1重要：本周内需要，影响目标
└── P2常规：可以排队，不紧急

请求方式：
├── 正式沟通：发邮件/工单
├── 面对面沟通：重要事项当面聊
├── 会议沟通：复杂事项开会讨论
└── 借助上级：必要时请领导协调
```

### 3.3 拒绝与妥协
```
拒绝原则：
1. 先理解对方立场
2. 清楚说明自己困难
3. 提供替代方案
4. 保持友好关系

妥协技巧：
├── 明确底线：哪些不能让步
├── 交换筹码：有所得有所失
├── 分步实施：先做一部分
└── 记录在案：明确双方责任
```

## 4. 协作推进

### 4.1 建立协作机制
```
协作机制：
├── 定期同步：周会/双周会
├── 进度共享：共享文档/看板
├── 关键节点：约定里程碑检查
├── 问题升级：明确升级流程
└── 结果反馈：及时告知结果
```

### 4.2 跟进技巧
```
跟进频率：
├── 重要事项：每天跟进
├── 常规事项：关键节点跟进
└── 小事项：完成后确认即可

跟进方式：
├── 面对面沟通：重要事项
├── 即时通讯：日常跟进
├── 邮件确认：重要决策
└── 会议回顾：阶段性复盘
```

### 4.3 关系维护
```
关系维护：
├── 平时多交流：建立信任
├── 主动提供帮助：互惠互利
├── 尊重对方：认可他们的工作
├── 及时反馈：告知结果和感谢
└── 分享信息：主动分享有用信息
```

## 5. 冲突处理

### 5.1 冲突原因
```
常见冲突：
├── 资源竞争：人力、资金、时间
├── 责任推诿：边界不清
├── 优先级冲突：紧急程度不同
├── 沟通误解：信息不对称
└── 利益冲突：影响部门利益
```

### 5.2 冲突处理
```
冲突处理5步法：
1. 冷静倾听：了解对方诉求
2. 表达立场：清楚说明自己的观点
3. 寻找共同点：明确双方都认可的
4. 协商方案：找到双方都能接受的
5. 达成共识：明确行动和责任

冲突升级：
├── L1：当事人协商
├── L2：请领导协调
└── L3：更高层决策
```
