## TL;DR

评价管理核心：好评引导→差评处理→口碑维护→回复优化，四步建立正向循环

## 1. 场景分析

### 请求摘要
用户需要管理外卖店铺评价，提升口碑

### 显性需求
- 掌握评价管理方法
- 获得可执行的解决方案

### 场景分类
- `scenario_classification`: 运营维护
- `urgency`: high

## 2. 评价管理体系

### 评价漏斗

```
用户下单 → 用餐 → 主动评价 → 好评/差评 → 商家回复
                              ↓
                          潜在用户看到 → 影响决策
```

### 评分影响

| 评分 | 对曝光影响 |
|------|------------|
| 4.8+ | 高权重曝光 |
| 4.5-4.7 | 正常曝光 |
| 4.0-4.4 | 降低曝光 |
| 4.0以下 | 流量受限 |

## 3. 专业建议

### A. 好评引导

#### 引导时机
```
1. 随餐小卡片：感谢+好评引导
2. 短信提醒：签收后提醒
3. 好评返券：激励好评
4. 包装惊喜：超出预期
```

#### 好评话术
```
"感谢您的支持！您的认可是我们最大的动力～
期待下次为您服务🌟"
```

### B. 差评处理

#### 差评类型

| 类型 | 原因 | 处理策略 |
|------|------|----------|
| 口味不合 | 个人偏好 | 解释+改进 |
| 配送问题 | 骑手/天气 | 道歉+改进 |
| 包装问题 | 破损/漏洒 | 赔偿+改进 |
| 份量问题 | 少送/不足 | 补发+改进 |
| 恶意差评 | 竞品/讹诈 | 申诉+证据 |

#### 差评响应原则
```
1. 速度：24小时内响应
2. 态度：真诚道歉
3. 解决：提出解决方案
4. 跟进：邀请再次体验
```

#### 差评回复模板
```
亲爱的顾客，非常抱歉给您带来不佳体验！
[具体问题说明]已收到并高度重视。
我们已经[改进措施]，期待下次为您带来更好的体验～
```

### C. 评价监控

#### 监控要点
```
1. 实时监控新评价
2. 分类统计差评原因
3. 定期分析评价趋势
4. 建立改进闭环
```

### D. 口碑优化

#### 长期策略
```
1. 稳定出品：口味一致性
2. 服务到位：响应及时性
3. 包装用心：用户体验
4. 持续改进：问题归因
```
