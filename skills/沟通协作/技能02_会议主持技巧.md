## TL;DR

会议主持 = 会前准备 + 会中控制 + 会后跟进 + 高效总结

## 1. 会议价值判断

### 1.1 是否要开会
```
开会判断标准：
□ 这个议题必须讨论才能解决吗？
□ 有没有办法不开会解决？
□ 能否用邮件/消息解决？
□ 是否需要集体讨论才能产出？
□ 是否需要同步决策？

不开会的替代方案：
├── 即时消息讨论
├── 邮件沟通决策
├── 文档异步讨论
├── 一对一沟通
└── 领导直接决定
```

### 1.2 会议分类
```
会议类型：
├── 决策会议：需要做出决定
├── 信息同步会议：传递信息
├── 问题解决会议：解决问题
├── 头脑风暴会议：产生创意
├── 进度跟踪会议：跟进进展
├── 培训会议：学习知识
└── 团建会议：增进关系
```

## 2. 会前准备

### 2.1 准备清单
```
会议准备清单：
□ 明确会议目的
□ 制定会议议程
□ 确定参会人员
□ 选择会议时间
□ 准备会议材料
□ 布置会议环境
□ 指定记录人
```

### 2.2 议程设计
```
议程设计原则：
├── 明确目的：每项议题要达成什么
├── 时间分配：给每项议题分配时间
├── 逻辑顺序：按重要程度或逻辑排列
├── 留有缓冲：预留时间调整
└── 提前发送：让参会者有准备

议程模板：
1. 开场（5分钟）
   - 会议目的
   - 议程介绍
   
2. 讨论议题1（15分钟）
   - 背景说明
   - 讨论问题
   - 得出结论

3. 讨论议题2（15分钟）
   ...

4. 总结（5分钟）
   - 回顾结论
   - 明确行动
```

### 2.3 参会人员
```
确定参会人员：
├── 必须参加：决策相关人员
├── 需要了解：信息同步对象
├── 邀请可选：可能贡献意见者
└── 排除在外：不需要知道的人

参会原则：
├── 人数精简：越少越好
├── 角色明确：决策者、参与者、旁听者
├── 提前通知：至少提前1天
└── 说明期望：让参会者知道要做什么
```

## 3. 会中主持

### 3.1 开场控制
```
开场要点：
├── 准时开始：不等待迟到者（除非重要）
├── 自我介绍：如有新人
├── 明确目的：为什么要开这个会
├── 介绍议程：时间安排
├── 约定规则：发言规则、时间控制
└── 指定记录：明确记录人

开场话术：
"大家好，感谢大家参会。今天会议目的是XX，
预计需要XX时间。议程有3项，首先XX，
请大家按议题讨论，我先讲一下基本规则..."
"
```

### 3.2 讨论引导
```
引导讨论技巧：
├── 聚焦议题：提醒不跑题
├── 鼓励参与：邀请沉默者发言
├── 控制时间：时间到了提醒
├── 归纳总结：把讨论要点归纳
├── 推进决策：引导做出决定
└── 处理冲突：调解分歧

常用话术：
├── "我们回到议题上来"
├── "XX你有什么看法？"
├── "时间差不多了，我们做个小结"
├── "这个分歧点，我们怎么处理"
├── "能否这样理解..."
└── "我们先确定XX，YY之后再讨论"
```

### 3.3 控制场面
```
处理常见问题：

跑题：
"XX话题很有价值，但我们现在先聚焦在XX上，
XX话题我们之后可以专门讨论"

一言堂：
"XX的观点很有价值，其他人怎么看？"
"我想听听不同角度的意见"

争论不休：
"我们先把分歧点列出来"
"能否先就共识部分达成一致？"
"这个分歧我们先记录，之后再处理"

沉默不语：
"XX你之前做过类似项目，有什么建议？"
"有没有其他想法？"

时间失控：
"时间不多了，我们抓紧"
"这个点很重要，我们超时5分钟"
```

### 3.4 会议节奏
```
时间控制技巧：
├── 按议程走：不要随意增加议题
├── 时间提醒：提前5分钟提醒
├── 果断收尾：到点就收尾
├── 记录未完成：没讨论完的记录下来
└── 延时补偿：超时后下个议题缩短
```

## 4. 会后跟进

### 4.1 会议纪要
```
纪要要素：
├── 会议基本信息
│   ├── 时间、地点
│   ├── 参会人员
│   └── 主持人、记录人
├── 讨论内容
│   ├── 议题
│   ├── 讨论要点
│   └── 分歧点
├── 会议结论
│   ├── 决策内容
│   └── 共识内容
├── 行动项
│   ├── 事项
│   ├── 负责人
│   └── 完成时间
└── 下次会议安排
```

### 4.2 跟进要点
```
跟进要点：
├── 及时发送：24小时内发出
├── 明确责任：每项行动落实到人
├── 跟踪进度：按时间节点跟踪
├── 反馈结果：完成后及时反馈
└── 公开透明：让相关人知道结果
```

## 5. 常见会议类型主持

### 5.1 决策会议
```
决策会议要点：
├── 提前准备方案
├── 明确决策标准
├── 给参会者决策权
├── 引导充分讨论
├── 明确决策结论
└── 记录决策理由
```

### 5.2 头脑风暴
```
头脑风暴要点：
├── 明确问题
├── 延迟评判
├── 追求数量
├── 鼓励发散
├── 记录所有想法
├── 组合创新
└── 快速收敛
```

### 5.3 进度会议
```
进度会议要点：
├── 提前收集进展
├── 关注阻塞点
├── 协助解决问题
├── 控制会议时间
├── 明确下一步
└── 避免变成讨论会
```
