## TL;DR

头脑风暴 = 充分准备 + 营造氛围 + 延迟评判 + 追求数量 + 组合创新 + 收敛评估

## 1. 头脑风暴认知

### 1.1 头脑风暴价值
```
头脑风暴价值：
├── 激发创意：不同思维碰撞
├── 集思广益：汇聚团队智慧
├── 参与感：增强团队凝聚力
├── 发现问题：暴露潜在问题
├── 建立共识：共同参与决策
└── 打破常规：突破思维定式
```

### 1.2 头脑风暴类型
```
头脑风暴类型：

常规头脑风暴：
├── 目标：产生大量创意
├── 原则：延迟评判、追求数量
└── 适合：产品创意、问题解决

逆向头脑风暴：
├── 目标：发现问题、预防风险
├── 方法：先想失败原因
└── 适合：风险评估、问题诊断

心智图风暴：
├── 目标：系统性思考
├── 方法：以主题为核心发散
└── 适合：复杂问题、系统规划

SCAMPER风暴：
├── 目标：改进现有方案
├── 方法：替代、组合、修改...
└── 适合：产品迭代、方案优化
```

### 1.3 传统头脑风暴问题
```
传统头脑风暴问题：
├── 评判太早：想法还没说完就被否定
├── 锚定效应：被第一个人带偏
├── 群体思维：少数人主导
├── 不敢发言：担心被嘲笑
├── 效率低下：讨论发散收不回来
└── 创意重复：想法相似不创新

这些问题导致：
├── 创意数量少
├── 创意质量低
├── 参与度不高
└── 时间浪费
```

## 2. 前期准备

### 2.1 准备清单
```
头脑风暴准备：
□ 明确目标：解决什么问题
□ 设定问题：清晰的问题陈述
□ 邀请人员：合适的人选
□ 选择时间：合适的时间段
□ 准备材料：白板、便签、笔
□ 选择场地：舒适、不被打扰
□ 确定主持：谁来主持
□ 准备破冰：开场热身
```

### 2.2 问题设计
```
好问题特点：
├── 具体明确：不要太大太泛
├── 开放式：允许多种答案
├── 可执行：有解决的可能
├── 有挑战性：需要动脑筋
└── 有价值：解决后有意义

问题设计示例：
❌ "怎么做好产品？"（太泛）
✅ "针对XX用户痛点，有什么创新解决方案？"

❌ "如何提升用户体验？"（太大）
✅ "用户在XX场景下，操作步骤能否减少50%？"
```

### 2.3 参与者选择
```
参与者选择：
├── 多元背景：不同岗位、不同经验
├── 合适人数：5-10人为宜
├── 角色明确：主持、记录、参与者
├── 提前通知：让参与者有准备
├── 排除权威：减少权威压制
└── 考虑性格：外向内向搭配
```

## 3. 实施过程

### 3.1 开场热身
```
开场热身：
├── 明确目的：今天要解决什么问题
├── 说明规则：头脑风暴规则
├── 破冰游戏：轻松的创意游戏
├── 减少压力：鼓励任何想法
└── 时间预告：说明大概时长

开场话术：
"大家好，今天我们来做一个头脑风暴
目标是想出10个解决XX问题的创意
规则很简单：1）延迟评判；2）追求数量；3）组合创新
先花2分钟热身，想一个疯狂的创意
不要担心可行性，现在只管想！
"
```

### 3.2 头脑风暴规则
```
核心规则：
1. 延迟评判
   └── 创意阶段不评价好坏
   
2. 追求数量
   └── 创意越多越好
   
3. 欢迎疯狂
   └── 疯狂的想法往往有价值
   
4. 组合改进
   └── 组合别人的想法
   
5. 平等参与
   └── 每个人都有发言权
```

### 3.3 创意激发技巧
```
创意激发技巧：
├── 随机词法：随机一个词激发联想
├── 逆向思考：从反面想
├── 假设无限：假设没有限制
├── 模仿借鉴：参考其他行业
├── 极端假设：极端情况会怎样
├── 换位思考：如果是XX会怎么做
└── 拆分问题：把大问题拆小
```

### 3.4 常见问题应对
```
冷场应对：
├── 主持人先说几个抛砖引玉
├── 用问题引导：如果是XX？
├── 点名邀请：邀请沉默者
├── 小组分组：先小组讨论
└── 休息一下：换个形式

偏离主题：
├── 记录偏离想法（可以之后讨论）
├── 拉回主题：我们回到XX问题
├── 提示：聚焦在XX方面

想法重复：
├── 鼓励组合：组合两个想法
├── 追问深化：能不能更进一步
└── 感谢已有：先记录这个
```

## 4. 创意收敛

### 4.1 收敛时机
```
收敛时机：
├── 数量达标：达到预期数量
├── 时间到：不要拖太长
├── 创意枯竭：确实想不出来了
├── 方向明确：已有好方向
└── 疲劳明显：大家开始走神
```

### 4.2 收敛方法
```
收敛方法：

分类归并：
├── 把相似想法归类
├── 每类取个主题名
└── 统计各类数量

投票筛选：
├── 每人选出3个最佳
├── 统计票数
└── 确定top创意

评估矩阵：
| 创意 | 可行性 | 价值 | 创新性 | 综合 |
|------|--------|------|--------|------|
| A    | 高     | 高   | 中     | ⭐⭐⭐ |
| B    | 中     | 高   | 高     | ⭐⭐⭐ |
| C    | 高     | 低   | 中     | ⭐⭐ |

可行性/创新性矩阵：
         ┌─────────┬─────────┐
         │  高创新  │  低创新  │
┌────────┼─────────┼─────────┤
│ 可行  │ 立即做   │  放弃   │
├────────┼─────────┼─────────┤
│不可行 │ 储备研究 │  放弃   │
└────────┴─────────┴─────────┘
```

### 4.3 收敛话术
```
收敛话术：
"我们现在已经有了30个创意，很棒！
现在进入收敛阶段
第一步，我们先把相似的归类
第二步，每人投3票选最重要的
第三步，我们选出top3深入讨论

先花5分钟归类...
"
```

## 5. 记录与跟进

### 5.1 记录要点
```
记录要点：
├── 所有创意：不要遗漏
├── 创意来源：谁提的
├── 分类整理：便于后续查看
├── 核心要点：创意精髓
├── 评估结论：讨论结果
└── 行动计划：后续安排
```

### 5.2 记录工具
```
记录工具：
├── 白板/白纸：实时记录
├── 便签：每人写自己的想法
├── 文档：飞书文档、石墨
├── 思维导图：XMind、MindMaster
├── 协作工具：Miro、Figma
└── 投票工具：投票小程序
```

### 5.3 跟进落实
```
跟进落实：
├── 明确责任：谁负责哪个创意
├── 制定计划：后续做什么
├── 设定时间：什么时候完成
├── 定期回顾：检查创意落地
├── 结果反馈：告知团队结果
└── 感谢参与：感谢大家的贡献
```
