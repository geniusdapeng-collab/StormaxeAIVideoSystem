## TL;DR

跨部门沟通 = 理解对方 + 明确诉求 + 互惠互利 + 高效沟通 + 结果导向

## 1. 跨部门沟通挑战

### 1.1 常见障碍
```
跨部门沟通障碍：
├── 目标不同：各部门的KPI不同
├── 优先级冲突：资源有限，时间紧张
├── 信息不对称：不了解对方的情况
├── 沟通风格不同：有的直接，有的委婉
├── 责任边界模糊：不知道谁负责
├── 利益冲突：可能影响对方利益
└── 缺乏信任：没有合作基础
```

### 1.2 沟通前认知
```
沟通前要了解：
├── 对方部门的核心职责是什么
├── 他们最关心什么指标
├── 他们面临什么困难和压力
├── 他们的工作流程是什么
├── 关键决策人是谁
├── 他们对你们部门的印象如何
└── 以前有没有合作过，合作如何
```

## 2. 沟通策略

### 2.1 沟通原则
```
跨部门沟通原则：
├── 平等尊重：不认为对方"应该"配合
├── 换位思考：理解对方的立场
├── 互惠互利：让对方也有收获
├── 提前沟通：不要等到最后一刻
├── 书面确认：重要事项要有记录
└── 关系维护：平时也要维护关系
```

### 2.2 沟通框架
```
沟通框架：
共同目标 + 具体需求 + 时间要求 + 互惠承诺 + 感谢

示例：
"为了Q3 GMV增长30%的共同目标（共同目标）
需要你们提供用户画像数据支持精准投放（具体需求）
请在周五前完成数据提取（时间要求）
后续我们可以提供转化数据帮助优化他们的模型（互惠承诺）
谢谢你们的支持！（感谢）
"
```

### 2.3 沟通渠道选择
```
沟通渠道选择：
├── 正式工单：重要、正式的需求
├── 邮件沟通：需要留记录的
├── 面对面沟通：重要、复杂的问题
├── 即时消息：日常沟通、跟进
├── 会议沟通：多方讨论、决策
└── 领导协调：无法达成一致时
```

## 3. 协作推进

### 3.1 建立协作机制
```
协作机制：
├── 明确责任人：谁是双方负责人
├── 定期同步：周会/双周会
├── 进度共享：共享文档/看板
├── 关键节点：约定里程碑
├── 问题升级：明确升级流程
└── 结果反馈：完成后及时反馈
```

### 3.2 跟进技巧
```
跟进原则：
├── 及时：不等待，主动跟进
├── 适度：不过于频繁也不忽视
├── 记录：跟进内容要有记录
├── 灵活：根据情况调整方式
└── 感谢：配合后要感谢

跟进话术：
"XX，关于XX事项，方便了解下进展吗？
我们这边需要在XX时间前完成，
目前遇到的问题是XX，
需要您这边协助的是XX，
请问有什么困难吗？
"
```

### 3.3 关系维护
```
平时维护关系：
├── 主动打招呼、聊天
├── 分享有用信息
├── 互相帮助支持
├── 理解对方难处
├── 取得成果后感谢
└── 不只在需要时才联系
```

## 4. 问题处理

### 4.1 被拒绝应对
```
被拒绝处理：
├── 理解原因：先了解为什么拒绝
├── 换位思考：理解对方的顾虑
├── 调整方案：看能否调整需求
├── 寻找替代：找其他方案解决
├── 提供支持：看能否减少对方负担
├── 升级处理：必要时请领导协调
└── 保持关系：即使被拒也要维护关系
```

### 4.2 冲突处理
```
冲突处理步骤：
1. 冷静倾听：了解对方的诉求
2. 表达立场：清楚说明自己的观点
3. 寻找共同点：明确双方都认可的
4. 协商方案：找到双方都能接受的
5. 达成共识：明确行动和责任
6. 记录确认：书面记录达成的共识

升级处理：
├── L1：当事人协商
├── L2：请领导协调
└── L3：更高层决策
```

### 4.3 推动困难事项
```
推动困难事项：
├── 充分准备：准备完整的方案和理由
├── 找到同盟：先和相关人员沟通
├── 借力打力：用数据、领导的话支持
├── 分解任务：把大任务分解
├── 小步推进：先推动一小步
├── 展示价值：让对方看到收益
└── 及时感谢：配合后要表达感谢
```
