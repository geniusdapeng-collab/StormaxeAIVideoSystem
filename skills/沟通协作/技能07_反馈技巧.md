## TL;DR

反馈 = 具体事实 + 影响说明 + 期望表达 + 共同改进

## 1. 反馈基础认知

### 1.1 反馈的价值
```
反馈的价值：
├── 提升绩效：帮助对方改进
├── 建立关系：真诚的反馈建立信任
├── 明确期望：让对方知道期望
├── 及时纠偏：避免小问题变大
├── 促进成长：帮助他人成长
└── 创造透明：透明沟通文化
```

### 1.2 反馈类型
```
反馈类型：

正向反馈：
├── 定义：肯定好的行为
├── 目的：强化正向行为
├── 时机：及时给予
└── 效果：激励和认可

建设性反馈：
├── 定义：指出改进空间
├── 目的：帮助改进提升
├── 时机：尽早发现尽早反馈
└── 效果：促进成长

纠正性反馈：
├── 定义：制止不当行为
├── 目的：纠正错误行为
├── 时机：立即指出
└── 效果：止损
```

### 1.3 反馈原则
```
反馈原则：
├── 具体而非抽象：具体的行为和事实
├── 对事不对人：关注行为而非人格
├── 及时而非延迟：趁热打铁
├── 平衡正负：正向和建设性结合
├── 双向沟通：倾听对方观点
├── 关注改变：目标是行为改变
└── 真诚友好：目的是帮助而非指责
```

## 2. 正向反馈技巧

### 2.1 正向反馈要素
```
正向反馈要素：
├── 具体行为：做得好的具体行为
├── 产生影响：带来的积极影响
├── 表达感谢：感谢对方的付出
├── 当众表扬：适时在团队面前认可
└── 持续关注：让对方知道你在关注
```

### 2.2 正向反馈话术
```
正向反馈话术：

通用模板：
"我注意到你XX（具体行为）
这让XX有了XX的改善/帮了团队大忙（影响）
真的很感谢你的付出（感谢）
继续保持，期待你更多的好表现（期望）
"

示例1：
"小王，这次项目你方案设计得很出色
特别是用户调研部分，方法选择很科学
数据报告也做得很清晰，让领导很满意
感谢你的专业和用心，期待你以后主导更多项目

示例2：
"小李，上周在客户会议上的表现太棒了
面对客户的追问，你的回答专业又有耐心
帮公司拿下了这个客户，太感谢了
你是我见过的最专业的销售之一
"
```

### 2.3 正向反馈场景
```
正向反馈场景：
├── 项目成功时
├── 帮助他人时
├── 克服困难时
├── 展现价值观时
├── 超越期望时
└── 日常工作中
```

## 3. 建设性反馈技巧

### 3.1 建设性反馈公式
```
建设性反馈公式：
观察（O）+ 影响（I）+ 期望（E）+ 讨论（D）

公式详解：
O（Observation）：观察具体行为
I（Impact）：说明影响
E（Expectation）：表达期望
D（Discussion）：开放讨论

示例：
"小王，我注意到这周的报告提交都比较晚（O）
这让团队的整体进度安排有点紧张（I）
希望以后能在周五之前提交报告（E）
有什么困难我们可以一起解决，你觉得呢？（D）
"
```

### 3.2 建设性反馈话术
```
建设性反馈话术：

开场：
"小王，有件事想和你聊一下
可能你觉得是小事，但我觉得有必要说一下
"

描述行为：
"我注意到这周的报告提交都比较晚
具体是周三和周五的报告都是在晚上11点后提交的
"

说明影响：
"这让团队的整体进度安排有点紧张
测试同事需要加班才能按时完成
"

表达期望：
"希望以后能在周五下午5点之前提交报告
这样有问题还可以当天修改
"

邀请讨论：
"你觉得有什么困难吗？
需要我这边提供什么支持吗？
"

结尾：
"先谢谢你理解，有问题随时沟通
"
```

### 3.3 反馈注意事项
```
反馈注意事项：
├── 选择时机：不要在公开场合
├── 选择地点：私下沟通更有效
├── 具体事实：不要模糊
├── 对事不对人：关注行为
├── 避免但是：正向后不要突然转折
├── 双向沟通：给对方表达机会
├── 达成共识：确认对方理解
└── 跟进改进：关注后续改变
```

## 4. 纠正性反馈

### 4.1 纠正性反馈时机
```
需要纠正性反馈的情况：
├── 违反公司政策
├── 行为影响他人
├── 严重绩效问题
├── 违反职业伦理
├── 持续重复犯错
└── 态度问题
```

### 4.2 纠正性反馈话术
```
纠正性反馈话术：

直接指出：
"小王，有件比较严肃的事需要和你谈
这关系到团队的合作
"

具体行为：
"我注意到你这周连续3天早上都迟到超过30分钟
今天的晨会你也缺席了
"

说明影响：
"迟到影响了XX项目的进度
团队其他同事也有反映
"

说明期望：
"公司的考勤制度需要严格遵守
准时参加会议是对团队的尊重
"

明确后果：
"如果再有类似情况，我需要按公司制度处理
"

要求承诺：
"你能承诺以后准时吗？
有什么困难需要我帮助解决吗？
"
```

## 5. 不同对象反馈

### 5.1 给下属反馈
```
给下属反馈要点：
├── 定期反馈：不要只在大项目后才反馈
├── 具体可执行：给出明确的改进建议
├── 关注成长：帮助下属成长
├── 平衡正负：正向反馈为主
├── 记录表现：做好绩效记录
└── 跟进改进：持续关注
```

### 5.2 给同事反馈
```
给同事反馈要点：
├── 私下反馈：避免公开场合
├── 尊重对方：不居高临下
├── 换位思考：理解对方角度
├── 建设态度：目的是解决问题
├── 提供帮助：一起找解决方案
└── 关系维护：保持友好
```

### 5.3 给领导反馈
```
给领导反馈要点：
├── 注意方式：选择合适的时机和方式
├── 数据支撑：用数据和事实说话
├── 换位思考：理解领导的处境
├── 建设方案：给出替代方案
├── 私下而非公开：不要让领导丢面子
└── 接受反馈：也主动向领导寻求反馈
```

## 6. 接收反馈

### 6.1 接收反馈态度
```
接收反馈态度：
├── 开放心态：不要防御
├── 倾听理解：先听完
├── 确认理解：复述确认
├── 感谢反馈：感谢对方的反馈
├── 提问澄清：有问题要问
├── 反思改进：认真思考
└── 跟进反馈：展示改变
```

### 6.2 接收反馈话术
```
接收反馈话术：

感谢反馈：
"谢谢你直接和我说，我很高兴你能指出
"

确认理解：
"我理解你的意思是XX，对吗？
"

询问详情：
"能举个例子说明吗？
"

反思承诺：
"我确实在这方面有提升空间
我会XX改进
"

感谢总结：
"感谢你的反馈，对我帮助很大
我会努力的
"
```
