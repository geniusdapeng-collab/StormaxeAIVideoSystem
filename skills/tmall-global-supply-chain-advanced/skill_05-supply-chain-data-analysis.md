# TL;DR

构建供应链数据驱动体系，从KPI定义到数据分析，从仪表盘设计到决策支持，让数据成为供应链优化的引擎。

# 场景分析

## 典型场景
- 老板问"为什么这个月利润下滑"，你只能说"可能是成本高了"
- 每天花2小时手动导数据做报表，效率低还容易出错
- 建了一堆数据看板，但没人看，因为没有洞察
- 数据分散在各个系统，不知道该看哪个

## 核心挑战
1. **数据孤岛**：系统分散，数据不互通
2. **指标混乱**：口径不统一，对不上数
3. **分析浅层**：只有描述，没有洞察
4. **决策脱节**：数据有了，但没人用

## 正确认知
- **数据是资产，分析是变现**
- **不是数据多，是分析透**
- **看板不是目的，决策支持才是**
- **数据质量是一切的基础**

# 决策框架

## 第一步：建立KPI体系

### 供应链KPI层级

```
战略层（高管看）
├── 供应链总成本
├── 库存周转天数
└── 订单履约率

战术层（经理看）
├── 采购成本
├── 物流成本
├── 库存结构
└── 供应商绩效

执行层（运营看）
├── 入库及时率
├── 出库准确率
├── 补货完成率
└── 异常处理时效
```

## 第二步：定义指标口径

### 指标定义表

| 指标 | 定义 | 计算公式 | 数据来源 |
|------|------|----------|----------|
| 库存周转天数 | 库存周转一次需要的天数 | 平均库存÷日均销量 | WMS |
| 订单履约率 | 按时完成订单的比例 | 按时完成÷总订单 | ERP |
| 入库及时率 | 按计划入库的比例 | 按时入库÷计划入库 | WMS |
| 采购成本率 | 采购成本占收入比 | 采购成本÷销售额 | 财务 |

## 第三步：设计数据看板

### 看板设计原则
| 原则 | 说明 |
|------|------|
| 目标导向 | 看板服务决策，不是展示 |
| 分层设计 | 不同层级看不同内容 |
| 实时更新 | 数据要新才有价值 |
| 异常高亮 | 异常数据要醒目 |

#### 库存健康度矩阵

```
           动销快
              ↑
              │
   滞销高价值 │    优化库存
   (立即清仓) │   (控制补货)
              │
              ├──────────────→ 动销快
              │
              │    快销高价值 │   主力销售
              │   (保持库存) │  (保障供应)
              │
              └──────────────→ 动销慢
              低价值              高价值
```

## 数据分析模型

### 一、同比环比分析

#### 分析模板

```
【同比分析】
本月 vs 上年同月
销售额：+15%（去年100万，今年115万）
成本：+12%（去年40万，今年44.8万）
毛利：+17%（去年60万，今年70.2万）
毛利率：+0.8pp（去年60%，今年60.8%）

【环比分析】
本月 vs 上月
销售额：+8%（上月106万，本月115万）
成本：+5%（上月42.7万，本月44.8万）
毛利：+10%（上月63.9万，本月70.2万）
毛利率：+0.8pp（上月60.3%，本月60.8%）

【分析结论】
1. 销售额增长主要由销量增长驱动（+12%）
2. 成本控制良好，增长低于收入增长
3. 毛利率小幅提升，成本优化见效
```

### 二、异常分析

#### 异常识别

```
Z-score异常检测：
Z = (X - μ) / σ
|X| > 2 → 异常
|X| > 3 → 严重异常

示例：某SKU日销量异常
- 近30天均值：100件
- 标准差：20件
- 今日销量：180件
- Z-score = (180-100)/20 = 4 → 异常！
```

### 三、归因分析

#### 毛利变动归因

```
毛利变动 = 销量贡献 + 价格贡献 + 成本贡献

示例：
上期毛利：60万
本期毛利：70万
变动：+10万

归因：
- 销量增长：+5万（销量↑15%，毛利贡献+5万）
- 价格变动：+2万（均价↑2%，毛利贡献+2万）
- 成本变动：+3万（成本↓5%，毛利贡献+3万）

验证：5+2+3 = 10万 ✓
```

### 四、预测分析

#### 趋势预测

```
方法：简单指数平滑
预测值 = α × 上期实际 + (1-α) × 上期预测

参数设置：
- α = 0.3（权重系数）

计算：
- Week1实际：100件，预测=100件
- Week2实际：110件，预测=0.3×100 + 0.7×100=100件
- Week3实际：105件，预测=0.3×110 + 0.7×100=103件
- Week4预测=0.3×105 + 0.7×103=103.6件
```

## 数据质量治理

### 数据质量维度

| 维度 | 定义 | 评估标准 |
|------|------|----------|
| 准确性 | 数据与实际一致 | 误差<1% |
| 完整性 | 数据无缺失 | 完整率>99% |
| 一致性 | 跨系统口径统一 | 口径统一 |
| 时效性 | 数据更新及时 | 更新频率达标 |
| 唯一性 | 无重复记录 | 重复率<0.1% |

### 数据治理措施

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据不准 | 人工录入错误 | 系统校验、抽检 |
| 数据缺失 | 接口问题 | 数据补录、接口优化 |
| 数据不一致 | 口径不统一 | 统一指标定义 |
| 数据滞后 | 更新频率低 | 实时同步 |
