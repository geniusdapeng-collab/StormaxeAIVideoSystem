# TL;DR

构建智能库存预警系统，实现断货前置预警、积压风险识别、补货时机精准把控，让库存管理从被动响应变为主动预防。

# 场景分析

## 典型场景
- 爆款突然爆单，库存来不及补，眼睁睁看着断货损失GMV
- 大促前备货太保守，活动效果超出预期，货卖光了
- 某个SKU不知不觉库存积压半年，占用大量资金
- 同时管理几百个SKU，人工盯不过来

## 核心挑战
1. **信息滞后**：发现断货时已经晚了
2. **预测不准**：不知道该备多少
3. **响应太慢**：发现问题了来不及反应
4. **维度太多**：SKU太多，人工盯不过来

## 正确认知
- **预警比补救更重要**
- **断货成本远高于积压成本**
- **数据驱动比经验判断更准确**
- **自动化是规模化的必由之路**

# 决策框架

## 第一步：建立预警指标体系

### 核心预警指标

| 指标 | 计算公式 | 预警阈值 | 应对 |
|------|----------|----------|------|
| 库存周转天数 | 平均库存÷日均销量 | >90天预警 | 促销清货 |
| 库存可售天数 | 当前库存÷日均销量 | <15天预警 | 紧急补货 |
| 断货风险指数 | 销量趋势×库存 | 高风险立即预警 | 锁定库存 |
| 滞销风险指数 | 库龄×销量趋势 | 高风险预警 | 促销处理 |
| 临期风险 | 效期预警 | <30天预警 | 优先出库 |

## 第二步：设置分级预警机制

### 预警分级

| 级别 | 触发条件 | 响应时限 | 处理方式 |
|------|----------|----------|----------|
| P0-紧急 | 库存=0 或 可售天数<5 | 2小时 | 立即补货/锁定流量 |
| P1-严重 | 可售天数<10 | 24小时 | 启动补货流程 |
| P2-警告 | 可售天数<15 | 72小时 | 计划补货 |
| P3-关注 | 可售天数<20 | 1周 | 观察趋势 |

## 第三步：建立自动化预警流程

### 预警推送机制

```
每日早9点 → 系统自动扫描 → 异常预警推送 → 责任人响应 → 闭环跟踪
```

# 专业建议

## 安全库存计算模型

### 基础安全库存公式
```
安全库存 = (最大日销量 - 平均日销量) × 备货周期 × 安全系数
```

### 动态安全库存
```
SS = Z × σ × √(LT)
其中：
- Z = 服务水平对应系数（95%→1.65，99%→2.33）
- σ = 需求标准差
- LT = 备货周期（天）
```

### 不同场景的安全系数

| 场景 | 安全系数 | 说明 |
|------|----------|------|
| 爆款/高优先级 | 1.3-1.5 | 宁多勿少 |
| 普通款 | 1.0-1.2 | 标准配置 |
| 长尾款 | 0.8-1.0 | 保守备货 |
| 大促期间 | 1.5-2.0 | 高风险加大 |

## 智能补货建议系统

### 补货点计算
```
ROP = 日均销量 × 备货周期 + 安全库存
```

### 经济订货量
```
EOQ = √(2 × 年需求量 × 订货成本 / 单位持有成本)
```

### 补货量计算
```
补货量 = 目标库存 - 当前库存 - 在途库存
目标库存 = 安全库存 + (备货周期 × 日均销量) + 最低库存
```

### 智能补货决策树

```
销量趋势分析
├── 稳定品 → 用历史均值预测
├── 上升品 → 用趋势预测，额外+20%
├── 下降品 → 用趋势预测，额外-10%
└── 波动品 → 用最大值预测
```

## 断货预警系统

### 断货风险评估模型
```
断货风险 = 销量强度 × (1 - 库存覆盖率)
库存覆盖率 = 当前库存 ÷ 预测需求
```

### 断货预警分级

#### P0-紧急断货
| 条件 | 影响 |
|------|------|
| 库存=0 | 已断货 |
| 可售天数<5 | 预计5天内断货 |

**应对策略**：
1. 立即通知运营停止推广
2. 启动紧急补货（空运）
3. 评估竞品替代方案
4. 向消费者致歉说明

#### P1-严重预警
| 条件 | 影响 |
|------|------|
| 可售天数5-10 | 预计5-10天断货 |

**应对策略**：
1. 评估库存消耗速度
2. 启动补货订单
3. 降低推广投入
4. 准备断货页面

#### P2-预警关注
| 条件 | 影响 |
|------|------|
| 可售天数10-15 | 预计10-15天断货 |

**应对策略**：
1. 纳入补货计划
2. 监控销售趋势
3. 调整推广预算

## 库存积压预警

### 积压风险评估
```
积压风险 = 滞销天数 × 库存量 × 资金占用
```

### 积压预警指标

| 指标 | 定义 | 预警阈值 |
|------|------|----------|
| 库龄结构 | 各库龄占比 | >60天>30% |
| 周转天数 | 库存周转速度 | >90天 |
| 动销率 | 有销量SKU占比 | <60% |
| 死库率 | 无动销SKU占比 | >20% |

### 积压处理策略

#### 轻度积压（30-60天）
| 策略 | 适用场景 |
|------|----------|
| 促销活动 | 价格敏感品 |
| 捆绑销售 | 可搭配品 |
| 会员专享 | 老客回馈 |
| 直播带货 | 冲动消费 |

#### 中度积压（60-90天）
| 策略 | 适用场景 |
|------|----------|
| 大幅降价 | 非必需品 |
| 批量清仓 | 资金压力 |
| 捐赠处理 | 品牌正向 |
| 退供处理 | 有协议支持 |

#### 重度积压（>90天）
| 策略 | 适用场景 |
|------|----------|
| 成本价清仓 | 止损 |
| 退运海外 | 有关税优惠 |
| 销毁处理 | 无价值商品 |

## 效期预警系统

### 效期管理原则
| 类型 | 预警周期 | 处理策略 |
|------|----------|----------|
| 食品/保健品 | 90天预警 | 优先出库 |
| 化妆品 | 60天预警 | 促销清货 |
| 一般商品 | 30天预警 | 关注跟进 |

### 效期预警分级
```
L-90：正常销售
L-60：促销出库
L-30：紧急处理
L-15：全力清仓
L-0：过期/销毁
```
