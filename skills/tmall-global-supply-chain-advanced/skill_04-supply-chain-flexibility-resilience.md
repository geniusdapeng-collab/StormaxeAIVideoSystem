# TL;DR

构建柔性响应的供应链韧性体系，从风险识别到应急响应，从多源供应到快速恢复，让供应链从"脆弱"变"韧性"。

# 场景分析

## 典型场景
- 单一供应商突然倒闭，货断了几个月
- 海运延误，货在海上漂了两个月
- 关税政策突变，备的货全部卡在海关
- 大促前供应商产能爆雷，货做不出来
- 汇率剧烈波动，利润被吃掉

## 核心挑战
1. **单点故障**：过度依赖单一供应商/物流商
2. **响应滞后**：发现问题才应对，来不及
3. **风险盲区**：只看到已知的风险，看不到黑天鹅
4. **恢复漫长**：出问题后恢复周期太长

## 正确认知
- **韧性比效率更重要**
- **冗余是成本，但救命时值回票价**
- **预案比应变更重要**
- **快速恢复比防止发生更现实**

## 第一步：风险识别

| 风险类别 | 风险点 | 发生概率 | 影响程度 |
|----------|--------|----------|----------|
| 供应端 | 供应商破产 | 低 | 高 |
| 供应端 | 产能不足 | 中 | 高 |

## 第二步：韧性评估

### 供应链韧性指数

| 指标 | 权重 | 评估内容 |
|------|------|----------|
| 多样性 | 25% | 供应商数量、品类分散度 |
| 冗余度 | 25% | 库存水位、安全库存 |
| 可视性 | 20% | 数据透明度、预警能力 |
| 响应速度 | 15% | 决策链路、应急机制 |
| 恢复能力 | 15% | 恢复时间、备份方案 |

### 韧性评估表

| 等级 | 综合分 | 特征 |
|------|--------|------|
| 优秀 | 90+ | 多供应商、充足库存、完备预案 |
| 良好 | 80-89 | 有备份、库存适中、应急机制 |
| 一般 | 70-79 | 单供应商、库存紧张、响应慢 |
| 脆弱 | <70 | 高度依赖、无备份、恢复慢 |

## 第三步：韧性建设

### 四维韧性模型

```
韧性 = 冗余性 + 敏捷性 + 可视性 + 协作性
```

### 典型应急预案

#### 供应商断供预案
```
触发条件：供应商无法按时供货

立即响应：
1. 确认断供原因和预计恢复时间（2小时内）
2. 评估库存可支撑天数
3. 启动备选供应商（24小时内）

短期应对：
1. 空运紧急补货
2. 调整推广，减少流量
3. 页面告知消费者

长期解决：
1. 切换主供应商
2. 分散供应商依赖
3. 增加安全库存
```

#### 物流延误预案
```
触发条件：物流时效超过预期5天以上

立即响应：
1. 确认延误原因和预计时间
2. 评估对消费者的影响
3. 启动消费者沟通

短期应对：
1. 协调加急处理
2. 启用备用物流通道
3. 灵活调整发货方式

长期解决：
1. 多物流商配置
2. 增加海外仓储备
3. 优化备货周期
```

#### 政策变化预案
```
触发条件：关税/准入政策突然变化

立即响应：
1. 评估政策影响范围
2. 统计受影响商品
3. 测算成本变化

短期应对：
1. 调整商品定价
2. 优化商品组合
3. 加快清库存

长期解决：
1. 分散货源地
2. 提前合规布局
3. 模式灵活切换
```
