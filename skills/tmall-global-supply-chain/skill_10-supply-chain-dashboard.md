# TL;DR

**供应链数据仪表盘的核心是通过构建覆盖库存/履约/采购/成本/客户体验五大维度的KPI体系，结合可视化看板和智能预警机制，实现供应链健康度的实时监控和异常快速响应，最终支撑数据驱动的补货决策、滞销处理和供应商管理。**

# 场景分析

### 典型场景

**场景一：每日供应链健康度检查**
运营团队每天早上需要快速了解供应链整体状态：库存是否充足、履约是否正常、有没有异常预警。

**场景二：库存告急时的补货决策**
系统提示某爆款库存只能支撑5天，需要快速评估补货紧急程度、计算最优补货量。

**场景三：大促前的备货分析**
双11大促前，需要全面评估各品类备货情况，识别可能的缺货风险点，制定应急预案。

**场景四：滞销库存清理决策**
某批长库龄库存已经积压超过90天，需要评估是继续持有还是立即清仓处理。

### 核心挑战

- **数据分散**：销售数据、库存数据、物流数据分属不同系统，整合困难
- **指标繁多**：KPI几十个，不知道哪些是关键、看哪些能解决问题
- **预警滞后**：异常已经发生了才告警，来不及反应
- **决策无据**：数据有了但不知道怎么用，拍脑袋决策
- **报告耗时**：每周每月做报告花大量时间，但价值有限

### 正确认知

- **仪表盘是工具，决策才是目的**：不是把数据堆上去就行，而是要支撑业务决策
- **关键指标不超过10个**：眉毛胡子一把抓等于没抓，要聚焦核心
- **预警要前置**：不是等断货了才告警，而是库存<7天就预警
- **数据要驱动行动**：看数据是为了发现问题、解决问题，不是为了看而看
- **仪表盘要持续迭代**：业务在变，仪表盘也要跟着变

# 决策框架

### Step 1：供应链健康度评估

**综合评分体系：**
| 维度 | 权重 | 优秀 | 良好 | 一般 | 差 |
|-----|-----|-----|-----|-----|---|
| 库存健康 | 25% | 90+ | 80-89 | 70-79 | <70 |
| 履约效率 | 25% | 98%+ | 95-97% | 90-94% | <90% |
| 供应保障 | 20% | 98%+ | 95-97% | 90-94% | <90% |
| 成本控制 | 15% | <5% | 5-8% | 8-10% | >10% |
| 客户体验 | 15% | 98%+ | 95-97% | 90-94% | <90% |

**综合评分 = Σ(维度评分 × 权重)**

### Step 2：库存异常识别

**预警等级体系：**
| 等级 | 标识 | 触发条件 | 响应时间 |
|-----|-----|---------|---------|
| 红色 | 🔴 | 库存=0 或 预计断货<3天 | 1小时内 |
| 橙色 | 🟠 | 库存<7天销售 | 4小时内 |
| 黄色 | 🟡 | 库存<14天销售 | 24小时内 |
| 蓝色 | 🔵 | 库存<21天销售 | 72小时内 |

### Step 3：补货决策

**补货优先级矩阵：**
| 优先级 | 条件 | 处理方式 |
|--------|------|---------|
| P0-紧急 | 库存=0 或 预计断货<3天 | 紧急空运 |
| P1-高优 | 库存<7天销售 | 优先海运+空运组合 |
| P2-正常 | 库存<14天销售 | 正常海运补货 |
| P3-优化 | 库存充足 | 暂缓补货，优化库存结构 |

**补货量计算：**
```
目标库存 = 日均销量 × 备货周期 × 备货系数
补货量 = 目标库存 - 当前库存 - 在途库存 + 安全库存
```

### Step 4：滞销处理决策

**库龄-毛利决策矩阵：**
| 库龄 | 毛利率 | 决策建议 |
|------|--------|---------|
| 60-90天 | >30% | 平台活动清仓 |
| 60-90天 | 10-30% | 直播/社群销售 |
| 60-90天 | <10% | 退回供应商 |
| >90天 | 任意 | 折扣清仓/捐赠 |

# 专业建议

### 策略层：KPI体系构建

1. **核心KPI不超过10个**：聚焦真正影响业务的关键指标，避免指标过载
2. **指标要分层**：战略层看综合评分，运营层看明细指标，执行层看具体动作
3. **指标要关联**：库存和履约要联动分析，断货率和GMV损失要关联计算
4. **指标要更新**：业务变化时指标体系也要迭代，不要一成不变

### 执行层：看板设计

1. **看板上只放关键信息**：不要把所有数据都堆上去，决策者只需要看关键指标
2. **红黄绿灯要清晰**：一眼能看出有没有问题，问题在哪里
3. **异常要能下钻**：点击异常指标能看到明细数据，定位到具体SKU
4. **更新频率要匹配**：实时指标实时更新，日报类指标每天更新

### 风控层：预警机制

1. **预警要前置**：不是等出问题了才告警，而是风险苗头出现时就告警
2. **预警要分级**：不同级别有不同的响应时效和处理方式
3. **预警要闭环**：告警发出后要有跟进、有处理、有闭环
4. **预警要复盘**：经常误报的预警要调整阈值，漏报的预警要补充规则

## 二、供应链核心KPI体系

### 2.1 库存健康指标

| KPI名称 | 计算公式 | 目标值 | 预警阈值 |
|---------|---------|--------|---------|
| 库存周转天数（DIO） | 平均库存成本 ÷ 日均销售额 × 100% | 45-60天 | >75天 |
| 库存周转率（ITO） | 销货成本 ÷ 平均库存 | 6-8次/年 | <4次/年 |
| 库存天数（DOS） | 期末库存 ÷ 日均销售量 | 30-50天 | >60天 |
| 滞销库存占比 | 滞销SKU库存额 ÷ 总库存额 | <10% | >15% |
| 临期库存占比 | 效期<90天库存 ÷ 总库存 | <5% | >8% |
| 库存准确率 | （盘点准确数 ÷ 盘点总数）× 100% | ≥99.5% | <99% |

### 2.2 履约效率指标

| KPI名称 | 计算公式 | 目标值 | 预警阈值 |
|---------|---------|--------|---------|
| 订单履约率 | （按时履约订单 ÷ 总订单）× 100% | ≥98% | <95% |
| 平均履约时长 | 支付到发货的平均小时数 | ≤24小时 | >36小时 |
| 发货及时率 | （T+1发货订单 ÷ 当日订单）× 100% | ≥95% | <90% |
| 仓库作业效率 | 人均每小时处理订单量 | ≥50单/时 | <40单/时 |
| 揽收及时率 | （24小时内揽收订单 ÷ 应发货订单）× 100% | ≥97% | <93% |

### 2.3 采购供应指标

| KPI名称 | 计算公式 | 目标值 | 预警阈值 |
|---------|---------|--------|---------|
| 供应商准时交货率 | （准时交货次数 ÷ 交货总次数）× 100% | ≥95% | <90% |
| 采购提前期（LT） | 下单到到货的平均天数 | 7-14天 | >21天 |
| 订单达成率 | （到货量 ÷ 采购量）× 100% | ≥98% | <95% |
| 缺货率 | （缺货SKU数 ÷ 总SKU数）× 100% | <3% | >5% |
| 安全库存覆盖率 | 设置安全库存的SKU占比 | ≥90% | <80% |

### 2.4 成本控制指标

| KPI名称 | 计算公式 | 目标值 | 预警阈值 |
|---------|---------|--------|---------|
| 库存持有成本率 | （持有成本 ÷ 平均库存值）× 100% | 20-25% | >30% |
| 仓储成本占比 | （仓储费用 ÷ 总销售额）× 100% | <3% | >4% |
| 物流成本占比 | （物流费用 ÷ 总销售额）× 100% | <8% | >10% |
| 缺货成本 | 因缺货导致的销售损失金额 | <1%GMV | >2%GMV |
| 报废成本率 | （报废金额 ÷ 总库存额）× 100% | <0.5% | >1% |

### 2.5 客户体验指标

| KPI名称 | 计算公式 | 目标值 | 预警阈值 |
|---------|---------|--------|---------|
| 晚到家赔付率 | （赔付订单 ÷ 总订单）× 100% | <0.5% | >1% |
| 逆向物流率 | （退货订单 ÷ 总订单）× 100% | <5% | >8% |
| 商品完好率 | （完好商品 ÷ 配送商品总数）× 100% | ≥99.9% | <99.5% |

## 四、库存分析模型

### 4.1 安全库存计算模型

#### 4.1.1 基础安全库存公式

**公式：**

```
SS = Z × σ × √(LT)

其中：
SS = 安全库存
Z = 服务水平对应的Z值（见下表）
σ = 需求标准差
LT = 采购提前期（天）
```

**Z值与服务水平对照表：**

| 服务水平 | Z值 | 适用场景 |
|---------|-----|---------|
| 90% | 1.28 | 普通商品 |
| 95% | 1.65 | 主力商品 |
| 97% | 1.88 | 爆款商品 |
| 99% | 2.33 | 限量商品 |

#### 4.1.2 动态安全库存模型

**考虑因素：**
- 季节性系数（Seasonal Factor）
- 促销活动影响（Promotion Lift）
- 竞品影响系数（Competitive Factor）
- 供应链风险系数（Supply Risk）

**修正公式：**

```
SS_adjusted = SS_base × SF × PF × CF × SRF

其中：
SS_base = 基础安全库存
SF = 季节性系数（1.0 + 季节波动幅度）
PF = 促销系数（1.0 + 促销预期增长%）
CF = 竞争系数（基于竞品上新/下架计划）
SRF = 供应链风险系数（供应商稳定性评分）
```

#### 4.1.3 天猫国际场景特殊处理

**保税仓备货模型：**
```
建议备货量 = 日均销量 × (补货周期 + 安全库存天数) × 备货系数

备货系数 = 1.2（进口商品物流不确定性补偿）
```

**海外仓备货模型：**
```
建议备货量 = 日均销量 × 备货周期 × 销售增长系数 × 风险系数

销售增长系数 = 1 + 预期增长率
风险系数 = 1.3（跨境物流风险补偿）
```

### 4.2 库存周转分析模型

#### 4.2.1 库存周转率计算

**方法一：成本法：**
```
ITO = 销售成本 ÷ 平均库存成本

平均库存成本 = (期初库存 + 期末库存) ÷ 2
```

**方法二：销售额法：**
```
ITO = 销售额 ÷ 平均库存金额
```

**天猫国际特殊考量：**
- 保税仓库存单独核算
- 海外仓库存计入在途
- 区分现货周转和预售周转

#### 4.2.2 库存周转优化策略

| 库存类型 | 周转天数 | 优化策略 |
|---------|---------|---------|
| 爆款SKU | <15天 | 保证现货，快速补货 |
| 主力SKU | 15-30天 | 稳定补货，保持安全库存 |
| 长尾SKU | 30-60天 | 控制采购量，优化陈列 |
| 滞销SKU | >60天 | 促销清仓，退回供应商 |
| 死库存 | 无限 | 强制处理（捐赠/销毁/折扣） |

### 4.3 需求预测模型

#### 4.3.1 基础预测方法

**移动平均法：**
```
F(t+1) = [D(t) + D(t-1) + ... + D(t-n+1)] ÷ n

其中：
D = 历史销量
n = 移动平均期数（建议7天或30天）
```

**指数平滑法：**
```
F(t+1) = α × D(t) + (1-α) × F(t)

其中：
α = 平滑系数（0<α<1，建议0.3）
D(t) = 实际销量
F(t) = 预测值
```

#### 4.3.2 天猫国际场景预测因素

| 影响因素 | 权重 | 数据来源 |
|---------|------|---------|
| 历史销量趋势 | 40% | ERP系统 |
| 促销活动计划 | 25% | 运营计划 |
| 搜索热度变化 | 15% | 生意参谋 |
| 竞品动态 | 10% | 第三方数据 |
| 季节性因素 | 10% | 历史规律 |

#### 4.3.3 预测准确性评估

**MAPE（平均绝对百分比误差）：**
```
MAPE = (1/n) × Σ|实际值 - 预测值| ÷ 实际值 × 100%

评估标准：
MAPE < 10% → 预测准确
MAPE 10-20% → 预测可用
MAPE > 20% → 预测偏差大，需调整模型
```

### 4.4 补货建议模型

#### 4.4.1 自动补货触发规则

**补货点公式：**
```
ROP = 日均销量 × 采购提前期 + 安全库存

当库存 ≤ ROP时，触发补货建议
```

#### 4.4.2 补货量计算

**经济订货量（EOQ）：**
```
Q = √(2 × D × S ÷ H)

其中：
Q = 最佳订货量
D = 年需求量
S = 每次订货成本
H = 单位库存持有成本
```

**天猫国际简化模型：**
```
建议补货量 = 目标库存 - 当前库存 - 在途库存 + 安全库存

目标库存 = 日均销量 × 备货周期 × 备货系数
```

#### 4.4.3 补货优先级排序

| 优先级 | 条件 | 处理方式 |
|--------|------|---------|
| P0-紧急 | 库存=0 或 预计断货<3天 | 紧急空运 |
| P1-高优 | 库存<7天销售 或 ROP触发 | 优先海运 |
| P2-正常 | 库存<14天销售 | 正常补货 |
| P3-优化 | 库存充足，优化库存结构 | 暂缓补货 |

### 5.3 决策场景库

#### 5.3.1 补货决策场景

**场景：爆款SKU库存告急**

**决策流程：**
1. 识别触发条件：库存<7天销售
2. 计算补货量：基于日均销量×备货周期
3. 评估物流选项：空运（快/贵）vs 海运（慢/便宜）
4. 权衡成本与收益：
   - 空运成本 = 空运费 + 可能的断货损失
   - 海运成本 = 仓储成本 + 可能的滞销风险
5. 做出决策：根据利润率和库存策略选择

**决策建议：**
- 高毛利商品（>50%）：优先空运保供
- 低毛利商品（<20%）：考虑海运控成本
- 季节性商品：以海运为主，备足货

#### 5.3.2 滞销处理决策场景

**场景：长库龄库存处理**

**决策矩阵：**

| 库龄 | 毛利率 | 决策建议 |
|------|--------|---------|
| 60-90天 | >30% | 平台活动清仓 |
| 60-90天 | 10-30% | 直播/社群销售 |
| 60-90天 | <10% | 退回供应商 |
| >90天 | 任意 | 折扣清仓/捐赠 |

**决策要点：**
1. 评估处理成本 vs 仓储成本
2. 考虑对品牌形象的潜在影响
3. 预留足够清仓窗口期
4. 制定止损底线

#### 5.3.3 供应商选择决策场景

**场景：新供应商准入评估**

**评估维度：**
1. **资质合规性**（一票否决）
   - 进口资质齐全
   - 产品合规认证
   - 质量体系认证

2. **供货能力**
   - 产能充足度
   - 交付稳定性
   - 应急响应能力

3. **成本竞争力**
   - 采购价格
   - 物流成本
   - 付款账期

4. **合作意愿**
   - 最低起订量
   - 退换货政策
   - 数据共享意愿

**综合评分≥80分 → 合格供应商**
