## TL;DR

权限 = 功能权限 + 数据权限 + 字段权限

## 1. 权限模型

### RBAC模型
| 概念 | 说明 |
|------|------|
| 用户 | 系统账号 |
| 角色 | 职能身份 |
| 权限 | 操作许可 |
| 用户-角色 | 一个用户多角色 |
| 角色-权限 | 一个角色多权限 |

### 权限层次
| 层级 | 管控内容 |
|------|----------|
| 功能权限 | 菜单、按钮、功能 |
| 数据权限 | 可见数据范围 |
| 字段权限 | 字段读写权限 |

## 2. 设计步骤

### 第一步：梳理角色
- 超级管理员：全部权限
- 企业管理员：企业级管理
- 部门管理员：部门级管理
- 业务角色：业务操作

### 第二步：定义权限
```markdown
## 权限矩阵

| 功能 | 管理员 | 经理 | 员工 |
|------|--------|------|------|
| 用户管理 | 全部 | 查看 | 无 |
| 审批 | 全部 | 本部门 | 创建 |
| 查看 | 全部 | 本部门 | 个人 |
```

### 第三步：数据权限
| 范围 | 说明 |
|------|------|
| 全部数据 | 所有数据 |
| 部门数据 | 本部门数据 |
| 个人数据 | 仅自己数据 |
| 自定义 | 按配置 |
