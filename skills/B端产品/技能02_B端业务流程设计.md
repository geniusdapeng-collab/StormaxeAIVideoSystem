## TL;DR

流程设计 = 角色 → 任务 → 路径 → 异常 → 优化

## 1. 流程设计原则

### 核心原则
| 原则 | 说明 |
|------|------|
| 最小权限 | 只给必要权限 |
| 闭环 | 有始有终 |
| 可追溯 | 记录可查 |
| 异常容错 | 异常可处理 |

### B端 vs C端流程
| 维度 | B端 | C端 |
|------|-----|-----|
| 复杂度 | 高 | 低 |
| 角色多 | 是 | 少 |
| 审批 | 常有 | 少 |
| 异常 | 需重点处理 | 简单处理 |

## 2. 流程设计步骤

### 第一步：识别角色和任务
```markdown
## 角色任务矩阵

| 角色 | 核心任务 | 触发条件 | 产出 |
|------|----------|----------|------|
| 提交人 | 创建/提交 | 业务需要 | 申请单 |
| 审批人 | 审核 | 收到待办 | 审批意见 |
| 执行人 | 落地 | 审批通过 | 执行结果 |
```

### 第二步：设计主流程
1. 正常路径：最常用、最简单的路径
2. 分支路径：不同条件不同处理
3. 并行路径：可同时进行

### 第三步：设计异常流程
| 异常类型 | 处理方式 |
|----------|----------|
| 审批拒绝 | 退回到上一步/发起人 |
| 审批超时 | 自动通过/转交/提醒 |
| 系统异常 | 记录日志、人工介入 |
| 角色缺失 | 代理/跳过/通知 |

### 第四步：审批流设计
```markdown
## 审批流配置

### 审批模式
- 顺序审批：按顺序逐个审批
- 会签审批：所有人通过
- 或签审批：任一人通过

### 审批规则
- 条件触发：满足条件才审批
- 金额阈值：不同金额不同流程
- 角色优先：高优先级先审批
```

## 3. 流程图规范

### 常用符号
| 符号 | 含义 |
|------|------|
| ○ | 开始/结束 |
| □ | 处理/操作 |
| ◇ | 判断/分支 |
| → | 流程方向 |

### 流程图模板
```
开始 → 提交申请 → [金额>1万?] → 是 → 部门审批 → 总经理审批 → 结束
                → 否 → 部门审批 → 结束
```
