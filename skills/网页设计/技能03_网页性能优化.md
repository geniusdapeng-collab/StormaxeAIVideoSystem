## TL;DR

网页性能优化核心：优化LCP（加载）、FID（交互）、CLS（稳定）三大指标

## 1. 场景分析

### 请求摘要
用户希望优化网页加载速度和运行性能

### 显性需求
- 提升页面加载速度
- 优化用户体验

### 场景分类
- `scenario_classification`: 前端性能
- `urgency`: high

## 2. Core Web Vitals 核心指标

| 指标 | 含义 | 良好标准 | 优化重点 |
|------|------|----------|----------|
| LCP | 最大内容绘制 | <2.5s | 服务器响应、图片 |
| FID | 首次输入延迟 | <100ms | JS执行、代码分割 |
| CLS | 累积布局偏移 | <0.1 | 图片尺寸、广告 |

## 3. 专业建议

### A. 加载优化（LCP）

#### 图片优化
```html
<!-- 现代格式 + 响应式 -->
<img src="image.webp" 
     srcset="image-400.webp 400w,
             image-800.webp 800w"
     sizes="(max-width: 600px) 100vw, 50vw"
     loading="eager" 
     width="800" height="600"
     alt="描述">
```

#### 关键CSS内联
```html
<style>
  /* 首屏CSS必须内联 */
  .header, .hero, .nav { ... }
</style>
```

#### 资源预加载
```html
<!-- 预加载关键资源 -->
<link rel="preload" href="critical.js" as="script">
<link rel="preload" href="hero.jpg" as="image">
```

### B. 交互优化（FID）

#### 代码分割
```javascript
// React示例
const LazyComponent = React.lazy(() => import('./HeavyComponent'));

// Vue示例
const HeavyComponent = () => import('./HeavyComponent')
```

#### 减少JS阻塞
```html
<!-- 异步加载 -->
<script src="analytics.js" async></script>

<!-- 延迟加载 -->
<script src="footer.js" defer></script>
```

### C. 视觉稳定（CLS）

#### 图片/视频尺寸
```html
<!-- 必须指定尺寸 -->
<img width="800" height="600" ...>
<video width="640" height="480" ...>
```

#### 广告/动态内容占位
```css
.ad-slot {
  min-height: 250px;
  background: #f0f0f0;
}
```

### D. 缓存策略

```javascript
// Service Worker缓存
self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(response => response || fetch(event.request))
  );
});
```
