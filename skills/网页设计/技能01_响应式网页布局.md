## TL;DR

创建响应式网页布局的核心方法：移动优先 + 弹性布局 + 媒体查询断点

## 1. 场景分析

### 请求摘要
用户需要创建或优化响应式网页，使其在手机、平板、桌面等不同设备上都能完美呈现

### 显性需求
- 掌握响应式设计核心概念
- 获取可执行的布局代码

### 隐性需求
- 性能优化
- 用户体验一致性

### 场景分类
- `scenario_classification`: 前端开发
- `urgency`: normal

### 约束
- 移动端优先原则
- 保持代码简洁
- 考虑性能

## 2. 决策摘要

### 主策略
采用"移动优先"策略，从最小屏幕开始设计，逐步增强

### 备选方案
- 桌面优先：从桌面版开始，逐步缩小
- 混合策略：根据项目特点选择

## 3. 专业建议

### A. 核心方法论

#### 移动优先 (Mobile First)
```css
/* 基础样式（移动端） */
.container { width: 100%; padding: 16px; }

/* 平板断点 */
@media (min-width: 768px) {
  .container { width: 750px; margin: 0 auto; }
}

/* 桌面断点 */
@media (min-width: 1024px) {
  .container { width: 960px; }
}

/* 大屏断点 */
@media (min-width: 1440px) {
  .container { width: 1200px; }
}
```

#### 视口设置
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

#### 弹性盒模型 (Flexbox)
```css
/* 垂直居中 */
.center-box {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}

/* 响应式导航 */
.nav {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
```

#### CSS Grid 栅格
```css
/* 响应式栅格 */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 20px;
}
```

### B. 最佳实践

| 实践 | 说明 |
|------|------|
| 断点选择 | 320px, 480px, 768px, 1024px, 1440px |
| 相对单位 | 使用 rem/em 替代 px |
| 图片适配 | 使用 srcset 和 sizes |
| 字体流式 | 使用 clamp() 函数 |

### C. 常见问题解决

1. **图片溢出**: `max-width: 100%; height: auto;`
2. **文字溢出**: `overflow: hidden; text-overflow: ellipsis;`
3. **触摸区域**: 最小 44x44px
4. **iOS 安全区**: `env(safe-area-inset-*)`
