## TL;DR

设计稿还原核心：提取设计规范 → 分析布局结构 → 编写结构化HTML/CSS → 逐项对比验证

## 1. 场景分析

### 请求摘要
用户需要将设计稿高质量还原为代码

### 显性需求
- 掌握设计稿分析方法
- 获得高效还原流程

### 场景分类
- `scenario_classification`: 前端开发
- `urgency`: normal

## 2. 还原流程

| 步骤 | 操作 | 工具 |
|------|------|------|
| 1 | 获取设计稿 | Figma/Sketch/PS |
| 2 | 提取设计规范 | 颜色/字体/间距 |
| 3 | 分析页面结构 | 布局/组件 |
| 4 | 编写HTML结构 | 编辑器 |
| 5 | 编写CSS样式 | 编辑器 |
| 6 | 逐项对比验证 | 测量工具 |

## 3. 专业建议

### A. 设计规范提取

#### 颜色提取
```css
/* 从设计稿提取主色 */
:root {
  --primary: #0066CC;      /* 主色 */
  --primary-hover: #0052A3;
  --secondary: #F5F7FA;    /* 背景色 */
  --text-primary: #1A1A1A; /* 文字色 */
  --text-secondary: #666666;
  --border: #E5E5E5;       /* 边框色 */
}
```

#### 字体提取
```css
/* 提取字体规范 */
body {
  font-family: 'Inter', -apple-system, sans-serif;
  font-size: 16px;
  line-height: 1.5;
  letter-spacing: 0;
}

h1 {
  font-size: 32px;
  font-weight: 700;
  line-height: 1.2;
}
```

#### 间距系统
```css
/* 提取间距规范 */
:root {
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
}
```

### B. 布局分析

#### 从设计稿读取的信息
- 整体布局：居中/全宽/栅格
- 组件排列：flex/grid/浮动
- 间距：内边距/外边距
- 尺寸：固定/响应式

#### 典型布局代码
```css
/* 常见布局 */
.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 24px;
}

.card {
  display: flex;
  gap: 16px;
  padding: 24px;
  background: #fff;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.08);
}
```

### C. 标注查看技巧

#### Figma开发模式
1. 点击开发模式(Dev Mode)
2. 查看自动标注
3. 一键复制CSS
4. 快速获取颜色/字体

#### 手动测量
- 选中元素 → 查看右侧面板
- 间距：X/Y轴位置
- 尺寸：W/H属性

### D. 切图导出

#### 导出设置
| 格式 | 用途 | 压缩 |
|------|------|------|
| PNG | 有透明需求 | 无损 |
| JPG | 照片/背景 | 80%质量 |
| WebP | 通用推荐 | 最佳压缩 |
| SVG | 图标/矢量 | 无损 |

#### 导出参数
- 1x / 2x / 3x 适配
- 64px 以下用 SVG
- 大图用 WebP

### E. 验证清单

| 验证项 | 方法 |
|--------|------|
| 颜色一致 | 取色器对比 |
| 字体一致 | 检查 font-family/size |
| 间距一致 | 测量工具 |
| 圆角一致 | 检查 border-radius |
| 阴影一致 | 检查 box-shadow |
| 交互状态 | 检查 hover/active/focus |

### F. 沟通技巧

#### 设计师沟通
- 问："这个间距是固定还是响应式？"
- 问："这个效果有详细参数吗？"
- 问："移动端有单独设计吗？"

#### 常见问题确认
- 动效参数是否确定？
- 字体是否有版权？
- 图片是否需要切图？
