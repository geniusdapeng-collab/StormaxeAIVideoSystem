## TL;DR

无障碍设计核心：可感知 + 可操作 + 可理解 + 健壮（WCAG四原则）

## 1. 场景分析

### 请求摘要
用户需要使网页对所有用户都可访问，包括残障人士

### 显性需求
- 了解WCAG标准
- 获得可执行的修复方案

### 场景分类
- `scenario_classification`: UI/前端
- `urgency`: high

## 2. WCAG 2.1 四原则

| 原则 | 说明 | 关键要求 |
|------|------|----------|
| 可感知(Perceivable) | 信息可被感知 | 替代文本、对比度、字幕 |
| 可操作(Operable) | 功能可通过键盘 | 键盘导航、足够时间、癫痫防护 |
| 可理解(Understandable) | 界面清晰易懂 | 可读性、可预测性、输入帮助 |
| 健壮(Robust) | 兼容辅助技术 | 兼容屏幕阅读器 |

## 3. 专业建议

### A. 替代文本

```html
<!-- 图片必须有alt -->
<img src="chart.png" alt="2024年用户增长趋势图">

<!-- 装饰性图片用空alt -->
<img src="decoration.png" alt="">

<!-- 复杂图片需要详细描述 -->
<img src="flowchart.png" alt="用户注册流程图：点击注册→填写信息→邮箱验证→完成">
```

### B. 键盘导航

```css
/* 焦点样式不可移除，可自定义 */
:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}

/* 隐藏但保持可聚焦 */
.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  border: 0;
}
```

### C. ARIA属性

```html
<!-- 按钮状态 -->
<button aria-expanded="false" aria-controls="menu">
  展开菜单
</button>
<div id="menu" hidden>...</div>

<!-- 表单错误 -->
<input aria-invalid="true" aria-describedby="email-error">
<span id="email-error" role="alert">邮箱格式错误</span>

<!-- 实时区域 -->
<div aria-live="polite" aria-atomic="true">
  已保存
</div>
```

### D. 对比度要求

WCAG AA级：
- 正常文本：4.5:1
- 大文本(18px+/14px粗体)：3:1
- UI组件和图形：3:1

### E. 焦点管理

```javascript
// 模态框打开时聚焦
const modal = document.getElementById('modal');
const focusable = modal.querySelectorAll(
  'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
);
focusable[0].focus();

// 关闭时返回触发元素
const trigger = document.activeElement;
modal.addEventListener('close', () => trigger.focus());
```
