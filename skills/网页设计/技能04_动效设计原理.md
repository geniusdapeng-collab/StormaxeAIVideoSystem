## TL;DR

动效设计核心：动效必须有意义、流畅且符合用户预期

## 1. 场景分析

### 请求摘要
用户需要为网页添加动效，提升用户体验

### 显性需求
- 掌握CSS动画技术
- 了解动效设计原则

### 场景分类
- `scenario_classification`: UI设计
- `urgency`: normal

## 2. 动效设计原则

| 原则 | 说明 |
|------|------|
| 有意义 | 引导注意力、反馈状态、提供连续性 |
| 响应及时 | 300ms内响应，60fps流畅 |
| 自然 | 符合物理世界的运动规律 |

## 3. 专业建议

### A. 基础动画技术

#### Transition（过渡）
```css
/* 按钮悬停 */
.btn {
  transition: transform 0.2s ease, background-color 0.2s ease;
}
.btn:hover {
  transform: scale(1.05);
  background-color: #0066cc;
}
```

#### Keyframes（关键帧）
```css
/* 加载动画 */
@keyframes spin {
  0% { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}
.loader {
  animation: spin 1s linear infinite;
}
```

### B. 微交互示例

#### 按钮点击波纹
```css
.btn-ripple {
  position: relative;
  overflow: hidden;
}
.btn-ripple::after {
  content: '';
  position: absolute;
  width: 100%;
  height: 100%;
  background: radial-gradient(circle, rgba(255,255,255,0.4) 0%, transparent 70%);
  transform: scale(0);
  transition: transform 0.5s;
}
.btn-ripple:active::after {
  transform: scale(2);
}
```

#### 卡片悬停浮起
```css
.card {
  transition: transform 0.3s cubic-bezier(0.34, 1.56, 0.64, 1);
}
.card:hover {
  transform: translateY(-8px);
  box-shadow: 0 20px 40px rgba(0,0,0,0.15);
}
```

### C. 缓动函数

| 效果 | 曲线 | 适用场景 |
|------|------|----------|
| 缓入 | ease-in | 退出动画 |
| 缓出 | ease-out | 启动动画 |
| 缓入缓出 | ease-in-out | 移动动画 |
| 弹性 | cubic-bezier(0.34, 1.56, 0.64, 1) | 强调效果 |

### D. 页面转场

```css
/* 淡入淡出 */
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
.page-enter {
  animation: fadeIn 0.3s ease-out;
}
```

### E. 性能优化

```css
/* 使用transform和opacity，避免动画属性 */
.good {
  transform: translateX(100px); /* GPU加速 */
  opacity: 0.5;
}

.bad {
  left: 100px; /* 触发重排 */
  background: red; /* 触发重绘 */
}
```
