## TL;DR

项目集管理 = 战略对齐 + 资源协调 + 依赖管理 + 收益实现 + 干系人管理

## 1. 项目集认知

### 1.1 什么是项目集
```
项目集定义：
一组相互关联的项目，通过协调管理以实现单独管理无法获得的收益

项目集特征：
├── 共同目标：所有项目服务于统一目标
├── 相互依赖：项目之间存在依赖关系
├── 协同效应：整体收益大于部分之和
├── 统一治理：统一的治理结构
└── 收益实现：关注收益而非交付物

项目集 vs 项目：
├── 项目：交付具体产出
├── 项目集：实现更大收益
├── 项目经理：管单个项目
└── 项目集经理：管多个项目的协调
```

### 1.2 项目集与项目组合
```
项目组合（Portfolio）：
为实现战略目标而管理的项目、项目集和运营的集合

项目集 vs 项目组合：

项目集：
├── 一组相关项目
├── 有直接关联
├── 共同收益
└── 协调管理

项目组合：
├── 无关的项目和项目集
├── 按战略优先级排序
├── 优化资源配置
└── 战略协调

三者关系：
├── 项目组合 → 项目集 → 项目
├── 组合协调战略
├── 项目集协调收益
└── 项目执行交付
```

## 2. 项目集治理

### 2.1 治理结构
```
项目集治理结构：

治理委员会：
├── 组成：高层领导、干系人代表
├── 职责：战略决策、重大审批
├── 权力：审批项目集章程、变更
└── 频率：每月/每季度

项目集管理办公室：
├── 组成：专业管理人员
├── 职责：日常管理、协调
├── 权力：日常决策、问题升级
└── 频率：持续

项目指导委员会：
├── 组成：各项目负责人
├── 职责：项目间协调
├── 权力：项目协调决策
└── 频率：每周/双周

各项目管理团队：
├── 组成：各项目团队
├── 职责：项目执行
├── 权力：项目内决策
└── 频率：每日
```

### 2.2 治理流程
```
项目集治理流程：

阶段1：项目集启动
├── 制定章程
├── 识别组件项目
├── 建立治理结构
├── 分配资源
└── 启动项目集

阶段2：项目集执行
├── 监控组件项目
├── 管理依赖
├── 协调资源
├── 管理风险
└── 汇报进展

阶段3：项目集收尾
├── 验证收益
├── 转移运营
├── 沉淀知识
├── 关闭项目集
└── 最终汇报
```

## 3. 资源管理

### 3.1 资源协调
```
项目集资源管理：

资源池管理：
├── 建立共享资源池
├── 统一资源调配
├── 优先级排序
├── 资源能力规划
└── 资源技能管理

资源分配原则：
├── 按战略优先级分配
├── 共享资源协调
├── 避免资源冲突
├── 平衡工作量
└── 考虑人员发展

跨项目资源协调：
├── 识别共享资源
├── 协调使用时间
├── 解决资源冲突
├── 监控资源使用
└── 优化资源配置
```

### 3.2 依赖管理
```
项目间依赖管理：

依赖类型：
├── 技术依赖：系统接口、数据格式
├── 进度依赖：任务先后顺序
├── 资源依赖：共享资源
├── 交付依赖：交付物依赖
└── 业务依赖：业务流程关联

依赖管理流程：
1. 识别依赖
2. 分析影响
3. 制定计划
4. 监控执行
5. 处理变更

依赖矩阵：
| 项目A → 项目B | 依赖类型 | 影响程度 | 管理策略 |
|--------------|----------|----------|----------|
| 设计 → 开发 | 技术 | 高 | 强协调 |
| 开发 → 测试 | 进度 | 高 | 紧密跟踪 |
| 测试 → 部署 | 交付 | 中 | 计划协调 |
```
