## TL;DR

项目复盘 = 回顾目标 + 分析结果 + 总结经验 + 沉淀知识 + 持续改进

## 1. 复盘认知

### 1.1 什么是项目复盘
```
项目复盘定义：
项目结束后，系统回顾项目全过程，分析得失，总结经验教训，沉淀知识用于未来改进

复盘的价值：
├── 避免重复犯错：同样的错误不犯第二次
├── 固化成功经验：好的做法传承下去
├── 促进团队学习：团队共同成长
├── 提升项目管理能力：不断提升
├── 知识沉淀：组织资产积累
└── 团队凝聚力：共同反思增强团队
```

### 1.2 复盘常见问题
```
复盘常见问题：

问题1：不复盘
├── 表现：项目结束就结束
├── 原因：太忙、觉得没必要
├── 后果：经验流失、重复犯错
└── 应对：建立机制、领导重视

问题2：流于形式
├── 表现：复盘变成批斗会
├── 原因：没有正确方法
├── 后果：没有实质收获
└── 应对：掌握方法、正向引导

问题3：归因不公
├── 表现：问题都归别人
├── 原因：不敢承担责任
├── 后果：没有真正学到
└── 应对：引导归因到自己

问题4：经验不沉淀
├── 表现：复盘完就完了
├── 原因：没有沉淀机制
├── 后果：经验流失
└── 应对：建立沉淀机制
```

## 3. 复盘流程

### 3.1 四步复盘法
```
四步复盘法：

Step 1：回顾目标（10%时间）
├── 原始目标是什么
├── 关键里程碑是什么
├── 成功标准是什么
└── 假设和约束是什么

Step 2：评估结果（20%时间）
├── 实际做到了什么
├── 和目标对比如何
├── 亮点是什么
├── 问题是什么
└── 偏差有多大

Step 3：分析原因（50%时间）
├── 为什么成功
├── 为什么有问题
├── 根本原因是什么
├── 主观原因是什么
├── 客观原因是什么
└── 可以改进什么

Step 4：总结经验（20%时间）
├── 学到了什么
├── 下次怎么做
├── 行动项是什么
├── 经验怎么沉淀
└── 谁来负责
```

### 3.2 STAR复盘法
```
STAR复盘法：

Situation（背景）：
├── 项目背景是什么
├── 当时情况如何
├── 面临什么挑战
└── 目标是什么

Task（任务）：
├── 我的职责是什么
├── 需要完成什么
├── 扮演什么角色
└── 负责什么部分

Action（行动）：
├── 我做了什么
├── 怎么做的
├── 为什么这么做
├── 做了哪些决策
└── 克服了什么困难

Result（结果）：
├── 结果如何
├── 做得好的原因
├── 做得不好的原因
├── 学到了什么
└── 下次怎么做
```

## 4. 复盘分析

### 4.1 成功因素分析
```
成功因素分析：

做得好的方面：
├── 目标达成：达成了什么目标
├── 效率提升：效率提高在哪里
├── 质量保证：质量如何
├── 团队协作：团队表现如何
├── 客户满意：客户反馈如何
├── 创新亮点：有什么创新
└── 成本控制：成本控制如何

成功原因：
├── 团队因素：能力、努力、协作
├── 管理因素：计划、控制、决策
├── 技术因素：方案、工具、能力
├── 资源因素：资源充足、利用高效
├── 环境因素：市场、竞争、政策
└── 运气因素：有时候也需要运气

可复制的经验：
├── 哪些经验可以复制
├── 需要什么条件
├── 如何标准化
└── 如何传承
```

### 4.2 问题分析
```
问题分析：

问题分类：
├── 计划问题：计划不准确、假设错误
├── 执行问题：执行不到位
├── 沟通问题：沟通不畅、误解
├── 技术问题：技术难度、方案问题
├── 资源问题：人员不足、资源不够
├── 外部问题：政策、市场、供应商
└── 管理问题：管理失误、决策错误

根因分析（5Why）：
├── 为什么1：问题是什么
├── 为什么2：原因是什么
├── 为什么3：更深层原因
├── 为什么4：根本原因
├── 为什么5：系统原因
└── 真正的原因

分析工具：
├── 5Why分析
├── 鱼骨图
├── 逻辑树
├── 帕累托分析
└── 复盘矩阵
```

### 6.2 复盘注意事项
```
复盘注意事项：

正向引导：
├── 目的是学习，不是追责
├── 关注改进，不是批评
├── 鼓励坦诚，不是隐瞒
├── 分享经验，不是推卸
└── 建设未来，不是抱怨过去

避免的坑：
├── 变成批斗会
├── 变成表功会
├── 变成抱怨会
├── 变成走过场
├── 归因都给别人
└── 不做行动计划
```
