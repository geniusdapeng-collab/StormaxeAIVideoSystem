# 异兽定妆照提交策略 v4.2-Peng

## 核心原则：生成8张 → 精选3-4张提交

**为什么不是8张全部提交？**
- Seedance的reference_image机制支持多张，但AI处理参考图时有信息容量限制
- 过多参考图可能导致：特征冲突、重点稀释、生成方向混乱
- 精选3-4张最能代表角色核心特征的图，反而一致性更高

---

## 生成阶段：8张全覆盖

| 角度 | 目的 |
|------|------|
| 正面全身 | 完整轮廓 + 所有特征可见 |
| 侧面全身 | 体型比例 + 侧面轮廓 |
| 背面全身 | 背部纹理 + 尾部特征 |
| 45度半身 | 头部细节 + 经典肖像 |
| 面部特写 | 五官/头部极端细节 |
| 动作奔跑 | 动态验证 + 姿态合理性 |
| 动作坐姿 | 栖息状态 + 自然感 |
| 手部特写 | 肢体纹理 + 材质细节 |

---

## 提交阶段：3-4张精选

### 决策标准（按优先级排序）

**1. 解剖正确性（一票否决）**
- ❌ 有5条腿的图 → 不提交
- ❌ 有突兀凸起/缺失部位的图 → 不提交
- ✅ 6足4翼完整可见 → 优先提交

**2. 视觉稳定性（一票否决）**
- ❌ 恐怖谷/恶心感严重 → 不提交
- ❌ 表情怪异/扭曲 → 不提交
- ✅ 神圣/神秘/庄严感 → 优先提交

**3. 特征完整性**
- 所有原著特征都在图中清晰展现
- 正面全身通常最佳（能看到全部特征）

**4. 轮廓清晰度**
- 全身图比半身图更能锁定整体形态
- 侧面图对体型比例锁定最关键

### 推荐组合

**标准4张组合（大多数异兽适用）**：
1. **正面全身** — 完整特征展示，轮廓锁定
2. **侧面全身** — 体型比例，侧面轮廓
3. **头部/面部特写** — 头部特征锁定（对帝江这种"无面目"角色尤其重要）
4. **动态姿态** — 验证运动时的形态一致性

**精简3张组合（特征简单的异兽）**：
1. 正面全身
2. 侧面全身
3. 头部特写

**扩展5张组合（特征复杂的异兽）**：
1-4 标准组合 + 5. 肢体特写（鳞片/毛发/爪翼细节）

## 特殊案例：混沌（帝江）

**为什么这组定妆照失败了？**
- 正面全身：5条腿（解剖错误）+ 恐怖谷严重
- 侧面全身：相对正常
- 背面全身：有大圆包凸起（形态漂移）
- 面部特写：恶心感最重（"无面目"处理失败）

**修复后应选哪3-4张提交？**
- 等v4.2-Peng生成完成后，人工审核8张
- 只选解剖正确 + 视觉稳定的图
- 预计可能只有2-3张合格

## 技术实现

```javascript
// 精选算法（未来可自动化）
function selectBestReferences(photos, criteria) {
  // 1. 解剖正确性筛选（AI视觉检测或人工）
  const anatomicallyCorrect = photos.filter(p => validateAnatomy(p));
  
  // 2. 视觉稳定性筛选
  const visuallyStable = anatomicallyCorrect.filter(p => !hasHorrorValley(p));
  
  // 3. 按特征完整性排序
  const ranked = visuallyStable.sort((a, b) => 
    scoreFeatureCompleteness(b) - scoreFeatureCompleteness(a)
  );
  
  // 4. 取前3-4张
  return ranked.slice(0, 4);
}
```
