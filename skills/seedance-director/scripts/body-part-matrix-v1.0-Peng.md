# 🐉 异兽部位标准化表（Body Part Matrix）v1.0-Peng

## 设计理念
把异兽的身体拆解为**标准化部位**，每个异兽根据自身体型"勾选"涉及的部位，填入对应材质描述。不带翅膀的异兽， wings 字段置空即可。

## 核心部位清单（16个标准化字段）

| 字段名 | 中文名 | 说明 | 示例值 |
|--------|--------|------|--------|
| **body** | 主体/躯干 | 躯干材质、形状、体积 | 囊状膜质、羊身毛皮、龙身鳞甲 |
| **head** | 头部 | 头部形态、与躯干连接方式 | 无头一体、人面、兽首 |
| **face** | 面部/五官 | 面部特征、表情 | 无面目、威严面容、狰狞 |
| **eyes** | 眼睛 | 位置、颜色、神态 | 腋下双眼、额头独眼、无眼 |
| **horns** | 犄角/冠 | 角的数量、形状、材质 | 无角、单角、双角分叉 |
| **ears** | 耳朵 | 形态、位置 | 无耳、尖耳、大耳扇形 |
| **mouth** | 嘴/牙齿 | 嘴型、牙齿特征 | 无口、虎牙外露、利齿 |
| **neck** | 颈部 | 颈长、粗细、与头连接 | 无颈、短粗颈、细长颈 |
| **forelimbs** | 前肢/手臂 | 前肢形态、数量 | 人爪、兽腿、鳍状 |
| **hindlimbs** | 后肢/腿 | 后肢形态、数量 | 六足、四足、无足 |
| **hands** | 手掌/爪子 | 末端形态、指甲 | 鹰爪、肉垫、蹄 |
| **wings** | 翅膀 | 翼型、膜质/羽质、数量 | 四翼膜质、双翼羽质、无翼 |
| **tail** | 尾巴 | 尾长、尾型、末端 | 无尾、长尾、尾分叉 |
| **coat** | 毛发/皮肤/鳞片 | 体表覆盖物材质 | 羊毛、鳞片、膜质 |
| **feet** | 足部/蹄 | 足部形态、着地方式 | 肉垫、蹄、利爪 |
| **special** | 特殊部位 | 独特器官或标记 | 发光经络、能量核心 |

## 材质描述标准化格式
每个部位包含5个属性：

```json
{
  "has": true,
  "material": "材质描述（物理属性）",
  "texture": "纹理细节（微观结构）",
  "color": "颜色及光泽",
  "light": "光影反应（次表面散射/透光/反射）",
  "detail": "特殊细节（厚度/硬度/透明度）"
}
```

## 动态组装逻辑

生成prompt时，系统自动：
1. 扫描部位表，只组装 `has=true` 的部位
2. 按固定顺序拼接：`body → head → eyes → mouth → limbs → wings → coat → special`
3. 每个部位按模板：`【部位名】material+texture+color+light+detail`
4. 缺失部位自动跳过，不生成描述

## 代码集成方案

在 `seedream-wrapper.js` 中：
- 新增 `--body-parts-json` 参数，接收部位表JSON
- 或者从 `--features` 解析结构化描述
- 每个角度自动选择对应部位的焦点（如面部特写优先 eyes+mouth+face）

## 优势

1. **100%通用**：任何异兽只要填表即可，无需硬编码
2. **部位级别一致性**：同一部位在所有角度保持相同材质描述
3. **角度智能对焦**：面部特写自动聚焦 eyes+mouth，动作奔跑聚焦 limbs+wings
4. **可扩展**：新增部位只需加一行，不影响已有异兽
