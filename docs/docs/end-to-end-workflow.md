## 标准交易工作流（端到端流程）

### 阶段1：市场扫描与信号生成（开盘前 09:00-09:30）

```
[定时触发] 每日 09:00 EST
    │
    ├─→ [01-09 Market Core Agents] 扫描大盘、行业、个股
    │       ├─→ 输出：市场状态报告、趋势方向、支撑阻力位
    │       └─→ 发送至：10 Orchestrator
    │
    ├─→ [新闻/事件/财报/宏观 Agents] 扫描消息面
    │       ├─→ 输出：今日催化事件清单、风险预警
    │       └─→ 发送至：10 Orchestrator
    │
    └─→ [10 Orchestrator] 汇总所有输入
            ├─→ 生成今日交易计划框架
            ├─→ 确定优先级：高/中/低机会
            └─→ 分发至：11-15 Sector Agents

[定时触发] 每2小时（09:30, 11:30, 13:30, 15:30）
    │
    └─→ [16-19 TSS技术信号 Agents] 扫描Watchlist
            ├─→ MA-Trend: 趋势方向信号
            ├─→ MACD-Momentum: 动量信号
            ├─→ RSI-OB: 超买超卖信号
            ├─→ KDJ-Stochastic: 随机指标信号
            ├─→ Volume-Profile: 量价信号
            └─→ Pattern-Rec: 形态识别信号
            │
            └─→ 所有信号发送至：21 TSS-Aggregator
```

### 阶段2：信号聚合与深度分析（实时）

```
[21 TSS-Aggregator] 接收所有技术信号
    │
    ├─→ 独立性检验（剔除相关性>0.7的冗余信号）
    ├─→ 动态加权计算（基于最近30天表现）
    ├─→ 共振等级判定（强/中/弱/冲突）
    │
    ├─→ 若 Consensus Score ≥ 75 且 共振等级 ≥ 中：
    │       └─→ 发送至：20 Options-Driver
    │
    └─→ 若 Consensus Score < 75：
            └─→ 记录为观望，2小时后重新评估

[20 Options-Driver] 接收高置信技术信号
    │
    ├─→ 扫描目标股票期权链
    ├─→ 分析IV、Skew、Smart Money流向
    ├─→ 计算Greeks（Delta/Gamma/Vega/Theta）
    ├─→ 评估期权定价优势
    │
    ├─→ 若 期权信号与技术面方向一致：
    │       └─→ 增强置信度，推荐期权策略
    │       └─→ 发送至：22 TOS-Calculator
    │
    └─→ 若 期权信号冲突或IV过高：
            └─→ 降低置信度或标记为观望
            └─→ 发送至：22 TOS-Calculator（附警告）
```

### 阶段3：交易方案计算与风险评估（实时）

```
[22 TOS-Calculator] 接收完整信号包
    │
    ├─→ 计算盈亏比（R:R）
    ├─→ 计算最优仓位（1/4 Kelly）
    ├─→ 计算期望值（EV）
    ├─→ 生成订单参数（入场/止损/止盈）
    ├─→ 生成情景概率矩阵
    │
    ├─→ 若 R:R ≥ 1.5:1 且 EV > 0：
    │       └─→ 发送至：23 Risk-Manager（预执行审查）
    │
    └─→ 若 R:R < 1.5:1 或 EV ≤ 0：
            └─→ 拒绝交易，记录原因

[23 Risk-Manager] 预执行风险评估
    │
    ├─→ 检查：风险叠加（加入后总Delta/Vega/行业暴露）
    ├─→ 检查：尾部风险（次日跳空-20%情景）
    ├─→ 检查：流动性（1小时能否平仓）
    ├─→ 检查：事件风险（未来5天是否有重大事件）
    │
    ├─→ 若 所有检查通过：
    │       ├─→ 输出：PASS
    │       └─→ 发送至：24 Execution-Agent
    │
    ├─→ 若 部分检查警告：
    │       ├─→ 输出：WARNING（附条件）
    │       ├─→ 条件可能：缩小仓位/收紧止损/附加对冲
    │       └─→ 发送至：24 Execution-Agent（附条件执行）
    │
    └─→ 若 检查未通过：
            ├─→ 输出：REJECT
            └─→ 通知Orchestrator，记录原因
```

### 阶段4：交易执行（实时）

```
[24 Execution-Agent] 接收通过审查的交易方案
    │
    ├─→ 选择订单类型（限价/市价/条件单）
    ├─→ 选择有效期（DAY/GTC）
    ├─→ 若大额订单：智能拆单（TWAP/VWAP/冰山）
    ├─→ 选择执行时段策略
    │
    ├─→ 提交入场订单
    ├─→ 成交后：
    │       ├─→ 提交止损单（GTC）
    │       ├─→ 提交止盈单（GTC）
    │       └─→ 通知：26 Position-Monitor（加入监控）
    │       └─→ 通知：23 Risk-Manager（更新暴露）
    │
    └─→ 生成执行报告
```

### 阶段5：持仓监控（持续）

```
[26 Position-Monitor] 每5分钟扫描所有持仓
    │
    ├─→ 计算持仓健康度（0-100）
    ├─→ 对比实际走势 vs 入场预期
    ├─→ 管理移动止损（盈利≥1R后启动）
    ├─→ 检查时间止损（持仓逾期）
    ├─→ 监控期权Theta burn和Delta漂移
    │
    ├─→ 若 健康度 < 40：
    │       └─→ 黄色预警 → 通知Orchestrator
    │
    ├─→ 若 移动止损触发：
    │       └─→ 通知Execution-Agent（更新止损单）
    │
    ├─→ 若 时间止损触发：
    │       └─→ 通知Execution-Agent（减仓/平仓）
    │
    └─→ 持续更新数据至：23 Risk-Manager

[23 Risk-Manager] 持续监控（与Position Monitor并行）
    │
    ├─→ 计算账户级VaR、Delta/Vega总暴露
    ├─→ 检查所有硬性限额
    ├─→ 生成每小时风险仪表盘
    │
    ├─→ 若 任何限额触及预警线：
    │       └─→ 通知Orchestrator + 建议减仓
    │
    └─→ 若 任何限额触及限制线：
            └─→ 通知25 KillSwitch（强制减仓）

[25 KillSwitch] 最高优先级实时监控
    │
    ├─→ 持仓级：价格触及止损 → 5秒内市价平仓
    ├─→ 持仓级：5分钟跌幅>10% → 立即平仓
    ├─→ 市场级：VIX 1小时飙升>40% → 减仓至30%
    ├─→ 市场级：指数熔断Level 2 → 减仓50%
    ├─→ 账户级：日内回撤≥5% → 强制减仓50%
    │
    └─→ 所有操作无需审批，直接执行
    └─→ 事后通知所有相关Agent
```

### 阶段6：平仓与复盘（交易结束后）

```
[触发条件] 任何持仓平仓（止盈/止损/时间止损/移动止损/KillSwitch）
    │
    ├─→ [24 Execution-Agent] 完成平仓执行
    │       └─→ 生成最终执行报告
    │
    ├─→ [27 Postmortem-Agent] 自动触发（平仓后2小时内）
    │       ├─→ 重建完整决策时间线
    │       ├─→ 归因分析（Alpha/Beta/执行/运气）
    │       ├─→ 执行质量评分
    │       ├─→ 行为模式识别
    │       └─→ 生成改进建议
    │
    └─→ [28 Backtest-Optimizer] 周度/月度汇总
            ├─→ 汇总所有Postmortem数据
            ├─→ 单Agent回测表现更新
            ├─→ TSS权重Walk-Forward优化
            ├─→ 参数敏感性分析
            └─→ 生成策略进化建议
```

## 异常工作流

### 异常1：Risk Manager拒绝交易

```
[23 Risk-Manager] 输出：REJECT
    │
    ├─→ 通知：10 Orchestrator
    ├─→ Orchestrator决策：
    │       ├─→ 选项A：接受拒绝，记录原因，扫描下一机会
    │       └─→ 选项B：若用户Override（不推荐）：
    │               ├─→ 要求用户确认风险
    │               └─→ 记录Override日志
    │
    └─→ 通知：22 TOS-Calculator（如可调整方案）
            └─→ 可能调整：缩小仓位/放宽止损/附加对冲
            └─→ 重新提交Risk审查
```

### 异常2：执行失败/部分成交

```
[24 Execution-Agent] 执行异常
    │
    ├─→ 场景A：限价单超时未成交
    │       └─→ 评估：修改价格 / 改为市价 / 取消交易
    │
    ├─→ 场景B：部分成交（<50%）
    │       └─→ 等待5分钟 / 修改价格 / 取消剩余
    │
    ├─→ 场景C：券商拒单（保证金不足）
    │       └─→ 通知Risk Manager → 缩减方案 → 重新提交
    │
    └─→ 场景D：股票停牌
            └─→ 标记冻结 → 通知Orchestrator → 复牌后评估
```

### 异常3：市场系统性风险

```
[市场事件] VIX飙升 / 指数熔断 / 个股闪崩
    │
    └─→ [25 KillSwitch] 自动触发
            ├─→ 根据触发级别执行减仓/清仓
            ├─→ 通知所有Agent进入保护模式
            │
            ├─→ [10 Orchestrator] 暂停所有新信号生成
            ├─→ [23 Risk Manager] 更新风险等级为红色
            ├─→ [26 Position Monitor] 标记所有持仓为"危机监控"
            │
            └─→ [27 Postmortem] 记录紧急事件时间线
                    └─→ [28 Backtest] 标记为极端市场环境数据点
```

## Agent通信协议

### 消息格式（标准信封）

```json
{
  "message_id": "uuid",
  "timestamp": "2026-04-30T15:30:00Z",
  "from_agent": "21-tss-aggregator",
  "to_agent": "22-tos-calculator",
  "message_type": "TRADE_SIGNAL",
  "priority": "HIGH",
  "payload": { ... },
  "requires_ack": true,
  "ttl_seconds": 300
}
```

### 消息类型清单

| 消息类型 | 发送方 | 接收方 | 说明 |
|---------|--------|--------|------|
| `MARKET_SCAN` | 01-09 | 10 | 市场扫描结果 |
| `CATALYST_ALERT` | 新闻Agent | 10 | 催化事件预警 |
| `SECTOR_ROTATION` | 11-15 | 10,21 | 行业轮动信号 |
| `TECH_SIGNAL` | 16-19 | 21 | 技术信号 |
| `OPTIONS_FLOW` | 20 | 21,22 | 期权流向分析 |
| `CONSENSUS_SIGNAL` | 21 | 22 | 聚合后的共识信号 |
| `TRADE_PLAN` | 22 | 23 | 交易方案 |
| `RISK_CHECK` | 23 | 24 | 风险审查结果 |
| `EXECUTION_ORDER` | 24 | 券商API | 执行订单 |
| `POSITION_UPDATE` | 24 | 26,23 | 持仓更新 |
| `HEALTH_ALERT` | 26 | 10,23 | 持仓健康预警 |
| `KILL_SWITCH` | 25 | 24,23,26 | 紧急平仓指令 |
| `RISK_REPORT` | 23 | 10 | 风险仪表盘 |
| `POSTMORTEM` | 27 | 28 | 交易复盘数据 |
| `OPTIMIZATION` | 28 | 10,21 | 参数优化建议 |

### 优先级体系

| 优先级 | 说明 | 处理时限 |
|--------|------|---------|
| `CRITICAL` | KillSwitch触发、Margin Call | 立即（<1秒） |
| `HIGH` | 止损触发、风险限额触及 | <5秒 |
| `MEDIUM` | 信号生成、执行确认 | <1分钟 |
| `LOW` | 复盘报告、优化建议 | <1小时 |

## 数据流与存储

### 实时数据流

```
行情API (Yahoo/WebSocket)
    │
    ├─→ [价格数据] → 16-19技术信号计算
    ├─→ [期权链] → 20 Options-Driver
    └─→ [VIX/指数] → 23 Risk Manager
```

### 持久化存储

| 数据类型 | 存储位置 | 保留期 | 说明 |
|---------|---------|--------|------|
| 原始行情 | `data/market/` | 2年 | OHLCV数据 |
| Agent信号 | `data/signals/` | 1年 | 所有Agent输出 |
| 交易记录 | `data/trades/` | 永久 | 完整交易生命周期 |
| 风险日志 | `data/risk/` | 2年 | Risk Manager输出 |
| 复盘报告 | `data/postmortem/` | 永久 | Postmortem输出 |
| 回测结果 | `data/backtests/` | 2年 | Backtest Optimizer输出 |
| 用户行为 | `data/behavior/` | 1年 | Override、情绪交易记录 |
