## 六、三个具体问题的根因

### 问题1：S00 Timeline 损坏（未知情绪、镜头为空、乱码）

**现象**：
```
【Timeline】 持续7秒 | 情绪: 未知情绪 | 镜头: ，G，，， | 【 — Nirath】
：0（，）
：Nirath，
```

**根因**：
1. `opening-title-designer.js` 生成 `seedancePrompt` 含有中文字段
2. fix19-21 的 `_translatePhasesToEnglish` 函数翻译后，phase 的 `description` 被替换为 `value.replace(CJK_REGEX, ' ').trim()`（清除了中文但内容残缺）
3. 重建的 `seedancePrompt` 中引用了翻译后的 phases，但 phases 的 description 已经变成破碎内容
4. LLM 生成 prompt 时把这个破碎的 seedancePrompt 内容抄进去了

### 问题2：字段重复（Action/Mood/Camera/Lighting 各出现2次）

**根因**：fix23 在 `_buildShotContext` 里输出 `P1 场景/P1 动作/P1 角色/情绪/运镜/灯光`，这些标签进入了 LLM 的 context。LLM 在生成 prompt 时，误把这些标签当成了 prompt 的一部分，在输出中重复生成了这些字段。

具体来说，LLM 看到 context 里包含：
```
P1 动作: 白泽从黑暗深处缓缓向前迈步...
情绪: 神秘、神圣、庄严...
运镜: 固定镜头，从正面缓慢推进...
灯光: ...
```

然后 LLM 在输出中既输出了这些字段的标准格式，又额外输出了 Action/Mood/Camera/Lighting 等。

### 问题3：S02/S03/S05 SPEAKER 与 Action 角色不一致

**现象**：
- S02：Action 描述"小G站在岩壁脚下..."，但 SPEAKER=白泽
- S03：Action 描述"小G在狭窄裂隙中谨慎前行..."，但 SPEAKER=白泽
- S05：Action 描述"小G鼓起勇气迈出一步..."，但 SPEAKER=白泽

**根因**：
1. story-plan.json 里 S02-S05 的 Dialogue 为空（只有 S01 和 S06 有对白）
2. Stage 7.5 对话标注因 `allShots is not defined` ReferenceError 失败，dialogue-annotation.json 未生成
3. Stage 8.3 闸机自愈的质检2：检测到没有 P0 Dialogue，执行强制注入
4. 4层来源全部为空，fallback 到 `storyPlan.title || '白泽'` → 注入了"白泽"的对白
5. S02/S03/S05 的 Action 是小G视角，但 SPEAKER 被注入了白泽的对白

**对白来源**：fix33 质检2 fallback 逻辑：
```javascript
const fbSp = storyPlan?.title || '白泽';
const fbTxt = fbSp === '白泽' ? '吾名白泽，通晓万物之情。' : '我在。';
injection = ` | 【Dialogue】 - SPEAKER: ${fbSp} | TYPE: ${type} | EMOTION: ${emotion} | TEXT: "${fbTxt}" | LIP_SYNC: YES | 口型情绪: 自然`;
```

## 九、修复方向分析

### 方向A：保守修复（保留 fix33 SPEAKER 修复，清除副作用）

1. **回退 `_buildShotContext` 改写**：恢复 fix17 的纯英文简短输出，移除 fix23 添加的中文标签和中文内容
2. **回退 stage8-pregeneration.js 的 P1 预填充**：移除 `【P1 Scene】【P1 Action】【P1 Character】` 注入

**优点**：改动量小，保持 SPEAKER 修复成果
**风险**：P1 预填充回退后，LLM 可能重新生成泛化 Nirath 神兽模板内容

### 方向B：修复上游根因（全链路修复）

1. **修复 Stage 2 PRD 解析**：从 PRD 正确提取 `beastId`（应为"baize"）和 `title`（应为"白泽"）
2. **修复 Stage 5**：确保 character-prompt-build 在 beastId=null 时也正确建立 character map
3. **修复 Stage 7.5 allShots ReferenceError**：让 dialogue-annotation.json 正常生成，S02-S05 获得各自的对白
4. **回退 fix23/fix28 的副作用代码**：清除 _buildShotContext 的中文标签和 P1 预填充

**优点**：彻底解决，从源头恢复数据流
**难点**：Stage 2/5/7.5 的问题可能涉及更深的代码逻辑，修复周期长

### 方向C：局部快速修复（仅修 S00 Timeline）

1. **修复 `_translatePhasesToEnglish`**：不要清除 phase description 中的中文内容，而是翻译或保留英文
2. **修复 seedancePrompt 重建逻辑**：确保生成干净的 seedancePrompt
